第5章 循环与分支程序设计

章节概述

循环 与 分支 是汇编程序控制执行流程的 核心结构，决定程序逻辑走向与重复执行规则；80386及后继机型的32位特性 ( 如32位寄存器、灵活寻址方式 ) 可在实模式下优化程序效率，兼顾兼容性与执行性能. 本章聚焦循环/分支程序的结构、设计方法，并结合32位CPU特性讲解优化思路.

5.1 循环程序设计

循环程序用于实现”重复执行某段逻辑”的需求，是处理批量数据、迭代运算 ( 如累加、排序 ) 的核心手段，核心要素为 循环初始化、循环体、循环控制、循环收尾.

5.1.1 循环程序的结构形式

循环程序包含4个核心部分，按”判断时机”分为两种典型结构，以下结合代码示例说明 :

1. 结构组成 ( 通用 )

结构部分 功能说明 示例操作
初始化部分 设置循环初始状态 ( 计数、地址、数据初始化 ) 循环计数器 \(CX\) 赋值、数据缓冲区地址指针 \(SI/DI\) 定位、累加器清零
循环体 重复执行的核心逻辑 数据运算、存储、\(I/O\) 读写、条件判断
循环控制 修改循环变量，判断是否终止循环 \(CX\) 自减、标志位检测、条件转移指令
循环结束 处理循环结果 ( 如数据汇总、寄存器复位 ) 存储最终结果、打印输出、寄存器清零

补充 : SI / DI 寄存器在循环中的应用

  • SI ( Source Index ) : 源变址寄存器, 常指向 源数据 地址.
  • DI ( Destination Index ) : 目的变址寄存器, 常指向 目标数据 地址.
  • 常用场景 :
    1. 数组/缓冲区访问 : 配合基址寄存器 ( BX/BP ) 进行 基址变址寻址 ( 如 MOV AX, [BX+SI] ).
    2. 串操作指令 : 在 MOVSB/MOVSW 等指令中, SI 自动指向 DS 段的源串, DI 自动指向 ES 段的目标串, 且会自动增减 ( 依据 DF 标志 ).

2. 两种结构形式 ( 附代码示例 )

环境说明 : 以下示例基于8086实模式，段定义为默认数据段 ( \(DATA\) ) 和代码段 ( \(CODE\) ) ，所有示例均包含完整段结构.

( 1 ) 先判断后执行 ( DO-WHILE型 )
  • 逻辑 : 先判断条件，满足则执行循环体 ( 循环体可能执行 \(0\) 次 ).
  • 适用场景 : 需先验证条件再执行的场景 ( 如”若缓冲区有数据则处理” ).
  • 代码示例 : 计算 \(1 \sim 10\) 的累加和 ( 先判断 \(CX\) 是否为 \(0\)，再执行累加 )
( 2 ) 先执行后判断 ( DO-UNTIL型 )
  • 逻辑 : 先执行循环体，再判断条件 ( 循环体至少执行 \(1\) 次 ).
  • 适用场景 : 必须执行至少一次的循环 ( 如”读取键盘输入，直到输入回车” ).
  • 代码示例 : 计算 \(1 \sim 10\) 的累加和 ( 先执行累加，再判断 )

5.1.2 循环程序设计方法

1. 核心设计步骤

  1. 明确循环目标 : 确定需重复执行的逻辑 ( 如”累加数组元素”“筛选偶数” );
  2. 定义循环要素 :
    • 循环变量 : 控制循环次数 ( 如 \(CX\) ) 或终止条件 ( 如”遇到 \(0\) 则停止” );
    • 终止条件 : 计数为 \(0\)、数据满足某条件 ( 如 \(ZF=1\) ) 、外部触发 ( 如键盘输入 );
  3. 编写初始化代码 : 赋值循环变量、地址指针、累加器…；
  4. 设计循环体 : 实现核心逻辑，避免”死循环” ( 确保循环变量会被修改 );
  5. 实现循环控制 : 选择合适的指令 ( 计数用 \(LOOP\)，条件用 \(JZ/JNZ\)… );
  6. 处理循环收尾 : 存储结果、复位寄存器、释放资源.

2. 常用循环控制指令 ( 8086/80386 )

指令 功能说明 依赖寄存器/标志位
\(LOOP\ label\) \(CX=CX-1\)，若 \(CX≠0\) 则跳转到 \(label\) \(CX\)
\(LOOPZ\ label\) \(CX=CX-1\)，若 \(CX≠0\) 且 \(ZF=1\) 则跳转 \(CX\)、\(ZF\)
\(LOOPNZ\ label\) \(CX=CX-1\)，若 \(CX≠0\) 且 \(ZF=0\) 则跳转 \(CX\)、\(ZF\)
\(JZ/JE\ label\) 若 \(ZF=1\) ( 结果为\(0\)/相… ) 则跳转 \(ZF\)
\(JNZ/JNE\ label\) 若 \(ZF=0\) ( 结果 非\(0\)/不相… ) 则跳转 \(ZF\)

3. 代码示例 : LOOP指令实现循环 ( 简化版累加 )

4. 代码示例 : LOOPNZ指令实现搜索 ( 查找非零值 )

5.1.3 多重循环程序设计

1. 核心概念

多重循环 ( 循环嵌套 ) : 一个循环体内包含另一个完整的循环结构，外层循环控制”轮数”，内层循环控制”每轮的迭代次数”.

  • 关键规则 :
    1. 内外循环的计数寄存器需独立 ( 如外层用 \(CX\)，内层用 \(DX\) );
    2. 内层循环需完整执行所有迭代后，外层循环才执行一次迭代；
    3. 禁止跨层跳转 ( 避免破坏循环结构 ).

2. 代码示例 : 冒泡排序 ( 双重循环 )

需求 : 对数组 \(ARR\) 中的5个16位整数进行升序排序 ( 外层控制轮数，内层控制每轮比较次数 ).

5.2 分支程序设计

分支程序用于”根据条件执行不同逻辑”，核心是通过 比较/测试指令生成标志位，结合 条件转移指令 实现流程跳转，分为 单分支、双分支、多分支 三类.

5.2.1 分支程序的结构形式

1. 单分支结构 ( IF-THEN )

  • 逻辑 : 条件成立则 执行 某段程序，否则 跳过.
  • 代码示例 : 判断数值是否为偶数 ( 是则存入 \(EVEN_NUM\) ).

2. 双分支结构 ( IF-THEN-ELSE )

  • 逻辑 : 条件成立执行程序段 \(1\)，否则执行程序段 \(2\).
  • 代码示例 : 比较两个数，将较大值存入 \(MAX\).

3. 多分支结构 ( CASE )

  • 逻辑 : 根据变量的不同取值，执行对应的多个程序段之一.
  • 基础实现 : 连续条件转移 ( 适合分支数少的场景 ).
  • 代码示例 : 根据 \(OP\) 的值执行加减乘运算 ( \(OP=1\) 加，\(OP=2\) 减，\(OP=3\) 乘 ).

5.2.2 分支程序设计方法

1. 核心设计步骤

  1. 确定分支条件 : 选择 \(CMP\) ( 比较 ) 、\(TEST\) ( 测试 )…指令生成标志位；
  2. 匹配条件转移指令 : 根据标志位 ( \(ZF\)、\(CF\)、\(SF\)… ) 选择对应指令 ( 如\(JG\)、\(JL\)、\(JC\) );
  3. 编写分支体 : 确保各分支逻辑独立，避免”掉分支” ( 遗漏条件 );
  4. 统一分支出口 : 通过 \(JMP\) 指令将所有分支汇总到同一出口 ( 提升可读性 ).

2. 常用条件转移指令 ( 按标志位分类 )

标志位 指令 功能说明
\(ZF\) \(JZ/JE\) 结果为 \(0\)/相等则跳转
\(JNZ/JNE\) 结果非\(0\)/不相等则跳转
\(CF\) \(JC\) 有进位/借位则跳转
\(JNC\) 无进位/借位则跳转
\(SF\) \(JS\) 结果为负则跳转
\(JNS\) 结果为正则跳转
无符号数 \(JA/JNBE\) \(>\)（不小于等于）则跳转
\(JAE/JNB\) \(≥\)（不小于）则跳转
\(JB/JNAE\) \(<\)（不大于等于）则跳转
\(JBE/JNA\) \(≤\)（不大于）则跳转
有符号数 \(JG/JNLE\) \(>\)（不小于等于）则跳转
\(JGE/JNL\) \(≥\)（不小于）则跳转
\(JL/JNGE\) \(<\)（不大于等于）则跳转
\(JLE/JNG\) \(≤\)（不大于）则跳转

补充说明 :

  1. 无符号数/有符号数的跳转指令逻辑一致，仅判断的数值类型不同 ( 无符号数不考虑正负，有符号数需结合符号位 \(SF\) 判断 )；
  2. 括号内”不小于等于/不大于等于”是指令别名的语义解释 ( 如\(JA = Jump Above\)，\(JNBE = Jump Not Below or Equal\) )，对应数学符号更直观；
  3. 所有\(ZF/CF/SF\)为CPU状态标志位 : \(ZF\) ( 零标志 )、\(CF\) ( 进位标志 )、\(SF\) ( 符号标志 ).

5.2.3 跳跃表法 ( 高效多分支 )

1. 核心原理

  • 构建”跳跃表” : 一个存储各分支程序入口地址的数组；
  • 按分支编号计算偏移量 : 通过”基址+变址”寻址定位目标分支地址；
  • 间接跳转 : 用 \(JMP [BX+SI]\)…指令直接跳转到目标分支 ( 避免连续条件判断 ).
  • 优势 : 分支数 \(≥4\) 时，效率远高于连续条件转移.

2. 代码示例 : 跳跃表实现多分支运算

需求 : 同5.2.1的多分支示例，用跳跃表优化 \(OP=1/2/3\) 的分支逻辑.

5.3 实模式下发挥80386及后继机型的优势

80386是32位 \(CPU\)，实模式下兼容16位程序，但可利用其32位特性 ( 寄存器、寻址方式、新指令 ) 优化程序，无需切换保护模式.

5.3.1 充分利用32位字长特性

1. 核心优势

80386提供32位通用寄存器 ( \(EAX\)、\(EBX\)、\(ECX\)、\(EDX\)、\(ESI\)、\(EDI\)、\(EBP\)、\(ESP\) ) ，一次可处理4字节数据，相比8086的16位寄存器，数据处理效率翻倍.

2. 代码示例 : 32位累加运算

5.3.2 通用寄存器可作为指针寄存器

1. 核心优势

8086中仅 \(BX\)、\(BP\)、\(SI\)、\(DI\) 可作为 地址指针；80386中所有通用寄存器 ( \(EAX\)、\(EBX\)、\(ECX\)… ) 均可作为基址寄存器，指针使用更灵活.

2. 代码示例 : 任意寄存器作为基址

5.3.3 与比例因子有关的寻址方式

1. 核心优势

80386新增”基址+变址+比例因子”寻址方式，公式 : [基址寄存器 + 变址寄存器 × 比例因子 + 位移量]，比例因子可选 \(1/2/4/8\)，适合快速访问数组元素 ( 无需手动计算偏移 ).

2. 代码示例 : 比例因子寻址访问数组

5.3.4 各种机型提供的新指令

80386及后继机型新增32位专用指令，简化32位数据处理 :

指令 功能说明 示例
\(LOOPD\ label\) 32位循环指令，\(ECX=ECX-1\)，非\(0\)则跳转 \(LOOPD\ LOOPBODY\)
\(MOVSD\) 32位串传送指令，\([EDI]=[ESI]，ESI/EDI±4\) \(REP\ MOVSD\)
\(CMPSD\) 32位串比较指令，比较 \([ESI]\) 和 \([EDI]\) \(REPE\ CMPSD\)
\(SCASD\) 32位串扫描指令，比较 \(EAX\) 和 \([EDI]\) \(REPNE\ SCASD\)
