第7章 高级汇编语言技术

章节概述

高级汇编技术是汇编器 ( 如MASM ) 提供的 预编译特性，通过宏、重复汇编、条件汇编实现 代码复用、重复代码自动生成、条件化编译，大幅提升汇编程序的开发效率与可维护性. 本章核心是掌握汇编器的”编译期逻辑控制”能力，区别于运行期的子程序/循环逻辑.

7.1 宏汇编

宏是 汇编期展开的代码模板，将重复的短代码封装为”宏指令”，调用时汇编器会将宏体直接替换到调用位置 ( 区别于子程序 : 子程序 是 运行期调用，宏 是 编译期展开 ).

7.1.1 宏定义、宏调用和宏展开

1. 核心概念

  • 宏定义 : 用 \(MACRO\) ( 宏开始 ) 和 \(ENDM\) ( 宏结束 ) 封装代码模板；
  • 宏调用 : 直接使用宏名 ( 类似指令 );
  • 宏展开 : 汇编器将宏体代码替换到调用位置，生成最终的机器指令.

2. 基本格式

3. 代码示例 ( 简单宏 )

4. 宏 vs 子程序 : 关键区别

特性 宏 子程序
执行时机 汇编期展开 ( 代码直接嵌入 ) 运行期调用 ( 栈跳转 )
代码体积 调用次数越多，代码越膨胀 代码仅一份，体积紧凑
运行效率 无调用/返回开销，速度快 有栈操作开销，速度略慢
适用场景 短代码、高频调用的简单逻辑 长代码、多参数的复杂逻辑

7.1.2 宏定义中的参数

宏的强大之处在于支持 参数，使得同一段代码模板可以适应不同的数据和场景. 宏参数在汇编期间进行 字符串形式的直接替换.

1. 基本参数的使用

宏定义时在宏名后列出 形参 ( 形式参数 ) ，宏调用时提供 实参 ( 实际参数 ).

代码示例 : 移位宏 实现”将指定寄存器逻辑左移 N 位”的通用操作.

2. 参数连接符 (&)

当参数不仅仅是独立的单词，而是 作为标识符的一部分 ( 如拼接变量名、标号 ) 时，必须使用 & 操作符作为分隔符，告诉汇编器”这里是参数”.

  • 场景 : 参数与前缀或后缀连在一起.
  • 规则 : 前缀&参数 或 参数&后缀.

7.1.7 PURGE 伪操作

PURGE 伪操作用于 解除宏定义，将指定的宏名从汇编器的内部表中删除.

主要用途 : 1. 释放内存 : 在宏不再使用时释放其占用的内存空间 ( 在早期内存受限的机器上很重要 ); 2. 重新定义 : 虽可直接重定义宏，但先 PURGE 再定义更安全，可确保宏名被彻底清除.

7.2 重复汇编

\(IFIDN\) <串1>, <串2> : 判断两个字符串是否 完全相同，用于根据参数值生成不同代码.

示例 ( 根据参数生成加减指令 )

本章核心总结

  1. 宏汇编 : 汇编期展开的代码模板，适合短代码复用，用 \(LOCAL\) 解决符号冲突，宏库实现跨文件复用；
  2. 重复汇编 : 自动生成重复代码，\(REPT\) 定次数，\(IRP/IRPC\) 按列表/字符串重复；
  3. 条件汇编 : 编译期判断条件，\(IF/IFNDEF/IFB\)…伪操作实现多版本代码、可选参数、防重复包含…功能.
