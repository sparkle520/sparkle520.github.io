键盘 INT 16H 00H 读键盘输入 返回: AH=扫描码, AL=字符码 游戏按键控制 (读取扫描码)
01H 查询键盘状态 返回: ZF=0 (有键), ZF=1 (无键) 非阻塞轮询按键

02H 设置光标位置 参数: BH=页, DH=行, DL=列 定位输出位置
09H 写字符+属性 参数: AL=字符, BL=属性, CX=次数 彩色文本界面
0EH TTY写字符 参数: AL=字符 简单流式输出 (自动滚屏)
INT 21H 02H 显示单个字符 参数: DL=字符 输出单个字母
09H 显示字符串 参数: DS:DX=串地址 (以$结尾) 输出提示信息
打印 INT 17H 00H 打印字符 参数: AL=字符, DX=端口 直接控制打印机
串口 INT 14H 00H 初始化串口 参数: AL=参数, DX=端口 设置波特率/校验位

第9章 BIOS和DOS中断

章节概述

(基本输入输出系统) 和 (磁盘操作系统) 通过 中断服务程序 封装了外设 (键盘、显示器、打印机...) 的硬件操作逻辑。开发者无需直接操作 端口，只需调用对应的中断 (如 INT 10H、INT 21H) 即可实现外设通信。本章聚焦常用外设的 中断与 功能调用，是汇编程序与外设交互的 "便捷工具"。

9.1 键盘I/O

键盘是最常用的输入外设，其输入过程需经过 "硬件扫描码 → 字符码" 的转换， 中断提供了封装好的键盘读取功能。

9.1.1 字符码与扫描码

键盘输入的核心数据分为两类:

  • 扫描码: 键盘硬件产生的 位编码，用于标识 "按下/松开的按键":
    • 通码: 按键按下时产生 (第 位为 );
    • 断码: 按键松开时产生 (第 位为 ，即通码 + )。
    • 示例: 字母 A 的通码是 1EH，断码是 9EH。
  • 字符码: 扫描码对应的 码 (仅字母、数字、符号等可打印字符有字符码)。
    • 示例: A 的字符码是 41H (对应 的 'A')。

9.1.2 BIOS键盘中断 (INT 16H)

通过 INT 16H 提供键盘服务，功能由 AH 寄存器指定，直接与键盘硬件交互，无回显/缓冲的底层控制。

AH 功能描述 返回值
00H 读取键盘输入 AH=扫描码，AL=字符码 (无按键则阻塞等待)
01H 查询键盘状态 若有按键: ZF=0，AH=扫描码，AL=字符码 (指针不移动)
若无按键: ZF=1
02H 读取特殊键状态 AL=状态字节 (位0: 右Shift，位1: 左Shift，位2: Ctrl...)

代码示例: 用 BIOS 中断读键盘

9.1.3 DOS键盘功能调用 (INT 21H)

通过 INT 21H 提供更上层的键盘服务，封装了回显、输入缓冲等功能，更适合用户交互。

AH 功能描述 参数 & 返回值
01H 带回显读字符 返回: AL=输入字符 (检测到 Ctrl-C 会中断)
08H 无回显读字符 返回: AL=输入字符 (适合密码输入)
0AH 读字符串 (带缓冲) 参数: DS:DX=缓冲区首地址
缓冲区格式: [最大长度] [实际长度] [数据区...]

代码示例: DOS 读字符串

9.2 显示器I/O

显示器分为 字符模式 (默认) 和图形模式， 中断主要支持字符模式的显示控制，核心是 "字符 + 属性" 的输出。

9.2.1 字符属性

字符模式下，每个字符对应 个 属性字节 (控制颜色、亮度)，结构为 位:

  • 颜色编码 (): =黑, =蓝, =绿, =青, =红, =紫, =棕, =白。
  • 示例: 黑色背景 () + 红色前景 () + 高亮 () 属性字节 00001100B (即 0CH)。

9.2.2 BIOS显示中断 (INT 10H)

通过 INT 10H 提供显示控制，支持模式设置、光标控制、字符输出等底层功能。

AH 功能描述 参数 返回值
00H 设置显示模式 AL=模式号 (03H=80x25 16色文本) 无
02H 设置光标位置 BH=页号 (默认0)，DH=行，DL=列 无
09H 写字符及属性 BH=页号，AL=字符，BL=属性，CX=次数 无 (光标不移动)
0EH TTY模式写字符 AL=字符，BL=前景色 (仅图形模式有效) 无 (光标自动移动/卷屏)

代码示例: BIOS 设置光标 + 写字符

9.2.3 DOS显示功能调用 (INT 21H)

通过 INT 21H 封装了简单的显示功能，无需关注属性/光标，更便捷。

AH 功能描述 参数
02H 显示单个字符 DL=字符 (光标跟随，检测 Ctrl-C)
09H 显示字符串 DS:DX=字符串首地址 (字符串必须以 $ 结尾)

代码示例: DOS 显示字符串

9.3 打印机I/O

打印机通过并行口 (如 ) 通信， 提供了字符打印的中断服务。

9.3.1 DOS打印功能 (INT 21H)

AH 功能描述 参数
05H 打印单个字符 DL=字符 (输出到 LPT1)

9.3.2 打印机的控制字符

打印时常用控制字符实现格式排版: - 0DH (回车): 打印头回到行首。 - 0AH (换行): 走纸一行。 - 0CH (换页): 走纸到新页开头。

9.3.3 BIOS打印功能 (INT 17H)

AH 功能描述 参数 返回值
00H 打印字符 AL=字符，DX=打印机号 (0=LPT1) AH=状态字节
01H 初始化打印机 DX=打印机号 AH=状态字节
02H 读取状态 DX=打印机号 AH=状态字节

9.4 串行通信口I/O

串行通信口 (如 ) 用于设备间异步通信 (如 Modem、单片机)，主要通过 中断控制。

9.4.1 串行通信接口

常用串行口为 标准，对应计算机的 (端口地址 )、 ()。 通信参数: 波特率、数据位、停止位、校验位 (需提前初始化)。

9.4.2 串行口功能调用 (INT 14H)

通过 INT 14H 提供串行口的初始化、发送、接收功能。

AH 功能描述 参数 返回值
00H 初始化串行口 AL=参数字节，DX=串口号 (0=COM1) AH=状态，AL=Modem状态
01H 发送字符 AL=字符，DX=串口号 AH bit 7 = 0 (成功) / 1 (失败)
02H 接收字符 DX=串口号 AH bit 7 = 0 (成功), AL=字符

初始化参数 (AL) 格式:

示例: 波特率、 数据位、 停止位、无校验 AL=11111000B (0F8H)。

本章核心总结

BIOS 中断 vs DOS 功能调用:

特性 BIOS 中断 DOS 功能调用 (INT 21H)
层级 接近硬件，依赖硬件配置 操作系统封装，与硬件无关
功能 底层控制 (如光标、串口初始化) 上层交互 (如带回显输入、串显示)
效率 高 (直接交互) 较低 (经过OS层)
适用 游戏开发、驱动编写、硬件控制 应用程序开发、文件操作、简单交互
