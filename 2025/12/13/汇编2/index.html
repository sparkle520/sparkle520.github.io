第2章 80x86计算机组织汇编笔记

2.1 80x86微处理器

  • 定义 : 80x86 是 \(Intel\) 推出的系列微处理器 ( 如 \(8086\)、\(80286\)、\(80386\)… )，是 \(x86\) 架构的基础，支持 \(16/32\) 位指令系统，是早期 \(PC\) 的核心运算部件.
  • 核心特点 :
    1. 向下兼容 : 新处理器支持旧型号的指令集；
    2. 分代演进 : 从 \(16\) 位 ( \(8086\) ) \(\to\) \(32\) 位 ( \(80386\) )，逐步扩展地址总线、寄存器位数.

2.2 基于微处理器的计算机系统构成

2.2.1 硬件

  • 核心组成 :

    1. 微处理器 : 运算与控制核心；
    2. 存储器 : 存放 数据 和 指令 ( 内存、缓存 )；
    3. 总线 : 连接各硬件的通信线路 ( 地址总线、数据总线、控制总线 )；
    4. 外部设备 : 输入 ( 键盘 )、输出 ( 显示器 )、存储 ( 硬盘 )…设备.

  • 关系 : 微处理器通过总线访问存储器和外设，实现数据/指令的传输与处理.

  • 总线详解 :

    • 地址总线 : 决定可 寻址空间大小，位宽为 \(n\) 则可寻址 \(2^n\) 个地址.
    • 数据总线 : 一次可 并行传输的位数，位宽为 \(m\) 则一次传输 \(m\) 位数据.
    • 控制总线 : 承载 读写控制、中断请求、总线仲裁… 控制信号.

  • 示例 : \(8086\) 的地址总线为 \(20\) 位，可寻址 \(1\) \(MB\) 空间；数据总线为 \(16\) 位，一次传输 \(16\) 位数据.

2.2.2 软件

  • 分类 :
    1. 系统软件 : 操作系统 ( 如 \(DOS\) )、汇编器、编译器…，管理硬件资源.
    2. 应用软件 : 基于系统软件运行的程序 ( 如 文本编辑器 ).
  • 与硬件的交互 : 软件的指令通过微处理器翻译成硬件可执行的电信号，完成运算或外设控制.

2.2.3 指令与伪指令对比

项目 定义 / 作用 处理者 是否生成机器码 示例 发生阶段 / 结果
伪指令 指导汇编器的编译行为 ( 数据定义、对齐、定位、结束 ) 汇编器 否 ( 本身无操作码 ，可能生成数据字节 ) \(DB\) / \(DW\) / \(ORG\) / \(END\) 编译阶段 : 影响目标文件布局或生成常量数据
真实指令 \(CPU\) 可执行的机器指令 ( 运算、传送、跳转、端口 \(I/O\) ) \(CPU\) 是 ( 生成操作码 + 操作数 ) \(MOV\) / \(ADD\) / \(JMP\) / \(IN\) / \(OUT\) 运行阶段 : 取指 \(→\) 译码 \(→\) 执行 ，更新寄存器 / \(FLAGS\)
汇编器处理伪指令 解析 / 展开伪指令 ，分配地址 ，生成常量数据 ，调整段位置 汇编器 伪指令不生成操作码 ，但可能生成数据字节 处理 \(DB\) 产生数据字节; \(ORG\) 调整当前位置; \(END\) 终止编译 编译阶段 : 形成可链接的目标文件
\(CPU\) 执行指令 按操作码与寻址方式执行 ，进行算术 / 逻辑 / 传送 / 控制 \(CPU\) 已生成 ( 指令码 ) 执行 \(MOV\) / \(ADD\) / \(JMP\) / \(IN\) / \(OUT\) 运行阶段 : 对数据与状态产生实际影响

2.3 中央处理机 ( CPU )

2.3.1 中央处理机CPU的组成

  • 核心模块 :
    1. 运算器 ( \(ALU\) ) : 执行 算术运算 ( 加/减 )、逻辑运算 ( 与/或 ).
    2. 控制器 : 解析指令、生成 控制信号，协调 硬件时序；
    3. 寄存器组 : 临时 存储 数据/地址/指令，提升运算效率 ( 80x86 的核心寄存器见 \(2.3.2\) ).
  • 典型数据通路 :
    • 取指 : \(CS:IP\) 指向 下一条指令，取到指令字节送入 指令队列.
    • 译码 : 控制器解析 操作码 与 寻址方式，产生执行所需 控制信号.
    • 执行 : \(ALU\) 完成算术/逻辑运算，结果写回寄存器或内存，对应标志位在 \(FLAGS\) 中更新.

2.3.2 80x86寄存器组

80x86 的寄存器按功能分为 \(4\) 类 ( 以 \(8086\) 为例，\(16\) 位寄存器 ).

  1. 通用寄存器 ( \(8\) 个 ).
    • 数据寄存器 : \(AX\) ( 累加器 )，\(BX\) ( 基址寄存器 )，\(CX\) ( 计数寄存器 )，\(DX\) ( 数据寄存器 ).
    • 指针/变址寄存器 : \(SP\) ( 栈指针 )，\(BP\) ( 基址指针 )，\(SI\) ( 源变址 )，\(DI\) ( 目的变址 ).
    • 典型寻址组合 : 有效地址 \(EA\) 可由 \(BX\)/\(BP\) 与 \(SI\)/\(DI\) 以及 位移 构成，如 \(EA = BX + SI + disp\) 或 \(EA = BP + DI + disp\).
  • 示例 : \(DS:BX + SI\) 指向 数据区，\(SS:BP + DI\) 指向 栈区.

注

\(AX\) : 累加器 ，用于 算术 / 逻辑运算 ; 下表总结乘除指令的 隐含寄存器 ; 串加载 \(LODSB\) / \(LODSW\) 使用 \(AL\) / \(AX\) .

指令 源位宽 输入 结果
\(MUL\) / \(IMUL\) \(8\) 位 \(AL \times\) 源 \(AX\)
\(MUL\) / \(IMUL\) \(16\) 位 \(AX \times\) 源 \(DX:AX\)
\(DIV\) / \(IDIV\) \(8\) 位 \(AX \div\) 源 商 \(AL\) ，余数 \(AH\)
\(DIV\) / \(IDIV\) \(16\) 位 \(DX:AX \div\) 源 商 \(AX\) ，余数 \(DX\)

\(CX\) : 计数寄存器 ，用于 循环 / 重复前缀 计数 ( \(LOOP\) / \(LOOPE\) / \(REP\) ); 移位 / 旋转 位数由 \(CL\) 指定.

\(DX\) : 数据寄存器 ，与 \(AX\) 组成 \(DX:AX\) 用于 乘除拓展; 端口号寄存器 ( 变址端口 \(IN\) / \(OUT\) 使用 \(DX\) 指定端口 ).

\(SP\) : 栈指针 ，指向 栈顶; 受 \(PUSH\) / \(POP\) 影响 ( \(SP = SP - 2\) / \(SP = SP + 2\) ).

\(AH\) / \(AL\) : \(AX\) 的 高 / 低字节; 常用于 \(DOS\) 中断 功能号 ( 如 \(INT 21H\) 的 \(AH\) ) 与 字符 \(I/O\) ( 使用 \(AL\) ).

  1. 段寄存器 ( \(4\) 个 ).
    • \(CS\) ( 代码段 )，\(DS\) ( 数据段 )，\(SS\) ( 栈段 )，\(ES\) ( 附加段 )，用于存储器 分段寻址.
  2. 控制寄存器 :
    • \(IP\) ( 指令指针 ) : 存放 下一条 要执行的 指令地址.
    • \(FLAGS\) ( 标志寄存器 ) : 记录 运算结果状态 ( 如 零标志 \(ZF\)，进位标志 \(CF\) ).

  • 标志位详解 (表格) :

    标志位 作用 典型置位示例 常见影响指令
    \(CF\) 无符号 进位/借位 \(8\) 位 \(255+2\) \(→\) \(CF=1\)；减法借位置 \(1\) \(ADD\)/\(ADC\)/\(SUB\)/\(SBB\)/\(SHL\)/\(SHR\)/\(RCL\)/\(RCR\)
    \(ZF\) 结果为 \(0\) \(5-5\) \(→\) \(ZF=1\)；比较相等 \(CMP\)/\(TEST\)/逻辑/移位，\(JZ\)/\(JNZ\)
    \(SF\) 结果符号 (最高位) 结果最高位为 \(1\) \(→\) \(SF=1\) 算术/逻辑，\(JS\)/\(JNS\)
    \(OF\) 有符号 溢出 \(127+1\) (\(8\) 位) \(→\) \(OF=1\) 算术/移位，\(JO\)/\(JNO\)
    \(PF\) 低 \(8\) 位 奇偶 (偶为 \(1\)) 低 \(8\) 位中 \(1\) 的个数为偶数 \(→\) \(PF=1\) 逻辑/\(TEST\)
    \(AF\) 低 \(4\) 位 进/借位 \(BCD\) 调整触发 \(DAA\)/\(DAS\)/\(AAA\)/\(AAS\)
    \(DF\) 串方向控制 \(STD\) \(→\) \(DF=1\) 自减；\(CLD\) \(→\) \(DF=0\) 自增 \(MOVS\)/\(CMPS\)/\(SCAS\)/\(LODS\)/\(STOS\)/\(REP\)
    \(IF\) 中断允许 \(STI\) \(→\) \(IF=1\)；\(CLI\) \(→\) \(IF=0\) 允许可屏蔽中断
    \(TF\) 单步陷阱 \(TF=1\) 每条指令后触发 \(INT\) \(1\) 调试/单步

2.4 存储器

2.4.1 存储单元的地址和内容

  • 存储单元 : 存储器的基本单位 ( \(8\) 位 \(= 1\) 字节 )，每个单元对应 唯一 的 地址 ( 由地址总线编码 ).
  • 地址与内容 :
    • 地址 : 标识存储单元的 编号 ( 如 \(8086\) 的地址总线为 \(20\) 位，可寻址 \(1\) \(MB\) 空间，地址范围 \(00000\mathrm{H} \sim FFFFF\mathrm{H}\) ).
    • 内容 : 存储单元中存放的 \(8\) 位 二进制 数据 ( 如 地址 \(1234H\) 的 内容 可能 是 \(5AH\) ).

2.4.2 实模式存储器寻址

  • 定义 : 80x86 的 \(16\) 位工作模式 ( 如 \(8086\) 默认模式 )，通过 段地址 + 偏移地址 组合寻址.
  • 寻址公式 : \[\text{物理地址} = \text{段地址} \times 16 + \text{偏移地址}\]
  • 示例 : 段地址 \(1000\mathrm{H}\) + 偏移地址 \(2000\mathrm{H}\) \(→\) 物理地址 = \(1000\mathrm{H} \times 16 + 2000\mathrm{H} = 12000\mathrm{H}\).
    • 特点 : 段地址由 段寄存器 提供，偏移地址由 通用寄存器 / \(IP\) 提供，最大寻址 \(1\) \(MB\).

注

可用于形成偏移地址的寄存器 : \(BX\) / \(SI\) / \(DI\) / \(BP\).

默认段寄存器 : \(BX\) / \(SI\) / \(DI\) \(→\) \(DS\); \(BP\) \(→\) \(SS\).

常见地址表达式 : [ \(BX\) ] , [ \(SI\) ] , [ \(DI\) ] , [ \(BP\) ] , [ \(BX + SI\) ] , [ \(BX + DI\) ] , [ \(BP + SI\) ] , [ \(BP + DI\) ] , [ \(disp\) ] , [ \(reg + disp\) ].

\(BX\) : 基址寄存器 , 默认搭配 \(DS\); 用于 数据区间接寻址 与 基址 + 变址组合 ( 如 [ \(BX\) ] , [ \(BX + SI\) ] , [ \(BX + DI\) ] , [ \(BX + disp\) ] ).

\(BP\) : 基址指针 , 默认搭配 \(SS\); 用于 栈帧 / 局部变量寻址 , 亦可 与变址寄存器组合 ( 如 [ \(BP\) ] , [ \(BP + SI\) ] , [ \(BP + DI\) ] , [ \(BP + disp\) ] ).

\(SI\) : 源变址 , 默认搭配 \(DS\); 串处理 指令的 源地址默认 \(DS:SI\) ( 如 \(MOVSB\) / \(LODSB\) ) , 亦可作一般间接寻址.

\(DI\) : 目的变址 , 一般间接寻址默认搭配 \(DS\); 串处理 指令的 目的地址默认 \(ES:DI\) ( 如 \(MOVSB\) / \(STOSB\) ).

\(IP\) : 指令指针 , 与 \(CS\) 组合给出 指令地址; 不用于一般数据寻址.

  • 段重叠与环绕 :
    • 多段可指向同一物理区域，如 \(1000\mathrm{H}:0200\mathrm{H}\) 与 \(0FF0\mathrm{H}:0300\mathrm{H}\) 的物理地址可能重叠.
    • \(A20\) 地址线 : 早期系统关闭 \(A20\) 时，超过 \(1\) \(MB\) 的地址按 \(1\) \(MB\) 取模环绕，即 \(\text{PA} = \text{PA} \bmod 2^{20}\).
    • 栈示例 : \(SS:SP\) 指向栈顶，\(PUSH\) 先将 \(SP\) 减 \(2\)，再写入数据; \(POP\) 先读出数据，再将 \(SP\) 加 \(2\).

2.4.3 保护模式存储器寻址

  • 定义 : \(80286\) 及以上处理器的 \(32\) 位工作模式，支持多任务、内存保护，寻址范围扩展至 \(4\) \(GB\).

  • 核心机制 :

    1. 段寄存器存 段选择子 ，指向 内存 中的 段描述符 ( 记录段的 基址 、长度 、权限 ).
    2. 物理地址 \(=\) 段描述符 中的 基址 \(+\) 偏移地址.

  • 特点 : 支持内存权限检查 ( 如只读段不可写 )，适合多任务操作系统 ( 如 \(Windows\) ).

  • 关键结构与字段 :

    • 段选择子 \(Selector\) : 由 索引 \(Index\) + 表标志 \(TI\) ( \(0\) 为 \(GDT\) , \(1\) 为 \(LDT\) ) + 请求者特权级 \(RPL\) 组成.
    • 全局描述符表 \(GDT\) / 局部描述符表 \(LDT\) : 存放 段描述符 \(Descriptor\).
    • 段描述符 : 包含 段基址 \(Base\) 、段界限 \(Limit\) 、访问权限 \(Access\) 与 标志位 \(Flags\) ( 如 \(DPL\) , \(Present\) , \(Granularity\) ).
    • 线性地址计算 : \(\text{Linear} = \text{Base} + \text{Offset}\) ，当启用 分页 ( \(Paging\) ) 时 , 线性地址 再经 页表 转换为 物理地址.
    • 特权级 \(Ring\) : \(0\) ( 内核 ) 至 \(3\) ( 用户 ) 四级，指令与数据访问需满足特权检查.
    • 使能保护模式 : 置位 控制寄存器 \(CR0\) 的 \(PE\) 位 为 \(1\) ，切换到 保护模式.

  • 示例 : \(CS\) 载入 选择子 指向 \(GDT\) 中 代码段描述符 ，从而确立 代码段 的 \(Base\) / \(Limit\) 与 权限.

2.4.4 存储器中数据的存放方式

  • 基本单位 : 字节 ( \(8\) 位 ) 、字 ( \(16\) 位 ) 、双字 ( \(32\) 位 ).
  • 字节序 : 80x86 采用 小端序 ( Little-Endian ) ，规则为 ” 低位字节存放在低地址 “.
    • 示例 ( 字 ) : 在地址 \(1000\mathrm{H}\) 开始存放 \(1234\mathrm{H}\) \(→\) [ \(1000\mathrm{H}\) ] = \(34\mathrm{H}\) ，[ \(1001\mathrm{H}\) ] = \(12\mathrm{H}\).
    • 示例 ( 双字 ) : 在地址 \(2000\mathrm{H}\) 开始存放 \(12345678\mathrm{H}\) \(→\) [ \(2000\mathrm{H}\) ] = \(78\mathrm{H}\) ，[ \(2001\mathrm{H}\) ] = \(56\mathrm{H}\) ，[ \(2002\mathrm{H}\) ] = \(34\mathrm{H}\) ，[ \(2003\mathrm{H}\) ] = \(12\mathrm{H}\).
  • 栈的存放方向 : 栈向 低地址 增长 ( \(PUSH\) 时 \(SP=SP-2\) ，\(POP\) 时 \(SP=SP+2\) ) ，栈内数据按 小端序 存放.
    • 示例 : \(PUSH\) \(1234\mathrm{H}\) 后 \(→\) [ \(SS:SP\) ] = \(34\mathrm{H}\) ，[ \(SS:SP+1\) ] = \(12\mathrm{H}\).
  • 字符与字符串 : 字符以 \(ASCII\) 按字节顺序存放; 字符串在内存中 连续 存放 , 常见以 \(00\mathrm{H}\) 或 $ 作为结束标记 ( 与 \(DOS\) 功能调用相关 ).
  • 数据对齐 : 在 \(8086\) 上 , 字 / 双字 不强制对齐; 较新处理器在 按类型对齐 时通常具有更好的性能.
  • 结构与数组 : 数据按 声明顺序 连续存放; 数组元素相邻存放.
    • 示例 : 在地址 \(3000\mathrm{H}\) 开始存放字节数组 \(11\mathrm{H}\) , \(22\mathrm{H}\) , \(33\mathrm{H}\) \(→\) [ \(3000\mathrm{H}\) ] = \(11\mathrm{H}\) ，[ \(3001\mathrm{H}\) ] = \(22\mathrm{H}\) ，[ \(3002\mathrm{H}\) ] = \(33\mathrm{H}\).
  • 说明 : 逻辑地址可写作 段 : 偏移 ( 如 \(DS:BX\) ); 对应 物理地址 计算为 段地址 × \(16\) + 偏移地址.

2.5 外部设备

  • 定义 : 连接在计算机外部的硬件，分为输入设备 ( 键盘、鼠标 )、输出设备 ( 显示器、打印机 )、存储设备 ( 硬盘、U盘 )….

  • 与 \(CPU\) 的交互 : 通过 接口电路 ( 如 \(I/O\) 端口 ) 与 \(CPU\) 通信，\(CPU\) 通过读写 \(I/O\) 端口的指令 ( 如 \(IN/OUT\) ) 控制外设.

  • \(I/O\) 端口与指令 :

    • 端口寻址 : \(8\) 位或 \(16\) 位端口号，常用指令 \(IN\) / \(OUT\) 访问.
    • 字符串 \(I/O\) : \(INS\) / \(OUTS\) 支持批量端口读写，配合 \(DX\) 指定端口，\(DI\) / \(SI\) 指向缓冲区，\(CX\) 为计数.
    • 中断与外设 : 外设通过中断控制器请求 \(CPU\) ，常见 \(PIC\) / \(APIC\) 结构用于管理中断.
