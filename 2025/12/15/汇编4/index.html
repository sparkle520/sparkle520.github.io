第4章 汇编语言程序格式

4.1 汇编程序功能

汇编程序 ( 如 \(MASM\) ) 是将 汇编语言源程序 ( \(.ASM\) ) 翻译成 机器语言 目标程序 ( \(.OBJ\) ) 的工具, 核心功能 :

  1. 解析 汇编指令 ( 翻译为机器码 )、伪操作 ( 指导汇编过程 )；
  2. 检查 语法错误；
  3. 分配 内存地址 ( 确定变量和指令的偏移量 )；
  4. 生成可被链接器处理的目标文件.

4.2 伪操作

伪操作 ( 伪指令 ) 是汇编程序识别的 “辅助指令”, 不对应机器指令, 仅在汇编阶段起作用 ( 如 定义段, 分配数据, 定义过程 ).

4.2.4 程序开始和结束伪操作

1. 模块定义 ( 可选 )

  • NAME : 定义模块名称 ( 若省略, 默认用文件名 ).
  • TITLE : 指定列表文件的标题 ( 打印时显示 ).

2. 程序入口与起始

  • ORG : 设置 位置计数器 的值 ( 指定下一条指令/数据的偏移地址 ).
  • 标号 : 通常用 START: 标记程序执行的第一条指令.

3. 程序结束

  • END [ 入口标号 ] : 标记 源文件结束, 汇编器停止处理后续内容.
    • 若带有标号 ( 如 END START ), 则指定程序的 入口地址 ( CS:IP 指向此处 ).
    • 若是模块链接, 仅主程序需要指定入口标号.

伪操作 含义 示例 ( 定义后分配的内存 )
\(DB\) 定义 字节 ( \(8\) 位 ) \(A\) \(DB\) \(10\mathrm{H}\), \('A'\) \(\to\) 内存存 \(10H, 41H\)
\(DW\) 定义 字 ( \(16\) 位 ) \(B DW\) \(1234\mathrm{H}\) \(\to\) 内存存 \(34H 12H\) ( 小端 )
\(DD\) 定义 双字 ( \(32\) 位 ) \(C DD\) \(12345678\mathrm{H}\) \(\to\) 内存存 \(78H 56H 34H 12H\)
\(DQ\) 定义 四字 ( \(64\) 位 ) \(Q DQ\) \(1234...5678\mathrm{H}\) \(\to\) 8 字节小端存储
\(DT\) 定义 十字节 ( \(80\) 位 ) \(T DT\) \(1234...5678\mathrm{H}\) \(\to\) 10 字节小端存储 ( 常用于 BCD )
\(DF\) 定义 6 字节远指针 ( \(48\) 位 ) \(F DF\) \(0\) \(\to\) 6 字节 ( 32 位偏移 + 16 位段, 386+ )

4.2.7 地址计数器与对准伪操作

  • 地址计数器 $
    • 含义 : 保存当前正在汇编的指令或数据的 偏移地址.
    • 常见用法 :
      1. 计算长度 : 利用当前地址减去起始地址.
      2. 死循环 : 跳转到当前指令地址.
  • ORG 伪操作
    • 格式 : ORG constant_expression
    • 作用 : 将 地址计数器 ($) 的值设置为指定数值. 后续指令或变量将从该地址开始存放.
    • 常见用法 :
      1. COM 文件 : 程序必须从偏移地址 100H 开始.
      2. 指定偏移 : 强制将数据放置在特定偏移处.
  • 对准伪操作
    • 作用 : 调整当前的地址计数器, 使其对齐到指定的边界, 以提升 16位/32位 CPU 的内存访问效率.
    • 指令 :
      • EVEN : 使下一个变量/指令的地址位于 偶地址.
        • 若当前为偶地址 : 不作处理.
        • 若当前为奇地址 : 插入一个字节 ( 00H 或 NOP ) 使其变为偶数.
      • ALIGN n : 使下一个地址位于 n 的倍数 ( n 必须是 2 的幂 ).

4.3 汇编语言程序格式

汇编语句的基本格式 : [名字项] [操作项] [操作数项] [; 注释项]

4.3.1 名字项

  • 含义 : 标号 ( 指令地址别名 ) 或 变量名 ( 数据地址别名 ).
  • 规则 : 字母/数字/下划线, 不能以数字开头, 标号后常跟冒号 :.
  • 示例 : START:, DATA_VAL

4.3.2 操作项

  • 含义 : 汇编指令 ( \(MOV\), \(ADD\) ) 或 伪操作 ( \(DB\), \(SEGMENT\) ).
  • 对应 : 机器码 ( 指令 ) 或 汇编动作 ( 伪操作 ).

4.3.3 操作数项

操作数项不仅可以是 立即数 / 寄存器 / 内存, 还可以由 操作符 构成的 表达式 ( 在汇编阶段计算出结果 ).

1. 算术操作符

  • 符号 : + ( 加 ), - ( 减 ), * ( 乘 ), / ( 除 ), MOD ( 取模 ).
  • 作用 : 在汇编阶段对 常量 或 地址 进行运算.
  • 示例 :

2. 逻辑与移位操作符

  • 符号 : AND, OR, XOR, NOT ( 逻辑 )；SHL, SHR ( 移位 ).
  • 作用 : 对常量进行位运算.
  • 示例 :

3. 关系操作符

  • 符号 : EQ (=), NE (!=), GT (>), LT (<), GE (>=), LE (<=).
  • 作用 : 比较两个操作数, 真返回 FFFFH ( -1 ), 假返回 0.
  • 示例 :

4. 数值回送操作符

  • 符号 :
    • OFFSET : 返回变量/标号的 偏移地址.
    • SEG : 返回变量/标号的 段基址.
    • TYPE : 返回变量/标号的 类型值 ( Byte=1, Word=2, Dword=4, Near=-1, Far=-2 ).
    • LENGTH : 返回变量定义时 第一层 DUP 的重复次数.
      • 若未使用 DUP, 则返回 1.
      • 若使用嵌套 DUP, 仅返回最外层次数.
    • SIZE : 返回变量占用的 总字节数 ( = LENGTH × TYPE ).
      • 注意: 若存在嵌套 DUP, SIZE 可能小于实际分配的内存空间.
  • 示例 :

5. 属性操作符

  • 符号 :
    • PTR : 临时改变操作数的类型属性.
      • 格式: type PTR expression
      • 常用类型: BYTE, WORD, DWORD, NEAR, FAR.
    • SHORT : 强制短跳转 ( -128 ~ +127 字节范围内 ).
    • THIS : 创建一个与当前位置地址相同, 但类型不同的标号/变量.
    • HIGH / LOW : 取字常量或地址的高字节/低字节.
  • 示例 :

4.3.4 注释项

  • 格式 : 以 ; 开头.
  • 作用 : 解释代码, 汇编时被忽略.

4.4 DOS 系统功能调用 ( INT 21H )

在 16 位汇编中, 常通过 软中断 调用 DOS 系统功能 ( 如 屏幕输出, 键盘输入, 返回 DOS ).

调用步骤 : 1. \(AH\) \(\leftarrow\) 功能号 2. 其他寄存器 \(\leftarrow\) 入口参数 ( 如 \(DX\) 存放字符串偏移 ) 3. \(INT\) \(21\mathrm{H}\) 4. 分析出口参数 ( 如 \(AL\) 存放输入字符 )

常用功能 :

功能号 (\(AH\)) 功能描述 入口参数 出口参数
\(01\mathrm{H}\) 键盘输入 (带回显) 无 \(AL\) = 字符 \(ASCII\)
\(02\mathrm{H}\) 字符显示 \(DL\) = 字符 \(ASCII\) 无
\(09\mathrm{H}\) 字符串显示 \(DS:DX\) = 串地址 ( 串以 $ 结束 ) 无
\(4\mathrm{C}\mathrm{H}\) 返回 DOS \(AL\) = 返回码 ( 通常 \(0\) ) 无

4.5.3 上机流程

  1. 编辑 : 编写 TEST.ASM.
  2. 汇编 : MASM TEST.ASM; \(\to\) 生成 TEST.OBJ.

  5. 调试 : DEBUG TEST.EXE.
