第1章 计算机系统概述

1.1 计算机发展历程

1.1.1 计算机硬件的发展

  1. 四代变化 :
    • 第一代 ( 1946-1957 ) : 电子管 计算机. ( 机器语言, 汇编语言 ).
    • 第二代 ( 1958-1964 ) : 晶体管 计算机. ( 高级语言 FORTRAN, 操作系统雏形 ).
    • 第三代 ( 1965-1971 ) : 中小规模集成电路 计算机. ( 分时操作系统 ).
    • 第四代 ( 1972-现在 ) : 超大规模集成电路 计算机. ( 微处理器, 并行, 分布式 ).
  2. 微型计算机的发展 : 以 微处理器 ( CPU ) 技术为标志.
    • 机器字长 : 8位 -> 16位 -> 32位 -> 64位.
  3. 摩尔定律 ( Moore's Law ) :
    • 集成电路上可容纳的晶体管数目，约每 18个月 增加一倍，性能也将提升一倍.

1.1.2 计算机软件的发展

  1. 编程语言 :
    • 机器语言 ( 0/1 代码 ) -> 汇编语言 ( 助记符 ) -> 高级语言 ( C, Java, Python ).
  2. 系统软件 :
    • 从无到有，发展出 DOS, Windows, UNIX, Linux 等操作系统.

1.2 计算机系统层次结构

1.2.1 计算机系统的组成

  • 计算机系统 = 硬件系统 + 软件系统.
    • 硬件 : 实体部件 ( 主机, 外设 ).
    • 软件 : 程序 + 数据 + 文档.
  • 软硬件关系 :
    • 逻辑功能等效性 : 某些功能既可以用硬件实现 ( 快, 成本高 ) ，也可以用软件实现 ( 慢, 灵活 ).

1.2.2 计算机硬件

1. 冯·诺依曼结构 ( Von Neumann Architecture )

  1. 五大部件 : 运算器、控制器、存储器、输入设备、输出设备.
  2. 指令和数据 以同等地位存储在存储器中，可按地址寻址.
  3. 指令和数据 均用 二进制 代码表示.
  4. 指令 由 操作码 ( 做什么 ) 和 地址码 ( 在哪做 ) 组成.
  5. 存储程序 : 程序预先存入内存，自动顺序执行 ( 核心思想 ).
  6. 以运算器为中心. ( 现代计算机 以存储器为中心 ).

2. 计算机的功能部件

  1. 输入/输出设备 ( I/O ) : 键盘, 鼠标, 显示器, 硬盘等.
  2. 存储器 :
    • 主存 ( 内存 ) : 存放正在运行的程序和数据.
      • MAR ( 存储器地址寄存器 ) : 位数反映存储单元个数 ( 寻址范围 ).
      • MDR ( 存储器数据寄存器 ) : 位数反映存储字长.
    • 辅存 ( 外存 ) : 永久存储.
  3. 运算器 ( ALU ) : 完成算术运算 ( +-*/ ) 和逻辑运算 ( 与或非 ).
    • 核心 : ALU ( 算术逻辑单元 ).
    • 寄存器 : ACC ( 累加器 ), MQ ( 乘商寄存器 ), X ( 操作数寄存器 ).
  4. 控制器 ( CU ) : 指挥各部件协调工作.
    • PC ( 程序计数器 ) : 存放 下一条 指令的地址.
    • IR ( 指令寄存器 ) : 存放 当前 正在执行的指令.
    • CU ( 控制单元 ) : 分析指令，发出控制信号.

3. 知识拓展：相联存储器 ( CAM )

  • 定义 : 相联存储器 ( Content Addressable Memory ) 是一种 按内容寻址 的存储器.
  • 区别 :
    • RAM ( 随机存取 ) : 给定 地址 读出 数据.
    • CAM ( 相联存取 ) : 给定 数据 ( 关键字 ) 返回 地址 或 命中标志.
  • 特点 : 内部采用 并行比较电路，所有存储单元同时与关键字比较，查找速度极快 ( ) ，但硬件复杂、成本高.
  • 应用 : 主要用于 TLB ( 快表 ) 和 Cache ( 高速缓存 ) 的全相联/组相联映射.

1.2.3 计算机软件

  1. 系统软件 : 保证系统运行，管理资源. ( 如 OS, 数据库管理系统, 编译程序, 连接程序 ).
  2. 应用软件 : 解决特定问题. ( 如 Office, 游戏 ).
  3. 翻译程序 :
    • 汇编程序 : 汇编语言 -> 机器语言.
    • 解释程序 : 高级语言 -> 逐条翻译执行 ( 不生成目标程序 ).
    • 编译程序 : 高级语言 -> 目标程序 ( 一次性翻译 ).

1.2.4 计算机系统的层次结构

从下到上 ( 虚线格以上为软件，以下为硬件 ) :

  1. 微程序机器 ( M0 ) : 硬件层，执行微指令.
  2. 传统机器 ( M1 ) : 机器语言机器，执行机器指令.
  3. 操作系统机器 ( M2 ) : 虚拟机器，向上提供广义指令 ( 系统调用 ).
  4. 汇编语言机器 ( M3 ) : 虚拟机器，执行汇编语言.
  5. 高级语言机器 ( M4 ) : 虚拟机器，执行高级语言.

1.2.5 计算机系统的工作原理

1. "存储程序" 概念

  • 核心思想 : 将指令以二进制代码的形式事先输入计算机的主存储器，然后由计算机自动地、连续地从主存储器中取出指令并执行.
  • 执行流程 :
    1. 程序装入 : 将程序和数据装入主存.
    2. 启动运行 : 将程序首地址送入 PC ( 程序计数器 ).
    3. 自动执行 : CPU 依据 PC 指示，自动取出指令、分析指令、执行指令，直到程序结束.

2. 指令执行过程 ( 冯·诺依曼机 )

一条指令的执行通常包含三个阶段 :

  1. 取指令 ( Fetch ) :
    • 根据 PC 中的地址，从内存中取出指令送入 IR ( 指令寄存器 ).
    • ( 指向下一条指令 ).
  2. 分析指令 ( Decode ) :
    • 指令译码器 ( ID ) 对 IR 中的操作码进行分析，识别是什么操作.
    • 形成操作数的有效地址.
  3. 执行指令 ( Execute ) :
    • 根据分析结果，取出操作数，通过 ALU 进行运算，并将结果写回寄存器或内存.

3. 典型指令执行举例

以 取数指令 LOAD ACC, [X] 为例 ( 将内存地址 X 中的数据取至累加器 ACC ) :

  1. 取指阶段 :
    • PC -> MAR : 将指令地址送入地址寄存器.
    • M(MAR) -> MDR : 存储器读出指令送入数据寄存器.
    • MDR -> IR : 将指令送入指令寄存器.
    • OP(IR) -> CU : 操作码送控制单元译码.
    • (PC) + 1 -> PC : 程序计数器加 1.
  2. 执行阶段 ( 取数 ) :
    • Ad(IR) -> MAR : 将指令中的地址码 ( X ) 送入地址寄存器.
    • M(MAR) -> MDR : 存储器读出数据 ( X的内容 ) 送入数据寄存器.
    • MDR -> ACC : 将数据送入累加器.

1.2.6 计算机系统的层次结构

计算机系统通常被划分为 五个层次，每一层都建立在下一层的基础上.

  1. 五层层次结构 :
    • L1 微程序机器 ( Microprogram Machine, M0 ) :
      • 硬件层.
      • 直接执行 微指令.
      • 是实际的物理机器.
    • L2 传统机器 ( Traditional Machine, M1 ) :
      • 机器语言机器.
      • 执行 机器指令 ( 二进制代码 ).
      • 通常是 软硬件的分界面 ( ISA ).
      • 微指令是对机器指令的解释和执行.
    • L3 操作系统机器 ( Operating System Machine, M2 ) :
      • 虚拟机器.
      • 由 操作系统 程序实现.
      • 向上提供 广义指令 ( 系统调用 ).
      • 管理系统资源.
    • L4 汇编语言机器 ( Assembly Language Machine, M3 ) :
      • 虚拟机器.
      • 由 汇编程序 支持.
      • 执行 汇编指令 ( 助记符 ).
    • L5 高级语言机器 ( High-level Language Machine, M4 ) :
      • 虚拟机器.
      • 由 编译程序 或 解释程序 支持.
      • 执行 高级语言 ( C, Java 等 ).
  2. 层次结构图 :

  1. 软件和硬件的逻辑等效性 :
    • 原理 : 任何由硬件实现的操作，理论上都可以由软件模拟实现；任何由软件实现的操作，理论上都可以由硬件直接实现.
    • 区别 :
      • 硬件实现 : 速度快，成本高. ( 频繁使用的功能，如乘除法 ).
      • 软件实现 : 速度慢，成本低，灵活性好. ( 复杂且不常用的功能 ).

1.3 计算机的性能指标

1.3.1 计算机的主要性能指标

  1. 机器字长 : CPU 一次能处理的二进制数据的位数 ( 通常等于 CPU 内部寄存器的位数 ).
    • 影响精度和范围.
  2. 数据通路带宽 : 数据总线一次能并行传送信息的位数.
  3. 主存容量 : 存储单元个数 × 存储字长.
  4. 运算速度 :
    • 吞吐量 : 单位时间内处理的请求数量.
    • 响应时间 : 从提交请求到获得结果的时间.
    • 主频 ( ) : CPU 时钟频率. ( Hz ).
    • 时钟周期 ( ) : ( s ).
    • CPI ( Cycles Per Instruction ) : 执行一条指令所需的时钟周期数.
    • IPS ( Instructions Per Second ) : 每秒执行指令数. .
    • MIPS : 每秒执行百万条指令. .
    • FLOPS : 每秒浮点运算次数 ( Floating-point Operations Per Second ).
      • MFLOPS ( Mega ) :
      • GFLOPS ( Giga ) :
      • TFLOPS ( Tera ) :
      • PFLOPS ( Peta ) :
      • EFLOPS ( Exa ) : ( 也是目前超算竞争的焦点 )
      • ZFLOPS ( Zetta ) : ( 未来目标 )

1.3.2 几个专业术语

  • 系列机 : 具有相同体系结构，但组成和实现不同的一系列计算机.
  • 兼容 : 软件或硬件的通用性. ( 主要是软件兼容，向后兼容 ).
  • 固件 :
    • 将程序固化在 ROM ( 只读存储器 ) 中.
    • 特性 : 具有软件的功能，但以硬件的形式存在 ( 速度快，不易修改 ).
    • 例子 : BIOS ( 基本输入输出系统 ).

1.4 本章小结

  1. 冯·诺依曼结构 是现代计算机的基础，核心是 存储程序.
  2. 计算机由 硬件 和 软件 组成，两者在逻辑功能上是等效的.
  3. 性能指标中，CPU 执行时间 是最综合的考量，取决于指令条数、CPI 和主频.

1.5 常见问题和易混淆知识点

  1. 主频高不一定速度快 : 还要看 CPI 和指令系统.
  2. 翻译 vs 解释 :
    • 翻译 ( Compile ) : 生成目标代码，执行速度快.
    • 解释 ( Interpret ) : 不生成目标代码，逐条解释，便于交互调试，慢.
  3. 机器字长 vs 存储字长 :
    • 机器字长 : CPU 处理能力 ( 寄存器位数 ).
    • 存储字长 : 存储器一个地址存放的位数 ( MDR 位数 ).
    • 两者可以不相等.
