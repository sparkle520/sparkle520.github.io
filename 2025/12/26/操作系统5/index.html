第5章 输入/输出管理

5.1 I/O 管理概述

5.1.1 I/O 设备

  1. 分类 :
    • 按传输速率分类 :
      • 低速设备 : 键盘、鼠标.
      • 中速设备 : 激光打印机.
      • 高速设备 : 磁盘机、光盘机.
    • 按信息交换单位分类 :
      • 块设备 : 以数据块为单位组织和传送数据.有结构，可寻址 ( 如磁盘 ).
      • 字符设备 : 以字符为单位传送数据. 无结构，不可寻址 ( 如键盘、打印机 ).
    • 按共享属性分类 :
      • 独占设备 : 一次只能被一个进程使用 ( 如打印机 ).
      • 共享设备 : 多进程可并发共享 ( 如磁盘 ).
      • 虚拟设备 : 通过 SPOOLing 技术将独占设备在逻辑上改造成可共享设备, 多个进程的请求进入输出井排队, 由守护进程按序送往实体设备.
    • 按用途分类 :
      • 人机交互设备 : 键盘、鼠标、显示器.
      • 存储设备 : 磁盘、SSD、磁带.
      • 通信设备 : 网卡、串口、USB.
    • 按工作方式 :
      • 同步/异步 : 是否与主机时序严格一致.
      • 半双工/全双工 : 传输方向是否同时进行.
  2. I/O 接口 ( 设备控制器 ) :
    • CPU 与设备之间的接口.
    • 功能 : 接收和识别 CPU 命令、数据交换、标识和报告设备状态、地址识别.
    • 组成 : 设备控制器与 CPU 的接口 ( 数据/状态/控制寄存器 )、设备控制器与设备的接口、I/O 逻辑.
    • 寄存器常见划分 :
      • 数据寄存器 : 负责主机与设备之间的数据读写.
      • 状态寄存器 : 反映设备当前状态 ( 如 BUSY/READY/DRQ/ERR 等标志位 ).
      • 控制寄存器 : 控制启动/复位/中断允许/模式设置等.
  3. I/O 接口的类型 :
    • 端口映射 I/O ( Port‑Mapped I/O, PMIO ) : 设备寄存器位于独立的 I/O 地址空间, 通过 IN/OUT 等专用指令访问.
      • 优点 : 与内存地址空间隔离; 硬件设计简单.
      • 限制 : 指令集相关; 编程不如内存读写直观.
    • 内存映射 I/O ( Memory‑Mapped I/O, MMIO ) : 将设备寄存器映射到物理地址空间, 使用普通 load/store 指令访问.
      • 优点 : 统一寻址; 易与缓存/虚拟内存协作 ( 需设置非缓存/强制序列 ).
      • 注意 : 需遵循内存屏障与访问顺序, 防止乱序导致状态错误.
    • 总线/接口标准 : PCI/PCIe, USB, SATA, NVMe 等, 决定带宽、寻址与热插拔能力.
    • 中断接口 : 线中断, MSI/MSI‑X ( 消息信号中断 ) 提供更高并发与更少抖动.
    • DMA 接口 : 设备作为总线主设备直接访问内存, 通过描述符/队列提交传输任务, 降低 CPU 参与度.

PMIO vs MMIO 对比

项目 端口映射 I/O ( PMIO ) 内存映射 I/O ( MMIO )
地址空间 独立 I/O 空间 物理/虚拟内存空间
访问指令 专用 IN/OUT 普通 load/store
编程模型 相对分离 统一读写更直观
缓存参与 通常不缓存 可配置非缓存/序列化
典型平台 x86 传统设备 现代设备/PCIe 设备

独立编址 vs 统一编址

  • 独立编址 : 将设备寄存器放在单独的 I/O 地址空间, 通过专用端口指令访问. 对应 PMIO; 优点是与内存地址隔离、解码简单; 缺点是指令受限、难以与通用内存模型融合.
  • 统一编址 : 将设备寄存器映射到内存地址空间, 通过普通读写访问. 对应 MMIO; 优点是编程统一、可配合虚拟内存与分页; 注意需设置非缓存/顺序访问并使用内存屏障保证时序.
  • 常见结合 : x86 同时支持两者; 现代高性能设备 ( PCIe/NVMe/高端网卡 ) 多采用 MMIO + MSI/MSI‑X + DMA 的组合以提升并发与吞吐.

块设备 vs 字符设备 对比

特性 块设备 字符设备
组织单位 块/扇区/页 字符/字节流
可寻址 是 ( 随机访问 ) 否 ( 顺序访问 )
缓冲 强依赖缓存/预读/写回 常见简单缓冲或直通
接口调用 read/write/seek get/put
典型设备 磁盘/SSD/光盘 键盘/鼠标/打印机/串口

性能指标速览

  • 带宽/吞吐量 : 单位时间内的数据传输量.
  • 响应时间 : 从发出请求到开始服务的时间.
  • 访问延迟 : 设备内部定位/准备时间 ( 磁盘的寻道+旋转延迟 ).
  • 利用率 : 设备忙碌时间占总时间的比例.

5.1.2 I/O 控制方式

  1. 程序直接控制方式 :
    • CPU 循环检查设备状态 ( 轮询 ).
    • 缺点 : CPU 和 I/O 设备只能串行工作，CPU 利用率低.
    • 数据通路 : 设备 → 控制器数据寄存器 → CPU → 内存.
    • 适用场景 : 简单外设、低速 I/O、一次性小量传输.
    • 特点 : 编程简单; 轮询开销大; 不能并行计算与 I/O.
  2. 中断驱动方式 :
    • 设备准备好数据后向 CPU 发中断信号.
    • 优点 : CPU 和 I/O 设备可并行工作.
    • 缺点 : 每个字/字节传输都要中断 CPU，中断处理消耗 CPU 时间.
    • 数据通路 : 设备 → 触发中断 → CPU 进入中断服务程序 (ISR) → 从数据寄存器读取/写入 → 内存.
    • 适用场景 : 字符/字节级设备 ( 键盘、串口、打印机 )、事件驱动.
    • 特点 : 并行度提升; 中断频繁带来开销; 需正确保存/恢复现场.
  3. DMA ( Direct Memory Access, 直接存储器存取 ) 方式 :
    • 数据传送单位是 块.
    • 数据流向 : 设备 <-> 内存 ( 不经过 CPU ).
    • 仅在传送一个或多个数据块的开始和结束时才需 CPU 干预.
    • 优点 : 数据传输效率高.
    • 数据通路 : CPU 配置 DMA 描述符/缓冲区 → DMAC 发起总线主控 → 设备 ↔︎ 内存 → 完成后中断通知.
    • 适用场景 : 磁盘/SSD、网卡、显存等批量数据传输.
    • 注意 : 缓存一致性与对齐; 写回策略; 非缓存/内存屏障配置.
  4. 通道控制方式 :
    • 通道是 专门负责输入/输出的处理机.
    • CPU 发出 I/O 指令，通道执行通道程序完成 I/O.
    • 优点 : CPU 干预更少，支持多台设备并行操作.
    • 数据通路 : CPU 下发通道程序 → 通道调度设备与内存传输 → 完成后汇报状态.
    • 适用场景 : 大型机/高端存储系统; 多设备并发与复杂 I/O 序列.
    • 特点 : 扩展性强; 通道程序复杂; 硬件成本高.

5.1.3 I/O 软件层次结构

从上到下依次为 :

  1. 用户层软件 : 发出 I/O 请求 ( 库函数 ).
  2. 设备独立性软件 : 实现通用 I/O 功能 ( 逻辑设备到物理设备的映射、缓冲、分配、保护 ).
  3. 设备驱动程序 : 将上层命令转换为具体设备的硬件指令.
  4. 中断处理程序 : 处理硬件中断.
  5. 硬件 : 执行 I/O 操作.

层次职责与示例

层次 主要职责 典型内容/例子
用户层软件 发起请求、处理结果 标准库 I/O、缓冲读写、阻塞/非阻塞设置
设备独立性软件 逻辑设备映射、缓冲/缓存、设备分配与保护 LUT、缓冲池、缓存策略、权限与排他控制、SPOOLing
设备驱动程序 适配具体硬件指令/协议 init/open/close/read/write/ioctl、队列、DMA、块/字符接口
中断处理程序 响应设备事件、完成数据传送、唤醒进程 识别中断源、读写寄存器、更新状态、发送 EOI、完成通知
硬件 执行物理 I/O 控制器寄存器、总线、设备介质

设备独立性软件的典型任务

  • 逻辑设备名 → 物理设备名的映射 ( LUT ).
  • 缓冲/缓存策略 ( 单/双/循环缓冲, 写回/直写 ).
  • 设备分配与保护 ( DCT/SDT/CHCT/COCT ), 排他与共享策略.
  • SPOOLing 虚拟设备化：将独占设备逻辑改造为共享设备.

驱动程序要点

  • 字符设备接口与块设备接口的差异：是否可寻址、是否有请求队列与调度 ( 块层 ).
  • 常见操作集：open/close/read/write/ioctl/mmap/fsync；队列与 DMA 配置；错误处理与重试策略.

中断处理程序要点

  • 识别中断来源与通道；读取/写入设备寄存器；更新状态与统计.
  • 唤醒阻塞的进程/完成队列；必要时发送 EOI；支持 MSI/MSI‑X/线中断.

易错点

  • 将设备驱动与设备独立性软件职责混淆；忘记缓冲与权限控制在独立性层完成.
  • 忽略缓存一致性与内存屏障导致数据错乱；对块设备读写缺少队列与调度支持.

5.1.4 应用程序 I/O 接口

  1. 字符设备接口 : get / put ( 或 read / write ).
    • 流式读写, 不可随机寻址; 常见键盘/串口/打印机等.
    • 典型特性 : 行/原始模式, 逐字节处理, 简单缓冲或直通.
  2. 块设备接口 : read / write / seek.
    • 支持随机寻址 ( seek ), 对齐与块大小相关; 常见磁盘/SSD.
    • 典型特性 : 页缓存/预读/写回, 可配直接 I/O 绕过缓存.
  3. 网络设备接口 : Socket (send / recv / connect / bind / listen / accept).
    • 面向连接 ( TCP ) 或无连接 ( UDP ), 配合缓冲队列与协议栈.
    • 典型特性 : 非阻塞与 I/O 复用 ( select / poll 等 ).
  4. 阻塞/非阻塞 I/O :
    • 阻塞 : 调用在数据就绪或操作完成后返回, 进程挂起等待.
    • 非阻塞 : 调用立即返回, 未就绪时返回错误码 ( 如 EAGAIN/EWOULDBLOCK ), 结合就绪通知与定时超时.

接口类型与特征

  • 字符设备 : 流式读写, 不可随机寻址, 常用于键盘/串口/打印机.
  • 块设备 : 支持随机访问与定位, 常配合缓冲与缓存策略, 用于磁盘/SSD.
  • 网络设备 : 面向连接或无连接的套接字读写, 涉及协议栈与缓冲队列.

同步/异步、缓冲/非缓冲

  • 同步 I/O : 系统调用在数据就绪后返回, 常与阻塞模式结合.
  • 异步 I/O : 提交请求后立即返回, 完成后事件/回调/队列通知应用.
  • 缓冲 I/O : 通过用户态/内核态缓冲区减少系统调用频次, 提升吞吐.
  • 非缓冲 I/O : 直接数据传送, 常用于对实时性或一致性要求较高场景.

I/O 复用

  • 多路复用 : 通过 select / poll / 高级接口监视多个描述符的可读/可写/异常状态, 单线程处理多连接.
  • 事件驱动 : 将就绪事件分发到处理逻辑, 配合非阻塞 I/O 构建高并发服务.

常见接口扩展

  • readv / writev : 散/聚 I/O, 一次调用读写多个缓冲区.
  • mmap : 将文件映射到进程地址空间, 通过内存访问实现 I/O, 利于随机访问与共享.
  • 直接 I/O : 绕过页缓存直接与设备交换, 降低双重缓存带来的拷贝与一致性问题.
  • 异步 I/O (AIO) : 接口提交后由系统异步完成, 通过完成事件获取结果.

接口对比

维度 字符设备 块设备 网络设备
寻址 不可寻址 可随机寻址 面向连接/报文
典型操作 get/put read/write/seek send/recv
缓冲 简单/直通 强缓存/预读/写回 套接字缓冲/队列
适用场景 输入输出外设 磁盘/SSD/光盘 客户端/服务端通信

5.2 设备独立性软件

5.2.1 设备独立性软件 ( Device-Independent I/O Software )

  1. 概念 :
    • 又称 设备无关性软件.
    • 核心目标 : 使得应用程序独立于具体的物理设备. 应用程序使用 逻辑设备名 请求 I/O，而无需关心物理设备的具体细节 ( 如制造商、型号、端口地址 ).
    • 好处 : 增加了设备分配的灵活性，易于实现 I/O 重定向 ( 如 Linux 中的 > ).
  2. 主要功能 :
    • 设备命名与映射 : 将逻辑设备名映射为物理设备名 ( 通过 LUT ).
    • 设备保护 : 检查用户是否有权访问该设备 ( 类似文件系统的权限检查 ).
    • 设备分配与回收 : 响应用户的设备请求，进行分配 ( SDT/DCT/COCT/CHCT ) 与回收.
    • 缓冲管理 : 管理缓冲区 ( 单/双/循环缓冲 )，解决速度不匹配问题.
    • 差错处理 : 处理设备驱动层上报的错误，向用户层提供统一的错误报告.
    • 屏蔽设备差异 : 向用户层提供统一的逻辑块大小 ( 如将不同扇区大小的磁盘统一抽象为 4KB 逻辑块 ).
  3. 逻辑设备表 ( LUT ) :
    • 作用 : 记录逻辑设备名到物理设备名的映射关系.
    • 表项结构 : | 逻辑设备名 | 物理设备名 | 驱动程序入口 |
    • 实现方式 :
      • 单用户系统 : 整个系统一张 LUT ( 不允许重名 ).
      • 多用户系统 : 每个用户一张 LUT ( 允许不同用户使用相同的逻辑名 ).

5.2.3 设备分配与回收

  1. 核心数据结构 :
    • 系统设备表 ( System Device Table, SDT ) :
      • 整个系统只有一张. 记录所有设备的物理属性.
      • 表项包含 : 设备类型、设备标识符、DCT 指针 ( 指向该设备的 DCT )、驱动程序入口.
    • 设备控制表 ( Device Control Table, DCT ) :
      • 每个设备一张. 记录设备的控制信息.
      • 表项包含 : 设备队列指针 ( 指向等待该设备的进程 )、设备状态 ( 忙 / 闲 )、COCT 指针 ( 指向连接的控制器 )、重复执行次数.
    • 控制器控制表 ( Controller Control Table, COCT ) :
      • 每个控制器一张.
      • 表项包含 : 控制器状态、CHCT 指针 ( 指向连接的通道 )、控制器队列指针.
    • 通道控制表 ( Channel Control Table, CHCT ) :
      • 每个通道一张.
      • 表项包含 : 通道状态、通道队列指针.

  2. 设备分配步骤 :
    • 步骤 1 ( 查询 LUT ) : 根据进程请求的 逻辑设备名，查询 逻辑设备表 ( LUT )，找到物理设备在 SDT 中的表项.
    • 步骤 2 ( 分设备 ) : 根据 SDT 找到 DCT. 若设备忙，将进程挂到设备等待队列; 若设备闲，则 分配设备 ( 修改 DCT 状态为忙 ).
    • 步骤 3 ( 分控制器 ) : 根据 DCT 找到 COCT. 若控制器忙，挂到控制器等待队列; 若闲，则 分配控制器 ( 修改 COCT 状态为忙 ).
    • 步骤 4 ( 分通道 ) : 根据 COCT 找到 CHCT. 若通道忙，挂到通道等待队列; 若闲，则 分配通道 ( 修改 CHCT 状态为忙 ).
    • 注 : 只有 设备 + 控制器 + 通道 三者都分配成功，I/O 操作才能开始.
  3. 分配方式 :
    • 静态分配 :
      • 作业开始执行前，一次性分配所有需要的设备、控制器和通道.
      • 优点 : 不会发生死锁.
      • 缺点 : 设备利用率低.
    • 动态分配 :
      • 进程在执行过程中根据需要申请设备.
      • 优点 : 设备利用率高.
      • 缺点 : 可能造成死锁 ( 需配合死锁避免算法，如银行家算法 ).
  4. 分配的安全性 :
    • 安全分配 :
      • 进程请求 I/O 后进入 阻塞 状态，直到 I/O 完成才被唤醒.
      • 特点 : 一个进程同一时刻只能请求一个 I/O，摒弃了请求和保持条件，不会死锁. 但 CPU 和 I/O 串行，效率低.
    • 不安全分配 :
      • 进程请求 I/O 后 继续运行，可发出第二个 I/O 请求.
      • 特点 : 效率高，但可能导致 死锁.
  5. 逻辑设备名到物理设备名的映射 :
    • 逻辑设备表 ( LUT ) : 每个表项包含 ( 逻辑设备名, 物理设备名, 驱动程序入口 ).
    • 设置方式 :
      • 整个系统一张 LUT : 不允许不同进程用相同的逻辑设备名 ( 单用户系统 ).
      • 每个用户一张 LUT : 允许不同用户用相同的逻辑设备名 ( 多用户系统 ).

5.2.4 SPOOLing 技术 ( 假脱机技术 )

  1. 定义 :
    • 全称 : Simultaneous Peripheral Operations On-Line ( 同时在外围设备上进行的联机操作 ).
    • 核心思想 : 在多道程序环境下，利用一道程序来模拟脱机输入/输出时的外围控制机，在 磁盘 上建立 输入井 和 输出井，缓解 CPU 与低速 I/O 设备的速度矛盾.
  2. 系统组成 :
    • 输入井/输出井 ( 在磁盘上 ) :
      • 输入井 : 暂存输入设备送来的数据.
      • 输出井 : 暂存用户进程输出的数据.
    • 输入缓冲区/输出缓冲区 ( 在内存中 ) :
      • 输入缓冲区 : 暂存从输入设备 -> 输入井的数据.
      • 输出缓冲区 : 暂存从输出井 -> 输出设备的数据.
    • 输入进程/输出进程 :
      • 输入进程 : 模拟脱机输入时的外围控制机，将数据从设备读入输入井.
      • 输出进程 : 模拟脱机输出时的外围控制机，将数据从输出井送往设备.

  3. 工作流程 ( 以共享打印机为例 ) :
    • Step 1 ( 申请 ) : 用户进程请求打印输出时，SPOOLing 系统同意请求，并在 磁盘输出井 中申请一块空间 ( 而不是直接分配打印机 ).
    • Step 2 ( 暂存 ) : 将要打印的数据存入 输出井，并生成一张 打印申请表 挂到打印队列.
    • Step 3 ( 伪装 ) : 对用户进程而言，打印操作已完成，进程可继续执行 ( 实际上数据还在磁盘 ).
    • Step 4 ( 实打 ) : 打印机空闲时，输出进程从队列取出一张申请表，将输出井中的数据送往打印机打印.
  4. 主要特点 :
    • 提高了 I/O 速度 : 缓和了 CPU 与低速设备的速度矛盾 ( CPU 只需把数据送到磁盘井即可 ).
    • 设备无关性 : 实现了虚拟设备功能.
    • 独占变共享 : 将 独占设备 ( 如打印机 ) 改造为逻辑上的 共享设备 ( 实际上是分时共享磁盘 ).

5.2.5 设备驱动程序接口

  1. 接口特点 :
    • 统一性 : 操作系统规定了统一的驱动程序接口标准 ( 如 Linux 的 file_operations 结构体 ), 要求所有驱动程序必须实现特定的函数 ( open, read, write, close, ioctl 等 ).
    • 抽象性 : 屏蔽了底层硬件的差异，使得上层软件 ( 文件系统、用户进程 ) 可以用统一的方式访问不同设备.
  2. 识别设备 :
    • 主设备号 ( Major Number ) : 标识设备 类型 ( 即使用哪个驱动程序 ).
    • 次设备号 ( Minor Number ) : 标识同类设备中的 具体个体 ( 如第 1 块硬盘、第 2 块硬盘 ).
  3. 实现机制 :
    • 设备开关表 ( Device Switch Table ) : 操作系统维护的一张函数指针表 ( 如 cdevsw, bdevsw ).
    • 映射过程 : 系统调用 ( 如 read ) 根据主设备号查表 调用对应驱动程序的 xxx_read 函数.
  4. 常见接口函数 :
    • open() : 初始化设备.
    • read() / write() : 数据传输.
    • ioctl() : I/O 控制 ( 设置波特率、弹出光盘等非读写操作 ).
    • close() : 释放设备.

5.2.6 I/O 操作举例 ( 以读取磁盘文件为例 )

场景 : 用户进程调用 read(fd, buffer, n) 从磁盘读取数据.

  1. 用户态阶段 :
    • 用户进程调用 C 标准库函数 read().
    • 库函数将系统调用号、参数 ( 文件描述符、缓冲区地址等 ) 放入寄存器.
    • 执行 trap 指令 ( 如 int 0x80 或 syscall ), CPU 从用户态切换到内核态.
  2. 内核态 ( 系统调用 & 独立软件 ) :
    • 系统调用入口 : 内核根据系统调用号找到 sys_read 处理函数.
    • 文件系统层 : 解析 fd 找到对应的文件索引节点 ( inode ), 确定逻辑块号.
    • 设备独立层 :
      • 查询 高速缓存 ( Page Cache )，若命中则直接返回 ( 零拷贝/快速路径 ).
      • 若未命中，将逻辑块号转换为物理块号 ( LBA ), 并构造 I/O 请求 ( BIO ).
      • 调用通用块层 ( Generic Block Layer ) 进行 I/O 调度 ( 电梯算法排序/合并 ).
  3. 内核态 ( 设备驱动 ) :
    • 驱动程序 : 从请求队列取出请求.
    • 启动 I/O : 向磁盘控制器的 寄存器 写入命令 ( 柱面、磁头、扇区、DMA 地址 ).
    • 进程阻塞 : 驱动程序将当前进程状态置为 阻塞 ( Blocked )，调度其他进程执行.
  4. 硬件阶段 :
    • 磁盘控制器 : 控制磁头移动 ( 寻道 ), 等待扇区旋转到磁头下，通过 DMA 将数据从磁盘传送到内核内存.
    • 中断发生 : 数据传输完成后，控制器向 CPU 发送 中断信号.
  5. 内核态 ( 中断处理 ) :
    • 中断服务程序 ( ISR ) : 保护现场，检查中断原因 ( 读取完成 ).
    • 数据拷贝 : ( 若非零拷贝 ) 将数据从内核缓冲区复制到用户缓冲区.
    • 唤醒进程 : 将等待该 I/O 的进程状态置为 就绪 ( Ready ).
  6. 返回用户态 :
    • 进程再次获得 CPU，从系统调用返回，read() 函数返回读取的字节数.

5.3 磁盘和固态硬盘

5.3.1 磁盘 ( Disk )

  1. 物理结构 :
    • 盘片 ( Platter ) : 磁盘由多个盘片叠加而成. 每个盘片有两个面 ( Surface ).
    • 磁道 ( Track ) : 盘面被划分成一个个同心圆.
    • 扇区 ( Sector ) : 磁道被划分成一个个圆弧段. 扇区是磁盘读写的最小单位 ( 通常 512B 或 4KB ).
    • 柱面 ( Cylinder ) : 所有盘片上相对位置相同的磁道组成一个柱面.
    • 磁头 ( Head ) : 每个盘面对应一个读写磁头.
  2. 磁盘地址 ( Addressing ) :
    • CHS 编址 : 柱面号 ( Cylinder ) - 磁头号 ( Head ) - 扇区号 ( Sector ).
      • 寻址过程 : 移动磁臂选柱面 ( 寻道 ) -> 激活磁头选盘面 -> 旋转磁盘选扇区.
    • LBA 编址 ( Logical Block Addressing ) : 逻辑块地址.
      • 操作系统使用线性逻辑块号 ( 0, 1, 2... ), 由磁盘控制器转换为物理 CHS 地址.
  3. 分类 :
    • 按磁头是否移动 :
      • 固定头磁盘 : 每个磁道都有一个磁头. ( 速度快，造价高 ).
      • 移动头磁盘 : 每个盘面只有一个磁头，需移动磁臂. ( 通用 ).
    • 按盘片是否可换 :
      • 固定盘磁盘 ( 硬盘 ) vs 可换盘磁盘 ( 光盘/软盘 ).

5.3.2 磁盘的管理

  1. 磁盘初始化 ( Initialization ) :
    • Step 1 低级格式化 ( Physical Formatting ) :
      • 将磁盘的各个磁道划分为特定格式的 扇区.
      • 写入扇区头/尾信息 ( 如校验码 ECC )，扇区数据区 通常填充 0x00.
      • ( 注: 出厂时通常已完成 ).
    • Step 2 磁盘分区 ( Partitioning ) :
      • 将物理磁盘分为由多个柱面组成的逻辑分区 ( 如 C:, D:, E: ).
      • 写入 分区表 ( MBR 或 GPT )，记录每个分区的起始/结束地址.
    • Step 3 高级格式化 ( Logical Formatting ) :
      • 在每个分区上建立 文件系统 ( 如 NTFS, FAT32, ext4 ).
      • 写入文件系统元数据 ( 如空闲空间位示图、根目录、i-node 表 ).
  2. 引导块 ( Boot Block ) :
    • 位置 : 磁盘的 0 号扇区 ( MBR ) 或分区的第一个扇区 ( PBR ).
    • 作用 : 存放 自举装入程序 ( Bootstrap Loader ).
    • 启动流程 :
      • ROM 中的 BIOS 自检 -> 读取 MBR -> 执行引导代码 -> 加载操作系统内核.
      • ( 注: 完整的操作系统代码太长，无法全部塞入引导块，引导块只负责"拉起"加载器 ).
  3. 坏块管理 ( Bad Blocks ) :
    • 硬件方案 ( 扇区备用 / 扇区重映射 ) :
      • 磁盘控制器维护一个坏块链表.
      • 出厂时保留一些 备用扇区 ( Spare Sectors ).
      • 当发现坏块时，控制器自动将其映射到备用扇区. ( 操作系统无感知，逻辑地址不变 ).
    • 软件方案 ( 文件系统标记 ) :
      • 文件系统 ( 如 FAT ) 在 FAT 表中标记坏簇 ( Bad Cluster ).
      • 操作系统分配空间时跳过这些坏簇.

5.3.3 磁盘调度算法

核心目标 : 减少 寻道时间 ( 这是机械运动，最耗时 ).

1. 磁盘访问时间 ( Disks Access Time )

一次磁盘读写操作的时间 由三部分组成 :

  1. 寻道时间 ( Seek Time, ) :
    • 磁头移动到指定磁道 ( 柱面 ) 所需的时间.
    • . ( : 跨越磁道数, : 常数, : 启动磁臂时间 ).
    • 注意 : 调度算法主要优化这一项.
  2. 旋转延迟时间 ( Rotational Latency, ) :
    • 指定扇区旋转到磁头下方所需的时间.
    • 平均旋转延迟 : . ( : 磁盘转速 ).
  3. 传输时间 ( Transfer Time, ) :
    • 数据从磁盘读出或写入的时间.
    • . ( : 读写字节数, : 每个磁道总字节数 ).

2. 典型调度算法

假设磁头当前位于 53 号磁道，请求队列 : 98, 183, 37, 122, 14, 124, 65, 67.

  1. 先来先服务 ( FCFS ) :
    • 规则 : 按请求到达的先后顺序进行调度.
    • 顺序 : 53 -> 98 -> 183 -> 37 -> 122 -> 14 -> 124 -> 65 -> 67.
    • 特点 : 最公平，但性能最差 ( 磁头乱跑 ).
  2. 最短寻找时间优先 ( SSTF ) :
    • 规则 : 优先选择距离当前磁头 最近 的请求.
    • 顺序 : 53 -> 65 -> 67 -> 37 -> 14 -> 98 -> 122 -> 124 -> 183.
    • 特点 : 平均寻道时间短，但可能导致 饥饿 ( Starvation ) 现象 ( 远处的请求一直得不到服务 ).
  3. 扫描算法 ( SCAN / 电梯算法 ) :
    • 规则 : 磁头按一个方向移动 ( 如向外 )，服务沿途请求，直到 磁盘边缘 ( 端点 )，才改变方向.
    • 特点 : 性能好，无饥饿. 缺点是对 两端磁道 的请求响应不如中间磁道 ( 中间被经过两次，两端只经过一次 ).
  4. LOOK 算法 ( SCAN 的优化 ) :
    • 规则 : 类似 SCAN，但 不需要走到磁盘边缘，只要该方向上没有请求了，就立即反向.
    • 地位 : 实际系统中常用的算法.
  5. 循环扫描算法 ( C-SCAN ) :
    • 规则 : 磁头单向移动服务，到达边缘后，立即快速返回 起点 ( 返回途中不服务 ).
    • 特点 : 使得所有磁道的 响应时间均匀 ( 类似单向循环公交车 ).
  6. C-LOOK 算法 ( C-SCAN 的优化 ) :
    • 规则 : 类似 C-SCAN，但 不需要走到磁盘边缘，只要该方向无请求就立即返回.

3. 算法对比总结

算法 优点 缺点 适用场景
FCFS 公平、简单 平均寻道时间长 负载较轻
SSTF 性能较好 可能导致 饥饿 中等负载
SCAN 性能好、无饥饿 边缘请求响应慢 负载较重
C-SCAN 响应时间均匀 平均性能略低 (多跑路) 负载较重
LOOK / C-LOOK 避免无谓移动 实现稍复杂 实际最常用

4. 减少延迟时间的方法

主要目标是 减少旋转延迟 ( Rotational Latency )，因为寻道时间主要靠调度算法优化.

  1. 交替编号 ( Interleaving ) :
    • 问题 : 磁头读完一个扇区后，需要短暂的处理时间。如果扇区物理连续 ( 1, 2, 3... )，磁头可能会错过下一扇区，必须多转一圈.
    • 策略 : 让逻辑上相邻的扇区，在物理上保持一定的 间隔.
    • 示例 : 物理顺序不是 1, 2, 3, 4，而是 1, 9, 2, 10... ( 间隔式排列 ).
  2. 错位命名 ( Cylinder Skew ) :
    • 问题 : 磁头从一个磁道 ( 柱面 ) 切换到另一个磁道需要 寻道时间. 切换完成后，可能错过了 0 号扇区.
    • 策略 : 相邻柱面的 0 号扇区位置 错开 ( Skew ) 一定角度.
    • 效果 : 磁头切换到位时，刚好能遇到目标扇区，无需等待旋转.

5. 提高磁盘 I/O 速度的方法

除了调度算法和物理优化，还可以通过以下技术提升整体性能 :

  1. 提前读 ( Read-Ahead ) :
    • 利用 空间局部性 原理.
    • 当请求读取扇区 时，顺便把 ... 也读入缓冲区.
  2. 延迟写 ( Delayed Write ) :
    • 写入数据时，先写入 缓冲区 ( Buffer )，并标记为 "脏 ( Dirty )".
    • 不立即写回磁盘，而是通过优化算法 ( 如电梯算法 ) 批量写回，或在系统空闲时写回.
  3. 磁盘高速缓存 ( Disk Cache ) :
    • 在磁盘控制器或内存中开辟一块 RAM 区域，缓存频繁访问的磁盘块.
    • 写策略 :
      • 写回 ( Write Back ) : 只修改 Cache，延迟写回磁盘 ( 速度快，断电易丢数据 ).
      • 全写 ( Write Through ) : 同时写入 Cache 和磁盘 ( 安全，速度慢 ).

5.3.4 固态硬盘 ( SSD )

核心特点 : 基于 电子 方式存储，无机械部件，随机读写速度极快.

1. 物理组成

  • 闪存翻译层 ( FTL, Flash Translation Layer ) :
    • 核心组件，相当于 SSD 的控制器.
    • 功能 : 将逻辑块号 ( LBA ) 映射到物理页号 ( PPN )，实现 磨损均衡、垃圾回收、坏块管理.
  • 存储介质 ( NAND Flash ) :
    • 由大量的 浮栅晶体管 组成.
    • 结构层级 : 芯片 ( Chip ) -> 块 ( Block ) -> 页 ( Page ).

2. 读写特性 ( 重要 )

SSD 的读写单位与擦除单位不一致，这是其最特殊的属性.

  1. 读写单位 : 页 ( Page ). ( 通常 4KB ~ 16KB ).
  2. 擦除单位 : 块 ( Block ). ( 一个块包含多个页，通常 128KB ~ 512KB ).
  3. 写前擦除 :
    • 如果目标页是空的，可以直接写入.
    • 如果目标页已有数据，必须先擦除 整个块，才能写入新数据.
    • 后果 : 导致 "写放大 ( Write Amplification )" 问题 ( 为了写 4KB 数据，可能需要迁移整个块的数据 ).

3. 磨损均衡技术 ( Wear Leveling )

闪存的 擦写次数有限 ( 如 TLC 颗粒约 1000~3000 次 )，如果频繁擦写同一个块，会导致该块损坏. FTL 必须保证所有块的磨损程度均匀.

  1. 动态磨损均衡 ( Dynamic Wear Leveling ) :
    • 写入数据时，优先选择 擦除次数少 的空闲块.
  2. 静态磨损均衡 ( Static Wear Leveling ) :
    • 更加高级. 将 长期不修改的数据 ( 如电影、操作系统核心文件 ) 从 "年轻" 的块搬移到 "老旧" 的块中.
    • 腾出 "年轻" 的块来存储 频繁修改的数据.

4. SSD vs HDD 对比

特性 机械硬盘 ( HDD ) 固态硬盘 ( SSD )
存储介质 磁性盘片 ( 机械 ) 闪存颗粒 ( 电子 )
读写速度 慢 ( 需寻道/旋转 ) 极快 ( 电路信号 )
随机 I/O 性能差 ( 严重依赖寻道 ) 性能极好 ( 无寻道 )
噪音/功耗 有噪音、功耗较高 0 噪音、低功耗
寿命 读写次数无限制 擦写次数有限
价格 便宜 ( 适合存冷数据 ) 较贵 ( 适合装系统/热数据 )

5.3.5 本节小结

  • 磁盘调度算法 ( SSTF, SCAN, C-SCAN )计算寻道长度是必考计算题.
  • SSD 的读写特性 ( 页读写、块擦除、写放大 ) 与机械硬盘不同.

5.4 本章疑难点

1. 设备控制器 vs 设备驱动程序

  • 设备控制器 ( Hardware ) :
    • 位置 : 硬件电路板上 ( 如显卡、网卡 ).
    • 职责 : 控制物理设备动作 ( 电机转动、电平信号 )，提供寄存器接口 ( 状态/命令/数据 ).
  • 设备驱动程序 ( Software ) :
    • 位置 : 操作系统内核中.
    • 职责 : 将 OS 的通用请求 ( read/write ) 翻译成控制器能听懂的指令 ( 读写寄存器 ).
    • 关系 : 驱动程序是 OS 驾驭控制器的 "骑手".

2. 块设备 vs 字符设备

  • 块设备 ( Block Device ) :
    • 单位 : 数据块 ( Block ).
    • 特性 : 可寻址 ( seekable )，支持随机访问.
    • 缓存 : 必须使用缓冲区 ( Page Cache ).
    • 例子 : 硬盘、SSD、光盘.
  • 字符设备 ( Character Device ) :
    • 单位 : 字符流 ( Byte Stream ).
    • 特性 : 不可寻址，只能顺序读写.
    • 缓存 : 通常不使用通用缓冲，直接读写.
    • 例子 : 键盘、鼠标、打印机.

3. I/O 软件层次的职责辨析

做题时常考 "某功能属于哪一层" :

  • 用户层 : 产生 I/O 请求，库函数调用 ( printf ).
  • 设备独立性软件 : 逻辑 设备名映射、缓冲 管理、分配 与回收、保护. ( 关键词 : 逻辑、通用 ).
  • 设备驱动程序 : 设置 寄存器、检查设备状态、生成具体指令. ( 关键词 : 物理细节 ).
  • 中断处理程序 : I/O 完成后的 善后 处理 ( 唤醒进程、拷贝数据 ).

4. I/O 控制方式的演变逻辑

  • 程序直接控制 : CPU 忙等 ( 轮询 ) -> CPU 利用率极低.
  • 中断驱动 : CPU 让出等待 -> 解决了 "忙等"，但中断频率太高 ( 每字/每字节一次 ).
  • DMA : 硬件接管数据传输 -> 解决了 "频繁中断" ( 每块一次中断 ).
  • 通道 : 专用处理机 -> 解决了 "CPU 即使是组织 DMA 也很累" ( 一组块/多组请求一次中断 ).
  • 核心趋势 : CPU 越来越 "懒"，I/O 越来越 "独立".

5. SPOOLing 技术 ( 假脱机 )

  • 本质 : 以空间换时间. 利用磁盘空间模拟独占设备.
  • 效果 : 将 独占设备 ( 如打印机 ) 改造为 共享设备.
  • 组成 :
    • 输入/输出井 : 磁盘上的区域.
    • 输入/输出进程 : 负责搬运数据.

6. 磁盘调度的核心

  • 瓶颈 : 寻道时间 ( 机械臂移动 ) 是毫秒级，远慢于电子操作.
  • 算法选择 :
    • SSTF : 贪心，快，但可能饿死.
    • SCAN (电梯) : 稳，无饿死，适合重负载.
    • C-SCAN : 稳，响应均匀，适合重负载.

7. 磁盘地址的连续性 ( 进位顺序 )

常考题型 : "某文件连续存放，当前块在 [C, H, S]，下一块在哪里?".

  • 读写优先级 : 扇区 ( Sector ) -> 磁头/盘面 ( Head ) -> 柱面 ( Cylinder ).
  • 进位规则 :
    1. 扇区号 . 若溢出 ( 超过每道扇区数 )，则 ->
    2. 磁头号 ( 扇区归零 ). 若溢出 ( 超过总磁头数 )，则 ->
    3. 柱面号 ( 磁头、扇区归零 ).
  • 物理原因 :
    • 同柱面切换磁头 : 仅需电子切换 ( 极快，微秒级 ).
    • 切换柱面 : 需要移动磁臂 ( 寻道，极慢，毫秒级 ).
    • 结论 : 连续存取应尽可能停留在同一个柱面内 ( 减少寻道 ).

    • 当前: [柱面5, 磁头1, 扇区7 (满)]. 下一块: [柱面5, 磁头2, 扇区0]. ( 优先换头 ).
