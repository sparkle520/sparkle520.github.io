第6章 子程序结构核心速查表

分类 概念 / 指令 核心功能 & 规则 备注
定义与结构 PROC / ENDP 定义子程序 ( 过程 ) 的开始与结束 必须成对出现
NEAR 段内调用 ( 默认 ) : 仅压入/弹出 IP ( 2字节 ) 适用于同一代码段内
FAR 段间调用 : 压入/弹出 CS 和 IP ( 4字节 ) 适用于不同代码段间
调用与返回 CALL Label 压入返回地址, 跳转至子程序 自动处理 IP / CS:IP
RET 弹出返回地址, 返回调用者 栈顶必须是合法的返回地址
RET n 返回并执行 SP = SP + n 用于栈传递参数时平栈 ( n=参数总字节数 )
参数传递 寄存器传递 用 AX, BX... 传参 速度最快, 但参数数量受限 ( ≤3个 )
栈传递 用 PUSH 压参, [BP+n] 取参 通用性强, 支持多参; 需 RET n 平栈
内存传递 定义 DATA 段变量共享 适合数组/大块数据, 速度较慢
现场保护 PUSH / POP 保存/恢复寄存器值 遵循 LIFO ( 后进先出 ) 原则
USES 伪指令: PROC USES AX BX 汇编器自动生成 PUSH/POP 代码
嵌套与递归 嵌套调用 子程序A调用子程序B 栈深度增加, 注意栈溢出风险
递归调用 子程序调用自身 必须有明确的终止条件 ( 出口 )

第6章 子程序结构

章节概述

子程序 ( 又称"过程" ) 是汇编程序 模块化设计 的核心，通过将重复逻辑封装为独立代码块，实现代码复用、逻辑拆分与程序可读性提升. 本章聚焦子程序的定义、调用/返回、现场保护、参数传递、嵌套及实际应用，是大型汇编程序开发的基础.

6.1.2 子程序的调用和返回

通过 ( 调用 ) 和 ( 返回 ) 指令实现子程序的执行流程跳转，核心是 栈的操作 ( 保存/恢复返回地址 ).

1. 调用指令 : CALL

根据调用类型， 会将 返回地址 压入栈中，再跳转到子程序入口.

  • 段内 : 压入当前 ( 2字节 ) ，跳转到子程序的段内偏移地址.
  • 段间 : 压入当前 ( 2字节 ) 和 ( 2字节 ) ，跳转到子程序的段地址+偏移地址.

2. 返回指令 : RET

从栈中弹出返回地址，回到调用者的下一条指令，分为两种形式 :

  • 基本形式 :
    • 段内 : 弹出栈顶2字节到 .
    • 段间 : 弹出栈顶2字节到 ，再弹出2字节到 .
  • 带参数形式 : ( 为偶数 ) 返回后自动将栈顶指针 加 ，用于 清理栈中传递的参数 ( 调用者压入的参数会留在栈中， 可自动释放空间 ).

传递方式 核心原理 核心优势 核心劣势 适用场景 典型示例 注意事项
寄存器传递 调用者将参数存入寄存器，子程序直接访问；结果存入寄存器返回 1. 速度最快 ( 无需内存/栈操作 )
2. 代码简洁，易调试		 1. 寄存器数量有限 ( 仅支持1~3个参数 )

2. 子程序需保护未作为参数的寄存器
栈传递 调用者将参数依次压入栈，子程序通过BP/SP定位栈中参数；结果可存寄存器/栈返回 1. 支持任意数量参数
2. 参数顺序固定，通用性强		 1. 需手动计算栈偏移 ( 易出错 )
2. 栈操作增加开销，速度慢于寄存器		 1. 参数数量多 ( ≥4个 )
2. 多模块程序的子程序调用		 三数求和、数组排序参数传递 1. 参数压栈顺序通常"从右到左"
2. 用RET n清理栈中参数 ( n=参数总字节数 )
3. 子程序中用BP固定栈基址，避免SP偏移错误
内存传递 调用者将参数存入指定内存单元，子程序直接访问；结果存入内存单元返回 1. 支持大量/大型参数 ( 数组、字符串 )
2. 参数持久化 ( 不占用寄存器/栈 )		 1. 速度最慢 ( 需内存寻址 )
2. 需定义专用内存单元，灵活性低		 1. 参数为数组、字符串...大型数据
2. 参数数量多且无需高频访问		 数组求和、字符串反转 1. 需确保调用者和子程序访问同一数据段
2. 可通过寄存器传递内存单元的地址 ( 折中方案 )
