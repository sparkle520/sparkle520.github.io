第8章 输入输出程序设计

章节概述

输入输出 ( ) 程序设计的核心是实现 与外部设备 ( 外设 ) 的数据交换，由于CPU与外设的速度、信号格式差异大，需通过 接口电路 和特定传送方式完成通信. 本章聚焦3种经典 数据传送方式 ( 程序直接控制、中断、 ) ，以及8086中断系统的实现逻辑，是硬件与软件交互的核心知识点.

8.1 I/O设备的数据传送方式

与外设的数据交换需解决"速度匹配""同步控制"问题，常见传送方式有3种，核心区别是" 参与程度".

8.1.1 CPU与外设

无法直接与外设通信，需通过 接口电路 ( 简称"接口" ) 中转，接口的核心功能 :

  1. 数据缓冲 : 暂存 或外设的数据，解决速度不匹配问题；
  2. 地址译码 : 识别 访问的 外设端口；
  3. 控制命令 : 传递 对设备的 控制信号 ( 如"启动""停止" )；
  4. 状态反馈 : 向 报告外设 状态 ( 如"忙""就绪" ).

8.1.2 直接存储器存取 ( DMA ) 方式

是 不参与的高速数据传送方式，由 控制器 ( ) 直接控制内存与外设的数据交换，适合批量数据 ( 如磁盘读写、显存刷新 ).

1. 核心特点

  • 不经过 ，直接在内存与外设间传数据；
  • 速度极快 ( 仅受硬件总线速度限制 )；
  • 会暂时释放总线控制权给 .

2. DMA 传送的完整过程

DMA 传送并非完全脱离 CPU，而是分为三个阶段：

  1. 预处理 ( CPU 执行 )
    • CPU 执行指令测试外设状态.
    • CPU 向 DMA 控制器 ( DMAC ) 写入 内存首地址、传送字数、传送方向 ( 读/写 ).
    • CPU 启动外设，允许 DMA 请求.
  2. 数据传送 ( DMAC 执行 )
    • 请求 : 外设准备好后，向 DMAC 发出 DMA请求 ( DREQ ).
    • 总线接管 : DMAC 向 CPU 发出 总线请求 ( HRQ )；CPU 执行完当前总线周期后，发送 总线响应 ( HLDA ) 并释放总线控制权 ( 挂起 ).
    • 传输 : DMAC 接管总线，发出一对地址和读写信号，数据直接在内存与外设间传输 ( 不经过 CPU ).
    • 循环 : 每传送一个数据，DMAC 自动修改内存地址并减小字数计数器，直到计数器为 0.
  3. 后处理 ( CPU 执行 )
    • 结束 : 计数器为 0 或外设结束，DMAC 停止传送，向 CPU 发送 中断请求 ( 或置状态位 ).
    • 收尾 : CPU 响应中断，检查传送正确性 ( 校验 )，决定是否继续或结束.

3. DMA 的传送方式 ( 工作模式 )

DMA 控制器获取总线控制权的方式主要有以下 3 种 :

方式 别名 工作原理 优点 缺点
停止 CPU 访问 突发传送 ( Burst ) DMA 独占总线直到 整个数据块传送完毕 传送速度最快 ( 连续 ) CPU 长时间挂起, 响应变慢
周期挪用 周期窃取 ( Cycle Stealing ) DMA 仅占用 1 个总线周期 传送 1 个字, 然后归还 CPU 仍可执行 ( 仅变慢 ) 传送速度不如突发模式
交替访问 透明 DMA DMA 利用 CPU 不使用总线 的周期 ( 如内部运算 ) CPU 毫无影响 ( 透明 ) 硬件控制逻辑复杂

8.2 程序直接控制I/O方式

又称 查询方式，是 通过程序主动查询外设状态，状态"就绪"后再进行数据传送，是最简单的 方式.

8.2.1 I/O端口

外设通过 端口 与 通信，端口是接口电路中的寄存器 ( 分3类 ) :

  • 数据端口 : 暂存 与外设交换的 数据；
  • 状态端口 : 存 放 外设状态 ( 如"忙""就绪" )；
  • 控制端口 : 存 放 对设备的 控制命令 ( 如"启动""复位" ).

端口编址方式 ( 8086采用独立编址 )

编址方式 特点 指令 地址范围
独立编址 ( 8086 ) 端口地址与内存地址 分离 ( 端口 )
存储器映射编址 端口作为内存单元，与内存 共享 地址空间 同内存地址范围

8.2.3 I/O程序举例 ( 查询方式控制打印机 )

需求 : 用查询方式向打印机发送字符 'A' ( 打印机端口 : 数据 378H、状态 379H、控制 37AH ).

  • 状态端口 (379H) : bit 7 = BUSY ( 0=忙, 1=空闲/就绪 ). 注: 不同打印机标准可能不同, 此处假设 D7=1 为就绪.
  • 控制端口 (37AH) : bit 0 = STROBE ( 选通脉冲, 需产生一个 1->0->1 的负脉冲通知打印机取数 ).

8.3 中断传送方式

中断是 外设主动通知 的传送方式 : 外设就绪后发"中断请求"， 暂停当前程序，转去执行"中断处理程序"，完成数据传送后返回原程序，解决了查询方式" 空..."的问题.

8.3.1 8086的中断分类

8086的中断分为 内部中断 ( 软件中断 ) 和 外部中断 ( 硬件中断 ) :

内部中断 执行指令触发 1. 除法错误 ( 除数为0 )
2. 指令 ( 软件调用 )
3. 溢出中断 ( 时执行 )		 高

2. 可屏蔽中断 ( ) : 键盘、打印机... ( 受 标志位控制 )		 低 ( )

8.3.3 中断过程

从"中断请求"到"返回原程序"的完整流程 :

  1. 中断请求 : 外设向 发 ( 或内部中断触发 )；
  2. 中断响应 : 若 的 ( 允许可屏蔽中断 ) ，则发"中断响应"信号，外设送中断类型号 ；
  3. 保护现场 : 自动压栈 保存当前 、、；
  4. 清标志位 : 自动清除 ( 禁止新的可屏蔽中断 ) 和 ( 禁止单步中断 )；
  5. 取中断向量 : 读取 地址处的 和 ，跳转 到 中断处理程序；
  6. 执行中断处理程序 : 处理 数据...核心逻辑；
  7. 中断返回 : 执行 指令，自动弹出 、、，回 到 原程序继续执行.

8.3.4 中断优先级和中断嵌套

1. 8086中断优先级 ( 从高到低 )

除法错误 > 指令 > 溢出中断 ( ) > 非屏蔽中断 ( ) > 可屏蔽中断 ( ) > 单步中断

2. 中断嵌套

高优先级中断可 打断低优先级中断的处理程序，实现"嵌套执行"，需满足 :

  • 高优先级中断请求发生时，低优先级处理程序未关中断 ( )；
  • 处理程序需正确保存/恢复现场，避免寄存器冲突.

8.3.5 中断处理程序

中断处理程序是响应中断时执行的代码，编写需遵循固定步骤 :

1. 编写步骤

  1. 保存现场 : 用 指令保存 寄存器 ( 避免覆盖原程序数据 )；
  2. 处理中断 : 实现 数据传送...核心逻辑；
  3. 发送 ( 中断结束信号 ) : 通知中断控制器"中断已处理" ( 仅可屏蔽中断需要 )；
  4. 恢复现场 : 用 指令恢复寄存器 ( 顺序与 相反 )；
  5. 中断返回 : 用 指令返回原程序 ( 自动恢复 、、 ).

2. 代码示例 ( 安装并实现中断处理程序 )

需求 : 安装 用户自定义中断 ( 类型号 60H ), 功能是显示字符 'X'. ( 注: 软中断指令 INT n 触发的中断无需发送 EOI 信号; 若是硬件 IRQ 中断则必须发送. )

  1. 传送方式对比 :
方式 参与度 速度 适用场景
程序直接控制 全程参与 ( 主动查询 ) 慢 简单外设 ( 、开关 )
中断 仅处理中断时参与 中 大多数外设 ( 键盘、打印机 )
不参与 ( 控制 ) 极快 批量数据 ( 磁盘、显存 )
  1. 中断系统核心 :
  • 中断向量表 : 内存 ，存储中断处理程序入口，向量地址=中断类型号 ；
  • 中断过程 : 请求响应保护现场执行处理程序恢复现场返回；
  • 中断处理程序 : 需保存/恢复现场，用 返回，可屏蔽中断需发送 信号.
