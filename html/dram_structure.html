各部件功能说明
T — NMOS 访问管
受 WL ( 字线 ) 控制. WL=1 时导通，连接存储电容 C 与位线 BL，允许读写操作.
C — 存储电容
利用电容充放电存储数据. 充电 ( 高压 ) = 逻辑 1，放电 ( 低压 ) = 逻辑 0.
漏电问题 ( 核心缺陷 )
电容会持续漏电，约 64ms 后数据丢失，因此必须周期性 刷新 ( Refresh ) ，这是 DRAM 名称中 Dynamic 的含义.
优势 : 极高集成度
每个单元仅需 1T1C，面积远小于 SRAM 的 6T，因此能以更低成本实现 GB 级 容量.

三种工作状态详解
📖
读操作 ( Read )
  • ①预充电 : BL 充至 VDD/2 ( 参考电压 )
  • ②WL 拉高，访问管 T 导通
  • ③电容 C 与 BL 发生电荷共享，BL 电压微小变化
  • ④灵敏放大器 SA 检测 BL 偏移方向并放大
  • ⑤SA 同时将数据回写到电容 ( 读操作具破坏性 )
  • ⚠读后必须刷新，否则数据丢失
✏️
写操作 ( Write )
  • ①写驱动器将 BL 强制拉至 VDD 或 GND
  • ②WL 拉高，访问管 T 导通
  • ③电容 C 被充电 ( 写 1 ) 或放电 ( 写 0 )
  • ④WL 拉低，T 截止，数据存于电容
  • ⚠写入后数据仍会漏电，需定期刷新
🔄
刷新操作 ( Refresh )
  • ①内存控制器定期发出 REFRESH 命令
  • ②依次激活每一行 WL
  • ③SA 读取并放大电容电荷
  • ④SA 将满电平回写至电容，补充漏失电荷
  • 周期典型刷新周期 64ms，刷新所有行
  • ⚠刷新期间内存无法响应其他访问请求

破坏性读取原理
⚠ 为什么读操作有破坏性？
读取时电容 C 与位线 BL 发生 电荷共享 :
Vc_new = ( C·Vc + CBL·VDDⅡ ) / ( C + CBL )

电容电压被拉向 VDD/2，原有数据 被破坏，必须由 SA 重新写回.
SA 灵敏放大器的作用
SA 检测 BL 偏离 VDD/2 的方向 :
• BL > VDD/2 → 判为 1，将 BL 拉至 VDD
• BL < VDD/2 → 判为 0，将 BL 拉至 GND

放大后再通过 WL 回写到电容，完成数据恢复.

地址复用原理
DRAM 通过 RAS↓ ( 行选通 ) 和 CAS↓ ( 列选通 ) 两个控制信号，分两次复用同一组地址引脚，先传行地址激活整行，再传列地址选定具体单元，节省了大量引脚，是 DRAM 高密度设计的关键技术之一.
