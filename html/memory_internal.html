各模块功能详解
地址寄存器 MAR
锁存 CPU 送来的地址码，保证整个存储周期内地址稳定，避免总线变化影响内部操作.
行 / 列译码器
将 n 位地址译为 2ⁿ 条选择线. 行译码器激活 WL 选定行; 列译码器选定 BL 列接入 SA.
存储矩阵 Cell Array
核心存储体. SRAM 用双稳态触发器 ( 6T ) ，DRAM 用电容 ( 1T1C ). 行激活整行，列选具体单元.
灵敏放大器 SA
检测 BL 与 BL̄ 间微弱电压差 ( 几十 mV ) ，放大为完整 VDD/GND 电平. DRAM 读后还负责回写.
数据缓冲器 MDR / I/O Buffer
读：暂存 SA 读出的数据并驱动数据总线. 写：接收外部总线数据，准备写入存储矩阵.
关键控制引脚
CS̄
片选 ( Chip Select )
低有效. 仅 CS̄=0 时芯片工作; 多片组合时由高位地址控制各片 CS̄.
WĒ
写使能 ( Write Enable )
WĒ=0 写操作，WĒ=1 读操作，决定数据流方向.
OĒ
输出使能 ( Output Enable )
OĒ=0 数据输出到总线; 否则呈高阻抗 ( 三态门断开 ).
A[n]
地址总线
n 根寻址 2ⁿ 个单元. DRAM 行列复用，分两次传入 ( RAS↓/CAS↓ ).
📐 地址线与容量关系
n 根地址线 + m 根数据线 →
容量 = 2ⁿ × m bit
例：14 根地址线 + 8 根数据线 → 2¹⁴ × 8 bit = 16 KB

存储周期 vs 存取时间
区分要点：
• 存取时间：发出地址到数据稳定，反映读速度
• 存储周期：两次访问间最小间隔，含恢复阶段
• DRAM 存储周期远大于存取时间
位扩展 — 增加字长 ( 数据位宽 )
字扩展 — 增加地址空间 ( 存储容量 )
字位同时扩展综合示例
用 1K×4bit 芯片构建 4K×8bit 存储器
  • ①位扩展：每组 2 片并联 → 每组 1K×8bit
  • ②字扩展：4 组串联 ( 片选 ) → 4K×8bit
  • ③共需 2×4 = 8 片芯片
  • ④地址线：低 10 位寻址组内，高 2 位选组
  • ⑤数据线：8 根 ( 2 片各出 4bit 并联 )
位扩展要点
地址线、CS̄、WĒ 并联共享; 数据线各自独立拼接. 所有芯片同时被选中同时工作.
字扩展要点
低位地址、数据线、WĒ 共享; 高位地址经译码器控制各片 CS̄，每次仅一片工作.
三态门的作用
未被选中的芯片呈高阻抗 ( Hi-Z ) ，防止多片并接时互相驱动冲突，是总线复用的基础.

三总线详解
地址总线 Address Bus
单向 ( CPU→存储器 ). 宽度决定寻址空间：n 位 → 2ⁿ 地址单元. 32 位→4GB，64 位→16EB.
数据总线 Data Bus
双向 ( CPU↔存储器 ). 宽度决定每次传输量：8/16/32/64 bit. 宽度越大带宽越高.
控制总线 Control Bus
传递读写控制 ( R/W̄ ) 、片选 ( CS̄ ) 、时钟 ( CLK ) 、中断 ( INT ) 等，各线方向不同.
ROM vs RAM 连接区别
ROM 连接特点
  • •地址总线、数据总线均接入
  • •WĒ 悬空或接高 ( 只读 )
  • •映射到低地址，启动时 PC 指向此处
  • •存 BIOS / 固件，掉电不丢
RAM 连接特点
  • •地址、数据、控制总线全接
  • •WĒ 由控制总线 R/W̄ 驱动
  • •DRAM 还需刷新控制器 ( RAS/CAS )
  • •掉电丢失，运行时存程序和数据

完整读写步骤分解
📖 读操作 ( 8步 )
  • ①CPU 将目标地址送入 MAR
  • ②MAR 将地址送上地址总线，CS̄=0，R/W̄=1
  • ③芯片 MAR 锁存总线地址
  • ④行译码器激活 WL，列译码器选定 BL
  • ⑤存储单元数据经 BL 流向灵敏放大器 SA
  • ⑥SA 放大差分信号，输出完整逻辑电平
  • ⑦数据经 I/O Buffer 驱动数据总线
  • ⑧CPU 从数据总线读入 MDR，完成读操作
✏️ 写操作 ( 7步 )
  • ①CPU 将地址→MAR，数据→MDR，R/W̄=0
  • ②地址送总线，CS̄=0，WĒ=0
  • ③芯片锁存地址，译码选定目标单元
  • ④MDR 数据经总线送入芯片 Buffer
  • ⑤写驱动器强制 BL/BL̄ 为所需电平
  • ⑥数据写入存储单元 ( 触发器翻转/电容充放电 )
  • ⑦WĒ 回高，CS̄ 回高，写操作完成
