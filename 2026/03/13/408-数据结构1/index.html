第1章 绪论

1.1 数据结构三要素

数据结构由三个核心要素构成. 理解三要素是掌握整门课程的基础.

逻辑结构 描述数据元素之间的逻辑关系，与具体计算机无关.

类型 说明 举例
线性结构 一对一 线性表、栈、队列、串
树形结构 一对多 二叉树、B 树、堆
图结构 多对多 有向图、无向图
集合结构 关系不显式 数学集合

存储结构 ( 物理结构 ) 是逻辑结构在计算机内存中的实现方式.

类型 特点
顺序存储 连续地址，随机访问快，插删慢
链式存储 离散地址，插删快，查找慢
索引存储 额外建索引表，查找快
散列存储 关键字映射地址，查找快 ( 依赖哈希函数 )

数据运算 是对数据施加的操作集合.

  • 运算的定义基于逻辑结构
  • 运算的实现依赖存储结构
  • 常见运算 增删改查、遍历、排序

记忆口诀 逻辑结构决定是什么，存储结构决定怎么存，数据运算决定能做什么.

1.2 基本术语与 ADT

1.2.1 核心概念

  • 数据 ( Data ) 计算机程序处理的所有信息的总称.
  • 数据元素 ( Data Element ) 数据的基本单位，通常作为一个整体处理，也称结点或记录.
  • 数据对象 ( Data Object ) 性质相同的数据元素的集合，是数据的子集.
  • 数据类型 ( Data Type ) 值的集合以及该集合上定义的运算的总称.

1.2.2 抽象数据类型 ( ADT )

抽象数据类型 ( Abstract Data Type ) 是一个数学模型及定义在该模型上的一组操作，与具体实现无关，强调做什么而非怎么做.

ADT 三要素

  • 数据对象 数据元素的集合
  • 数据关系 元素之间的关系集合
  • 基本操作 对数据对象施加的操作集合

1.3 算法与特性

1.3.1 定义

算法是对特定问题求解步骤的一种描述，是指令的有限序列.

1.3.2 五大特性

① 有穷性 ( Finiteness )

算法执行有限步骤后必须终止，不能无限循环.

② 确定性 ( Definiteness )

每条指令含义唯一明确，相同输入必定产生相同输出，不存在二义性.

③ 可行性 ( Feasibility )

每一步操作都可以在有限时间内完成，操作切实可执行.

④ 输入 ( Input )

有零个或多个输入，来自外部环境.

  • 零输入 初始条件已内置于算法之中
  • 多输入 来自键盘、文件、参数等

⑤ 输出 ( Output )

有一个或多个输出，与输入有确定的关系.

  • 无输出的算法毫无意义
  • 输出可以是打印、返回值、文件写入等形式

有穷性针对算法，程序不一定满足有穷性，例如操作系统是长期运行的系统程序.

1.4 算法分析

1.4.1 时间复杂度

衡量算法运行时间随输入规模 \(n\) 增长的变化趋势，使用大 \(O\) 表示法描述数量级.

分析角度 含义 常用程度
最好情况 输入最有利时的时间复杂度 ( 下界 ) ★☆☆
最坏情况 输入最不利时的时间复杂度 ( 上界 ) ★★★
平均情况 所有输入的期望时间复杂度 ★★☆

做题时最常使用最坏情况时间复杂度，因为它能给出任何输入下的性能上界.

\[O(1) < O(\log n) < O(n) < O(n \log n) < O(n^2) < O(2^n) < O(n!)\]

复杂度 名称 典型算法
\(O(1)\) 常数阶 数组随机访问
\(O(\log n)\) 对数阶 二分查找
\(O(n)\) 线性阶 顺序查找
\(O(n \log n)\) 线性对数阶 归并排序、快速排序 ( 平均 )
\(O(n^2)\) 平方阶 冒泡排序、插入排序
\(O(2^n)\) 指数阶 汉诺塔
\(O(n!)\) 阶乘阶 全排列
  1. 只保留最高次项，忽略低次项与常数系数.
  2. 循环嵌套，复杂度为各层循环次数之积.
  3. 顺序执行，复杂度取最大值.
  4. 递归算法，使用递推式或主定理求解.

时间复杂度求解技巧

核心思路是先选定一个最能代表运行时间的基本操作 ( 比较、赋值、数组访问等 )，然后数它执行了多少次.

  • 外层循环次数 \(\times\) 内层循环次数 只适用于内层次数与外层无关的情况
  • 内层次数依赖外层变量时，用求和而不是直接相乘
  • 看到 \(i*=2\)、\(i<<=1\)、\(i/=2\) 这类倍增或折半，循环次数通常是 \(O(\log n)\)

出现依赖关系的嵌套循环，常见结果来自三类求和.

  • 等差求和 \(\displaystyle\sum_{i=1}^n i = \frac{n(n+1)}{2} = O(n^2)\)
  • 等比求和 \(\displaystyle\sum_{k=0}^{\log n} 2^k = O(n)\)
  • 调和级数 \(\displaystyle\sum_{i=1}^n \frac{1}{i} = O(\log n)\)，因此 \(\sum_{i=1}^n \frac{n}{i} = O(n \log n)\)

先写递推式 (T(n))，再用主定理或递归树估计每层工作量.

  • 归并排序 \(T(n)=2T(n/2)+n \Rightarrow O(n \log n)\)
  • 二分递归 \(T(n)=T(n/2)+O(1) \Rightarrow O(\log n)\)
  • 分治 + 额外代价 \(T(n)=aT(n/b)+f(n)\) 用主定理对比 \(f(n)\) 与 \(n^{\log_b a}\)

摊还分析关注的是一连串操作的总成本再平均到每次操作上.

  • 动态数组扩容 ( 容量翻倍 ) 单次最坏 \(O(n)\)，但 \(m\) 次 push 的总移动次数是 \(O(m)\)，因此摊还为 \(O(1)\)

1.4.2 空间复杂度

衡量算法运行时所需额外内存空间随 \(n\) 增长的变化趋势.

  • \(O(1)\) 原地算法，不随 \(n\) 增大而增加额外空间.
  • \(O(n)\) 如递归调用栈深度为 \(n\).
  • \(O(n^2)\) 如邻接矩阵存储图.

时空权衡是做题常用思想，空间换时间或时间换空间都可能成为最优方案.

1.5 好算法的标准

✓ 正确性 ( Correctness )

算法能正确解决问题，对合法输入产生正确输出，并能覆盖边界情况.

✓ 健壮性 ( Robustness )

对非法输入能做出适当处理而非崩溃，具有容错能力.

✓ 可读性 ( Readability )

结构清晰易懂，便于阅读、理解与维护.

✓ 高效率 ( Efficiency )

时间复杂度低，避免不必要的重复计算.

✓ 低存储 ( Low Storage )

空间复杂度低，适合资源受限场景.

优先级排序 正确性 > 健壮性 > 可读性 > 高效率 > 低存储.

1.6 例题精选

例题 1 ( 逻辑结构判断 ) 下列数据关系分别属于哪类逻辑结构.

  1. 每个元素有且仅有一个直接前驱和一个直接后继 ( 除两端元素外 ).
  2. 每个元素可以对应多个后继，但每个元素的前驱唯一.
  3. 任意两个元素之间都可能存在关系.

解

  1. 一对一关系，线性结构.
  2. 一对多关系，树形结构.
  3. 多对多关系，图结构.

例题 2 ( 存储结构选择 ) 要求频繁进行插入与删除操作，同时希望结点移动尽量少，应该优先考虑哪种存储结构.

解

优先考虑链式存储. 顺序存储插入删除通常需要大量移动元素，链式存储通过修改指针完成，移动代价小.

例题 3 ( 时间复杂度 ) 分析下列代码段的时间复杂度.

解 内层循环总执行次数为 \(1+2+\cdots+n = \frac{n(n+1)}{2}\)，因此时间复杂度为 \(O(n^2)\).

例题 4 ( 空间复杂度 ) 设递归函数每次调用只使用常数个局部变量，且递归深度为 \(n\)，求空间复杂度.

解 递归调用栈深度为 \(n\)，每层常数空间，因此空间复杂度为 \(O(n)\).

例题 5 ( 依赖外层的步长 ) 分析下列代码段的时间复杂度.

解 固定 \(i\) 时，内层循环大约执行 \(\frac{n}{i}\) 次. 总执行次数 \[ \sum_{i=1}^n \frac{n}{i} = n \sum_{i=1}^n \frac{1}{i} = O(n \log n). \] 因此时间复杂度为 \(O(n \log n)\).

例题 6 ( 倍增循环 + 线性循环 ) 分析下列代码段的时间复杂度.

解 外层循环每次将 \(i\) 翻倍，执行次数为 \(\lfloor \log_2 n \rfloor + 1 = O(\log n)\). 内层每次执行 \(n\) 次. 因此总次数为 \(n \cdot O(\log n)\)，时间复杂度为 \(O(n \log n)\).

例题 7 ( 递推式包含额外对数项 ) 某分治算法满足 \[ T(n) = 2T(n/2) + n\log n, \quad T(1)=1. \] 求 \(T(n)\) 的渐进复杂度.

解 主定理中 \(a=2, b=2\)，因此 \(n^{\log_b a} = n\). 比较 \(f(n)=n\log n\) 与 \(n\)，属于 \(f(n)=\Theta(n \log^1 n)\) 的情形. 因此 \[ T(n) = \Theta(n \log^{2} n). \]

例题 8 ( 非对称子问题的递推 ) 某递归算法满足 \[ T(n) = T(n/2) + T(n/4) + n, \quad T(1)=1. \] 求 \(T(n)\) 的渐进复杂度.

解 用递归树估计每一层的总工作量.

第 0 层代价为 \(n\). 第 1 层两个子问题代价之和为 \(\frac{n}{2} + \frac{n}{4} = \frac{3n}{4}\). 第 2 层总规模继续按同样比例衰减，不会超过上一层的 \(\frac{3}{4}\) 倍.

因此各层总代价形成等比级数 \[ n + \frac{3n}{4} + \left(\frac{3}{4}\right)^2 n + \cdots < \frac{n}{1-3/4} = 4n. \] 递归深度为 \(O(\log n)\)，但每层总代价在等比下降，整体被常数倍 \(n\) 上界住. 因此 \(T(n)=\Theta(n)\).

1.7 本章疑难点

1. 数据元素 vs 数据对象

  • 数据元素是基本单位，数据对象是一类数据元素的集合.
  • 数据对象是数据的子集，不等同于数据元素.

2. 逻辑结构 vs 存储结构

  • 逻辑结构描述关系模型，不依赖计算机.
  • 存储结构是逻辑结构在内存中的实现，直接影响运算效率.

3. 时间复杂度的合并规则

  • 顺序执行取最大值而非相加.
  • 多层循环看乘积，常见求和则化为最高阶.

1.8 本章小结

  • 三要素从逻辑结构、存储结构、数据运算三个角度刻画数据结构.
  • ADT 强调抽象层面的模型与操作，是实现与复用的基础.
  • 算法五大特性与复杂度分析是后续所有章节的底层工具.
