第2章 线性表

2.1 定义与基本操作

线性表 ( Linear List ) 是具有 相同数据类型 的 \(n\) ( \(n \geq 0\) ) 个数据元素的 有限序列，记作 \(L = (a_1, a_2, \ldots, a_n)\).

基本特征

  • 元素个数 \(n\) 称为线性表的 长度，\(n = 0\) 时为 空表.
  • \(a_1\) 为 表头元素，\(a_n\) 为 表尾元素.
  • 除 \(a_1\) 外每个元素有且仅有一个 直接前驱; 除 \(a_n\) 外每个元素有且仅有一个 直接后继.

2.1.1 基本操作

操作 功能说明
InitList(L) 初始化一个空线性表 \(L\)
DestroyList(L) 销毁线性表，释放内存
ClearList(L) 清空线性表，将 \(L\) 置为空表
操作 功能说明
GetElem(L, i, e) 按位查找，取第 \(i\) 个元素赋给 \(e\)
LocateElem(L, e) 按值查找，返回元素 \(e\) 的位序
Length(L) 返回线性表的长度
Empty(L) 判断线性表是否为空
操作 功能说明
ListInsert(L, i, e) 在第 \(i\) 个位置插入元素 \(e\)
ListDelete(L, i, e) 删除第 \(i\) 个元素，并用 \(e\) 返回其值
操作 功能说明
PrintList(L) 按顺序输出线性表的所有元素
Traverse(L, visit()) 遍历线性表，对每个元素调用 visit()

不同教材操作命名可能不同，408 考试中重点掌握 插入、删除、查找 的实现逻辑与复杂度，而非死记操作名称.

2.1.2 408 常考点

  • 位序 通常从 1 开始 ( 第 \(i\) 个元素 )， 数组下标 通常从 0 开始 ( data[i-1] )
  • 题目里若写「第 \(i\) 个元素」，默认 \(1 \le i \le n\)
  • 「直接前驱 / 直接后继」是逻辑概念，和存储方式无关
  • 顺序表判空通常看 length == 0
  • 带头结点的链表判空通常看 L->next == NULL
  • 单链表中「删除结点 p 本身但不知道其前驱」只能用后继覆盖法，且 p 不能是尾结点
结构 按位访问 按值查找 插入删除 ( 已定位结点 ) 插入删除 ( 需先定位 )
顺序表 \(O(1)\) \(O(n)\) \(O(n)\) \(O(n)\)
单链表 \(O(n)\) \(O(n)\) \(O(1)\) \(O(n)\)
  • 不要混淆数组下标与位序 题目中说「删除第 \(i\) 个元素」，对应的数组下标往往是 \(i-1\).
  • 看清题目限制 如果题目要求「空间复杂度 \(O(1)\)」，通常意味着你不能开辟新的数组或链表，必须在原结构上进行 指针操作 或 元素交换.
  • 边界条件必须写 写代码题时，第一步永远是判断 L == NULL 或 length == 0，这通常占 1~2 分.

2.2 顺序存储 ( 顺序表 )

顺序表 用一组 地址连续 的存储单元依次存储线性表的数据元素，逻辑相邻的元素物理地址也相邻.

存储地址公式

\[\text{LOC}(a_i) = \text{LOC}(a_1) + (i - 1) \times \text{sizeof}(\text{ElemType})\]

实现方式对比

方式 特点
静态分配 ( 数组 ) 大小固定，简单，可能溢出或浪费
动态分配 ( malloc ) 可扩容，更灵活，需手动管理内存

2.2.1 插入操作

在第 \(i\) 个位置插入元素 \(e\) ( \(1 \leq i \leq n+1\) ) :

  1. 判断 \(i\) 的合法性，表满则报错.
  2. 将第 \(i \sim n\) 个元素 依次后移 一位.
  3. 将 \(e\) 放入第 \(i\) 个位置.
  4. 表长加 1.

设插入位置等概率，每个位置概率为 \(\dfrac{1}{n+1}\).

\[E_{\text{insert}} = \sum_{i=1}^{n+1} \frac{1}{n+1}(n-i+1) = \frac{n}{2}\]

情况 移动次数 时间复杂度
最好 ( 表尾插入 ) \(0\) \(O(1)\)
最坏 ( 表头插入 ) \(n\) \(O(n)\)
平均 \(n/2\) \(O(n)\)

2.2.2 删除操作

删除第 \(i\) 个元素 ( \(1 \leq i \leq n\) )

  1. 判断 \(i\) 的合法性，表空则报错.
  2. 取出第 \(i\) 个元素保存.
  3. 将第 \(i+1 \sim n\) 个元素 依次前移 一位.
  4. 表长减 1.

设删除位置等概率，每个位置概率为 \(\dfrac{1}{n}\).

\[E_{\text{delete}} = \sum_{i=1}^{n} \frac{1}{n}(n-i) = \frac{n-1}{2}\]

情况 移动次数 时间复杂度
最好 ( 删表尾 ) \(0\) \(O(1)\)
最坏 ( 删表头 ) \(n-1\) \(O(n)\)
平均 \((n-1)/2\) \(O(n)\)

2.2.3 查找操作

类型 时间复杂度 说明
按位查找 \(O(1)\) 直接计算地址，随机访问
按值查找 \(O(n)\) 顺序扫描比较

顺序表的 按位查找 是 \(O(1)\)，这是顺序存储相较于链式存储最大的优势.

2.2.4 常考题型与技巧

删除满足条件的元素且要求 (O(1)) 额外空间时，优先考虑「快慢指针」.

  • 设 i 为写入位置 ( 慢指针 )，j 为扫描位置 ( 快指针 )
  • 当 data[j] 保留时，执行 data[i++] = data[j]
  • 遍历结束后令 length = i

线性表循环左移 / 右移常用「三次逆置」实现，时间 (O(n))，空间 (O(1)).

将长度为 \(n\) 的顺序表左移 \(k\) 位 ( \(0 \le k < n\) ).

  1. 逆置前 \(k\) 个元素 Reverse(0, k-1)
  2. 逆置后 \(n-k\) 个元素 Reverse(k, n-1)
  3. 逆置整个表 Reverse(0, n-1)

真题示例 (2010年) 将数组中的两个序列前后位置互换，核心就是利用上述的三次逆置法.

  • 有序顺序表删除重复元素可以一次扫描完成，时间 \(O(n)\)
  • 合并两个有序顺序表使用双指针，时间 \(O(m+n)\)
  • 求两个等长有序表的中位数
    • 最优解法 (分治法) 比较两表中位数，舍弃不可能包含全局中位数的一半，时间复杂度 \(O(\log n)\).
    • 次优解法 (双指针) 类似于归并排序的合并过程，当合并到第 \(n\) 个元素时即为中位数，时间复杂度 \(O(n)\)，空间 \(O(1)\).

2.3 链式存储 ( 链表 )

链表用一组 任意 的存储单元存储线性表，每个结点除数据域外还有 指针域 指向后继结点，逻辑相邻的元素物理地址不一定相邻.

2.3.1 头结点与头指针

头指针 指向链表第一个结点的指针.

  • 链表必须有头指针，是链表的 唯一标识.
  • 若链表有头结点，头指针指向 头结点; 否则指向第一个数据结点.
  • 头指针不可为 NULL ( 空表时指向头结点 ).

头结点 链表第一个数据结点之前附设的一个额外结点，数据域可不存有效数据.

  • 头结点是 可选的，但引入后能简化操作.
  • 有头结点时，对第一个数据结点的插删与其他位置 统一处理，无需特判.
  • 头结点的后继为 NULL 时链表为空.
比较项 头指针 头结点
是否必须 必须 可选
空表判断 L == NULL L->next == NULL
操作统一性 首元素需特判 首元素无需特判

2.3.2 单链表

尾插法 新结点插入到链表 末尾，建立的链表元素顺序与输入顺序 相同.

头插法 新结点插入到链表 头部，建立的链表元素顺序与输入顺序 相反.

提示

头插法天然可以实现 链表逆置，常作为逆置算法的基础.

后插操作 在结点 \(p\) 之后插入结点 \(s\)

前插操作 在结点 \(p\) 之前插入结点 \(s\) ( 借助后插实现 )

时间复杂度均为 \(O(1)\) ( 已知前驱结点时 ).

删除结点 \(p\) 的后继结点

删除结点 \(p\) 本身 ( 不知前驱，借助后继实现 ) :

时间复杂度均为 \(O(1)\).

类型 时间复杂度 说明
按位查找 \(O(n)\) 从头遍历，寻找第 \(i\) 个结点
按值查找 \(O(n)\) 从头遍历，比较数据域

单链表 不支持随机访问，按位查找必须从头遍历，这是相较于顺序表的劣势.

头插法逆置 取原链表每个结点头插入新链表.

时间复杂度 \(O(n)\)，空间复杂度 \(O(1)\). 技巧 遇到要求“空间复杂度 \(O(1)\)” 且需要改变元素顺序的链表题，第一时间考虑 就地逆置.

2.3.3 双链表

双链表 每个结点有两个指针域 prior 指向前驱， next 指向后继. 可从任意结点向前或向后遍历.

插入操作 在结点 \(p\) 之后插入结点 \(s\)

插入时指针修改顺序很关键 先处理 \(s\) 的指针，再修改 \(p\) 的 next，避免断链.

删除操作 删除结点 \(p\) 的后继结点 \(q\)

操作 时间复杂度
按值查找 \(O(n)\)
按位查找 \(O(n)\)
已知结点的插入 / 删除 \(O(1)\)

2.3.4 循环链表

循环单链表 尾结点的 next 指向 头结点 而非 NULL.

  • 判空条件 L->next == L
  • 判表尾 p->next == L
  • 任意结点出发可遍历整表，适合 约瑟夫问题 等循环场景.

循环双链表 头结点 prior 指向尾结点，尾结点 next 指向头结点.

  • 判空条件 L->next == L 且 L->prior == L
  • 合并两个循环双链表只需修改 4 个指针，\(O(1)\) 完成.

2.3.5 静态链表

静态链表 用 数组 模拟链表，结点包含数据域 data 和游标 next，next 存放后继元素的 数组下标 而非指针.

  • 适用于 不支持指针 的语言 ( 如早期 Basic/Fortran ).
  • 数组第一个元素通常作为 头结点，游标指向第一个空闲结点.
  • 优点 插入删除不移动元素. 缺点 容量固定，不能随机访问.
比较项 顺序表 链表 ( 动态 ) 静态链表
随机访问 ✓ ✗ ✗
插删效率 低 高 高
存储连续 ✓ ✗ ✓
容量可变 ✗ ✓ ✗

2.3.6 高频题型与方法

对「删除」「插入」「分割」等操作，先引入一个虚拟头结点 dummy 往往能避免首结点特判. 尤其是当头结点可能被删除或者链表本身没有头结点时，虚拟头结点极其好用.

  • 初始化 LNode* dummy = (LNode*)malloc(sizeof(LNode)); dummy->next = head;
  • 用 prev 指向待处理结点的前驱，删结点时执行 prev->next = cur->next
  • 最后返回 head = dummy->next; free(dummy); return head;

非常高频的考点，常用于解决各种链表定位问题.

  • 找中间结点 (2009年真题) fast 每次走两步，slow 每次走一步. 当 fast 走到表尾时，slow 正好在中间.
  • 找倒数第 \(k\) 个结点 先让 fast 领先 \(k\) 步，然后 slow 和 fast 同步移动，直到 fast == NULL，此时 slow 就是倒数第 \(k\) 个结点.
  • 判环与找环入口 fast 和 slow 相遇说明有环. 相遇后，让一个指针回到头结点，然后两个指针都每次走一步，再次相遇的点即为环的入口.

共享尾意味着从某个结点开始，两个链表的 结点地址相同，而不是仅仅值相同. 相交后的拓扑结构必然是 “Y” 型.

找两个单链表的公共结点 * 长度差法 分别遍历求出长度 \(L_1\) 和 \(L_2\). 让较长的链表先走 \(|L_1 - L_2|\) 步，然后两个指针同步前进，第一次遇到的相同结点（地址相同）即为公共结点. 时间 \(O(L_1+L_2)\). * 指针切换法 ( 浪漫相遇法 ) 指针 A 遍历链表 1，结束后跳转到链表 2 的头部；指针 B 遍历链表 2，结束后跳转到链表 1 的头部. 两者必然在交点相遇.

如果题目允许 (O(N)) 的空间复杂度，且要求时间复杂度尽可能低（如 (O(N))），优先考虑 以空间换时间.

  • 绝对值去重 要求删除绝对值相等的结点. 可以开辟一个大小为 \(N\) 的数组（由于绝对值范围已知且较小，通常用 int hash[MAX_SIZE] = {0}）. 遍历链表，若 hash[abs(p->data)] == 1 则删除该结点；否则将 hash[abs(p->data)] 置为 1 并保留结点. 时间 \(O(N)\)，空间 \(O(N)\).

2.4 顺序表 vs. 链表 综合对比

类型 按位存取 按值查找
顺序表 \(O(1)\) 随机存取 \(O(n)\)
链表 \(O(n)\) 顺序存取 \(O(n)\)
类型 操作 时间复杂度 说明
顺序表 插入 / 删除 \(O(n)\) 需移动大量元素
链表 插入 / 删除 \(O(1)\) 已知前驱结点时
链表 插入 / 删除 \(O(n)\) 需先查找目标位置
类型 空间分配 存储密度 适用场景
顺序表 预分配，静态 高 ( 无指针开销 ) 元素个数已知，查找频繁
链表 按需分配，动态 低 ( 有指针域 ) 元素个数未知，插删频繁

选择原则 频繁查找选顺序表; 频繁插删选链表; 元素个数变化大选链表; 存储密度要求高选顺序表.

2.5 线性表的应用

2.5.1 多项式表示与相加

设多项式 \(P(x) = a_0 + a_1 x + a_2 x^2 + \cdots + a_n x^n\).

顺序表表示 下标 \(i\) 对应系数 \(a_i\)，适合 稠密多项式.

链表表示 每个结点存 ( 系数，指数 ) 对，按指数降序排列，适合 稀疏多项式.

设 \(P_a\) 和 \(P_b\) 均按指数降序存储，双指针 \(p\)、\(q\) 分别扫描

  1. 若 \(p \to \text{exp} > q \to \text{exp}\)，取 \(p\) 结点，\(p\) 后移.
  2. 若 \(p \to \text{exp} < q \to \text{exp}\)，取 \(q\) 结点，\(q\) 后移.
  3. 若指数相等，系数相加 非零则保留，为零则丢弃，\(p\)、\(q\) 同时后移.
  4. 剩余部分直接链入结果链表.

时间复杂度 \(O(m + n)\)，\(m\)、\(n\) 为两多项式项数.

2.5.2 有序表合并

将两个非递减有序顺序表 \(A\)、\(B\) 合并为非递减有序顺序表 \(C\)

时间复杂度 \(O(m + n)\)，空间复杂度 \(O(m + n)\).

将两个有序单链表 \(L_a\)、\(L_b\) 合并为有序单链表 \(L_c\)， 利用原结点，不申请新空间

时间复杂度 \(O(m + n)\)，空间复杂度 \(O(1)\) ( 原地合并 ).

2.6 例题精选

例题 1 ( 顺序表删除 ) 设顺序表中元素递增有序，请删除其中所有值为 \(x\) 的元素，要求时间复杂度 \(O(n)\)，空间复杂度 \(O(1)\).

解 顺序表可用双指针压缩.

令 i 为写入位置，j 为扫描位置.

由于每个元素最多被访问与写入常数次，时间复杂度为 \(O(n)\)，空间复杂度为 \(O(1)\).

例题 2 ( 链表逆置 ) 将单链表 \(L\) 就地逆置，不申请额外空间.

解 迭代头插逆置.

时间复杂度为 \(O(n)\)，空间复杂度为 \(O(1)\).

例题 3 ( 找公共结点 ) 两个单链表 \(L_a\)、\(L_b\) 共享一段尾部，找到第一个公共结点.

解 指针切换法更直接.

两指针各自走过的路径长度都为 \(m+n\)，因此会在相交点相遇或同时为 NULL. 时间复杂度为 \(O(m+n)\)，空间复杂度为 \(O(1)\).

例题 4 ( 判断循环链表 ) 判断单链表是否有环，若有环，返回环的入口结点.

解 Floyd 判圈.

  1. slow 每次走 1 步，fast 每次走 2 步，若相遇则有环.
  2. 令 p 从头结点出发，与 slow 同步每次走 1 步，再次相遇点就是入口.

时间复杂度为 \(O(n)\)，空间复杂度为 \(O(1)\).

例题 5 ( 删除倒数第 k 个结点 ) 已知带头结点的单链表 \(L\)，删除倒数第 \(k\) 个结点 ( \(1 \le k \le n\) )，要求一次遍历完成.

解 双指针.

令 fast 先走 \(k\) 步，然后 fast 与 slow 同步走，直到 fast 到达表尾，此时 slow 指向倒数第 \(k\) 个结点的前驱.

时间复杂度为 \(O(n)\)，空间复杂度为 \(O(1)\).

例题 6 ( 有序顺序表删除重复元素 ) 设顺序表元素非递减有序，删除所有重复元素，使得每个值只保留一个，要求时间复杂度 \(O(n)\)，空间复杂度 \(O(1)\).

解 有序性意味着相同元素相邻，可用双指针保留首个出现位置.

时间复杂度为 \(O(n)\)，空间复杂度为 \(O(1)\).

例题 7 ( 链表稳定分割 ) 给定单链表 head 与值 \(x\)，将链表分成两段，使得小于 \(x\) 的结点在前，大于等于 \(x\) 的结点在后，并保持各段内部的相对顺序不变.

解 用两个虚拟头结点分别维护两条链，最后拼接.

每个结点仅处理一次，时间复杂度为 \(O(n)\)，空间复杂度为 \(O(1)\).

例题 8 ( 两链表原地交错合并 ) 已知两个带头结点的单链表 \(A\)、\(B\)，将 \(B\) 的结点依次插入到 \(A\) 中形成交错序列 \(a_1,b_1,a_2,b_2,\ldots\)，要求复用原结点且不申请新结点.

解 同时扫描两表，逐个摘下 \(B\) 的结点插入到 \(A\) 当前结点之后.

时间复杂度为 \(O(m+n)\)，空间复杂度为 \(O(1)\).

2.7 本章疑难点

1. 头插法与尾插法顺序问题

  • 头插法建表后，链表顺序是输入的 逆序，常用于逆置.
  • 尾插法建表需维护 尾指针 \(r\)，链表顺序与输入 相同.

2. 双链表插入指针顺序

  • 必须先完成 \(s\) 的两个指针赋值，再修改 \(p->\) next，否则会丢失后继结点地址.

3. 静态链表与动态链表对比

  • 静态链表游标是 数组下标，不是地址，不能用 p = p->next 遍历，而是 p = A[p].next.

4. 顺序表插删的时间复杂度

  • 时间复杂度是 \(O(n)\)，但若 只在表尾操作 则是 \(O(1)\). 栈的顺序实现正是利用这一点.

5. 单链表删结点但不知道前驱的限制

  • 后继覆盖法本质是删除后继结点并拷贝数据，因此要求 \(p\) 不是尾结点.
  • 若 \(p\) 是尾结点，必须从头遍历找到前驱才能删除.

6. 判环后的入口定位

  • Floyd 判圈第一次相遇只说明有环.
  • 从表头与相遇点分别出发同步走，二者再次相遇的位置才是入口.

7. 相交链表是地址相交

  • 共享尾部指的是两个指针最终指向同一块结点地址，不是结点值相同.
  • 指针切换法不需要显式求长度，适合手写推导与代码实现.

2.8 本章小结

  • 线性表逻辑结构为 一对一，顺序存储和链式存储各有优劣.
  • 顺序表 随机访问 效率高，适合查找密集场景; 链表 插删 效率高，适合频繁修改场景.
  • 双链表、循环链表在单链表基础上扩展，静态链表是数组模拟链表.
  • 多项式相加、有序表合并是链表的经典应用，双指针、虚拟头结点、快慢指针是 408 高频方法.
