第4章 树与二叉树

4.1 树的基本概念

4.1.1 核心术语

术语 定义
结点 树的基本单位，包含数据域和指向子树的指针
根结点 没有前驱的唯一结点
叶结点 ( 终端结点 ) 度为 0 的结点，无子树
分支结点 ( 非终端结点 ) 度大于 0 的结点
结点的度 结点拥有的子树个数
树的度 树中所有结点度的最大值
术语 定义
层次 根结点为第 1 层，其子结点为第 2 层，以此类推
深度 ( 高度 ) 树中结点的最大层次数，即树的层数
结点深度 从根到该结点的路径长度 ( 根深度为 0 或 1，视定义 )
结点高度 从该结点到叶结点的最长路径长度
路径 从结点 \(A\) 到结点 \(B\) 经过的结点序列
路径长度 路径上边的条数
术语 定义
祖先 从根到某结点路径上的所有结点
子孙 某结点子树中的所有结点
双亲 ( 父结点 ) 直接上层的结点
孩子 直接下层的结点
兄弟 同一双亲的结点
森林 \(m\) ( \(m \geq 0\) ) 棵互不相交的树的集合

深度与高度的区别 深度从根往下数层次，高度从叶往上数层次. 整棵树的深度 = 整棵树的高度 = 最大层数.

重要性质

  • 树中 结点数 = 所有结点度之和 + 1 ( 即 \(n = \sum d_i + 1\) ).
  • 度为 \(m\) 的树中，第 \(i\) 层至多有 \(m^{i-1}\) 个结点.
  • 高度为 \(h\)、度为 \(m\) 的树，最多有 \(\dfrac{m^h - 1}{m - 1}\) 个结点.

4.1.2 树的存储结构

双亲表示法 用一维数组存储结点，每个结点记录 双亲的下标，根结点双亲下标为 \(-1\).

  • 优点 找双亲 \(O(1)\).
  • 缺点 找孩子需遍历全表 \(O(n)\).

孩子表示法 每个结点的孩子构成一个单链表，用数组存储各结点的孩子链表头指针.

  • 优点 找孩子方便.
  • 缺点 找双亲需遍历所有链表，可结合双亲表示法形成 双亲孩子表示法.

孩子兄弟表示法 ( 二叉链表表示法 ) 每个结点含两个指针域 — firstchild ( 第一个孩子 ) 和 nextsibling ( 右兄弟 ).

  • 该表示法可将任意树 转换为二叉树，是树与二叉树互转的基础.
  • 优点 实现方式与二叉链表相同，统一处理.

4.2 二叉树

4.2.1 定义与五大性质

二叉树 是每个结点 最多有两棵子树 的有序树，左子树和右子树严格区分，不能互换.

性质 1 第 \(i\) 层 ( \(i \geq 1\) ) 至多有 \(2^{i-1}\) 个结点.

性质 2 高度为 \(h\) 的二叉树，至多有 \(2^h - 1\) 个结点.

性质 3 对任意二叉树，设叶结点数为 \(n_0\)，度为 2 的结点数为 \(n_2\)，则

\[n_0 = n_2 + 1\]

性质 3 的推导 设度为 1 的结点数为 (n_1)，则总结点数 (n = n_0 + n_1 + n_2). 又边数 = (n - 1 = 0 n_0 + 1 n_1 + 2 n_2)，联立得 (n_0 = n_2 + 1).

性质 4 ( 完全二叉树 ) 含 \(n\) 个结点的完全二叉树，高度为 \(\lceil \log_2(n+1) \rceil\).

性质 5 ( 完全二叉树顺序存储编号规律 ) 对 \(n\) 个结点按层从 1 开始编号，对编号 \(i\) 的结点

条件 结论
\(i > 1\) 双亲编号为 \(\lfloor i/2 \rfloor\)
\(2i \leq n\) 左孩子编号为 \(2i\)
\(2i > n\) 无左孩子 ( 叶结点 )
\(2i+1 \leq n\) 右孩子编号为 \(2i+1\)
\(i \leq \lfloor n/2 \rfloor\) 分支结点

4.2.2 特殊二叉树

满二叉树 每层结点都达到最大值，高度为 \(h\) 的满二叉树有 \(2^h - 1\) 个结点.

  • 所有叶结点都在同一层.
  • 编号规律与完全二叉树相同.
  • 满二叉树 一定是 完全二叉树.

完全二叉树 只有最后一层结点可以不满，且最后一层结点 从左到右连续排列.

  • 完全二叉树 不一定是 满二叉树.
  • 适合 顺序存储，无空间浪费，按 \(2i\)、\(2i+1\) 规律索引父子.
  • \(n\) 个结点的完全二叉树中，若 \(n\) 为奇数，则每个分支结点都有左、右孩子; 若 \(n\) 为偶数，则有一个分支结点只有左孩子.

平衡二叉树 ( AVL ) 任意结点的左、右子树 高度差 绝对值不超过 1，即 \(|\text{BF}| \leq 1\).

  • 平衡因子 \(\text{BF}(v) = h_L - h_R\)，只能取 \(-1,0,1\).
  • 含 \(n\) 个结点的 AVL 树高度 \(O(\log n)\)，查找效率有保证.
  • 最少结点数 \(N(h)\) 满足 \(N(0)=0, N(1)=1, N(2)=2, N(h)=N(h-1)+N(h-2)+1\).

4.2.3 存储结构

顺序存储 按层编号存入数组，编号 \(i\) 的结点存于 data[i]，根存于 data[1].

  • 适合 完全二叉树，空间利用率高.
  • 对一般二叉树，需为空结点占位，最坏情况 ( 右单链树 ) 需 \(2^n - 1\) 个空间存 \(n\) 个结点.

二叉链表 ( 最常用 ) 每个结点包含数据域、左孩子指针 lchild 和右孩子指针 rchild.

  • \(n\) 个结点共有 \(2n\) 个指针域，其中 \(n-1\) 个非空，\(n+1\) 个空指针 ( 可用于 线索化 ).

三叉链表 在二叉链表基础上增加 双亲指针 parent.

  • 方便从任意结点向上追溯祖先，但增加额外空间开销.

4.2.4 遍历 ( 必考 )

二叉树遍历是 整章最高频考点，三种递归遍历、两种非递归遍历、层次遍历以及由遍历序列还原二叉树，务必全部掌握.

前序 ( Pre-order ) 根 → 左 → 右

递归实现

非递归实现 ( 显式栈 )

中序 ( In-order ) 左 → 根 → 右

递归实现

非递归实现

中序遍历 BST 得到的序列是 有序递增序列，常用于判断一棵树是否为 BST.

后序 ( Post-order ) 左 → 右 → 根

递归实现

非递归实现 ( 双栈法 / 标记法 )

层次遍历 ( BFS ) 按层从左到右依次访问，借助 队列 实现.

  • 层次遍历序列可直接用于判断 完全二叉树 出现空结点后不再有非空结点.

由两种遍历序列唯一确定二叉树的条件

已知序列组合 能否唯一确定
前序 + 中序 ✓ 可以
后序 + 中序 ✓ 可以
层次 + 中序 ✓ 可以
前序 + 后序 ✗ 不可以 ( 无法确定左右子树 )

构造方法 ( 以前序 + 中序为例 )

  1. 前序第一个元素为 根.
  2. 在中序序列中找到根的位置，左侧为左子树，右侧为右子树.
  3. 递归构造左、右子树.

4.2.5 线索二叉树

线索二叉树 利用 \(n+1\) 个空指针存储结点的 前驱和后继信息，将空指针改造为线索 ( Thread )，从而无需递归即可遍历.

结点结构

中序线索化 遍历时用 pre 指针记录刚访问的结点，建立前驱 / 后继线索.

中序线索二叉树中查找后继

  • 若 rtag == 1，则 rchild 即为 后继.
  • 若 rtag == 0，则后继为 右子树最左下角 的结点.

中序线索二叉树中查找前驱

  • 若 ltag == 1，则 lchild 即为 前驱.
  • 若 ltag == 0，则前驱为 左子树最右下角 的结点.
线索类型 后继查找 前驱查找 常考类型
中序线索 容易 容易 ★★★
前序线索 容易 需找双亲 ★★☆
后序线索 需找双亲 容易 ★☆☆

4.2.6 408 常考点

  • 二叉链表中，\(n\) 个结点共有 \(2n\) 个孩子指针域
  • 其中非空指针数为 \(n-1\) ( 一棵树的边数 )
  • 因此空指针数为 \(2n-(n-1)=n+1\)，这些空指针可用于线索化
  • 三种深度优先遍历 ( 前/中/后序 ) 时间复杂度均为 \(O(n)\)
  • 递归版本的辅助空间是递归深度 \(h\)，最坏 \(O(n)\)，平衡时 \(O(\log n)\)
  • 非递归版本的辅助栈同理是 \(O(h)\); 层次遍历的队列最坏是该层最大结点数

层次遍历时，如果把空孩子也按位置入队，出现第一个 NULL 之后，后续所有出队元素都必须是 NULL，否则不是完全二叉树.

常见陷阱是只看结点编号规律而不验证「从左到右连续」这一条.

  • 前序 + 中序 / 后序 + 中序 / 层次 + 中序 可以唯一确定一棵二叉树
  • 前序 + 后序 一般不唯一，除非额外给出「满二叉树」等限制
  • 构造时先在中序里定位根，分出左右子树长度，再回到另一序列按长度切分

4.3 树、森林与二叉树的转换

4.3.1 树 ↔︎ 二叉树

规则 ( 孩子兄弟表示法 )

  1. 每个结点的 第一个孩子 成为二叉树中的 左孩子.
  2. 每个结点的 右兄弟 成为二叉树中的 右孩子.
  3. 根结点的右孩子为空 ( 根无兄弟 ).

记忆 左孩子，右兄弟.

规则 ( 逆向操作 )

  1. 二叉树根结点的 左孩子链 还原为原树的 孩子链.
  2. 沿左孩子一直向下，遇到的每个结点的 右孩子链 还原为 兄弟链.

4.3.2 森林 ↔︎ 二叉树

  • 森林 → 二叉树 将森林中每棵树各自转为二叉树，然后将第 2 棵树的根作为第 1 棵树二叉树根的 右孩子，第 3 棵树的根作为第 2 棵树二叉树根的 右孩子，依此类推.
  • 二叉树 → 森林 沿根结点的右孩子链拆分，每棵以右孩子链各结点为根的子树还原为独立的树.

4.3.3 树 / 森林的遍历

遍历方式 规则 对应二叉树遍历
先根遍历 先访问根，再依次先根遍历子树 前序遍历
后根遍历 先依次后根遍历子树，再访问根 中序遍历

树的后根遍历对应其转化后二叉树的 中序遍历，不是后序遍历，常作为考题陷阱.

遍历方式 规则 对应二叉树遍历
先序遍历 访问第一棵树根，先序遍历第一棵树子树，再先序遍历剩余森林 前序遍历
中序遍历 中序遍历第一棵树，再访问根，再中序遍历剩余森林 中序遍历

4.4 二叉排序树 ( BST )

4.4.1 定义与性质

二叉排序树 ( BST )

若左子树非空，左子树所有结点值 < 根;

若右子树非空，右子树所有结点值 > 根;

左、右子树也分别为二叉排序树.

  • 中序遍历 BST 得到的序列是 递增有序序列.
  • BST 不允许重复关键字 ( 408 默认 ).

4.4.2 查找、插入与删除

  • 成功 找到关键字等于 key 的结点.
  • 失败 到达 NULL.
  • 时间复杂度 最好 \(O(\log n)\)，最坏 \(O(n)\) ( 单链树 )，平均 \(O(\log n)\).
  • 新结点 一定插入为叶结点.
  • 插入序列不同，BST 形态不同，性能差异巨大.

删除结点 \(v\) 的三种情况

  1. \(v\) 是叶结点 直接删除.
  2. \(v\) 只有左子树或只有右子树 用子树替代 \(v\) 的位置.
  3. \(v\) 有左右子树 用 \(v\) 的 中序前驱 ( 左子树最右结点 ) 或 中序后继 ( 右子树最左结点 ) 替换 \(v\)，再删除替换结点.

4.4.3 平衡二叉树 ( AVL )

AVL 树 是在 BST 基础上加入 平衡条件 的查找树，任意结点的平衡因子 \(|\text{BF}| \leq 1\)，保证树高 \(O(\log n)\).

四种失衡旋转

LL 型 在 A 的 左孩子的左子树 中插入导致失衡，A 的 BF 由 1 变为 2.

解法 对 A 做 右单旋.

RR 型 在 A 的 右孩子的右子树 中插入导致失衡，A 的 BF 由 \(-1\) 变为 \(-2\).

解法 对 A 做 左单旋.

LR 型 在 A 的 左孩子的右子树 中插入导致失衡.

解法 先对 B 做 左旋，再对 A 做 右旋 ( 双旋 ).

RL 型 在 A 的 右孩子的左子树 中插入导致失衡.

解法 先对 B 做 右旋，再对 A 做 左旋 ( 双旋 ).

判断旋转类型的口诀 看新结点插入的位置 — 在失衡结点的左左 → LL 右旋; 右右 → RR 左旋; 左右 → LR 双旋; 右左 → RL 双旋.

4.4.4 408 常考点

  • 充要条件 ( 关键字互异 ) 一棵二叉树是 BST ⇔ 其中序遍历序列严格递增
  • 已知插入序列构造 BST 时，树形高度取决于序列「有序程度」，极端情况下退化为单链
  • 删除度为 2 的结点时，用 中序前驱 或 中序后继 替代都合法
  • 替代后要继续删除「被搬上来」的那个结点，且该结点一定最多只有一棵子树
  • AVL 的最少结点数 \(N(h)\) 满足 \[N(0)=0,\ N(1)=1,\ N(h)=N(h-1)+N(h-2)+1\]
  • 常用结论 AVL 高度为 \(O(\log n)\)，因此查找/插入/删除最坏均为 \(O(\log n)\)
  • BST 的查找复杂度与树高 \(h\) 成正比 \(O(h)\)
  • 最坏 \(h=n\) ( 单链树 )，平均常取 \(O(\log n)\) 的前提是插入序列近似随机

4.5 哈夫曼树 ( Huffman Tree )

4.5.1 定义与 WPL

哈夫曼树 ( 最优二叉树 ) 带权路径长度 ( WPL ) 最小的二叉树.

\[\text{WPL} = \sum_{i=1}^{n} w_i \times l_i\]

  • \(w_i\) 第 \(i\) 个叶结点的 权值.
  • \(l_i\) 第 \(i\) 个叶结点到根的 路径长度 ( 层数 - 1 ).

4.5.2 构造算法

  • 含 \(n\) 个叶结点的哈夫曼树共有 \(2n - 1\) 个结点，度只有 0 和 2，无度为 1 的结点.
  • 哈夫曼树 不唯一，但 WPL 唯一.

4.5.3 哈夫曼编码

哈夫曼编码 根据哈夫曼树，左分支编 0，右分支编 1，从根到叶结点的路径即为该字符的编码.

  • 前缀编码 任何编码都不是其他编码的前缀，保证解码唯一. 哈夫曼编码天然满足前缀编码.
  • 高频字符路径短 → 编码短; 低频字符路径长 → 编码长. 平均编码长度最短.

权值集合 \(\{1, 3, 5, 7\}\) 的哈夫曼树构造过程

WPL \(= 1 \times 3 + 3 \times 3 + 5 \times 2 + 7 \times 1 = 3 + 9 + 10 + 7 = 29\).

编码 权 1 → 000，权 3 → 001，权 5 → 01，权 7 → 1.

解码过程 从根结点出发，读入一位编码，0 走左、1 走右，到达叶结点输出字符，再返回根继续，直到编码串读完.

  • 前缀编码保证解码过程 无歧义.

4.5.4 408 常考点

  • 叶结点层数从 1 开始时，路径长度 \(l = \text{层数}-1\)
  • WPL 只看叶结点，不把内部结点权值计入
  • 合并两棵树时，新根权值会在后续合并中重复参与，等价于把被合并的两权值各自「深度 +1」
  • 反复取最小两棵树合并，最适合用 小顶堆 维护权值集合
  • 若用小顶堆，构造复杂度为 \(O(n \log n)\); 若线性扫描取最小两项则为 \(O(n^2)\)
  • 权值相等时合并顺序可不同，因此树形与编码可能不同
  • 但 WPL 不变，且编码仍满足前缀编码

4.6 堆 ( Heap )

4.6.1 定义与存储

堆 是满足如下性质的 完全二叉树 * 大顶堆 ( 最大堆 ) 每个结点的值 ≥ 其子结点的值，根为最大值. * 小顶堆 ( 最小堆 ) 每个结点的值 ≤ 其子结点的值，根为最小值.

堆用 数组 存储，下标从 1 开始，结点 \(i\) 的左孩子为 \(2i\)，右孩子为 \(2i+1\)，双亲为 \(\lfloor i/2 \rfloor\).

4.6.2 建堆

建堆 ( 从最后一个分支结点 \(\lfloor n/2 \rfloor\) 开始向上做下沉 siftDown )

  • 时间复杂度 \(O(n)\) ( 逐层分析求和 )，优于逐个插入的 \(O(n \log n)\).

4.6.3 插入与删除

插入 将新元素追加到数组末尾，然后 上浮 ( siftUp ) 到合适位置.

时间复杂度 \(O(\log n)\).

删除堆顶 将堆顶与末尾元素互换，堆大小减 1，然后对新堆顶做 下沉 ( siftDown ).

时间复杂度 \(O(\log n)\).

堆排序

  1. 建大顶堆 \(O(n)\).
  2. 循环 \(n-1\) 次 每次将堆顶与末尾元素交换，堆大小减 1，再对堆顶 siftDown \(O(n \log n)\).

总时间复杂度 \(O(n \log n)\)，空间复杂度 \(O(1)\)，不稳定.

4.6.4 堆的应用 ( 2025/2026 新增 )

优先队列 用堆实现，每次出队 最高优先级 ( 最大或最小值 ) 的元素.

  • 入队 插入堆，\(O(\log n)\).
  • 出队 删除堆顶，\(O(\log n)\).
  • 取最值 读堆顶，\(O(1)\).
  • 应用 任务调度、Dijkstra 最短路、Prim 最小生成树.

TopK 问题 从 \(n\) 个元素中找最大的 \(K\) 个.

最优方案 — 小顶堆

  1. 用前 \(K\) 个元素建小顶堆，\(O(K)\).
  2. 遍历剩余 \(n-K\) 个元素，若当前元素 > 堆顶，则替换堆顶并 siftDown，\(O(\log K)\).
  3. 遍历结束后堆中即为最大的 \(K\) 个元素.

总时间复杂度 \(O(n \log K)\)，优于排序 \(O(n \log n)\)，适合 \(K \ll n\) 的场景.

找最大 (K) 个 → 用 小顶堆; 找最小 (K) 个 → 用 大顶堆. 堆的大小始终维持在 (K).

动态中位数 数据流中随时查询当前中位数.

双堆方案

  • 大顶堆 \(L\) 存较小的一半，堆顶为较小一半的最大值.
  • 小顶堆 \(R\) 存较大的一半，堆顶为较大一半的最小值.
  • 维持 \(|L| - |R| \leq 1\)，中位数 = \(L\) 的堆顶 ( 奇数时 ) 或 \((L\text{堆顶} + R\text{堆顶}) / 2\) ( 偶数时 ).

插入操作

  1. 新元素 \(x \leq L\) 堆顶 → 插入 \(L\); 否则插入 \(R\).
  2. 若两堆大小差超过 1，将较大堆的堆顶移入较小堆.

每次插入 \(O(\log n)\)，查询中位数 \(O(1)\).

4.6.5 408 常考点

  • 从 \(\lfloor n/2 \rfloor\) 到 1 依次 siftDown，总复杂度为 \(O(n)\)
  • 直观理解 越靠近叶子的结点高度越小，下沉步数越少，总步数求和是线性的
  • 时间 \(O(n \log n)\)，空间 \(O(1)\)
  • 不稳定排序
  • 相比快排，最坏时间更稳，但常数与缓存友好性通常更差
  • \(K \ll n\) 时用堆做 TopK 是典型最优解 \(O(n \log K)\)
  • \(K\) 接近 \(n\) 时堆与排序差距缩小，可视实现与常数选择

4.7 例题精选

例题 1 ( 二叉树结点数 ) 已知二叉树有 50 个叶结点，则该树中度为 2 的结点数为多少 ?

解

由性质 \(n_0 = n_2 + 1\)，得 \(n_2 = n_0 - 1 = 49\).

例题 2 ( 由遍历序列构造二叉树 )

前序序列 ABCDEFG，中序序列 CBDAEFG，求后序序列.

解

前序第一个 A 为根; 中序中 A 左侧 CBD 为左子树，右侧 EFG 为右子树.

左子树 前序 BCD，中序 CBD → B 为根，C 为左孩子，D 为右孩子.

右子树 前序 EFG，中序 EFG → E 为根，F 为左孩子，G 为右孩子.

后序 CDBFGEA.

例题 3 ( AVL 旋转 )

在 AVL 树中依次插入 10、20、30，说明插入过程并指出旋转类型.

解

插入 10 树为单结点，平衡.

插入 20 右孩子，BF(10) = \(-1\)，平衡.

插入 30 BF(10) = \(-2\)，失衡，10 的右孩子 20 的右子树中 插入，属于 RR 型，对 10 做 左单旋，20 成为新根，10 为左孩子，30 为右孩子.

例题 4 ( 哈夫曼树 WPL )

字符频率 A→4，B→2，C→6，D→3，E→5，求哈夫曼树的 WPL.

解

排序 {2,3,4,5,6}.

合并 2+3=5 → {4,5,5,6}.

合并 4+5=9 → {5,6,9}.

合并 5+6=11 → {9,11}.

合并 9+11=20 → {20}.

WPL \(= 2\times3 + 3\times3 + 4\times2 + 5\times2 + 6\times2 = 6+9+8+10+12 = 45\).

例题 5 ( TopK )

在 \(10^7\) 个整数中找最大的 100 个，比较三种方案的效率.

解

方案 时间复杂度 空间复杂度
全部排序 \(O(n \log n)\) \(O(n)\)
全部扫描 + 堆 \(O(n \log K)\) \(O(K)\)
分治 + 快选 \(O(n)\) 期望 \(O(\log n)\)

最优实践 维护大小为 100 的 小顶堆，扫描一遍，\(O(n \log K)\) 且空间仅 \(O(K)\).

4.8 本章疑难点

1. 树的后根遍历 ≠ 二叉树后序遍历

  • 树的后根遍历对应其孩子兄弟表示法转化后二叉树的中序遍历. 这是高频易错点.

2. 完全二叉树与满二叉树

  • 满二叉树 ⊂ 完全二叉树，满二叉树一定是完全二叉树，反之不一定.

3. BST 删除操作

  • 删除有两棵子树的结点时，选择中序前驱或后继替代均合法，但两者得到的 BST 形态不同.

4. AVL 旋转后子树归属

  • 旋转时注意被旋转结点的子树重新挂接，不能遗漏. 例如 LL 旋转后，B 的右子树变为 A 的左子树.

5. 哈夫曼树无度为 1 的结点

  • 哈夫曼树只有叶结点 ( 度 0 ) 和度为 2 的结点，不存在度为 1 的结点.

6. TopK 的堆类型选择

  • 找最大 \(K\) 个用小顶堆 ( 淘汰小的 ) ; 找最小 \(K\) 个用大顶堆 ( 淘汰大的 ). 逻辑相反，不能弄错.

4.9 本章小结

  • 树是非线性结构，孩子兄弟表示法是树与二叉树互转的桥梁.
  • 二叉树五大性质、四种遍历 ( 含非递归 ) 、线索化是核心考点，必须完全掌握.
  • BST 中序有序，AVL 四种旋转保持平衡，两者共同构成查找树基础.
  • 哈夫曼树保证最小 WPL，哈夫曼编码是前缀编码，是数据压缩的理论基础.
  • 堆是完全二叉树的特化，建堆 \(O(n)\)、增删 \(O(\log n)\)，支撑优先队列、TopK、动态中位数等重要应用.
