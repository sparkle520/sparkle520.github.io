第4章 树与二叉树

4.1 树的基本概念

4.1.1 核心术语

术语 定义
结点 树的基本单位，包含数据域和指向子树的指针
根结点 没有前驱的唯一结点
叶结点 ( 终端结点 ) 度为 0 的结点，无子树
分支结点 ( 非终端结点 ) 度大于 0 的结点
结点的度 结点拥有的子树个数
树的度 树中所有结点度的最大值
术语 定义
层次 根结点为第 1 层，其子结点为第 2 层，以此类推
深度 ( 高度 ) 树中结点的最大层次数，即树的层数
结点深度 从根到该结点的路径长度 ( 根深度为 0 或 1，视定义 )
结点高度 从该结点到叶结点的最长路径长度
路径 从结点 \(A\) 到结点 \(B\) 经过的结点序列
路径长度 路径上边的条数
术语 定义
祖先 从根到某结点路径上的所有结点
子孙 某结点子树中的所有结点
双亲 ( 父结点 ) 直接上层的结点
孩子 直接下层的结点
兄弟 同一双亲的结点
森林 \(m\) ( \(m \geq 0\) ) 棵互不相交的树的集合

注意 深度与高度的区别 深度从根往下数层次，高度从叶往上数层次. 整棵树的深度 = 整棵树的高度 = 最大层数.

重要性质

  • 树中 结点数 = 所有结点度之和 + 1 ( 即 \(n = \sum d_i + 1\) ).
  • 度为 \(m\) 的树中，第 \(i\) 层至多有 \(m^{i-1}\) 个结点.
  • 高度为 \(h\)、度为 \(m\) 的树，最多有 \(\dfrac{m^h - 1}{m - 1}\) 个结点.

4.1.2 树的存储结构

双亲表示法 用一维数组存储结点，每个结点记录 双亲的下标，根结点双亲下标为 \(-1\).

  • 优点 找双亲 \(O(1)\).
  • 缺点 找孩子需遍历全表 \(O(n)\).

孩子表示法 每个结点的孩子构成一个单链表，用数组存储各结点的孩子链表头指针.

  • 优点 找孩子方便.
  • 缺点 找双亲需遍历所有链表，可结合双亲表示法形成 双亲孩子表示法.

孩子兄弟表示法 ( 二叉链表表示法 ) 每个结点含两个指针域 — firstchild ( 第一个孩子 ) 和 nextsibling ( 右兄弟 ).

  • 该表示法可将任意树 转换为二叉树，是树与二叉树互转的基础.
  • 优点 实现方式与二叉链表相同，统一处理.

4.2 二叉树

4.2.1 定义与五大性质

定义 二叉树 是每个结点 最多有两棵子树 的有序树，左子树和右子树严格区分，不能互换.

性质 1 第 \(i\) 层 ( \(i \geq 1\) ) 至多有 \(2^{i-1}\) 个结点.

性质 2 高度为 \(h\) 的二叉树，至多有 \(2^h - 1\) 个结点.

性质 3 对任意二叉树，设叶结点数为 \(n_0\)，度为 2 的结点数为 \(n_2\)，则

\[n_0 = n_2 + 1\]

性质 3 的推导 设度为 1 的结点数为 \(n_1\)，则总结点数 \(n = n_0 + n_1 + n_2\). 又边数 = \(n - 1 = 0 \cdot n_0 + 1 \cdot n_1 + 2 \cdot n_2\)，联立得 \(n_0 = n_2 + 1\).

性质 4 ( 完全二叉树 ) 含 \(n\) 个结点的完全二叉树，高度为 \(\lceil \log_2(n+1) \rceil\).

性质 5 ( 完全二叉树顺序存储编号规律 ) 对 \(n\) 个结点按层从 1 开始编号，对编号 \(i\) 的结点

条件 结论
\(i > 1\) 双亲编号为 \(\lfloor i/2 \rfloor\)
\(2i \leq n\) 左孩子编号为 \(2i\)
\(2i > n\) 无左孩子 ( 叶结点 )
\(2i+1 \leq n\) 右孩子编号为 \(2i+1\)
\(i \leq \lfloor n/2 \rfloor\) 分支结点

4.2.2 特殊二叉树

满二叉树 每层结点都达到最大值，高度为 \(h\) 的满二叉树有 \(2^h - 1\) 个结点.

  • 所有叶结点都在同一层.
  • 编号规律与完全二叉树相同.
  • 满二叉树 一定是 完全二叉树.

完全二叉树 只有最后一层结点可以不满，且最后一层结点 从左到右连续排列.

  • 完全二叉树 不一定是 满二叉树.
  • 适合 顺序存储，无空间浪费，按 \(2i\)、\(2i+1\) 规律索引父子.
  • \(n\) 个结点的完全二叉树中，若 \(n\) 为奇数，则每个分支结点都有左、右孩子; 若 \(n\) 为偶数，则有一个分支结点只有左孩子.

平衡二叉树 ( AVL ) 任意结点的左、右子树 高度差 绝对值不超过 1，即 \(|\text{BF}| \leq 1\).

  • 平衡因子 \(\text{BF}(v) = h_L - h_R\)，只能取 \(-1,0,1\).
  • 含 \(n\) 个结点的 AVL 树高度 \(O(\log n)\)，查找效率有保证.
  • 最少结点数 \(N(h)\) 满足 \(N(0)=0, N(1)=1, N(2)=2, N(h)=N(h-1)+N(h-2)+1\).

4.2.3 存储结构

顺序存储 按层编号存入数组，编号 \(i\) 的结点存于 data[i]，根存于 data[1].

  • 适合 完全二叉树，空间利用率高.
  • 对一般二叉树，需为空结点占位，最坏情况 ( 右单链树 ) 需 \(2^n - 1\) 个空间存 \(n\) 个结点.

二叉链表 ( 最常用 ) 每个结点包含数据域、左孩子指针 lchild 和右孩子指针 rchild.

  • \(n\) 个结点共有 \(2n\) 个指针域，其中 \(n-1\) 个非空，\(n+1\) 个空指针 ( 可用于 线索化 ).

三叉链表 在二叉链表基础上增加 双亲指针 parent.

  • 方便从任意结点向上追溯祖先，但增加额外空间开销.

4.2.5 线索二叉树

定义 线索二叉树 利用 \(n+1\) 个空指针存储结点的 前驱和后继信息，将空指针改造为线索 ( Thread )，从而无需递归即可遍历.

结点结构

中序线索化 遍历时用 pre 指针记录刚访问的结点，建立前驱 / 后继线索.

中序线索二叉树中查找后继

  • 若 rtag == 1，则 rchild 即为 后继.
  • 若 rtag == 0，则后继为 右子树最左下角 的结点.

中序线索二叉树中查找前驱

  • 若 ltag == 1，则 lchild 即为 前驱.
  • 若 ltag == 0，则前驱为 左子树最右下角 的结点.

4.2.6 408 常考点

  • 二叉链表中，\(n\) 个结点共有 \(2n\) 个孩子指针域
  • 其中非空指针数为 \(n-1\) ( 一棵树的边数 )
  • 因此空指针数为 \(2n-(n-1)=n+1\)，这些空指针可用于线索化
  • 三种深度优先遍历 ( 前/中/后序 ) 时间复杂度均为 \(O(n)\)
  • 递归版本的辅助空间是递归深度 \(h\)，最坏 \(O(n)\)，平衡时 \(O(\log n)\)
  • 非递归版本的辅助栈同理是 \(O(h)\); 层次遍历的队列最坏是该层最大结点数

层次遍历时，如果把空孩子也按位置入队，出现第一个 NULL 之后，后续所有出队元素都必须是 NULL，否则不是完全二叉树.

常见陷阱是只看结点编号规律而不验证「从左到右连续」这一条.

  • 前序 + 中序 / 后序 + 中序 / 层次 + 中序 可以唯一确定一棵二叉树
  • 前序 + 后序 一般不唯一，除非额外给出「满二叉树」等限制
  • 构造时先在中序里定位根，分出左右子树长度，再回到另一序列按长度切分

仅先序序列的情况

  • 给定先序序列，能确定的不同二叉树个数 = 卡特兰数 \(C_n\)
  • 卡特兰数递归关系 : \(C(n) = \sum_{k=0}^{n-1} C(k) \times C(n-1-k)\)
  • 常见卡特兰数 : \(C_0=1, C_1=1, C_2=2, C_3=5, C_4=14, C_5=42\)
  • 快速计算 : 对于选择题，记住前几个卡特兰数即可

408 考察重点

  1. 选择题 : 给定结点数 \(n\)，选择对应的卡特兰数 \(C_n\)
  2. 判断题 : 判断”仅先序序列能唯一确定二叉树”的真假
  3. 计算题 : 计算给定结点数的不同二叉树个数
  4. 简答题 : 解释为什么先序序列不能唯一确定二叉树

解题技巧

  • 记住卡特兰数的前几个值
  • 理解递归关系：左子树 \(k\) 个结点，右子树 \(n-1-k\) 个结点
  • 注意：先序序列的第一个元素一定是根结点

4.3 树、森林与二叉树的转换

4.3.1 树 ↔︎ 二叉树

定义 树与二叉树的转换基于 孩子兄弟表示法，这种转换保持了树的结构信息，使得树的操作可以在二叉树上进行.

转换规则 ( 孩子兄弟表示法 )

  1. 左孩子规则 : 每个结点的 第一个孩子 成为二叉树中的 左孩子.
  2. 右兄弟规则 : 每个结点的 右兄弟 成为二叉树中的 右孩子.
  3. 根结点规则 : 根结点的右孩子为空 ( 根无兄弟 ).

记忆口诀 : 左孩子，右兄弟

详细步骤

  1. 在树中，对每个结点，将其 第一个孩子 作为二叉树中的 左孩子
  2. 将该结点的 其他孩子 ( 从左到右 ) 依次作为前一个孩子的 右孩子
  3. 根结点没有兄弟，所以其右孩子为空

示例 树结构

转换过程

  • A 的第一个孩子 B → A 的左孩子
  • B 的兄弟 C → B 的右孩子
  • C 的兄弟 D → C 的右孩子
  • B 的第一个孩子 E → B 的左孩子
  • E 的兄弟 F → E 的右孩子

转换结果

性质 : 转换后的二叉树 根结点没有右子树

转换规则 ( 逆向操作 )

  1. 孩子链还原 : 二叉树根结点的 左孩子链 还原为原树的 孩子链
  2. 兄弟链还原 : 沿左孩子一直向下，遇到的每个结点的 右孩子链 还原为 兄弟链

详细步骤

  1. 二叉树的根结点作为树的根结点
  2. 根结点的左孩子作为树的第一个孩子
  3. 左孩子的右孩子作为树的第二个孩子
  4. 继续处理右孩子链，直到遇到空指针
  5. 对每个结点的左子树递归处理

示例 二叉树结构 :

转换过程

  • A 的左孩子 B → A 的第一个孩子
  • B 的右孩子 C → A 的第二个孩子
  • C 的右孩子 D → A 的第三个孩子
  • B 的左孩子 E → B 的第一个孩子
  • E 的右孩子 F → B 的第二个孩子

转换结果

性质 : 只有根结点没有右子树的二叉树才能转换为树

结构对应关系

树结构 二叉树结构
根结点 根结点
第一个孩子 左孩子
其他孩子 右孩子链
兄弟关系 右孩子关系

遍历对应关系

  • 树的 先根遍历 ↔︎ 二叉树的 前序遍历
  • 树的 后根遍历 ↔︎ 二叉树的 中序遍历

应用场景

  • 树的操作可以在二叉树上实现
  • 二叉树的算法可以应用于树结构
  • 简化树结构的存储和操作

4.3.2 森林 ↔︎ 二叉树

定义 森林是 m ( m ≥ 0 ) 棵互不相交的树的集合. 森林与二叉树的转换可以看作是树与二叉树转换的扩展.

转换规则

  1. 单树转换 : 将森林中的每棵树分别转换为二叉树
  2. 连接规则 : 将第 2 棵树的根作为第 1 棵树二叉树根的 右孩子
  3. 链式连接 : 将第 3 棵树的根作为第 2 棵树二叉树根的 右孩子，依此类推

详细步骤

  1. 将森林中的第一棵树转换为二叉树
  2. 将第二棵树的根结点作为第一棵树根结点的右孩子
  3. 将第三棵树的根结点作为第二棵树根结点的右孩子
  4. 重复此过程，直到所有树都连接完成

示例 森林结构

转换过程

  1. 将树1转换为二叉树
  2. 将树2的根D作为A的右孩子
  3. 将树3的根G作为D的右孩子
  4. 分别转换树2和树3的内部结构

最终结果

性质 : 森林转换后的二叉树，根结点的右子树非空

转换规则

  1. 拆分规则 : 沿根结点的右孩子链拆分
  2. 子树还原 : 每棵以右孩子链各结点为根的子树还原为独立的树
  3. 递归处理 : 对每个子树进行二叉树到树的转换

详细步骤

  1. 二叉树的根结点作为森林中第一棵树的根
  2. 根结点的左子树转换为第一棵树的结构
  3. 根结点的右孩子作为第二棵树的根
  4. 继续处理右孩子链，直到遇到空指针

示例 二叉树结构

转换过程

  1. A 作为第一棵树的根
  2. A 的左孩子 B 转换为 A 的孩子
  3. A 的右孩子 D 作为第二棵树的根
  4. D 的右孩子 E 作为第三棵树的根
  5. E 的右孩子 F 作为第四棵树的根
  6. F 的右孩子 G 作为第五棵树的根
  7. 分别转换各子树的结构

转换结果

性质 : 任何二叉树都可以转换为森林

结构对应关系

森林结构 二叉树结构
第一棵树的根 二叉树的根
其他树的根 根结点的右孩子链
每棵树的结构 对应子树的结构

遍历对应关系

  • 森林的 先序遍历 ↔︎ 二叉树的 前序遍历
  • 森林的 中序遍历 ↔︎ 二叉树的 中序遍历

应用场景

  • 森林的操作可以在二叉树上实现

4.4 二叉排序树 ( BST )

4.4.1 定义与性质

定义 二叉排序树 ( BST )

若左子树非空，左子树所有结点值 < 根;

若右子树非空，右子树所有结点值 > 根;

左、右子树也分别为二叉排序树.

  • 中序遍历 BST 得到的序列是 递增有序序列.
  • BST 不允许重复关键字 ( 408 默认 ).

4.4.2 查找、插入与删除

  • 成功 找到关键字等于 key 的结点.
  • 失败 到达 NULL.
  • 时间复杂度 最好 \(O(\log n)\)，最坏 \(O(n)\) ( 单链树 )，平均 \(O(\log n)\).
  • 新结点 一定插入为叶结点.
  • 插入序列不同，BST 形态不同，性能差异巨大.

删除结点 \(v\) 的三种情况

  1. \(v\) 是叶结点 直接删除.
  2. \(v\) 只有左子树或只有右子树 用子树替代 \(v\) 的位置.
  3. \(v\) 有左右子树 用 \(v\) 的 中序前驱 ( 左子树最右结点 ) 或 中序后继 ( 右子树最左结点 ) 替换 \(v\)，再删除替换结点.

4.4.3 平衡二叉树 ( AVL )

定义 AVL 树 是在 BST 基础上加入 平衡条件 的查找树，任意结点的平衡因子 \(|\text{BF}| \leq 1\)，保证树高 \(O(\log n)\).

四种失衡旋转

LL 型 在 A 的 左孩子的左子树 中插入导致失衡，A 的 BF 由 1 变为 2.

解法 对 A 做 右单旋.

RR 型 在 A 的 右孩子的右子树 中插入导致失衡，A 的 BF 由 \(-1\) 变为 \(-2\).

解法 对 A 做 左单旋.

LR 型 在 A 的 左孩子的右子树 中插入导致失衡.

解法 先对 B 做 左旋，再对 A 做 右旋 ( 双旋 ).

RL 型 在 A 的 右孩子的左子树 中插入导致失衡.

解法 先对 B 做 右旋，再对 A 做 左旋 ( 双旋 ).

注意 判断旋转类型的口诀 看新结点插入的位置 — 在失衡结点的左左 → LL 右旋; 右右 → RR 左旋; 左右 → LR 双旋; 右左 → RL 双旋.

4.4.4 408 常考点

  • 充要条件 ( 关键字互异 ) 一棵二叉树是 BST ⇔ 其中序遍历序列严格递增
  • 已知插入序列构造 BST 时，树形高度取决于序列「有序程度」，极端情况下退化为单链
  • 删除度为 2 的结点时，用 中序前驱 或 中序后继 替代都合法
  • 替代后要继续删除「被搬上来」的那个结点，且该结点一定最多只有一棵子树
  • AVL 的最少结点数 \(N(h)\) 满足 \[N(0)=0,\ N(1)=1,\ N(h)=N(h-1)+N(h-2)+1\]
  • 常用结论 AVL 高度为 \(O(\log n)\)，因此查找/插入/删除最坏均为 \(O(\log n)\)
  • BST 的查找复杂度与树高 \(h\) 成正比 \(O(h)\)
  • 最坏 \(h=n\) ( 单链树 )，平均常取 \(O(\log n)\) 的前提是插入序列近似随机

4.5 哈夫曼树 ( Huffman Tree )

定义 哈夫曼树 ( Huffman Tree ) 由 David A. Huffman 于 1952 年提出，是一种 带权路径长度 ( WPL ) 最小 的二叉树，也称为 最优二叉树 ( Optimal Binary Tree ). 哈夫曼树在数据压缩、编码等领域有广泛应用.

4.5.1 核心概念与定义

带权路径长度 ( Weighted Path Length, WPL ) 是衡量二叉树效率的重要指标

\[\text{WPL} = \sum_{i=1}^{n} w_i \times l_i\]

  • \(w_i\) : 第 \(i\) 个叶结点的 权值 ( 通常表示字符频率或重要性 )
  • \(l_i\) : 第 \(i\) 个叶结点到根的 路径长度 ( 边数，即层数 - 1 )
  • \(n\) : 叶结点总数

示例 对于权值集合 \(\{2, 3, 4, 5\}\)，不同二叉树的 WPL 不同

哈夫曼树的目标是找到 WPL 最小 的二叉树结构.

哈夫曼树的基本性质

  1. WPL 最小性 : 在所有可能的二叉树中，哈夫曼树的 WPL 最小
  2. 度分布特性 : 含 \(n\) 个叶结点的哈夫曼树共有 \(2n - 1\) 个结点，度只有 0 和 2，无度为 1 的结点
  3. 结构不唯一性 : 当权值相等时，合并顺序可能不同，导致树形不同，但 WPL 唯一
  4. 贪心算法 : 哈夫曼树的构造采用贪心策略，每次合并权值最小的两棵树

数学性质

  • 哈夫曼树的平均编码长度 \(L_{\text{avg}} = \frac{\text{WPL}}{\sum w_i}\)
  • 对于给定的权值集合，哈夫曼编码的平均长度最短

哈夫曼树的应用场景

  1. 数据压缩 : 哈夫曼编码是最经典的无损压缩算法
  2. 文件编码 : ZIP、GZIP、JPEG 等格式使用哈夫曼编码
  3. 通信协议 : 在数据传输中优化编码效率
  4. 图像处理 : 用于图像压缩和编码
  5. 数据库系统 : 优化数据存储和查询效率

历史意义

  • 1952 年由 David A. Huffman 在 MIT 研究生课程中提出
  • 解决了当时数据压缩领域的关键问题
  • 至今仍是计算机科学教育中的重要算法

4.5.2 构造算法详解

  • 时间复杂度 : 使用小顶堆时为 \(O(n \log n)\)，线性扫描为 \(O(n²)\)

4.5.3 哈夫曼编码详解

哈夫曼编码 ( Huffman Coding ) 是一种基于哈夫曼树的变长编码方法，具有最优的平均编码长度.

编码规则

  1. 路径编码 : 从根结点到叶结点的路径，左分支编 0，右分支编 1
  2. 叶结点对应 : 每个叶结点对应一个字符及其编码
  3. 频率优化 : 高频字符路径短 ( 编码短 )，低频字符路径长 ( 编码长 )

前缀编码性质

  • 定义 : 任何编码都不是其他编码的前缀
  • 重要性 : 保证解码过程无歧义，无需分隔符
  • 哈夫曼编码天然满足前缀编码

数学性质

  • 平均编码长度 \(L_{\text{avg}} = \frac{\text{WPL}}{\sum w_i}\)
  • 对于给定的字符频率，哈夫曼编码的平均长度最短
  • 压缩率 \(R = 1 - \frac{L_{\text{avg}}}{L_{\text{fixed}}}\)，其中 \(L_{\text{fixed}}\) 是定长编码长度

4.5.4 408 常考点与扩展

WPL 计算的简化方法

  1. 层数定义 : 叶结点层数从 1 开始时，路径长度 \(l = \text{层数}-1\)
  2. 只计叶结点 : WPL 只看叶结点，不把内部结点权值计入
  3. 合并深度法 : 每次合并相当于把被合并的两权值各自「深度 +1」

快速计算技巧

  • 对于权值集合 \(\{w_1, w_2, ..., w_n\}\)，WPL 等于所有合并操作中权值之和
  • 示例 : 权值 {2, 3, 5, 7} 的合并过程
    • 合并 2 和 3 → 5 ( 深度 +1 )
    • 合并 5 和 5 → 10 ( 深度 +1 )
    • 合并 7 和 10 → 17 ( 深度 +1 )
    • WPL = 5 + 10 + 17 = 32

4.6 堆 ( Heap )

堆 是一种特殊的 完全二叉树，满足堆序性质，能够高效支持插入、删除最值等操作，在优先队列、排序、TopK 等问题中有广泛应用.

4.6.1 定义与存储

定义 堆 是满足如下性质的 完全二叉树

  • 大顶堆 ( 最大堆 ) 每个结点的值 ≥ 其子结点的值，根为最大值.
  • 小顶堆 ( 最小堆 ) 每个结点的值 ≤ 其子结点的值，根为最小值.

堆的核心特性

完全二叉树性质

  • 除最后一层外，其他层结点数都达到最大值
  • 最后一层结点都集中在左侧连续位置
  • 高度为 \(h\) 的堆最多有 \(2^h - 1\) 个结点，最少有 \(2^{h-1}\) 个结点

堆序性质 ( Heap Property ) * 大顶堆 : 任意结点值 ≥ 其子树中所有结点的值 * 小顶堆 : 任意结点值 ≤ 其子树中所有结点的值 * 堆序性质保证了根结点是最大值或最小值

存储结构

堆用 数组 存储，利用完全二叉树的特性实现高效存储和访问.

下标约定

  • 下标从 1 开始 ( 常用约定 )
    • 结点 \(i\) 的左孩子 : \(2i\)
    • 结点 \(i\) 的右孩子 : \(2i + 1\)
    • 结点 \(i\) 的双亲 : \(\lfloor i/2 \rfloor\)
  • 下标从 0 开始 ( 编程实现常用 )
    • 结点 \(i\) 的左孩子 : \(2i + 1\)
    • 结点 \(i\) 的右孩子 : \(2i + 2\)
    • 结点 \(i\) 的双亲 : \(\lfloor (i-1)/2 \rfloor\)

存储优势

  • 空间效率 : 无需指针，仅用数组存储数据
  • 访问效率 : 通过下标计算快速定位父子结点
  • 缓存友好 : 连续存储，提高缓存命中率

4.6.2 核心操作算法

下沉操作 ( SiftDown / Heapify )

功能 : 当某个结点的值不满足堆序性质时，将其下沉到合适位置

算法步骤 ( 大顶堆 )

  1. 比较当前结点与其左右孩子的值
  2. 如果当前结点值小于某个孩子，与较大的孩子交换
  3. 递归处理交换后的位置，直到满足堆序性质

上浮操作 ( SiftUp )

功能 : 当某个结点的值大于其父结点时，将其上浮到合适位置

4.6.3 建堆算法

Floyd 建堆算法

思想 : 从最后一个分支结点开始，自底向上对每个结点做下沉操作

时间复杂度分析

  • 直观理解 : 越靠近叶子的结点高度越小，下沉步数越少
  • 数学推导
    • 高度为 \(h\) 的结点最多有 \(2^h\) 个
    • 每个高度为 \(h\) 的结点最多下沉 \(h\) 步
    • 总步数 \(T(n) = \displaystyle\sum_{h=0}^{\lfloor \log n \rfloor} 2^h \cdot h\)
    • 通过级数求和可得 \(T(n) = O(n)\)

与逐个插入对比

  • Floyd 建堆 : \(O(n)\)，更优
  • 逐个插入 : \(O(n \log n)\)，每次插入 \(O(\log n)\)

4.6.4 插入与删除操作

插入 将新元素追加到数组末尾，然后 上浮 到合适位置

时间复杂度 : \(O(\log n)\)

操作流程

  1. 将新元素放在数组末尾
  2. 比较新元素与其父结点的值
  3. 如果违反堆序性质，与父结点交换
  4. 重复步骤 2-3，直到满足堆序性质

删除堆顶 将堆顶与末尾元素互换，堆大小减 1，然后对新堆顶做 下沉

时间复杂度 : \(O(\log n)\)

操作流程

  1. 保存堆顶元素
  2. 用末尾元素覆盖堆顶
  3. 堆大小减 1
  4. 对新的堆顶元素做下沉操作

堆排序算法

算法步骤

  1. 建堆 : 将无序数组构建成大顶堆，\(O(n)\)
  2. 排序 : 循环 \(n-1\) 次
    • 将堆顶 ( 最大值 ) 与当前末尾元素交换
    • 堆大小减 1
    • 对新的堆顶做下沉操作，恢复堆序

时间复杂度

  • 建堆 : \(O(n)\)
  • 排序循环 : \(n-1\) 次，每次 \(O(\log n)\)，总 \(O(n \log n)\)
  • 总复杂度 : \(O(n \log n)\)

空间复杂度 : \(O(1)\) ( 原地排序 )

稳定性 : 不稳定 ( 交换操作可能改变相等元素的相对顺序 )

性能特点

  • 优点 : 最坏情况 \(O(n \log n)\)，空间效率高
  • 缺点 : 常数因子较大，缓存不友好
  • 适用场景 : 对最坏时间复杂度有要求的场景

4.6.5 堆的应用场景

堆作为一种高效的数据结构，在算法设计和系统应用中有着广泛的应用，特别是在需要频繁获取最值或维护优先级的场景中.

优先队列 ( Priority Queue ) 是堆最经典的应用，用堆实现可以高效支持插入、删除最值操作.

操作复杂度

  • 入队 ( enqueue ) : 插入堆，\(O(\log n)\)
  • 出队 ( dequeue ) : 删除堆顶，\(O(\log n)\)
  • 取最值 ( peek ) : 读堆顶，\(O(1)\)
  • 判空 : 检查堆大小，\(O(1)\)

应用场景

  • 任务调度 : 操作系统进程调度，按优先级执行
  • 网络路由 : 数据包按优先级转发
  • 事件驱动 : 按时间戳处理事件
  • 资源分配 : 按优先级分配系统资源

算法应用

  • Dijkstra 最短路 : 用小顶堆维护未访问结点的最短距离
  • Prim 最小生成树 : 用小顶堆维护候选边的最小权重
  • A* 搜索算法 : 用优先队列维护搜索结点的估价函数值

实现示例

TopK 问题 从 \(n\) 个元素中找出最大或最小的 \(K\) 个元素，是堆的经典应用场景.

算法选择策略

  • 找最大 \(K\) 个 : 用 小顶堆 ( 维护当前最大的 \(K\) 个元素 )
  • 找最小 \(K\) 个 : 用 大顶堆 ( 维护当前最小的 \(K\) 个元素 )

小顶堆方案 ( 找最大 \(K\) 个 )

时间复杂度分析

  • 建堆 : \(O(K)\)
  • 遍历剩余元素 : \(n-K\) 次，每次 \(O(\log K)\)
  • 总复杂度 : \(O(n \log K)\)

空间复杂度 : \(O(K)\) ( 仅需维护大小为 \(K\) 的堆 )

适用场景

  • \(K \ll n\) 时优势明显
  • 数据流处理，无法一次性加载全部数据
  • 内存受限，无法存储全部数据

对比其他方案

  • 全部排序 : \(O(n \log n)\)，空间 \(O(n)\)
  • 快选算法 : \(O(n)\) 期望，但最坏 \(O(n^2)\)
  • 堆方案 : \(O(n \log K)\)，稳定高效

动态中位数 在数据流中随时查询当前中位数，双堆方案是最优解.

双堆设计

  • 大顶堆 \(L\) : 存储较小的一半数据，堆顶为较小一半的最大值
  • 小顶堆 \(R\) : 存储较大的一半数据，堆顶为较大一半的最小值
  • 平衡条件 : \(|L| - |R| \leq 1\)

中位数计算

  • 当总数为奇数时 : 中位数 = \(L\) 的堆顶
  • 当总数为偶数时 : 中位数 = \((L\text{堆顶} + R\text{堆顶}) / 2\)

插入算法

性能分析

  • 插入操作 : \(O(\log n)\)
  • 查询中位数 : \(O(1)\)
  • 空间复杂度 : \(O(n)\)

应用场景

  • 实时数据流分析
  • 系统性能监控
  • 金融数据统计
  • 网络流量分析

多路归并排序

  • 问题 : 将 \(K\) 个有序数组合并成一个有序数组
  • 堆方案 : 用小顶堆维护每个数组的当前最小元素
  • 复杂度 : \(O(n \log K)\)，其中 \(n\) 为总元素数

定时器管理

  • 问题 : 管理系统中的定时任务，按触发时间执行
  • 堆方案 : 用小顶堆按触发时间排序
  • 优势 : 快速获取下一个触发的定时器

外部排序

  • 问题 : 数据量太大无法全部加载到内存
  • 堆方案 : 用堆进行多路归并，减少磁盘 I/O
  • 应用 : 大数据处理，数据库排序

负载均衡

  • 问题 : 将任务分配给负载最小的服务器
  • 堆方案 : 用小顶堆维护服务器负载
  • 优势 : 快速找到负载最小的服务器

游戏 AI

  • 问题 : 在游戏中选择最优的行动路径
  • 堆方案 : 用优先队列维护候选行动
  • 应用 : A* 寻路算法，决策树搜索

4.6.5 408 常考点

  • 从 \(\lfloor n/2 \rfloor\) 到 1 依次 siftDown，总复杂度为 \(O(n)\)

4.7 虚节点 ( Virtual Node / Dummy Node )

介绍

虚节点 ( 虚拟节点 / 哑节点 / 哨兵节点 ) 是一种特殊的技术手段，通过在数据结构中添加不存储实际数据的节点来简化算法实现、处理边界情况、提高代码可读性.

4.7.1 基本概念与定义

定义 虚节点 是数据结构中不存储实际数据内容的特殊节点，主要用于

  • 简化边界条件处理
  • 统一算法逻辑
  • 提高代码可读性和健壮性
  • 优化特定操作的性能

虚节点的核心特性

数据特性

  • 不存储实际数据 : 虚节点本身不包含有效的数据内容
  • 仅提供结构支持 : 主要用于维护数据结构的完整性
  • 辅助功能 : 简化算法实现，处理特殊情况

功能特性

  • 边界处理 : 避免对空结构的特殊判断
  • 逻辑统一 : 使算法逻辑更加一致和简洁
  • 性能优化 : 在某些场景下减少条件判断

虚节点的常见类型

  1. 链表虚头节点 : 单链表、双链表的头节点前添加的虚节点
  2. 树结构虚根节点 : 某些树结构中添加的虚根节点
  3. 哈希表虚节点 : 解决哈希冲突的虚节点技术
  4. 图结构虚节点 : 图的虚拟起点或终点

4.7.2 虚节点在链表中的应用

  • 循环链表 : 虚节点可以简化循环链表的实现

4.8 例题精选

例题 1 ( 二叉树结点数 ) 已知二叉树有 50 个叶结点，则该树中度为 2 的结点数为多少 ?

解

由性质 \(n_0 = n_2 + 1\)，得 \(n_2 = n_0 - 1 = 49\).

例题 2 ( 由遍历序列构造二叉树 )

前序序列 ABCDEFG，中序序列 CBDAEFG，求后序序列.

解

前序第一个 A 为根; 中序中 A 左侧 CBD 为左子树，右侧 EFG 为右子树.

左子树 前序 BCD，中序 CBD → B 为根，C 为左孩子，D 为右孩子.

右子树 前序 EFG，中序 EFG → E 为根，F 为左孩子，G 为右孩子.

后序 CDBFGEA.

例题 3 ( AVL 旋转 )

在 AVL 树中依次插入 10、20、30，说明插入过程并指出旋转类型.

解

插入 10 树为单结点，平衡.

插入 20 右孩子，BF(10) = \(-1\)，平衡.

插入 30 BF(10) = \(-2\)，失衡，10 的右孩子 20 的右子树中 插入，属于 RR 型，对 10 做 左单旋，20 成为新根，10 为左孩子，30 为右孩子.

例题 4 ( 哈夫曼树 WPL )

字符频率 A→4，B→2，C→6，D→3，E→5，求哈夫曼树的 WPL.

解

排序 {2,3,4,5,6}.

合并 2+3=5 → {4,5,5,6}.

合并 4+5=9 → {5,6,9}.

合并 5+6=11 → {9,11}.

合并 9+11=20 → {20}.

WPL \(= 2\times3 + 3\times3 + 4\times2 + 5\times2 + 6\times2 = 6+9+8+10+12 = 45\).

例题 5 ( TopK )

在 \(10^7\) 个整数中找最大的 100 个，比较三种方案的效率.

解

方案 时间复杂度 空间复杂度
全部排序 \(O(n \log n)\) \(O(n)\)
全部扫描 + 堆 \(O(n \log K)\) \(O(K)\)
分治 + 快选 \(O(n)\) 期望 \(O(\log n)\)

最优实践 维护大小为 100 的 小顶堆，扫描一遍，\(O(n \log K)\) 且空间仅 \(O(K)\).

4.8 本章疑难点

1. 树的后根遍历 ≠ 二叉树后序遍历

  • 树的 后根遍历 对应其孩子兄弟表示法转化后二叉树的 中序遍历. 这是高频易错点.

2. 完全二叉树与满二叉树

  • 满二叉树 ⊂ 完全二叉树，满二叉树一定是完全二叉树，反之不一定.

3. BST 删除操作

构造方法 ( 以前序 + 中序为例 )

  1. 前序第一个元素为 根.
  2. 在中序序列中找到根的位置，左侧为左子树，右侧为右子树. 删除有两棵子树的结点时，选择中序前驱或后继替代均合法，但两者得到的 BST 形态不同.

4. AVL 旋转后子树归属

  • 旋转时注意被旋转结点的 子树重新挂接，不能遗漏. 例如 LL 旋转后，B 的右子树变为 A 的左子树.

5. 哈夫曼树无度为 1 的结点

  • 哈夫曼树只有叶结点 ( 度 0 ) 和度为 2 的结点，不存在度为 1 的结点.

6. TopK 的堆类型选择

  • 找最大 \(K\) 个用 小顶堆 ( 淘汰小的 ); 找最小 \(K\) 个用 大顶堆 ( 淘汰大的 ). 逻辑相反，不能弄错.

4.9 本章小结

  • 树是非线性结构，孩子兄弟表示法是树与二叉树互转的桥梁.
  • 二叉树五大性质、四种遍历 ( 含非递归 ) 、线索化是核心考点，必须完全掌握.
  • BST 中序有序，AVL 四种旋转保持平衡，两者共同构成查找树基础.
  • 哈夫曼树保证最小 WPL，哈夫曼编码是前缀编码，是数据压缩的理论基础.
  • 堆是完全二叉树的特化，建堆 \(O(n)\)、增删 \(O(\log n)\)，支撑优先队列、TopK、动态中位数等重要应用.
  • 遍历序列确定性 : 仅先序序列能确定的不同二叉树个数 = 卡特兰数 \(C_n\)，其中 \(C_0=1, C_1=1, C_2=2, C_3=5, C_4=14, C_5=42\).
