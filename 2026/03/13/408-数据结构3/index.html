第3章 栈、队列和数组

3.1 栈 ( Stack )

3.1.1 定义与特性

定义

栈 是只允许在 一端 进行插入或删除操作的线性表. 允许操作的一端称为 栈顶 ( Top )，不允许操作的一端称为 栈底 ( Bottom ).

  • LIFO ( Last In First Out ) — 后进先出，最后入栈的元素最先出栈.
  • \(n\) 个不同元素入栈，出栈序列共有 \(\dfrac{1}{n+1}\dbinom{2n}{n}\) 种 ( Catalan 数 ).

3.1.2 基本操作

操作 功能说明
InitStack(S) 初始化一个空栈 \(S\)
StackEmpty(S) 判断栈是否为空，空返回 true
Push(S, x) 入栈，将元素 \(x\) 压入栈顶
Pop(S, x) 出栈，弹出栈顶元素并用 \(x\) 返回
GetTop(S, x) 读栈顶，将栈顶元素赋给 \(x\) 但不弹出
DestroyStack(S) 销毁栈，释放内存

区别

GetTop 与 Pop 的区别 GetTop 只 读取 栈顶，栈不变化; Pop 会 删除 栈顶元素.

3.1.3 顺序栈

  • S.top == -1 时栈空; S.top == MaxSize - 1 时栈满.
  • 栈顶元素为 S.data[S.top].

共享栈 两个栈共享同一块数组空间，栈 1 的栈底在 data[0]，栈 2 的栈底在 data[MaxSize-1]，分别向中间增长.

优点 两个栈的空间互补，总存储利用率更高，适合两栈大小互为消长的场景.

3.1.4 链栈

链栈 用单链表实现，规定所有操作在 链表头部 进行，无需头结点. 链栈不存在栈满问题.

  • 入栈 = 在链表头部插入结点，\(O(1)\).
  • 出栈 = 删除链表头部结点，\(O(1)\).
  • 适合栈的大小无法预估的场景.
比较项 顺序栈 链栈

指针开销 无 每个结点有指针域
访问速度 稍快 ( 数组连续 ) 稍慢 ( 指针跳转 )

3.1.6 408 常考点

已知入栈序列，判断某出栈序列是否合法，核心是用栈模拟.

设入栈序列为 \(1,2,\ldots,n\)，待验证出栈序列为 \(b_1,b_2,\ldots,b_n\).

  • 从 1 到 \(n\) 依次入栈
  • 每次入栈后，只要栈顶等于当前需要输出的 \(b_k\) 就持续出栈并令 \(k\) 后移
  • 扫描结束后若成功输出完全部 \(b\)，则合法

常见误区是只看「局部可出」而不做全流程模拟.

定义

卡特兰数 ( Catalan Number ) 是组合数学中的重要序列，描述了满足特定约束条件的组合结构的数量.

卡特兰数定义

显式公式 \[C_n = \dfrac{1}{n+1}\dbinom{2n}{n} = \dfrac{(2n)!}{(n+1)!n!}\]

递归定义 \[C_0 = 1,\quad C_{n+1} = \sum_{k=0}^{n} C_k C_{n-k}\]

数学推导

方法一 : 反射原理 ( 经典推导 )

问题 : 计算 \(n\) 个元素的合法出栈序列数

步骤

  1. 总序列数 : \(n\) 次 Push 和 \(n\) 次 Pop 的排列数为 \(\binom{2n}{n}\)
  2. 非法序列 : 存在某个时刻 Pop 次数 > Push 次数的序列
  3. 反射原理 : 将第一个非法点后的操作取反 ( Push ↔︎ Pop )
  4. 一一对应 : 每个非法序列对应一个 \(n+1\) 次 Push 和 \(n-1\) 次 Pop 的序列
  5. 非法序列数 : \(\binom{2n}{n+1}\)
  6. 合法序列数 : \(\binom{2n}{n} - \binom{2n}{n+1} = \dfrac{1}{n+1}\dbinom{2n}{n}\)

方法二 : 生成函数法

设生成函数 \(C(x) = \displaystyle\sum_{n\geq 0} C_n x^n\)

由递归关系 \[C(x) = 1 + xC(x)^2\]

解得 \[C(x) = \dfrac{1 - \sqrt{1 - 4x}}{2x}\]

展开得卡特兰数的显式公式.

组合含义详解

卡特兰数计数的组合结构都具有 递归分解 和 平衡约束 的本质特征.

1. 栈操作序列 ( 出栈序列 )

  • 约束 : 任何时刻 Pop 次数 ≤ Push 次数
  • 递归 : 第一次 Pop 将序列分为左右两部分
  • 意义 : 描述了栈操作的合法性和有序性

2. 二叉树形态

  • 约束 : 每个结点最多有两个孩子
  • 递归 : 根结点 + 左子树 + 右子树
  • 意义 : 反映了二叉树结构的组合多样性

3. 括号匹配

  • 约束 : 任何位置左括号数 ≥ 右括号数
  • 递归 : 最外层括号内的表达式
  • 意义 : 语法结构的合法性检查

4. Dyck 路径

  • 约束 : 路径不穿过对角线
  • 递归 : 第一次触及对角线的点
  • 意义 : 组合几何中的路径计数

深刻数学意义

卡特兰数体现了组合数学的统一性

  • 递归结构 : 问题可以分解为相似子问题
  • 平衡约束 : 解必须满足某种平衡条件

  • 算法复杂度 : 许多递归算法的时间复杂度与卡特兰数相关

增长速率 \[C_n \sim \dfrac{4^n}{n^{3/2}\sqrt{\pi}}\] 这表明卡特兰数问题具有指数级复杂性，但比纯指数增长稍慢.

  • 两栈相向增长，栈满条件是 top1 + 1 == top2

3.2 队列 ( Queue )

3.2.1 定义与特性

队列 是只允许在 一端 插入、在 另一端 删除的线性表. 允许插入的一端称为 队尾 ( Rear )，允许删除的一端称为 队头 ( Front ).

  • FIFO ( First In First Out ) — 先进先出，最先入队的元素最先出队.

3.2.3 顺序队列与假溢出

用数组实现队列时，频繁入队出队后 rear 到达数组末尾，而数组前端有空间，这种现象称为 假溢出.

解决假溢出的方法是使用 循环队列 将数组首尾相接，下标运算采用取模 % MaxSize.

  • 队空 Q.front == Q.rear.

3.2.5 链队列

链队列 用带头结点的单链表实现，front 指向头结点，rear 指向尾结点. 不存在队满问题.

  • 入队 在链表 尾部 插入新结点，\(O(1)\).
  • 出队 删除头结点之后的第一个数据结点，\(O(1)\).
  • 队空条件 Q.front == Q.rear ( 均指向头结点 ).

3.2.6 双端队列

双端队列 ( Deque ) 两端均可进行插入和删除操作的线性表.

  • 可看作栈与队列的推广.
  • 完全双端队列包含所有输入 / 输出受限的情形.

输入受限双端队列 只允许在 一端插入，两端均可删除.

  • 该队列能输出的序列是普通队列的超集，但不是所有栈能输出的序列都能输出.

输出受限双端队列 两端均可插入，只允许在 一端删除.

  • 该队列能输出的序列是普通队列的超集，但不是所有栈能输出的序列都能输出.

双端队列考题常考以下判断

  1. 若序列可由 栈 得到，则一定能由 双端队列 得到 ( 双端队列更强大 ).
  2. 某序列 不能 由栈得到，但 能 由输出受限双端队列得到 → 典型区分点.
  3. 判断方法 画出入队过程，验证各时刻队列状态是否合法.

3.2.7 队列的应用

  • 广度优先搜索 ( BFS ) 从起始结点出发，逐层访问，每层结点依次入队，出队时将未访问邻居入队.
  • 进程调度 操作系统用队列管理等待 CPU 的进程 ( 就绪队列 ).
  • 打印缓冲区、消息队列 先到先服务，典型 FIFO 应用.
  • 层次遍历二叉树 利用队列按层输出各结点.

3.3 数组与特殊矩阵

3.3.1 数组的存储与地址计算

数组 是线性表的推广，由一组 类型相同 的数据元素构成的有序集合，每个元素受 \(n\) ( \(n \geq 1\) ) 个线性关系的约束，每个元素在 \(n\) 个线性关系中的序号称为该元素的 下标. 数组是 随机存取 结构.

定义 数组 \(A[0 \ldots n-1]\)，首地址为 \(\text{LOC}(A[0])\)，每个元素占 \(L\) 字节

核心思想 地址计算的核心是 分析当前下标前面有多少个元素

地址公式推导

  • 元素 \(A[i]\) 前面有 \(i\) 个元素 : \(A[0], A[1], \ldots, A[i-1]\)
  • 这些元素总共占用了 \(i \times L\) 字节
  • 因此 : \(\text{LOC}(A[i]) = \text{LOC}(A[0]) + i \times L\)

直观理解

示例 数组 int A[10]，首地址为 1000，每个 int 占 4 字节

  • \(\text{LOC}(A[5])\) : 前面有 5 个元素，占用 \(5 \times 4 = 20\) 字节
  • \(\text{LOC}(A[5]) = 1000 + 20 = 1020\)
  • \(\text{LOC}(A[9])\) : 前面有 9 个元素，占用 \(9 \times 4 = 36\) 字节
  • \(\text{LOC}(A[9]) = 1000 + 36 = 1036\)

代码实现

时间复杂度 \(O(1)\)，随机访问

定义 数组 \(A[0 \ldots m-1][0 \ldots n-1]\)，首地址为 \(\text{LOC}(A[0][0])\)

核心思想 分析 \(A[i][j]\) 前面有多少个元素

行优先存储 ( C 语言、Python、Java 默认 )

  • 前面有多少行 : \(i\) 行完整行，每行有 \(n\) 个元素
  • 当前行前面有多少列 : \(j\) 个元素
  • 总共前面元素数 : \(i \times n + j\)
  • 地址公式 : \(\text{LOC}(A[i][j]) = \text{LOC}(A[0][0]) + (i \times n + j) \times L\)

列优先存储 ( Fortran、MATLAB 默认 )

  • 前面有多少列 : \(j\) 列完整列，每列有 \(m\) 个元素
  • 当前列前面有多少行 : \(i\) 个元素
  • 总共前面元素数 : \(j \times m + i\)
  • 地址公式 : \(\text{LOC}(A[i][j]) = \text{LOC}(A[0][0]) + (j \times m + i) \times L\)

示例 数组 int A[3][4]，首地址 1000，int 占 4 字节

  • 行优先 : \(A[1][2]\) 前面有 1 整行 (4个元素) + 当前行前 2 个元素 = 6 个元素
  • \(\text{LOC}(A[1][2]) = 1000 + 6 \times 4 = 1024\)
  • 列优先 : \(A[1][2]\) 前面有 2 整列 (每列3个元素) + 当前列前 1 个元素 = 7 个元素
  • \(\text{LOC}(A[1][2]) = 1000 + 7 \times 4 = 1028\)

内存布局对比

定义 \(n\) 维数组 \(A[d_1][d_2]\ldots[d_n]\)，首地址 \(\text{LOC}(A[0][0]\ldots[0])\)

核心思想 逐维分析 前面有多少个完整的超平面

行优先存储

  • 第1维前面 : \(i_1\) 个完整 \((d_2 \times d_3 \times \cdots \times d_n)\) 维超平面
  • 第2维前面 : \(i_2\) 个完整 \((d_3 \times \cdots \times d_n)\) 维超平面
  • …
  • 第n-1维前面 : \(i_{n-1}\) 个完整 \(d_n\) 维超平面
  • 第n维前面 : \(i_n\) 个元素
  • 地址公式 : \(\text{LOC}(A[i_1][i_2]\ldots[i_n]) = \text{LOC}(A[0][0]\ldots[0]) + \left( i_1 \times d_2 \times d_3 \times \cdots \times d_n + i_2 \times d_3 \times \cdots \times d_n + \cdots + i_{n-1} \times d_n + i_n \right) \times L\)

列优先存储

  • 类似思想，但从最后一维开始计算
  • 地址公式 : \(\text{LOC}(A[i_1][i_2]\ldots[i_n]) = \text{LOC}(A[0][0]\ldots[0]) + \left( i_n \times d_1 \times d_2 \times \cdots \times d_{n-1} + i_{n-1} \times d_1 \times d_2 \times \cdots \times d_{n-2} + \cdots + i_2 \times d_1 + i_1 \right) \times L\)

示例 三维数组 int A[2][3][4]，首地址 1000，int 占 4 字节

  • 行优先 : \(A[1][2][3]\) 前面有
    • 1 个完整 \(3 \times 4 = 12\) 元素的平面
    • 2 个完整 \(4\) 元素的列
    • 3 个元素
    • 总共 : \(12 + 8 + 3 = 23\) 个元素
  • \(\text{LOC}(A[1][2][3]) = 1000 + 23 \times 4 = 1092\)

若数组 \(A[1 \ldots m][1 \ldots n]\)，行优先存储

核心思想 将任意下标范围 归一化 到从 0 开始计算

公式推导

  • 将下标 \(i\) 归一化 : \(i' = i - 1\)，这样就从 \([1,m]\) 映射到 \([0,m-1]\)
  • 将下标 \(j\) 归一化 : \(j' = j - 1\)，这样就从 \([1,n]\) 映射到 \([0,n-1]\)
  • 前面元素数 : \(i' \times n + j' = (i-1) \times n + (j-1)\)
  • 地址公式 : \(\text{LOC}(A[i][j]) = \text{LOC}(A[1][1]) + [(i-1) \times n + (j-1)] \times L\)

通用公式 ( 下标范围 \([l_1 \ldots h_1][l_2 \ldots h_2]\) )

  • 归一化思想 : 将任意范围映射到从 0 开始
  • 行数 : \(m = h_1 - l_1 + 1\)
  • 列数 : \(n = h_2 - l_2 + 1\)
  • 归一化下标 : \(i' = i - l_1\)，\(j' = j - l_2\)
  • 地址公式 : \(\text{LOC}(A[i][j]) = \text{LOC}(A[l_1][l_2]) + [(i-l_1) \times n + (j-l_2)] \times L\)

示例 数组 int A[5..10][3..8]，首地址 1000，int 占 4 字节

  • 归一化计算
    • 行数 \(m = 10-5+1 = 6\)，列数 \(n = 8-3+1 = 6\)
    • \(A[7][5]\) 归一化 : \(i' = 7-5 = 2\)，\(j' = 5-3 = 2\)
    • 前面元素数 : \(2 \times 6 + 2 = 14\) 个元素
  • \(\text{LOC}(A[7][5]) = 1000 + 14 \times 4 = 1056\)

408 考试重点

  • 下标起始值不固定，务必 仔细读题 确认下标范围
  • 注意区分行优先和列优先存储方式
  • 题目常给元素大小 (L) 与首地址，最终答案常以字节偏移给出
  • 多维数组计算时注意各维度的乘积关系

地址计算核心思想总结

地址计算总结

“依靠分析当前下标前面有多少个元素来计算地址” 是数组地址计算的 根本原理

统一的计算框架

  1. 确定存储顺序 : 行优先还是列优先
  2. 归一化下标 : 将任意范围映射到从 0 开始
  3. 计算前面元素数 : 按存储顺序逐维计算
  4. 乘以元素大小 : 得到字节偏移量
  5. 加上首地址 : 得到最终地址

记忆技巧

  • 一维数组 : 前面有 \(i\) 个元素
  • 二维数组行优先 : 前面有 \(i\) 整行 + \(j\) 个元素
  • 二维数组列优先 : 前面有 \(j\) 整列 + \(i\) 个元素
  • 多维数组 : 逐维计算完整的超平面

避免的错误

  • 忘记归一化下标 ( 特别是下标从 1 开始时 )
  • 混淆行优先和列优先的计算顺序
  • 忽略元素大小 \(L\) 的单位转换

3.3.2 特殊矩阵压缩存储

压缩存储定义

压缩存储 对具有规律分布的特殊矩阵，只存储 有效元素，其余元素通过计算得到，以节省空间. 压缩存储的核心思想是 用计算换空间.

定义 \(n\) 阶方阵满足 \(A[i][j] = A[j][i]\)，\(\forall i,j \in [1,n]\)

存储策略 只需存储 下三角 部分 ( 含主对角线 )，共 \(\dfrac{n(n+1)}{2}\) 个元素

压缩数组 \(B[0 \ldots \frac{n(n+1)}{2}-1]\)

下标映射公式 ( \(1 \leq i,j \leq n\)，行优先存储下三角 ) \[k = \begin{cases} \dfrac{i(i-1)}{2} + j - 1 & i \geq j \text{ ( 下三角 )} \\[6pt] \dfrac{j(j-1)}{2} + i - 1 & i < j \text{ ( 上三角，用对称性 ) } \end{cases}\]

示例 4 阶对称矩阵，求 \(A[3][2]\) 在压缩数组中的位置

  • \(i=3, j=2\)，\(i \geq j\)，\(k = \dfrac{3 \times 2}{2} + 2 - 1 = 3 + 1 = 4\)

代码实现

空间节省 从 \(n^2\) 压缩到 \(\dfrac{n(n+1)}{2}\)，节省约 50% 空间

下三角矩阵 上三角区域 ( 不含对角线 ) 全为常数 \(c\)

存储策略 存储下三角 + 对角线元素及常数 \(c\)，共 \(\dfrac{n(n+1)}{2} + 1\) 个元素

下标映射公式 \[k = \begin{cases} \dfrac{i(i-1)}{2} + j - 1 & i \geq j \\[6pt] \dfrac{n(n+1)}{2} & i < j \text{ ( 存常数 } c \text{ 的位置 ) } \end{cases}\]

上三角矩阵 下三角区域全为常数，类似处理

行优先存储上三角 ( \(i \leq j\) ) \[k = \dfrac{(i-1)(2n-i+2)}{2} + (j-i)\]

示例 4 阶下三角矩阵，求 \(A[3][4]\) 在压缩数组中的位置

  • \(i=3, j=4\)，\(i < j\)，\(k = \dfrac{4 \times 5}{2} = 10\) ( 常数位 )

空间节省 从 \(n^2\) 压缩到 \(\dfrac{n(n+1)}{2} + 1\)，节省约 50% 空间

定义 ( 带状矩阵 ) 非零元素集中在主对角线及其两侧共 \(d\) 条对角线的带状区域内

带宽 \(d = 2b+1\) ( \(b\) 为半带宽 )

三对角矩阵 ( \(b=1\) ) 只存储 \(3n-2\) 个非零元素

下标映射公式 ( \(|i-j| \leq 1\) ) \[k = 2(i-1) + (j - i + 1) = 2i + j - 3\]

反向推导 已知 \(k\)，求 \((i,j)\) \[i = \left\lfloor \dfrac{k+1}{3} \right\rfloor + 1, \quad j = k - 2(i-1) + i - 1\]

通用对角矩阵公式 ( 带宽 \(d=2b+1\) ) \[k = (i-1) \times d + (j - i + b) \quad (|i-j| \leq b)\]

示例 5 阶三对角矩阵，求 \(A[3][2]\) 在压缩数组中的位置

  • \(i=3, j=2\)，\(|3-2|=1 \leq 1\)，\(k = 2 \times 2 + (2 - 3 + 1) = 4 + 0 = 4\)

空间节省 从 \(n^2\) 压缩到 \(n \times d\)，当 \(d \ll n\) 时节省大量空间

定义 矩阵中非零元素个数 \(t\) 远少于零元素个数

稀疏度判断 通常 \(t \leq \dfrac{m \times n}{3}\) 时视为稀疏

稀疏因子 \(\delta = \dfrac{t}{m \times n}\)，当 \(\delta < 0.05\) 时通常认为稀疏

常见稀疏矩阵类型

  • 对角占优矩阵 主对角线元素绝对值大于同行其他元素绝对值之和
  • Laplacian 矩阵 图论中图的拉普拉斯矩阵
  • 有限差分矩阵 数值计算中离散微分算子

应用场景

  • 有限元分析
  • 电路网络分析
  • 社交网络分析
  • 图像处理

  • 通用公式 掌握 \(k = (i-1) \times d + (j - i + b)\)

示例题目 4 阶对称矩阵，元素按行优先存储在下三角，求 A[2][3] 的存储位置

  • \(i=2, j=3\)，\(i < j\)，用对称性 \(A[2][3] = A[3][2]\)
  • \(k = \dfrac{3 \times 2}{2} + 2 - 1 = 3 + 1 = 4\)

快速转置算法

  1. 统计 num[col] 每列非零元个数
  2. 前缀和 pos[col] 每列在转置后的起始位置
  3. 放置 扫描原数组，直接放到目标位置

  • 普通转置 \(O(n \times t)\)
  • 快速转置 \(O(n + t)\)
  • 十字链表转置 \(O(t)\)

3.4 例题精选

例题 1 ( 循环队列队长 ) 循环队列 MaxSize = 10，front = 6，rear = 3，求队列中元素个数.

解 \(\text{len} = (\text{rear} - \text{front} + \text{MaxSize}) \bmod \text{MaxSize} = (3 - 6 + 10) \bmod 10 = 7\).

例题 2 ( 出栈序列合法性 ) 入栈序列为 \(1,2,3\)，判断 \(3,1,2\) 是否为合法出栈序列.

解 模拟过程 压 1、压 2、压 3、弹 3，此时栈为 \([1,2]\)，栈顶为 2，但需弹出 1，不合法. 因此 \(3,1,2\) 不是合法出栈序列.

例题 3 ( 对称矩阵下标映射 ) 对称矩阵 \(A[1\ldots 6][1\ldots 6]\) 按行优先存储下三角于 \(B[0\ldots 20]\)，求 \(A[4][2]\) 对应 \(B\) 中下标 \(k\).

解 \(i = 4 > j = 2\)，属于下三角，\(k = \dfrac{i(i-1)}{2} + j - 1 = \dfrac{4 \times 3}{2} + 2 - 1 = 7\).

例题 4 ( 中缀转后缀 ) 将中缀表达式 A + B * (C - D) - E / F 转换为后缀表达式.

解 扫描过程 ( 运算符栈记为 OP，输出序列记为 OUT )

  1. A → OUT: A
  2. + → OP: +
  3. B → OUT: A B
  4. * → OP: + *
  5. ( → OP: + * (
  6. C → OUT: A B C
  7. - → OP: + * ( -
  8. D → OUT: A B C D
  9. ) → 弹出直到 ( 输出 -，丢弃 (
  10. - → 弹出优先级不小于 - 的运算符 依次输出 *、+，再将 - 入栈
  11. E → 输出
  12. / → 入栈
  13. F → 输出
  14. 扫描结束 → 弹出剩余运算符

结果为 A B C D - * + E F / -.

例题 5 ( 出栈序列判定 ) 入栈序列为 \(1,2,3,4,5\)，判断出栈序列 \(4,5,3,2,1\) 是否合法.

解 模拟入栈出栈.

依次压入 1,2,3,4，弹出 4. 再压入 5，弹出 5. 此时栈顶为 3，弹出 3. 继续弹出 2,1. 可完成输出，因此为 合法 序列.

例题 6 ( 出栈序列判定 ) 入栈序列为 \(1,2,3,4,5\)，判断出栈序列 \(3,1,2,5,4\) 是否合法.

解 压入 1,2,3，弹出 3. 此时栈顶为 2，但下一次需要输出 1，且 1 被 2 压住，无法直接弹出. 继续入栈只能压入 4,5，使 1 更不可能弹出. 因此该序列 不合法.

例题 7 ( 两栈实现队列的均摊 ) 用两个栈实现队列，入队执行 push(A, x)，出队若栈 B 为空则把 A 全部倒入 B，再 pop(B). 证明出队操作均摊为 \(O(1)\).

解 每个元素最多经历一次从栈 A 到栈 B 的转移.

  • 入队时压入 A 一次
  • 转移时从 A 弹出一次并压入 B 一次
  • 出队时从 B 弹出一次

因此每个元素总共参与常数次栈操作，总成本为 \(O(m)\)，\(m\) 次队列操作的均摊时间复杂度为 \(O(1)\).

例题 8 ( 循环队列操作序列 ) 循环队列 MaxSize = 6 采用牺牲一格方案，初始 front = rear = 0. 依次执行入队 1,2,3,4，出队两次，再入队 5,6，求此时 front、rear 与队列长度.

解 牺牲一格方案下，队列长度 \[\text{len} = (\text{rear} - \text{front} + \text{MaxSize}) \bmod \text{MaxSize}.\]

模拟过程后队列中为 3,4,5,6.

此时 front = 2，rear = 0，因此 \[\text{len} = (0 - 2 + 6) \bmod 6 = 4.\]

例题 9 ( 地址计算 ) 二维数组 \(A[1\ldots 10][1\ldots 20]\) 行优先存储，元素占 4 字节，\(\text{LOC}(A[1][1])=1000\). 求 \(\text{LOC}(A[3][5])\).

解 下标从 1 开始，行优先 : \[\text{LOC}(A[i][j]) = \text{LOC}(A[1][1]) + [(i-1)\times n + (j-1)]\times L.\]

代入 \(i=3,j=5,n=20,L=4\) : \[1000 + [(2\times 20 + 4)\times 4] = 1000 + 176 = 1176.\]

例题 10 ( 三角矩阵压缩 ) 下三角矩阵 \(A[1\ldots n][1\ldots n]\) 的上三角元素均为常数 \(c\)，按行优先压缩到一维数组 \(B\). 求 \(A[i][j]\) 对应的存储下标.

解 只存下三角含主对角线的 \(\\dfrac{n(n+1)}{2}\) 个元素，以及常数位.

\[k = \begin{cases} \dfrac{i(i-1)}{2} + j - 1 & i \ge j \\\\[4pt] \dfrac{n(n+1)}{2} & i < j \end{cases}\]

例题 11 ( 快速转置定位 ) 稀疏矩阵三元组表中共有 \(t\) 个非零元，列数为 \(n\). 说明快速转置中 pos[col] 的含义，并写出其递推关系.

解 pos[col] 表示转置后三元组数组中「第 col 列 ( 转置后为第 col 行 ) 的第一个非零元」存放的位置.

若 num[col] 是原矩阵第 col 列非零元个数，则

  • pos[1] = 0
  • pos[col] = pos[col-1] + num[col-1] ( \(2 \le col \le n\) )

这样扫描原三元组数组时，可以把每个元素直接放到目标位置并递增该列的当前位置指针.

3.5 本章疑难点

1. 循环队列判空 / 判满

  • 牺牲一格方案最常用，务必记住队满条件 (rear+1)%MaxSize == front 而非 rear == front.
  • 三种方案各有优缺点，选择时结合题目要求.

2. 栈与递归的关系

  • 每次函数调用都会在系统栈上开辟一个 栈帧，保存局部变量、参数和返回地址.
  • 递归深度过大会导致 栈溢出，可改用显式栈消除递归.

3. 后缀表达式求值与中缀转后缀

  • 中缀转后缀用 运算符栈; 后缀表达式求值用 操作数栈. 两者使用的栈类型不同，注意区分.

4. 特殊矩阵压缩下标公式

  • 408 考试中常给出 从 1 开始 的下标，务必仔细读题后代入正确公式，不要混淆行列.

5. 十字链表 vs. 三元组

  • 三元组转置更快 ( 快速转置 \(O(n+t)\) ); 十字链表插删更方便. 按照操作类型选择.

3.6 本章小结

  • 栈 ( LIFO ) 和队列 ( FIFO ) 都是操作受限的线性表，本质上是特殊的线性表.
  • 顺序栈 / 链栈、循环队列 / 链队列是四种实现方式，各有适用场景.
  • 双端队列是栈和队列的推广，输入 / 输出受限双端队列是重要变体.
  • 数组地址计算和特殊矩阵压缩存储是纯粹的计算题，掌握公式即可.
  • 稀疏矩阵三元组与十字链表是空间优化的重要思想，十字链表插删优势明显.
