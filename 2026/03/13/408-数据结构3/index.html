第3章 栈、队列和数组

3.1 栈 ( Stack )

3.1.1 定义与特性

栈 是只允许在 一端 进行插入或删除操作的线性表. 允许操作的一端称为 栈顶 ( Top )，不允许操作的一端称为 栈底 ( Bottom ).

  • LIFO ( Last In First Out ) — 后进先出，最后入栈的元素最先出栈.
  • \(n\) 个不同元素入栈，出栈序列共有 \(\dfrac{1}{n+1}\dbinom{2n}{n}\) 种 ( Catalan 数 ).

3.1.2 基本操作

操作 功能说明
InitStack(S) 初始化一个空栈 \(S\)
StackEmpty(S) 判断栈是否为空，空返回 true
Push(S, x) 入栈，将元素 \(x\) 压入栈顶
Pop(S, x) 出栈，弹出栈顶元素并用 \(x\) 返回
GetTop(S, x) 读栈顶，将栈顶元素赋给 \(x\) 但不弹出
DestroyStack(S) 销毁栈，释放内存

GetTop 与 Pop 的区别 GetTop 只 读取 栈顶，栈不变化; Pop 会 删除 栈顶元素.

3.1.3 顺序栈

  • S.top == -1 时栈空; S.top == MaxSize - 1 时栈满.
  • 栈顶元素为 S.data[S.top].

入栈 ( Push )

出栈 ( Pop )

时间复杂度均为 \(O(1)\).

共享栈 两个栈共享同一块数组空间，栈 1 的栈底在 data[0]，栈 2 的栈底在 data[MaxSize-1]，分别向中间增长.

优点 两个栈的空间互补，总存储利用率更高，适合两栈大小互为消长的场景.

3.1.4 链栈

链栈 用单链表实现，规定所有操作在 链表头部 进行，无需头结点. 链栈不存在栈满问题.

  • 入栈 = 在链表头部插入结点，\(O(1)\).
  • 出栈 = 删除链表头部结点，\(O(1)\).
  • 适合栈的大小无法预估的场景.
比较项 顺序栈 链栈
空间 预分配，可能溢出或浪费 按需分配，灵活
指针开销 无 每个结点有指针域
访问速度 稍快 ( 数组连续 ) 稍慢 ( 指针跳转 )

3.1.5 栈的应用

算法思路

  1. 遇到左括号 (、[、{ → 入栈.
  2. 遇到右括号 → 检查栈顶 若匹配则出栈; 若不匹配或栈空则失败.
  3. 扫描结束后栈为空则匹配成功.

规则 ( 运算符栈 )

  1. 遇到 操作数 → 直接输出.
  2. 遇到 左括号 ( → 入栈.
  3. 遇到 右括号 ) → 弹出并输出直到遇到 (，左括号不输出.
  4. 遇到 运算符 → 弹出并输出所有优先级 大于等于 当前运算符的栈顶运算符，再将当前运算符入栈.
  5. 扫描结束 → 弹出并输出栈中所有运算符.

运算符优先级 *、/ > +、- > (. 括号内的运算符暂时优先级视为最低.

示例 A + B * C - D → A B C * + D -

算法思路 ( 操作数栈 )

  1. 遇到 操作数 → 入栈.
  2. 遇到 运算符 → 弹出栈顶两个元素，执行运算，将结果入栈.
  3. 扫描结束 → 栈中唯一元素即为结果.

示例 2 3 4 * + → 弹出 3、4 相乘得 12，再弹出 2 与 12 相加得 14.

递归调用 系统用 调用栈 保存每次函数调用的返回地址、局部变量和参数. 递归调用本质是栈的隐式应用.

  • 递归深度过大会导致 栈溢出 ( Stack Overflow ).
  • 所有递归算法均可用栈显式模拟，消除递归.

深度优先搜索 ( DFS )

  1. 将起始结点入栈.
  2. 弹出栈顶结点，标记已访问，将其未访问邻居依次入栈.
  3. 重复直到栈为空.

3.1.6 408 常考点

已知入栈序列，判断某出栈序列是否合法，核心是用栈模拟.

设入栈序列为 \(1,2,\ldots,n\)，待验证出栈序列为 \(b_1,b_2,\ldots,b_n\).

  • 从 1 到 \(n\) 依次入栈
  • 每次入栈后，只要栈顶等于当前需要输出的 \(b_k\) 就持续出栈并令 \(k\) 后移
  • 扫描结束后若成功输出完全部 \(b\)，则合法

常见误区是只看「局部可出」而不做全流程模拟.

  • \(n\) 个元素的合法出栈序列数为 \[C_n = \dfrac{1}{n+1}\dbinom{2n}{n}.\]
  • 典型等价模型是「括号匹配」或「Dyck 路径」
  • 408 常以小 \(n\) 让你列举或判断序列数量
  • 两栈相向增长，栈满条件是 top1 + 1 == top2
  • 本质是把两个栈的空闲空间合并成一个池，空间利用率优于两个独立顺序栈
  • 出题点常在于初始化与判满条件写错
  • 中缀转后缀用 运算符栈，后缀求值用 操作数栈
  • 运算符处理看两件事 优先级 与 结合性 ( 408 常按左结合处理 + - * / )
  • 遇到右括号只弹出到左括号为止，左括号不输出

3.2 队列 ( Queue )

3.2.1 定义与特性

队列 是只允许在 一端 插入、在 另一端 删除的线性表. 允许插入的一端称为 队尾 ( Rear )，允许删除的一端称为 队头 ( Front ).

  • FIFO ( First In First Out ) — 先进先出，最先入队的元素最先出队.

3.2.2 基本操作

操作 功能说明
InitQueue(Q) 初始化一个空队列 \(Q\)
QueueEmpty(Q) 判断队列是否为空
EnQueue(Q, x) 入队，将元素 \(x\) 插入队尾
DeQueue(Q, x) 出队，删除队头元素并用 \(x\) 返回
GetHead(Q, x) 读队头，将队头元素赋给 \(x\) 但不删除

3.2.3 顺序队列与假溢出

用数组实现队列时，频繁入队出队后 rear 到达数组末尾，而数组前端有空间，这种现象称为 假溢出.

解决假溢出的方法是使用 循环队列 将数组首尾相接，下标运算采用取模 % MaxSize.

3.2.4 循环队列

  • 初始化 Q.front = Q.rear = 0.
  • 队空 Q.front == Q.rear.
  • 队满 (Q.rear + 1) % MaxSize == Q.front ( 牺牲一个存储单元区分队空 / 满 ).

入队

出队

队列长度

\[\text{len} = (\text{rear} - \text{front} + \text{MaxSize}) \bmod \text{MaxSize}\]

区分队空 / 满的三种方案

方案 说明
牺牲一个存储单元 队满 (rear+1)%MaxSize == front ( 最常用 )
增加 size 计数器 size == 0 队空，size == MaxSize 队满
增加 tag 标志位 最后操作为删除且 front==rear 时队空; 最后操作为插入时队满

3.2.5 链队列

链队列 用带头结点的单链表实现，front 指向头结点，rear 指向尾结点. 不存在队满问题.

  • 入队 在链表 尾部 插入新结点，\(O(1)\).
  • 出队 删除头结点之后的第一个数据结点，\(O(1)\).
  • 队空条件 Q.front == Q.rear ( 均指向头结点 ).
比较项 顺序队列 ( 循环 ) 链队列
空间 预分配 按需分配
队满 有 ( 容量固定 ) 无
指针开销 无 每结点一个指针
适用场景 大小已知 大小不定

3.2.6 双端队列

双端队列 ( Deque ) 两端均可进行插入和删除操作的线性表.

  • 可看作栈与队列的推广.
  • 完全双端队列包含所有输入 / 输出受限的情形.

输入受限双端队列 只允许在 一端插入，两端均可删除.

  • 该队列能输出的序列是普通队列的超集，但不是所有栈能输出的序列都能输出.

输出受限双端队列 两端均可插入，只允许在 一端删除.

  • 该队列能输出的序列是普通队列的超集，但不是所有栈能输出的序列都能输出.

双端队列考题常考以下判断

  1. 若序列可由 栈 得到，则一定能由 双端队列 得到 ( 双端队列更强大 ).
  2. 某序列 不能 由栈得到，但 能 由输出受限双端队列得到 → 典型区分点.
  3. 判断方法 画出入队过程，验证各时刻队列状态是否合法.

3.2.7 队列的应用

  • 广度优先搜索 ( BFS ) 从起始结点出发，逐层访问，每层结点依次入队，出队时将未访问邻居入队.
  • 进程调度 操作系统用队列管理等待 CPU 的进程 ( 就绪队列 ).
  • 打印缓冲区、消息队列 先到先服务，典型 FIFO 应用.
  • 层次遍历二叉树 利用队列按层输出各结点.

3.2.8 408 常考点

方案 队空 队满 备注
牺牲一格 front == rear (rear+1)%MaxSize == front 最常见
加 size size == 0 size == MaxSize 不浪费空间
加 tag front==rear && tag==0 front==rear && tag==1 易写错
  • 假溢出是数组前端有空但 rear 到末尾导致无法入队
  • 真溢出是容量确实不够
  • 循环队列解决的是假溢出，不会改变固定容量的上限
  • 两个栈实现队列 入队压入栈 A，出队时若栈 B 为空则把 A 全部倒入 B，再从 B 弹出
  • 两个队列实现栈 维持一个空队列作为辅助，把除最后一个元素外的元素转移到辅助队列，再弹出最后一个
  • 408 常问最坏与均摊复杂度 两栈实现队列的出队均摊为 \(O(1)\)
  • 题目通常给「入队序列」与「出队序列」，要求判断能否由某类双端队列得到
  • 有效做法是画状态或做过程模拟，避免凭直觉判断

3.3 数组与特殊矩阵

3.3.1 数组的存储与地址计算

设数组 \(A[0 \ldots n-1]\)，首地址为 \(\text{LOC}(A[0])\)，每个元素占 \(L\) 字节

\[\text{LOC}(A[i]) = \text{LOC}(A[0]) + i \times L\]

设数组 \(A[0 \ldots m-1][0 \ldots n-1]\)，首地址为 \(\text{LOC}(A[0][0])\).

行优先存储 ( C 语言默认 )

\[\text{LOC}(A[i][j]) = \text{LOC}(A[0][0]) + (i \times n + j) \times L\]

列优先存储 ( Fortran )

\[\text{LOC}(A[i][j]) = \text{LOC}(A[0][0]) + (j \times m + i) \times L\]

若数组 \(A[1 \ldots m][1 \ldots n]\)，行优先存储 :

\[\text{LOC}(A[i][j]) = \text{LOC}(A[1][1]) + [(i-1) \times n + (j-1)] \times L\]

408 考题中下标起始值不固定，务必 仔细读题 确认下标范围，再套公式.

3.3.2 特殊矩阵压缩存储

压缩存储 对具有规律分布的特殊矩阵，只存储 有效元素，其余元素通过计算得到，以节省空间.

对称矩阵 \(A[i][j] = A[j][i]\)，\(n\) 阶矩阵只需存储 下三角 部分 ( 含主对角线 )，共 \(\dfrac{n(n+1)}{2}\) 个元素，存入一维数组 \(B[0 \ldots \frac{n(n+1)}{2}-1]\).

下标对应关系 ( \(1 \leq i,j \leq n\)，行优先存储下三角 ) :

\[k = \begin{cases} \dfrac{i(i-1)}{2} + j - 1 & i \geq j \text{ ( 下三角 )} \\[6pt] \dfrac{j(j-1)}{2} + i - 1 & i < j \text{ ( 上三角，用 } A[j][i] \text{ ) } \end{cases}\]

下三角矩阵 上三角区域 ( 不含对角线 ) 全为常数 \(c\)，只存下三角 + 对角线元素及常数 \(c\)，共 \(\dfrac{n(n+1)}{2} + 1\) 个元素.

\[k = \begin{cases} \dfrac{i(i-1)}{2} + j - 1 & i \geq j \\[6pt] \dfrac{n(n+1)}{2} & i < j \text{ ( 存常数 } c \text{ 的位置 ) } \end{cases}\]

上三角矩阵 下三角区域全为常数，类似处理，只存上三角 + 对角线 + 常数.

对角矩阵 ( 带状矩阵 ) 非零元素集中在主对角线及其两侧共 \(d\) 条对角线的带状区域内，带宽为 \(d = 2b+1\) ( \(b\) 为半带宽 ).

以三对角矩阵 ( \(b=1\) ) 为例，只存储 \(3n-2\) 个非零元素

\[k = 2(i-1) + (j - i + 1) \quad (|i-j| \leq 1)\]

反向推导 已知 \(k\)，求 \((i,j)\)

\[i = \left\lfloor \dfrac{k+1}{3} \right\rfloor + 1, \quad j = k - 2(i-1) + i - 1\]

3.3.3 稀疏矩阵

稀疏矩阵 矩阵中非零元素个数远少于零元素个数. 通常非零元素个数 \(t \leq \dfrac{m \times n}{3}\) 时视为稀疏.

三元组表 ( Triple ) 将每个非零元素用 \((i,\ j,\ v)\) 表示，顺序存入一维数组，同时记录矩阵规模 \(m\)、\(n\) 和非零元素总数 \(t\).

  • 存储量 \(O(t)\)，\(t\) 为非零元个数.
  • 转置算法 按列扫描原矩阵，时间复杂度 \(O(n \times t)\) ( 普通转置 ) 或 \(O(n+t)\) ( 快速转置 ).

十字链表 ( Orthogonal List ) 每个非零元素用一个结点表示，结点含 5 个域

  • right 指向同行下一个非零元素.
  • down 指向同列下一个非零元素.
  • 另有 rhead[] 行头指针数组和 chead[] 列头指针数组.
  • 优点 插入、删除非零元素方便，无需移动大量数据.
  • 缺点 指针开销大，结构复杂，不如三元组直观.
表示方法 存储量 转置复杂度 插删方便性 适用场景
三元组表 \(O(t)\) \(O(n+t)\) 一般 静态稀疏矩阵
十字链表 \(O(t)\) \(O(t)\) 好 动态修改频繁

3.3.4 408 常考点

  • 先确认下标起始 ( 0 还是 1 ) 与存储方式 ( 行优先还是列优先 )
  • 地址公式里最容易错的是把 \(m\)、\(n\) 写反
  • 题目常给元素大小 \(L\) 与首地址，最终答案常以字节偏移给出
  • 对称矩阵常让你算 \(k\) 或反算 \((i,j)\)
  • 三角矩阵会额外出现一个「常数位」存 \(c\)，容易漏
  • 带状矩阵要先判断 \((i,j)\) 是否落在带内，否则该位置为 0 或常数

快速转置把「按列找元素」改为「一次统计 + 前缀和定位」，把复杂度降到 (O(n+t)).

  • num[col] 统计每列非零元个数
  • pos[col] 记录该列在转置后数组中的起始位置 ( 前缀和 )
  • 扫描原三元组数组时直接把元素放到转置数组的目标下标

3.4 例题精选

例题 1 ( 循环队列队长 ) 循环队列 MaxSize = 10，front = 6，rear = 3，求队列中元素个数.

解 \(\text{len} = (\text{rear} - \text{front} + \text{MaxSize}) \bmod \text{MaxSize} = (3 - 6 + 10) \bmod 10 = 7\).

例题 2 ( 出栈序列合法性 ) 入栈序列为 \(1,2,3\)，判断 \(3,1,2\) 是否为合法出栈序列.

解 模拟过程 压 1、压 2、压 3、弹 3，此时栈为 \([1,2]\)，栈顶为 2，但需弹出 1，不合法. 因此 \(3,1,2\) 不是合法出栈序列.

例题 3 ( 对称矩阵下标映射 ) 对称矩阵 \(A[1\ldots 6][1\ldots 6]\) 按行优先存储下三角于 \(B[0\ldots 20]\)，求 \(A[4][2]\) 对应 \(B\) 中下标 \(k\).

解 \(i = 4 > j = 2\)，属于下三角，\(k = \dfrac{i(i-1)}{2} + j - 1 = \dfrac{4 \times 3}{2} + 2 - 1 = 7\).

例题 4 ( 中缀转后缀 ) 将中缀表达式 A + B * (C - D) - E / F 转换为后缀表达式.

解 扫描过程 ( 运算符栈记为 OP，输出序列记为 OUT )

  1. A → OUT: A
  2. + → OP: +
  3. B → OUT: A B
  4. * → OP: + *
  5. ( → OP: + * (
  6. C → OUT: A B C
  7. - → OP: + * ( -
  8. D → OUT: A B C D
  9. ) → 弹出直到 ( 输出 -，丢弃 (
  10. - → 弹出优先级不小于 - 的运算符 依次输出 *、+，再将 - 入栈
  11. E → 输出
  12. / → 入栈
  13. F → 输出
  14. 扫描结束 → 弹出剩余运算符

结果为 A B C D - * + E F / -.

例题 5 ( 出栈序列判定 ) 入栈序列为 \(1,2,3,4,5\)，判断出栈序列 \(4,5,3,2,1\) 是否合法.

解 模拟入栈出栈.

依次压入 1,2,3,4，弹出 4. 再压入 5，弹出 5. 此时栈顶为 3，弹出 3. 继续弹出 2,1. 可完成输出，因此为 合法 序列.

例题 6 ( 出栈序列判定 ) 入栈序列为 \(1,2,3,4,5\)，判断出栈序列 \(3,1,2,5,4\) 是否合法.

解 压入 1,2,3，弹出 3. 此时栈顶为 2，但下一次需要输出 1，且 1 被 2 压住，无法直接弹出. 继续入栈只能压入 4,5，使 1 更不可能弹出. 因此该序列 不合法.

例题 7 ( 两栈实现队列的均摊 ) 用两个栈实现队列，入队执行 push(A, x)，出队若栈 B 为空则把 A 全部倒入 B，再 pop(B). 证明出队操作均摊为 \(O(1)\).

解 每个元素最多经历一次从栈 A 到栈 B 的转移.

  • 入队时压入 A 一次
  • 转移时从 A 弹出一次并压入 B 一次
  • 出队时从 B 弹出一次

因此每个元素总共参与常数次栈操作，总成本为 \(O(m)\)，\(m\) 次队列操作的均摊时间复杂度为 \(O(1)\).

例题 8 ( 循环队列操作序列 ) 循环队列 MaxSize = 6 采用牺牲一格方案，初始 front = rear = 0. 依次执行入队 1,2,3,4，出队两次，再入队 5,6，求此时 front、rear 与队列长度.

解 牺牲一格方案下，队列长度 \[\text{len} = (\text{rear} - \text{front} + \text{MaxSize}) \bmod \text{MaxSize}.\]

模拟过程后队列中为 3,4,5,6.

此时 front = 2，rear = 0，因此 \[\text{len} = (0 - 2 + 6) \bmod 6 = 4.\]

例题 9 ( 地址计算 ) 二维数组 \(A[1\ldots 10][1\ldots 20]\) 行优先存储，元素占 4 字节，\(\text{LOC}(A[1][1])=1000\). 求 \(\text{LOC}(A[3][5])\).

解 下标从 1 开始，行优先 : \[\text{LOC}(A[i][j]) = \text{LOC}(A[1][1]) + [(i-1)\times n + (j-1)]\times L.\]

代入 \(i=3,j=5,n=20,L=4\) : \[1000 + [(2\times 20 + 4)\times 4] = 1000 + 176 = 1176.\]

例题 10 ( 三角矩阵压缩 ) 下三角矩阵 \(A[1\ldots n][1\ldots n]\) 的上三角元素均为常数 \(c\)，按行优先压缩到一维数组 \(B\). 求 \(A[i][j]\) 对应的存储下标.

解 只存下三角含主对角线的 \(\\dfrac{n(n+1)}{2}\) 个元素，以及常数位.

\[k = \begin{cases} \dfrac{i(i-1)}{2} + j - 1 & i \ge j \\\\[4pt] \dfrac{n(n+1)}{2} & i < j \end{cases}\]

例题 11 ( 快速转置定位 ) 稀疏矩阵三元组表中共有 \(t\) 个非零元，列数为 \(n\). 说明快速转置中 pos[col] 的含义，并写出其递推关系.

解 pos[col] 表示转置后三元组数组中「第 col 列 ( 转置后为第 col 行 ) 的第一个非零元」存放的位置.

若 num[col] 是原矩阵第 col 列非零元个数，则

  • pos[1] = 0
  • pos[col] = pos[col-1] + num[col-1] ( \(2 \le col \le n\) )

这样扫描原三元组数组时，可以把每个元素直接放到目标位置并递增该列的当前位置指针.

3.5 本章疑难点

1. 循环队列判空 / 判满

  • 牺牲一格方案最常用，务必记住队满条件 (rear+1)%MaxSize == front 而非 rear == front.
  • 三种方案各有优缺点，选择时结合题目要求.

2. 栈与递归的关系

  • 每次函数调用都会在系统栈上开辟一个 栈帧，保存局部变量、参数和返回地址.
  • 递归深度过大会导致 栈溢出，可改用显式栈消除递归.

3. 后缀表达式求值与中缀转后缀

  • 中缀转后缀用 运算符栈; 后缀表达式求值用 操作数栈. 两者使用的栈类型不同，注意区分.

4. 特殊矩阵压缩下标公式

  • 408 考试中常给出 从 1 开始 的下标，务必仔细读题后代入正确公式，不要混淆行列.

5. 十字链表 vs. 三元组

  • 三元组转置更快 ( 快速转置 \(O(n+t)\) ); 十字链表插删更方便. 按照操作类型选择.

3.6 本章小结

  • 栈 ( LIFO ) 和队列 ( FIFO ) 都是操作受限的线性表，本质上是特殊的线性表.
  • 顺序栈 / 链栈、循环队列 / 链队列是四种实现方式，各有适用场景.
  • 双端队列是栈和队列的推广，输入 / 输出受限双端队列是重要变体.
  • 数组地址计算和特殊矩阵压缩存储是纯粹的计算题，掌握公式即可.
  • 稀疏矩阵三元组与十字链表是空间优化的重要思想，十字链表插删优势明显.
