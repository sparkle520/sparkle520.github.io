一、物理层基本概念

1.1 物理层的功能与任务

物理层的地位

物理层是 OSI 参考模型的 最底层，直接与物理传输介质相连. 它不是指具体的物理设备，也不是指传输介质本身，而是指在传输介质上传输 原始比特流 的规则集合.

物理层的主要任务

在数据链路层实体之间建立、维护和释放物理连接，并透明地传输比特流. 所谓”透明”，是指物理层对上层来说就如同不存在一样——不论实际传输使用何种介质，数据链路层看到的都是一条能传送 0/1 比特流的逻辑信道.

物理层解决的核心问题

  1. 使用多大的电压代表 “1” / “0”?
  2. 每个比特持续多长时间?
  3. 数据传输是单向还是双向?
  4. 如何建立和断开物理连接?
  5. 物理接口有多少针脚，每个针脚有什么功能?

1.2 物理层的四大特性

物理层协议的四个特性 ( 接口特性 )

物理层协议 ( 也称 物理层标准 或 物理层接口规范 ) 规定了以下四个方面

特性 含义 典型示例
机械特性 规定接口的形状、尺寸、引脚数目与排列 RJ-45 接口 ( 8针 ) 、DB-25 接口
电气特性 规定接口电缆的各条线上出现的电压范围 \(+15\text{V} \sim -15\text{V}\) 表示逻辑电平
功能特性 规定某条线上出现某一电平的电压所代表的含义 某引脚高电平代表”请求发送”
规程特性 规定各个物理接口部件的工作规程和时序关系 握手信号的先后顺序

记忆口诀 机 ( 接口形状 ) 、电 ( 电压范围 ) 、功 ( 信号含义 ) 、规 ( 时序规程 ).

二、数据通信基础知识

2.1 数据通信系统模型

通信系统的三要素

\[\text{信源} \xrightarrow{\text{发送设备}} \text{信道} \xrightarrow{\text{接收设备}} \text{信宿}\]

组成部分 含义 示例
信源 ( Source ) 产生并发出信息的源头 发送方计算机、话筒
发送设备 将信源信号转换成适合在信道上传输的信号 调制解调器、编码器
信道 ( Channel ) 传输信号的通道 ( 包含传输介质 ) 电话线、光纤、无线信道
噪声源 对信道中信号造成干扰的来源 热噪声、电磁干扰
接收设备 将信道中的信号还原为原始信息 解调器、解码器
信宿 ( Destination ) 接收信息的终点 接收方计算机、扬声器

2.2 信号与信道的基本概念

模拟信号 vs 数字信号

模拟信号 ( Analog Signal ) 数字信号 ( Digital Signal )
特征 连续变化的电磁波 离散的电脉冲序列 ( 方波 )
表示 幅度连续变化 只有有限个离散值 ( 如 0 和 1 )
抗干扰性 弱 ( 噪声直接叠加 ) 强 ( 易通过门限判决恢复 )
适用介质 电话网、有线电视 计算机网络

信道的分类

  • 模拟信道 传输模拟信号 ( 如传统电话网的本地环路 ).
  • 数字信道 传输数字信号 ( 如光纤、数字中继 ).
  • 单工通信 只能单向传输 ( 如广播、电视 ).
  • 半双工通信 可双向传输，但不能同时 ( 如对讲机 ).
  • 全双工通信 可同时双向传输 ( 如电话、现代以太网 ).

2.3 码元、波特、速率 ( 重点 )

核心概念辨析

码元 ( Symbol ) 数字通信中表示一个 离散状态 的信号单元，是数字传输的基本时间单位.

波特率 ( Baud Rate ) 又称 符号率 或 调制速率，单位为 波特 ( Baud )，表示每秒传输的码元个数. \[R_B = \frac{1}{T}\ \text{ ( Baud ) }, \quad T \text{ 为码元持续时间 ( s ) }\]

比特率 ( Bit Rate ) 又称 数据率，单位为 bps，表示每秒传输的二进制位数.

两者关系

\[R_b = R_B \times \log_2 N\]

其中 \(N\) 为每个码元可表示的离散状态数 ( 电平数 ).

码元状态数 \(N\) 每个码元携带比特数 \(\log_2 N\) 关系
2 ( 二进制 ) 1 bit \(R_b = R_B\)
4 ( 四进制 ) 2 bit \(R_b = 2R_B\)
8 ( 八进制 ) 3 bit \(R_b = 3R_B\)
16 ( 十六进制 ) 4 bit \(R_b = 4R_B\)

举例 某信道波特率为 \(2400\ \text{Baud}\)，采用 4 种状态 ( \(N=4\) ) ，则比特率 \(= 2400 \times 2 = 4800\ \text{bps}\).

易混淆点 带宽的两种含义

语境 含义 单位
通信/模拟信号领域 信号占用的频率范围 ( 最高频率 - 最低频率 ) Hz
计算机网络领域 信道的最高数据传输速率 bps

两者通过 香农定理 相互关联 带宽 ( Hz ) 越大，可支持的最大比特率 ( bps ) 越高.

408 考试中，两种含义均会出现，需根据语境判断.

2.4 奈奎斯特定理与香农定理 ( 核心公式 )

奈奎斯特定理 ( Nyquist Theorem ) —— 理想无噪声信道的“内部规定”

在 理想低通 ( 无噪声、带宽有限 ) 的信道中，最高码元传输速率为

\[R_{B,max} = 2W\ \text{ ( Baud ) }\]

其中 \(W\) 为信道带宽 ( Hz ). 对应的最大比特率 ( 信道容量 ) 为

\[C = 2W\log_2 N\ \text{ ( bps ) }\]

其中 \(N\) 为信号的离散电平数 ( 或称状态数 ).

🧠 理解与记忆技巧

  • 物理含义 想象一条宽度为 \(W\) 的公路，奈奎斯特规定了这条路上车流的“极限速度”是 \(2W\) ( 码元速率 ). 车速再快就会发生“追尾” ( 码间串扰 ).
  • 关于 \(N\) \(\log_2 N\) 就是“每辆车装载的货物量” ( 每个码元携带的比特数 ). \(N\) 越大，总运货量 ( 比特率 \(C\) ) 就越大. 在理想状态下，只要 \(N\) 无限大，比特率就能无限大.
  • 奈奎斯特采样定理 ( PCM 基础 ) 对模拟信号进行数字化采样时，采样频率 \(f_s\) 必须满足 \(f_s \ge 2f_{max}\). ( 只有采样频率达到最高频率的2倍，才能无失真恢复原信号 )

香农定理 ( Shannon Theorem ) —— 有噪声信道的“自然法则”

在有随机热噪声的信道中，信道的极限信息传输速率 ( 信道容量 ) 为

\[C = W\log_2\left(1 + \frac{S}{N}\right)\ \text{ ( bps ) }\]

其中

  • \(W\) 信道带宽 ( Hz )
  • \(S\) 信号功率 ( W )
  • \(N\) 噪声功率 ( W )
  • \(S/N\) 信噪比 ( 无量纲比值，注意公式里绝对不能代入 dB 值！ )

信噪比的 dB 换算 ( 考试常设陷阱 ) \[\text{信噪比}(\text{dB}) = 10 \lg \frac{S}{N}\] 技巧 \(10\text{dB} \rightarrow 10^1 = 10\) 倍; \(20\text{dB} \rightarrow 10^2 = 100\) 倍; \(30\text{dB} \rightarrow 10^3 = 1000\) 倍.

🧠 理解与记忆技巧

  • 物理含义 真实世界是有噪声的 ( 下雨起雾 ). 香农定理告诉你，不管你的“车” ( 电平数 \(V\) ) 造得多大，由于看不清 ( 噪声 \(N\) 掩盖了信号 \(S\) )，总的运货量 \(C\) 是有绝对天花板的.
  • 重要结论 1. 噪声不为零时，信道容量是有限的. 它给出了无论采用多么复杂的编码都无法突破的上限.
    1. 提高容量的途径 拓宽公路 ( 增大 \(W\) )、提高信号强度 / 降低噪声 ( 提高 \(S/N\) ).

奈奎斯特定理 vs 香农定理对比 ( 考研必考综合应用 )

奈奎斯特定理 ( Nyquist ) 香农定理 ( Shannon )
信道环境 理想、无噪声 真实、有噪声
公式 \(C = 2W\log_2 N\) \(C = W\log_2(1+S/N)\)
限制因素 带宽 \(W\)，电平数 \(N\) 带宽 \(W\)，信噪比 \(S/N\)
理论极限 \(N\) 趋于无穷，\(C\) 趋于无穷 受限于 \(S/N\)，\(C\) 有绝对上限

💡 做题套路 ( 联合使用 ) 考研大题常常同时给出带宽 \(W\)、信噪比 \(S/N\) 和电平数 \(N\).

原则 真正的最大数据率，必须同时满足奈奎斯特和香农的要求，因此取两者计算结果的较小值！

  • 套路 1 先用香农定理算出理论天花板 \(C_{shannon}\)，再用奈奎斯特算出 \(C_{nyquist}\)，取 \(\min(C_{shannon}, C_{nyquist})\).
  • 套路 2 问“要达到香农极限，至少需要多少个电平？”，令 \(2W\log_2 N = W\log_2(1+S/N)\) 解出 \(N\) 即可.

示例 1 ( 奈奎斯特 + 香农联合计算 )

已知某电话信道带宽 \(W = 3100\ \text{Hz}\)，信噪比 \(S/N = 30\ \text{dB}\).

  1. 用香农定理求信道容量.
  2. 若要达到此容量，用奈奎斯特定理求需要多少个离散电平.

解 (1) 将 dB 换算为比值 \(S/N = 10^{30/10} = 1000\).

\[C = W\log_2(1+S/N) = 3100 \times \log_2(1001) \approx 3100 \times 9.97 \approx 30907\ \text{bps} \approx 30\ \text{kbps}\]

  1. 由奈奎斯特定理，信道最大码元率为 \(2W = 6200\ \text{Baud}\). 要使比特率达到 \(30907\ \text{bps}\)

\[\log_2 N = \frac{C}{2W} = \frac{30907}{6200} \approx 4.99 \implies N \approx 2^5 = 32\]

即需要约 32 个离散电平 才能达到香农容量上限 ( 实际上 30 dB 的信噪比意味着噪声较大，难以可靠区分 32 个电平 ).

结论 这也说明了为何现实通信系统通常工作在香农容量的 70%~80%，留有余量.

三、编码与调制

3.1 基本概念

编码与调制的区别

编码 ( Encoding ) 调制 ( Modulation )
含义 将数字/模拟数据转换为 数字信号 将数字/模拟数据转换为 模拟信号
输出 数字信号 ( 方波、脉冲 ) 模拟信号 ( 正弦波 )
适用信道 数字信道 ( 基带传输 ) 模拟信道 ( 宽带传输 )

四种转换关系

3.2 数字数据→数字信号 ( 基带编码 )

编码方式 同步能力 带宽效率 典型应用
NRZ 无 ( 连 0/1 时 ) 高 ( 100% ) 串行通信
曼彻斯特 好 50% 10 Mbps 以太网
差分曼彻斯特 好 50% 令牌环网

3.3 数字数据→模拟信号 ( 数字调制 )

3.4 模拟数据→数字信号 ( PCM )

脉冲编码调制 ( PCM, Pulse Code Modulation )

PCM 是将模拟信号转换为数字信号的标准方法，分三步

第一步 采样 ( Sampling ) 按照奈奎斯特采样定理，以频率 \(f_s \ge 2f_{max}\) 对模拟信号采样.

  • 示例 电话语音频率范围 \(300 \sim 3400\ \text{Hz}\)，取 \(f_{max} = 4000\ \text{Hz}\)，故采样频率 \(f_s = 8000\ \text{Hz}\) ( 每秒 8000 个样本 ).

第二步 量化 ( Quantization ) 将采样值映射到有限个离散幅度等级 ( 量化电平 ).

  • 将幅度范围划分为 \(2^n\) 个等级 ( \(n\) 为量化比特数 ).
  • 量化误差 ( 量化噪声 ) 不可避免，\(n\) 越大，误差越小.

第三步 编码 ( Encoding ) 将量化后的值用 \(n\) 位二进制数表示.

PCM 数据率计算 \[R_b = f_s \times n\ \text{ ( bps ) }\]

示例 电话语音 PCM，\(f_s = 8000\ \text{Hz}\)，\(n = 8\ \text{bit}\)，则 \[R_b = 8000 \times 8 = 64000\ \text{bps} = 64\ \text{kbps}\]

这就是一路电话的标准数字化速率，即 DS0 ( 基本数字信道 ).

四、交换技术

4.1 电路交换 ( Circuit Switching )

工作原理与过程

电路交换在通信双方之间建立一条 专用的物理通路，通信期间独占该通路.

三个阶段

  1. 建立连接 通话前必须先建立从源到目的的物理链路 ( 拨号过程 ).
  2. 数据传输 整个通信期间独占该电路，通信双方实时传输.
  3. 释放连接 通话结束后断开连接，释放占用的资源.

优点

  • 通信时延小 ( 数据直通，无存储转发延迟 ).
  • 实时性强，适合语音、视频等实时应用.
  • 通信有序，无失序问题.

缺点

  • 资源利用率低 建立连接后，即使不传输数据，电路资源也被占用 ( “说话停顿”时浪费信道 ).
  • 建立连接时间长 需要端到端逐段建立路径，延迟较大.
  • 灵活性差 固定带宽，无法适应突发流量.
  • 网络中一个节点故障会中断通信.

适用场景 传统公共电话网 ( PSTN ) 、电话会议.

4.2 报文交换 ( Message Switching )

工作原理

报文交换将整个报文作为一个数据单元，在网络中采用 存储-转发 方式传输

  1. 发送方将完整报文发给第一个节点 ( 路由器 ).
  2. 节点 存储 整个报文，待输出链路空闲时再转发到下一节点.
  3. 直到报文到达目的地.

优点

  • 无需建立连接，信道利用率比电路交换高.
  • 可以实现多目标投递 ( 广播 ).

缺点

  • 存储整个报文需要 大量缓存，不适合长报文.
  • 时延大且不确定，不适合实时通信.
  • 若某节点存储能力不足，报文可能被丢弃.

现代已基本被分组交换取代，但电子邮件、电报等非实时应用仍体现报文交换思想.

4.3 分组交换 ( Packet Switching )

工作原理

分组交换将报文 拆分为固定大小的分组 ( 包 )，每个分组独立存储-转发.

核心优势来自 统计复用 ( Statistical Multiplexing )

多个用户的分组按需共享同一链路，链路利用率大幅提升. 当某用户暂无数据发送时，其他用户可使用全部带宽.

分组交换的流程 ( 以 3 跳网络为例 )

流水线效应 多个分组可以在不同节点间并行传输，总时延远小于报文交换.

🧠 理解与记忆技巧

  • 生活比喻 假设你要搬 1000 块砖 ( 一个大报文 ) 到 3 公里外的地方，中间有 2 个中转站.
    • 报文交换 就是一辆大卡车装满 1000 块砖，开到中转站 1 卸下，再装上开到中转站 2… 这样太慢了.
    • 分组交换 就是把 1000 块砖分成 100 份，每份 10 块交给一辆小推车. 第一辆小推车到了中转站 1 后，立刻出发去中转站 2，此时第二辆小推车也出发去中转站 1. 这样多辆车在路上同时跑，就是 流水线效应，大大节省了时间！
  • 本质 以空间 ( 额外的头部开销 ) 换时间 ( 并行传输降低时延 ).

💡 做题套路 ( 时延计算 ) 考研大题常考“分组交换总时延”的计算，核心公式是基于流水线效应推导的.

假设报文被分为 \(n\) 个分组，经过 \(k\) 段链路 ( 经过 \(k-1\) 个路由器 ).

总时延 = 最后一个分组的发送与传播时延 + 前面 \(n-1\) 个分组的发送时延

  1. 算出单个分组在一跳的发送时延 \(t_s = \frac{\text{分组长度 ( 含首部 )}}{\text{带宽}}\)
  2. 算出单跳的传播时延 \(t_p = \frac{\text{距离}}{\text{传播速度}}\)
  3. 套用流水线公式 \(t_{total} = (n + k - 1) \times t_s + k \times t_p\) ( 注意 : 题目若说忽略传播时延，则 \(t_p = 0\) )

优点

  • 存储-转发时延小 ( 仅需缓存单个小分组 ).
  • 灵活高效，可适应突发流量 ( 统计复用 ).
  • 可靠性高，某链路故障可绕道.
  • 链路利用率高.

缺点

  • 存在排队时延，且时延可变 ( 不适合严格实时 ).
  • 分组头部开销 ( 每个分组需携带目的地址等控制信息 ).
  • 可能出现失序 ( 各分组路径不同 ) ，需接收方重组.
  • 网络拥塞时会丢包.

互联网 ( IP 网络 ) 采用分组交换.

三种交换方式对比 ( 考研必考 )

比较维度 电路交换 报文交换 分组交换
是否需要建立连接 是 ( 必须 ) 否 否
传输单位 比特流 ( 连续 ) 整个报文 分组 ( 固定大小 )
传输方式 直通 ( 无存储转发 ) 存储转发 存储转发
信道利用率 低 中 高 ( 统计复用 )
时延 小 ( 无排队，数据实时到达 ) 大 较小但有波动
实时性 最好 差 较好 ( 不如电路 )
适用场景 语音电话 电子邮件 互联网数据通信
典型网络 PSTN 电话网 电报网 Internet ( IP 网 )

4.4 数据报与虚电路

分组交换的两种实现方式

分组交换内部有两种转发机制，两者在网络层的体现尤为重要.

1. 数据报 ( Datagram )

每个分组称为 数据报，携带 完整的目的地址，在网络中 独立路由.

  • 每到一个路由器，查路由表决定下一跳.
  • 不同分组可走不同路径，到达顺序可能与发送顺序不同 ( 失序 ).
  • 无需建立连接，网络无需保存连接状态.
  • 路由器故障时，后续分组可自动绕道 ( 高可靠性 ).

互联网的 IP 协议采用数据报方式.

2. 虚电路 ( Virtual Circuit )

在通信前先建立一条从源到目的的 逻辑通路 ( 虚电路 )，数据传输期间，所有分组沿此路径按序传输.

  • 分组只需携带 虚电路编号 ( VCI )，不需完整地址，头部开销小.
  • 分组 沿同一路径按序到达，无失序问题.
  • 网络中的节点需保存虚电路状态 ( 连接表 ) ，资源开销大.
  • 某节点故障会导致 该虚电路中断，可靠性不如数据报.

ATM 网络采用虚电路方式 ( 异步传输模式 ).

数据报 vs 虚电路对比 ( 重要 )

比较维度 数据报 虚电路
连接建立 不需要 需要 ( 通信前建立 )
路由 逐包独立路由 建立时确定，分组沿固定路径
地址 完整目的地址 虚电路标识符 ( VCI )
分组顺序 可能失序 保证有序
节点存储 只需路由表 需保存虚电路表 ( 连接状态 )
故障影响 可绕道，自适应 虚电路中断，需重建
适用场景 突发数据，容忍失序 ( IP ) 要求有序、固定时延 ( ATM )
典型代表 IP ( 互联网 ) ATM、X.25、帧中继

记忆关键 虚电路类似电路交换 ( 有连接、有序 ) 但共享信道 ( 分组交换 ); 数据报类似邮局投递信件 ( 每封信独立路由，不保证顺序 ).

示例 2 ( 三种交换方式的时延分析 )

设有 3 个节点 A、B、C ( A→B→C ) ，A 向 C 发送 \(10^7\) bit 的数据，每段链路带宽 \(1\ \text{Mbps}\)，传播时延 \(0.001\ \text{s}\).

(1) 电路交换时延 ( 忽略建立连接时间 )

A 直通发送，路径确定，总时延 \[t = \frac{10^7}{10^6} + 2 \times 0.001 = 10 + 0.002 = 10.002\ \text{s}\]

(2) 报文交换时延 ( 整体存储转发 )

B 需先完全收到再转发 \[t = 2 \times \frac{10^7}{10^6} + 2 \times 0.001 = 20 + 0.002 = 20.002\ \text{s}\]

(3) 分组交换时延 ( 分组大小 \(10^3\) bit，共 \(10^4\) 个分组 )

利用流水线效应，最后一个分组到达 C 的时延 \[t = (n+k-1) \times \frac{L}{B} + k \times t_p\]

其中 \(n=10^4\) 个分组，\(k=2\) 跳，\(L=10^3\) bit，\(B=10^6\) bps，\(t_p=0.001\ \text{s}\) \[t = (10^4 + 2 - 1) \times 10^{-3} + 2 \times 0.001 = (10001) \times 0.001 + 0.002 \approx 10.003\ \text{s}\]

结论 分组交换时延 ( 10.003 s ) 接近电路交换 ( 10.002 s ) ，远优于报文交换 ( 20.002 s ).

五、传输介质

5.1 导向传输介质 ( 有线介质 )

双绞线 ( Twisted Pair )

将两根绝缘铜导线按一定规律 螺旋状绞合 在一起，绞合可以减少电磁干扰 ( 相邻导线的干扰相互抵消 ).

类型 全称 特点 典型应用
UTP 非屏蔽双绞线 无金属屏蔽层，成本低，灵活 以太网 ( Cat5e/Cat6 )
STP 屏蔽双绞线 有金属屏蔽层，抗干扰强，成本高 工业环境、高速网络

双绞线类别 ( Cat ) 与支持速率

类别 支持频率 最大速率 典型应用
Cat 3 16 MHz 10 Mbps 传统以太网
Cat 5e 100 MHz 1 Gbps 百兆/千兆以太网
Cat 6 250 MHz 10 Gbps ( 短距 ) 数据中心
Cat 7 600 MHz 10 Gbps 高速局域网

双绞线 最大传输距离约 100 m ( 以太网标准规定 ) ，超过需中继器或交换机.

同轴电缆 ( Coaxial Cable )

由 中心铜导线、绝缘层、金属屏蔽网、外绝缘层 构成的同轴结构，抗干扰能力强于双绞线.

类型 阻抗 典型应用
基带同轴电缆 \(50\ \Omega\) 早期以太网 ( 10BASE-5，10BASE-2 )
宽带同轴电缆 \(75\ \Omega\) 有线电视 ( CATV ) ，频分复用传输多路信号

现状 在局域网领域已被双绞线取代，但仍广泛用于有线电视 ( CATV ) 网络.

光纤 ( Optical Fiber )

利用 全反射 原理，光信号在折射率较高的纤芯中以接近光速传播，带宽极大，不受电磁干扰.

光纤通信系统

  • 光源 ( LED / 激光二极管 LD )
  • 光纤 ( 纤芯 + 包层 )
  • 光检测器 ( 光电二极管 )

单模光纤 vs 多模光纤

单模光纤 ( Single-Mode ) 多模光纤 ( Multi-Mode )
纤芯直径 \(\sim 9\ \mu m\) ( 极细 ) \(50 \sim 62.5\ \mu m\)
光源 激光 ( LD ) LED / 激光
传输模式 只有一条光路 ( 基模 ) 多条光路 ( 色散较大 )
带宽 极大，近乎无限 较小 ( 色散限制 )
传输距离 长 ( 几十至上百 km ) 短 ( \(\le 2\ \text{km}\) )
成本 高 ( 需精密连接器 ) 低
典型应用 主干网、城域网、广域网 楼宇内局域网

光纤的主要优点

  1. 带宽极大 ( 理论上可达 Tbps 级别 ).
  2. 传输距离远，损耗低 ( 光纤损耗约 \(0.2\ \text{dB/km}\)，铜线约 \(2\ \text{dB/km}\) ).
  3. 完全不受电磁干扰 ( 光信号，非电信号 ).
  4. 安全性好 ( 难以窃听，弯曲就会泄漏光，易于检测 ).
  5. 体积小，重量轻.

光纤的主要缺点

  1. 质地脆，弯折半径有要求，安装维护成本高.
  2. 光电接口成本高.
  3. 单向传播 ( 全双工需两根光纤或 WDM ).

5.2 非导向传输介质 ( 无线介质 )

主要无线传输方式

类型 频段 传播方式 特点与应用
无线电波 \(300\ \text{kHz} \sim 3\ \text{GHz}\) 地面传播 / 天波 穿透能力强，广播、LTE/5G

卫星通信的传播时延

同步轨道卫星高度约 \(36000\ \text{km}\)，电磁波传播时间 \[t_p = \frac{36000 \times 10^3}{3 \times 10^8} \approx 0.12\ \text{s} = 120\ \text{ms}\]

上下行各一次，单向时延 \(\approx 270\ \text{ms}\) ( 信号要从地面上行到卫星，再下行到接收站 ).

七、综合真题解析 ( Selected Problems )

真题 1 ( 奈奎斯特定理综合计算 )

若某无噪声信道的带宽为 \(4000\ \text{Hz}\)，采用 8 种不同的物理状态来表示数据，求该信道的最大数据传输速率.

解 由奈奎斯特定理 \[R_{B,max} = 2W = 2 \times 4000 = 8000\ \text{Baud}\]

每个码元携带 \(\log_2 8 = 3\ \text{bit}\)，所以最大比特率 \[C = 2W\log_2 N = 8000 \times 3 = 24000\ \text{bps} = 24\ \text{kbps}\]

真题 2 ( 香农定理 + 奈奎斯特联合 )

已知某信道带宽 \(W = 4000\ \text{Hz}\)，信噪比 \(S/N = 255\). (1) 求该信道的香农容量. (2) 若用奈奎斯特定理达到此容量，至少需要多少个信号电平?

解 (1) 香农容量 \[C = W\log_2(1+S/N) = 4000 \times \log_2(256) = 4000 \times 8 = 32000\ \text{bps} = 32\ \text{kbps}\]

  1. 奈奎斯特最大码元率为 \(2W = 8000\ \text{Baud}\). 为使比特率达到 \(32000\ \text{bps}\) \[\log_2 N = \frac{32000}{8000} = 4 \implies N = 2^4 = 16\]

即至少需要 16 个离散电平 ( 4 bit 每码元 ).

真题 3 ( PCM 编码计算 )

电话系统对语音信号进行 PCM 编码 语音带宽 \(0 \sim 4000\ \text{Hz}\)，采用 8 位编码. (1) 求最低采样频率. (2) 求一路数字化语音的比特率. (3) 一条 T1 线路可复用多少路语音? ( T1 速率 \(1.544\ \text{Mbps}\)，其中 8 kbps 用于帧同步 )

解 (1) 最低采样频率 \[f_s = 2f_{max} = 2 \times 4000 = 8000\ \text{Hz}\]

  1. 每路语音比特率 \[R_b = f_s \times n = 8000 \times 8 = 64000\ \text{bps} = 64\ \text{kbps}\]

  2. T1 可用于语音的带宽 \(1544 - 8 = 1536\ \text{kbps}\). 路数 \[\frac{1536\ \text{kbps}}{64\ \text{kbps}} = 24\ \text{路}\]

T1 线路标准确实复用 24 路 DS0 ( 64 kbps ) 语音，总速率 \(24 \times 64 + 8 = 1544\ \text{kbps} = 1.544\ \text{Mbps}\).

真题 4 ( 分组交换时延计算 )

主机 A 向主机 B 发送报文，经过 2 个路由器 ( 共 3 段链路 ) ，每段链路带宽 \(1\ \text{Mbps}\)，每段传播时延 \(1\ \text{ms}\)，报文大小 \(3 \times 10^6\) bit，分组大小 ( 含首部 ) \(10^3\) bit.

  1. 每个分组的发送时延 ( 每段 ) 是多少?
  2. 使用分组交换，总时延是多少 ( 忽略排队时延和处理时延 ) ?
  3. 与报文交换相比，分组交换节省了多少时延?

解 共 \(n = 3 \times 10^6 / 10^3 = 3000\) 个分组，\(k = 3\) 段链路.

  1. 每个分组每段的发送时延 \[t_s = \frac{10^3}{10^6} = 10^{-3}\ \text{s} = 1\ \text{ms}\]

  2. 分组交换总时延 ( 流水线 )

最后一个分组到达目的地 \[t_{packet} = (n + k - 1) \times t_s + k \times t_p = (3000 + 3 - 1) \times 1 + 3 \times 1 = 3002 + 3 = 3005\ \text{ms}\]

  1. 报文交换总时延 ( 整体存储转发 )

\[t_{message} = k \times \frac{3 \times 10^6}{10^6} + k \times t_p = 3 \times 3000 + 3 \times 1 = 9000 + 3 = 9003\ \text{ms}\]

分组交换节省 \(9003 - 3005 = \mathbf{5998}\ \textbf{ms} \approx 6\ \textbf{s}\).

体现了分组交换流水线的巨大优势.

真题 5 ( 曼彻斯特编码分析 )

  1. 简述曼彻斯特编码与差分曼彻斯特编码的区别.
  2. 若某信道传输曼彻斯特编码信号，则实际数据率与信号的最高频率 ( 带宽 ) 之间是什么关系?
  3. 10 Mbps 以太网 ( 采用曼彻斯特编码 ) 占用的信道带宽是多少?

解

  1. 曼彻斯特编码 每个比特的中间时刻有且仅有一次跳变， 用跳变的方向 ( 下降/上升 ) 表示数据值 ( 1 或 0 ). 接收方从跳变中提取时钟，也识别数据.

差分曼彻斯特编码 每个比特的中间时刻 一定有跳变 ( 用于时钟同步，不携带数据 ). 比特值由每个比特开始处是否有跳变决定 有额外跳变表示 “0”，无跳变表示 “1” ( 或相反定义 ). 其抗干扰能力更强，因为判决基于相对跳变，而非绝对电平.

  1. 曼彻斯特编码每个比特需要 2 次电平变化 ( 中间必有跳变 ) ，相当于将信号频率 翻倍. 因此 \[\text{信道带宽 ( Hz ) } = 2 \times \text{数据率 ( bps ) }\] 即带宽效率仅为 50%.

  2. 10 Mbps 以太网需要的信道带宽 \[W = 2 \times 10\ \text{Mbps} = 20\ \text{MHz}\]

真题 6 ( 传输介质综合判断 )

某城市需要建设一条高速主干光纤链路，连接两个数据中心，距离约 \(80\ \text{km}\)，要求带宽 \(\ge 100\ \text{Gbps}\)，高可靠性. 请分析应选用何种光纤，并说明理由.

解

应选用 单模光纤 ( SMF ).

理由

  1. 距离 \(80\ \text{km}\) 远超多模光纤的有效传输距离 ( \(\le 2\ \text{km}\) ) ，多模光纤因 模色散 导致信号严重失真，无法使用.
  2. 带宽 单模光纤纤芯极细 ( 约 \(9\ \mu m\) ) ，只允许一条光路， 几乎无模色散，带宽极大，可支持 \(100\ \text{Gbps}\) 乃至更高速率 ( 配合 WDM 波分复用可达 Tbps 级 ).
  3. 可靠性 单模光纤通常与 WDM ( 波分复用 ) 技术结合，多个波长承载不同信号，某个波长故障不影响其他，可靠性高.
  4. 成本 虽然单模光纤接口成本较高，但主干链路的带宽、距离和可靠性需求使其成为唯一合理选择，长期运维成本低于铺设多条多模光纤.

补充 若要进一步提高容量，可在单模光纤上使用 DWDM ( 密集波分复用 )，在单根光纤上同时传输 80 路及以上的 100 Gbps 信号，总容量可达 \(8\ \text{Tbps}\).
