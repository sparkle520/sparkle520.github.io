一、网络层的功能

网络层的核心任务

网络层位于 OSI 第三层，负责将分组从 源主机传送到目的主机 ，可能需要跨越多个异构网络 ( 不同类型的链路、不同的数据链路层协议 ).

三大核心功能 :

1. 路由 ( Routing ) 选择分组从源到目的的 最优路径. 路由算法运行在路由器上，维护路由表 ( Routing Table ).

2. 转发 ( Forwarding ) 将到达路由器某端口的分组，根据路由表 ( 或转发表 FIB ) 从正确的出口端口发出. 转发是数据平面的操作，速度要求极高.

路由 vs 转发的区别 :

路由 ( Routing ) 转发 ( Forwarding )
时间尺度 慢 ( 秒级，需要计算 ) 快 ( 纳秒级，查表即发 )
所在平面 控制平面 数据平面
输入 网络拓扑信息 单个到达的分组
输出 路由表 正确的出口端口

3. 拥塞控制 ( Congestion Control ) 当网络中分组过多导致性能下降时，采取措施减少拥塞 ( 如 ECN 显式拥塞通知、令牌桶等 ).

其他功能 :

  • 异构网络互联 : 通过 IP 协议屏蔽底层网络差异 ( IPv4/IPv6 是”网络层黏合剂” ).
  • 提供无连接的尽力而为服务 : IP 协议不提供可靠传输保证，可靠性由运输层 ( TCP ) 保证.

二、路由算法

2.1 路由算法分类

静态路由 vs 动态路由

静态路由 动态路由
配置方式 管理员手动配置 路由协议自动维护
适应拓扑变化 不能自适应 能自动适应
开销 无额外通信开销 需要路由协议报文开销
适用场景 小型网络、特定路径控制 大型动态网络
安全性 较高 ( 无协议报文暴露 ) 相对较低 ( 协议报文可伪造 )

特殊静态路由 :

  • 默认路由 ( 0.0.0.0/0 ) : 匹配所有目的地址，当路由表无更具体匹配时使用，相当于”其他所有目的地都往这里走” ( 常用于末节路由器指向 ISP ).
  • 主机路由 ( /32 ) : 指向单个主机的路由.

2.2 距离向量算法 ( Distance Vector, DV )

基本思想 ( Bellman-Ford 方程 )

每个节点维护一张 距离向量表 ，记录到所有已知目的地的最短距离估计值及下一跳. 节点定期将自己的距离向量 广播给直接相邻的邻居 ，邻居根据收到的信息更新自己的距离向量.

更新规则 ( Bellman-Ford ) :

\[d_x(y) = \min_v \{ c(x,v) + d_v(y) \}\]

其中 \(c(x,v)\) 是 \(x\) 到邻居 \(v\) 的链路代价，\(d_v(y)\) 是邻居 \(v\) 到目的 \(y\) 的距离估计.

算法流程 : 1. 初始化 : 到自身距离为 0，到直接邻居距离为链路代价，到其他节点距离为 \(\infty\). 2. 每个节点将自己的距离向量发给所有邻居. 3. 收到邻居的距离向量后，用 Bellman-Ford 公式更新自己的距离向量. 4. 若有变化，再次发给邻居; 若无变化则收敛.

优点 : 实现简单，计算开销小.

缺点 — 无穷计数 ( Count to Infinity ) 问题 :

当链路失效时，距离向量算法收敛极慢，节点间通过相互”误导”逐步将距离计数到无穷大.

典型场景 : A-B-C 线性拓扑，B-C 断开，B 得知 C 不可达后，从 A 处得到 A 说”到 C 距离 2”，B 更新为 3; A 再从 B 得到 3，更新为 4…… 两者相互引用，距离不断增大，直到超过最大值 ( RIP 中为 16 ).

解决方法 :

  • 水平分割 ( Split Horizon ) : 从某邻居学到的路由，不再发回给该邻居.
  • 毒性逆转 ( Poison Reverse ) : 将从某邻居学到的路由以距离无穷大发回给该邻居，更明确地声明”不要通过我到达此目的地”.
  • 触发更新 ( Triggered Update ) : 拓扑变化时立即发送更新，而不等待定时器.

2.3 链路状态算法 ( Link State, LS )

基本思想 ( Dijkstra 算法 )

每个节点通过 泛洪 ( Flooding ) 将自己的链路状态信息 ( 直接相连的邻居及链路代价 ) 发给 全网所有节点 ，从而每个节点都拥有完整的网络拓扑图，然后各自独立运行 Dijkstra 最短路径算法 计算到所有目的节点的最短路径.

算法步骤 ( Dijkstra ) :

  1. 初始化 : 源节点到自身距离 0，到其他节点距离 \(\infty\).
  2. 从未确定集合中选取距离最小的节点 \(u\)，加入确定集合.
  3. 对 \(u\) 的每个邻居 \(v\) : 若 \(d(u) + c(u,v) < d(v)\)，则更新 \(d(v) = d(u) + c(u,v)\)，前驱 = \(u\).
  4. 重复步骤 2~3 直到所有节点确定.

优点 :

  • 快速收敛 : 网络变化后，泛洪机制使全网快速同步，重新计算时间短.
  • 无环路 : 每个节点基于完整拓扑计算，不存在 DV 算法中的路由环路.
  • 支持分层 : OSPF 中的区域 ( Area ) 机制.

缺点 :

  • 开销大 : 泛洪需要消耗带宽，全网拓扑数据库需要较大存储空间.
  • 实现复杂 : 比 DV 算法复杂.

DV vs LS 对比 ( 考研必考 ) :

比较维度 距离向量 ( DV ) 链路状态 ( LS )
信息传播 只发给直接邻居 ( 路由向量 ) 泛洪给全网 ( 链路状态 )
拓扑知识 局部 ( 只知道邻居告诉的 ) 全局 ( 知道整个网络 )
计算算法 Bellman-Ford ( 分布式 ) Dijkstra ( 各自独立计算 )
收敛速度 慢 ( 可能计数到无穷 ) 快
路由环路 可能存在 不会存在
通信开销 小 ( 只传向量给邻居 ) 大 ( 泛洪全网 )
典型协议 RIP OSPF

网络拓扑如下，求从节点 A 到所有节点的最短路径 ( 链路代价标注 ) :

解 :

初始化 : \(d(A)=0\)，\(d(B)=\infty\)，\(d(C)=\infty\)，\(d(D)=\infty\)，\(d(E)=\infty\)，\(d(F)=\infty\).

第1轮 : 选 A ( 距离 0 ) ，确定 A.

  • 更新 B : \(0+2=2\); 更新 C : \(0+5=5\).
  • 状态 : \(\{A:0\}\)，待定 \(\{B:2, C:5, D:\infty, E:\infty, F:\infty\}\).

  • 状态 : \(\{A:0, B:2, E:3\}\)，待定 \(\{C:5, D:5, F:4\}\).

第4轮 : 选 F ( 距离 4 ) ，确定 F.

  • 更新 D : \(\min(5, 4+2)=5\) ( 无改变 ).
  • 状态 : \(\{A:0, B:2, E:3, F:4\}\)，待定 \(\{C:5, D:5\}\).

第5轮 : 选 C 或 D ( 距离均为 5 ) ，确定 C. 第6轮 确定 D.

最终最短路径 :

  • A→B : 2，路径 A-B
  • A→C : 5，路径 A-C ( 或 A-B-E-C，相同代价 )
  • A→D : 5，路径 A-B-D
  • A→E : 3，路径 A-B-E
  • A→F : 4，路径 A-B-E-F

3.1 IPv4 数据报格式

IPv4 数据报首部 ( 最小 20 字节 )

关键字段说明 :

字段 长度 说明
版本 4 bit IPv4 = 4，IPv6 = 6
首部长度 4 bit 单位为 4 字节，最小值 5 ( = 20 B ) ，最大 15 ( = 60 B )
区分服务 8 bit DSCP/ECN，QoS 服务质量标记
总长度 16 bit 首部 + 数据 的总字节数，最大 65535 字节
标识 16 bit 同一数据报的所有分片具有 相同 标识
标志 3 bit DF ( 不允许分片 ) 、MF ( 后面还有分片 )
片偏移 13 bit 该分片在原数据报中的偏移量，单位为 8 字节
TTL 8 bit 每经过一个路由器减 1，为 0 时丢弃并发 ICMP 超时报文
协议 8 bit 上层协议 : TCP=6，UDP=17，ICMP=1，OSPF=89

源/目的 IP 32 bit 各 4 字节

IP 数据报分片 ( 重要 ) :

当数据报总长度超过链路 MTU ( 最大传输单元 ) 时，路由器对数据报进行分片.

  • 分片只在目的主机重组 ( 不在中间路由器重组 ).
  • 分片时， 数据部分 必须是 8 字节的整数倍 ( 因片偏移单位为 8 B ).
  • 除最后一片外，MF=1; 最后一片 MF=0.
  • 所有分片标识相同，片偏移记录各片在原数据报数据部分的起始位置.

示例 2 ( IP 分片计算 )

一个 IP 数据报，总长度 3800 字节 ( 首部 20 字节，数据 3780 字节 ) ，MTU = 1500 字节，求分片情况.

解 :

每片最大数据 = \(1500 - 20 = 1480\) 字节，且须为 8 字节整数倍，\(1480 / 8 = 185\) ( 整除 ) ，合法.

分片编号 数据长度 总长度 MF 片偏移 ( 字节 ) 片偏移字段值
1 1480 B 1500 B 1 0 0
2 1480 B 1500 B 1 1480 185
3 820 B 840 B 0 2960 370

验证 : \(1480 + 1480 + 820 = 3780\) B ✓. 第 3 片 MF=0，片偏移 = \(2960/8 = 370\).

3.3 子网划分与 CIDR

子网划分 ( Subnetting )

将一个网络的主机号部分进一步划分为 子网号 + 主机号 ，通过 子网掩码 指示哪些位是网络/子网号.

子网掩码 : 32 位，网络号和子网号对应位为 1，主机号对应位为 0.

\[\text{网络地址} = \text{IP 地址} \text{ AND } \text{子网掩码}\]

划分子网的步骤 ( 从一个大网中划出若干子网 ) : 1. 确定需要的子网数量，求所需子网号位数 \(k\) ( \(2^k \ge\) 子网数 ). 2. 确定需要的主机数，求所需主机号位数 \(h\) ( \(2^h - 2 \ge\) 主机数 ). 3. 从原主机号中借 \(k\) 位作子网号，剩 \(h\) 位作主机号. 4. 新的子网掩码 = 原掩码加上 \(k\) 位 1.

每个子网可用主机数 = \(2^h - 2\) ( 去掉全 0 网络地址和全 1 广播地址 ).

CIDR ( 无类域间路由，Classless Inter-Domain Routing )

CIDR 废弃了有类 IP 地址的概念，允许以 任意长度 的前缀 ( 网络前缀 ) 划分地址块，用斜线表示法 ( 如 192.168.1.0/24 ).

CIDR 地址块 :

  • /n 表示前 \(n\) 位为网络前缀，后 \(32-n\) 位为主机号.
  • 地址块大小 = \(2^{32-n}\) 个地址.
  • 地址聚合 ( 路由聚合/超网 ) : 将多个连续的 CIDR 地址块合并为一个更大的块，缩短前缀，减少路由表条目.

最长前缀匹配 ( Longest Prefix Match ) :

路由器转发时，若多条路由均能匹配目的 IP，则选择 前缀最长 ( 最具体 ) 的路由，这是 CIDR 中路由查找的基本规则.

示例 : 目的 IP 192.168.1.100，路由表中有 192.168.0.0/16 和 192.168.1.0/24，选 /24 ( 更具体 ).

示例 3 ( 子网划分综合计算 )

某公司分配到地址块 192.168.10.0/24，需划分 4 个子网，每个子网最多容纳 50 台主机. 设计子网方案.

解 :

4 个子网需 \(k\) 位 : \(2^k \ge 4 \implies k = 2\). 50 台主机需 \(h\) 位 : \(2^h - 2 \ge 50 \implies h = 6\) ( \(2^6-2=62\) ). 验证 : \(24 + k + h = 24 + 2 + 6 = 32\) ( 刚好用满 32 位 ) ✓.

新子网掩码 : /26 ( 255.255.255.192，原 /24 加 2 位 1 ).

子网编号 子网地址 可用 IP 范围 广播地址
子网 0 192.168.10.0/26 192.168.10.1 ~ 192.168.10.62 192.168.10.63
子网 1 192.168.10.64/26 192.168.10.65 ~ 192.168.10.126 192.168.10.127
子网 2 192.168.10.128/26 192.168.10.129 ~ 192.168.10.190 192.168.10.191
子网 3 192.168.10.192/26 192.168.10.193 ~ 192.168.10.254 192.168.10.255

每子网可用 62 台主机 ( \(> 50\)，满足需求 ) ✓.

示例 4 ( CIDR 路由聚合 )

某路由器路由表包含以下四条路由，尝试聚合 :

  • 192.168.0.0/24，192.168.1.0/24，192.168.2.0/24，192.168.3.0/24

解 :

转为二进制 ( 第三字节 ) :

  • 0 = 00000000，1 = 00000001，2 = 00000010，3 = 00000011

前两位均为 00，即前 22 位 ( 192.168. 的 16 位 + 第三字节的前 6 位 000000 ) 相同.

等等，准确地说，192.168.0~192.168.3 的第三字节前 6 位均为 000000，后 2 位变化.

聚合后 : 192.168.0.0/22 ( 前 22 位相同，涵盖 192.168.0.0 ~ 192.168.3.255 ).

验证 : 192.168.0.0/22 包含 \(2^{32-22} = 2^{10} = 1024\) 个地址 ( 192.168.0.0 ~ 192.168.3.255 ) ，正好覆盖这 4 个 /24 网络 ✓.

3.4 ARP 协议

ARP ( 地址解析协议，Address Resolution Protocol )

ARP 解决同一链路上 已知 IP 地址，求对应 MAC 地址 的问题.

工作过程 :

  1. 主机 A 需要发送分组给同网段的主机 B ( IP 已知 ) ，先查 ARP 缓存 ( ARP Cache ) .
  2. 若缓存无记录，A 发送 ARP 广播请求 ( 目的 MAC = FF:FF:FF:FF:FF:FF ) ，内容 : “谁的 IP 是 192.168.1.2? 请告诉我你的 MAC 地址.”
  3. 同一链路上所有主机收到请求，但只有 B 回复一个 ARP 单播应答 ( 目的 MAC = A 的 MAC ) ，内容 : “我的 IP 是 192.168.1.2，MAC 是 00:1A:2B:3C:4D:5E.”
  4. A 将 B 的 IP↔︎MAC 映射写入 ARP 缓存 ( 有超时，通常 2 分钟 ).

跨路由器通信时的 ARP :

A ( 192.168.1.1 ) 向 C ( 10.0.0.1 ) 发送分组 ( 不同网段 ) :

  • A 先用 ARP 解析 默认网关 的 MAC ( 而非目的主机 C 的 MAC！ ).
  • 路由器收到后，再次 ARP 解析下一跳的 MAC，依次转发.
  • IP 地址全程不变，MAC 地址逐跳改变.

免费 ARP ( Gratuitous ARP ) :

主机发送 ARP 请求，目标 IP = 自身 IP，用于 : 检测 IP 地址冲突、更新网络中其他节点的 ARP 缓存 ( 如 IP 地址变更后 ).

代理 ARP ( Proxy ARP ) :

路由器代替远程主机回应 ARP 请求，使不支持子网的主机也能通信. 现代网络较少使用.

3.5 DHCP 协议

DHCP ( 动态主机配置协议，Dynamic Host Configuration Protocol )

DHCP 基于 UDP ( 服务器端口 67 ，客户端端口 68 ) ，用于 动态分配 IP 地址 及其他配置信息 ( 子网掩码、默认网关、DNS 服务器等 ).

DHCP 四步握手 ( DORA ) :

广播原因 : 客户端尚无 IP，只能广播; 广播也能让其他 DHCP 服务器知道请求已被满足 ( Request 仍广播 ).

租约机制 :

  • DHCP 分配的 IP 有 租约期 ( Lease Time ) ，到期前客户端需续租 ( 发 DHCP Request ).
  • 服务器续租则发 DHCP ACK，否则客户端需重新走 DORA 流程.
  • 若客户端离开网络，发 DHCP Release 主动释放 IP.

DHCP 中继代理 ( Relay Agent ) :

路由器通常不转发广播，跨子网的主机无法直接找到 DHCP 服务器. 通过在路由器上配置 DHCP 中继代理 ，将 DHCP 广播转换为单播转发给 DHCP 服务器，实现跨网段 DHCP 服务.

3.6 ICMP 协议

ICMP ( 互联网控制报文协议，Internet Control Message Protocol )

ICMP 封装在 IP 数据报中 ( 协议字段 = 1 ) ，用于传递 差错报告 和 网络诊断 信息.

ICMP 报文分为两类 :

1. 差错报告报文 ( Error Reporting )

类型 含义 常见原因
目的不可达 ( Type 3 ) 分组无法送达目的地 无路由、端口未监听、DF 但需分片
源点抑制 ( Type 4 ) 请求发送方降速 路由器缓冲区溢出 ( 现已废弃 )
时间超过 ( Type 11 ) TTL=0 或重组超时 路由环路、ttraceroute 工具基础
参数问题 ( Type 12 ) IP 首部字段有误 非法字段值
重定向 ( Type 5 ) 通知主机更优路由 路由器发现更短路径

2. 查询报文 ( Query )

类型 含义 工具
回声请求/应答 ( Type 8/0 ) 测试可达性 ping 命令
时间戳请求/应答 ( Type 13/14 ) 时钟同步 —
地址掩码请求/应答 ( Type 17/18 ) 查询子网掩码 —

不发 ICMP 差错报文的情况 :

  • ICMP 差错报文本身出错 ( 防止 ICMP 风暴 ).
  • 目的地址为广播或组播地址.
  • 分片数据报 ( 非第一片 ).
  • 源地址为 0.0.0.0 或回环地址.

traceroute/tracert 工作原理 :

依次发送 TTL = 1, 2, 3, … 的 UDP ( 或 ICMP ) 分组，每个路由器 TTL 减为 0 时返回 ICMP 时间超过 报文 ( Type 11 ) ，发送方记录路由器 IP 和往返时延，逐跳探测路径.

3.7 NAT ( 网络地址转换 )

NAT ( Network Address Translation )

NAT 允许 多台私有 IP 主机共享一个 ( 或少数几个 ) 公有 IP 地址 访问互联网，缓解 IPv4 地址耗尽问题.

NAPT ( 网络地址端口转换，PAT ) — 最常用形式 :

通过 IP地址 + 端口号 的组合，实现多个内网主机共享同一公网 IP.

工作过程示例 :

内网主机 192.168.1.10:1234 访问 8.8.8.8:80 :

NAT 表记录 : 203.0.113.1:5001 ↔︎ 192.168.1.10:1234

公网服务器 8.8.8.8 的回复发往 203.0.113.1:5001，NAT 路由器查表转换为 192.168.1.10:1234 转发给内网主机.

NAT 的优缺点 :

内容
优点 节省公网 IP 地址; 隐藏内网拓扑 ( 一定程度安全 ); 内网 IP 可自由更改
缺点 破坏端到端连接 ( 外网无法主动连内网主机，P2P 困难 ); 修改 IP/端口，影响某些协议 ( FTP、SIP ); NAT 表有状态，路由器负担增加; 与 IPSec 不兼容

NAT 穿透技术 : STUN、TURN、ICE ( 用于 WebRTC、VoIP 等 P2P 应用 ).

四、IPv6

IPv6 的产生背景与特性

IPv4 地址空间 ( \(2^{32} \approx 43\) 亿 ) 已耗尽，IPv6 将地址扩展至 128 位 ( \(2^{128}\) 个地址，约 \(3.4 \times 10^{38}\) ) ，从根本上解决地址问题.

IPv6 的主要改进 : 1. 地址空间极大扩展 : 128 位地址. 2. 简化首部 ( Fixed Header，40 字节 ) : 取消分片字段 ( 分片由端系统处理 ) 、取消首部校验和 ( 可靠性交给上层或链路层 ) 、取消可选字段 ( 改用扩展首部 ). 3. 内置安全 : IPSec 为 IPv6 必选功能 ( IPv4 中可选 ). 4. 自动配置 : 无状态地址自动配置 ( SLAAC ) ，不需要 DHCP 即可获取地址. 5. 更好的 QoS 支持 : 流标签 ( Flow Label ) 字段.

IPv6 地址表示 :

128 位地址，用冒号分隔的 8 组十六进制数表示 : 2001:0DB8:0000:0000:0000:FF00:0042:8329

缩写规则 : 1. 每组前导 0 可省略 : 2001:DB8:0:0:0:FF00:42:8329 2. 连续的全 0 组用 :: 代替 ( 每个地址只能用一次 ) : 2001:DB8::FF00:42:8329

IPv6 地址分类 :

类型 前缀 说明
全球单播地址 2000::/3 公网可路由，类似 IPv4 公有地址
链路本地地址 FE80::/10 只在单链路有效，自动配置，类似 169.254.x.x
组播地址 FF00::/8 代替 IPv4 广播 ( IPv6 无广播！ )
回环地址 ::1 类似 IPv4 的 127.0.0.1
未指定地址 :: 类似 IPv4 的 0.0.0.0

IPv4 到 IPv6 的过渡技术 :

技术 机制 说明
双栈 ( Dual Stack ) 主机同时运行 IPv4 和 IPv6 最简单，两套协议栈
隧道 ( Tunneling ) 将 IPv6 数据报封装在 IPv4 数据报中 跨越 IPv4 网络传输 IPv6
翻译 ( NAT64 ) IPv6 与 IPv4 地址/协议翻译 纯 IPv6 主机访问 IPv4 网络

五、路由协议

5.1 RIP 协议

RIP ( 路由信息协议，Routing Information Protocol )

RIP 是最早的动态路由协议，基于 距离向量算法 ，使用 跳数 作为度量 ( 每经过一个路由器，代价加 1 ).

RIP 基本特点 :

  • 度量 : 跳数， 最大 15 跳 ( 16 表示不可达 ) ，适合小型网络.
  • 路由更新 : 每 30 秒 广播 ( 或组播 ) 一次路由表给相邻路由器 ( 基于 UDP，端口 520 ).
  • 收敛速度慢 : 存在无穷计数问题，收敛可能需要数分钟.
  • 简单，配置方便.

RIP 版本 :

RIPv1 RIPv2
地址类型 有类 ( 不携带子网掩码 ) 无类 ( 携带子网掩码 )
广播/组播 广播 255.255.255.255 组播 224.0.0.9
认证 不支持 支持 ( 明文/MD5 )
VLSM 不支持 支持

RIP 的定时器 :

  • 更新定时器 : 30 s，定期发送完整路由表.
  • 无效定时器 : 180 s，路由未被更新则标记为不可达 ( 度量 = 16 ).
  • 垃圾回收定时器 : 120 s，不可达路由在 120 s 后从路由表删除.

5.2 OSPF 协议

OSPF ( 开放最短路径优先，Open Shortest Path First )

OSPF 基于 链路状态算法 ( Dijkstra ) ，是大型 IP 网络中最广泛使用的内部路由协议.

OSPF 的关键特性 :

  • 度量 ( Cost ) : 默认 \(= 10^8 / 带宽\) ( Mbps ) ，可自定义. 千兆链路 cost=1，百兆 cost=10，10M cost=100.
  • 快速收敛 : 拓扑变化时立即触发 LSA 泛洪，而不是等待定时器.
  • 直接封装在 IP 中 ( 协议号 89 ，不使用 UDP/TCP ).
  • 无环路 : 基于完整拓扑图的 Dijkstra 算法天然无环.
  • 支持 VLSM 和 CIDR : 路由更新携带子网掩码.
  • 支持多路径 : 等价多路径 ( ECMP ).

OSPF 区域 ( Area ) 机制 :

为了扩展性，OSPF 将网络划分为多个 区域 ，减少 LSA 泛洪范围 :

  • 骨干区域 ( Area 0 ) : 所有其他区域必须直接连到 Area 0 ( 或通过虚链路连接 ).
  • 非骨干区域 : Area 1、Area 2 等，区域内部运行 Dijkstra.
  • ABR ( 区域边界路由器 ) : 连接骨干区域与非骨干区域.
  • ASBR ( 自治系统边界路由器 ) : 连接 OSPF 自治系统与外部 ( 如 BGP ).

OSPF 报文类型 :

类型 名称 作用
Hello Hello 报文 发现邻居、建立和维护邻居关系 ( 每 10 s 一次，Dead Interval 40 s )
DBD 数据库描述 初始化时同步 LSDB 的摘要信息
LSR 链路状态请求 请求完整 LSA 信息
LSU 链路状态更新 发送完整 LSA ( 泛洪更新 )
LSAck 链路状态确认 确认收到 LSU

DR/BDR 选举 ( 广播网络 ) :

在以太网等广播多路访问网络中，所有路由器两两建立邻居关系开销太大 ( \(n(n-1)/2\) 条 ). OSPF 选举 指定路由器 ( DR ) 和 备用指定路由器 ( BDR ) ，其他路由器只与 DR/BDR 交换 LSA，减少报文数量.

5.3 BGP 协议

BGP ( 边界网关协议，Border Gateway Protocol )

BGP 是互联网上 自治系统 ( AS ) 间 的路由协议，负责在不同 ISP、机构之间交换路由信息.

自治系统 ( AS ) : 由单一管理机构控制的 IP 网络集合，有唯一的 AS 号 ( ASN，16 位或 32 位 ).

路由协议分类 :

  • IGP ( 内部网关协议 ) : AS 内部使用，如 RIP、OSPF、IS-IS.
  • EGP ( 外部网关协议 ) : AS 之间使用， BGP 是目前唯一实际使用的 EGP.

BGP 的关键特性 :

  • 路径向量算法 ( Path Vector ) : 不只传距离，而是传完整的 AS 路径 ，避免路由环路 ( 若 AS 路径中已包含自己的 AS 号，则丢弃该路由 ).
  • 基于 TCP ( 端口 179 ) : 可靠传输，无需额外差错控制.
  • 策略路由 : BGP 支持基于商业政策 ( 而非纯技术指标 ) 选择路由，是互联网商业运营的基础.
  • 增量更新 : 只在路由变化时发送受影响的路由 ( 不像 RIP 每 30 s 全量发送 ).

BGP 邻居类型 :

  • eBGP ( 外部 BGP ) : 不同 AS 之间的 BGP 会话 ( 直连链路 ).
  • iBGP ( 内部 BGP ) : 同一 AS 内部的 BGP 会话 ( 用于传递从外部学到的路由 ).

三种路由协议对比 ( 考研必考 ) :

比较维度 RIP OSPF BGP
类型 IGP IGP EGP
算法 距离向量 ( DV ) 链路状态 ( LS ) 路径向量 ( PV )
度量 跳数 ( ≤15 ) Cost ( 带宽相关 ) 路径属性 ( 策略 )
传输协议 UDP 520 直接 IP ( 协议号 89 ) TCP 179
更新方式 全量，每 30 s 触发更新，泛洪 增量更新
收敛速度 慢 快 较慢 ( 策略复杂 )
规模 小型网络 中大型企业网 互联网骨干
路由环路 可能 不可能 不可能 ( 路径向量 )

六、组播

组播 ( Multicast ) 基本概念

传输类型 目的地 适用场景
单播 ( Unicast ) 一个主机 点对点通信 ( HTTP、FTP )
广播 ( Broadcast ) 所有主机 局域网内广播 ( ARP、DHCP )
组播 ( Multicast ) 一组特定主机 视频会议、IPTV、金融行情推送

组播的优势 : 发送方只发一份数据，网络在 分叉点 复制，避免单播的重复发送和广播的无差别分发，兼顾效率与精准.

D 类组播地址 ( 224.0.0.0 ~ 239.255.255.255 ) :

范围 用途
224.0.0.0/24 链路本地组播 ( 不跨路由器 ) ，如 224.0.0.5 ( OSPF All Routers )
224.0.1.0/24 互联网协议控制块
232.0.0.0/8 源指定组播 ( SSM )
239.0.0.0/8 私有组播地址 ( 类似私有单播 )

IGMP ( Internet Group Management Protocol )

IGMP 用于主机与 本地组播路由器 之间通信，让路由器知道哪些主机加入了哪些组播组.

IGMPv2 工作过程 : 1. 主机加入组播组时，发送 IGMP Membership Report ( 目的地址 = 组播地址 ). 2. 组播路由器定期发送 IGMP Membership Query ( 询问哪些主机还在组里 ). 3. 主机收到 Query 后，在随机延时后回复 Report ( 抑制重复回复机制 ). 4. 主机离开组时，发送 IGMP Leave Group ; 路由器发 Group-Specific Query 确认是否还有其他成员.

组播路由协议 :

协议 全称 特点
DVMRP 距离向量组播路由协议 基于 RPF，泛洪+剪枝，早期协议
PIM-DM 密集模式独立组播 适合组成员密集分布，推模式
PIM-SM 稀疏模式独立组播 适合组成员稀疏分布，需 RP 汇聚点， 最常用

逆向路径转发 ( RPF ) : 组播路由器只转发从 到达组播源的最短路径接口 收到的组播数据报，防止环路.

七、移动 IP

移动 IP 的基本概念

移动 IP 使主机在移动到不同网络时，仍能保持 原有 IP 地址不变 ，维持正在进行的通信.

关键角色 :

角色 描述
移动节点 ( MN ) 在不同网络间移动的主机
归属代理 ( HA, Home Agent ) 移动节点归属网络中的路由器，负责跟踪 MN 位置，转发数据报
外地代理 ( FA, Foreign Agent ) MN 当前所在网络中的路由器，为 MN 提供转交地址
转交地址 ( CoA ) MN 在外地网络的临时地址 ( 从 FA 处获得 )
归属地址 MN 的永久 IP 地址 ( 不变 )

移动 IP 通信过程 ( 三角路由 ) :

  1. MN 移动到外地网络，从 FA 获得转交地址 ( CoA ) ，向 HA 注册 ( 绑定更新 ).
  2. CN 向 MN 的归属地址发送数据报，HA 收到后 用隧道 ( IP-in-IP 封装 ) 转发给 CoA ( FA ).
  3. FA 解封装，将数据报转发给 MN.
  4. MN 回复时，直接以 归属地址为源 发给 CN ( 不需要经过 HA ) ，因此称为 三角路由.

三角路由的问题与优化 :

  • 问题 : 每个数据报都要先绕到 HA，再隧道到 MN，路径不优，增加时延.
  • 优化 ( 路由优化 ) : HA 将 MN 的当前 CoA 通知 CN，此后 CN 可直接用隧道发给 CoA ( 直接通信 ) ，无需再绕 HA.

八、路由器

路由器的结构

路由器是工作在 网络层 的设备，核心功能是 路由 ( 维护路由表 ) 和 转发 ( 交换分组 ).

路由器的主要组成部分 :

输入端口功能 : 接收比特流、成帧、查转发表 ( FIB ) 、将分组送往正确的输出端口. 交换结构 ( Switching Fabric ) : 连接输入和输出端口，有总线式、交叉开关矩阵、共享内存三种. 输出端口功能 : 缓冲 ( 队列 ) 、成帧、发送. 路由处理器 : 运行路由协议 ( RIP/OSPF/BGP ) ，计算最短路径，生成和维护路由表.

路由表 vs 转发表 :

  • 路由表 ( RIB ) : 路由协议维护，包含所有路由来源 ( 直连、静态、RIP、OSPF 等 ) ，含度量、下一跳等.
  • 转发表 ( FIB ) : 从路由表中提取最优路由，只包含目的前缀→出口端口→下一跳 MAC，供数据平面高速查找.

路由器 vs 交换机 vs 集线器 ( 三层设备对比 )

比较维度 集线器 ( Hub ) 交换机 ( Switch ) 路由器 ( Router )
工作层次 物理层 ( L1 ) 数据链路层 ( L2 ) 网络层 ( L3 )
地址识别 无 MAC 地址 IP 地址
冲突域 不隔离 每端口独立 每接口独立
广播域 不隔离 不隔离 ( 无 VLAN ) 隔离 ( 每接口独立广播域 )
转发决策 广播所有端口 MAC 表查找 路由表查找
网络互联 同一 LAN 延伸 同一 LAN 内 异构网络互联
IP 地址 无 无 ( 管理 IP 除外 ) 每接口有 IP 地址

九、综合真题解析 ( Selected Problems )

真题 1 ( 子网划分 + 最长前缀匹配 )

路由器的路由表如下 :

目的网络/前缀 下一跳
192.168.0.0/22 接口 0
192.168.1.0/24 接口 1
192.168.2.0/25 接口 2
0.0.0.0/0 接口 3 ( 默认 )

分析目的 IP 地址 192.168.2.100 将从哪个接口转发?

解 :

对 192.168.2.100 进行各条路由匹配 ( 最长前缀匹配 ) :

  • 192.168.0.0/22 : 覆盖 192.168.0.0 ~ 192.168.3.255 ( 前 22 位匹配 ) ，192.168.2.100 在此范围 ✓，前缀长度 22.
  • 192.168.1.0/24 : 第三字节为 1，但 192.168.2.100 第三字节为 2， 不匹配 ✗.
  • 192.168.2.0/25 : 子网掩码 /25 即 255.255.255.128，子网地址 192.168.2.0，广播 192.168.2.127，192.168.2.100 在此范围 ✓，前缀长度 25.
  • 0.0.0.0/0 : 默认路由，全部匹配 ✓，前缀长度 0.

最长前缀匹配 : 选 192.168.2.0/25 ( 前缀最长，25 > 22 > 0 ) ，从 接口 2 转发.

真题 2 ( ARP 工作过程分析 )

主机 A ( 192.168.1.10/24，MAC = AA:AA ) 向主机 B ( 192.168.1.20/24，MAC = BB:BB ) 发送 IP 数据报. 主机 C ( 192.168.1.30/24，MAC = CC:CC ) 也在同一子网. 描述 ARP 的完整工作过程，并说明哪些主机会收到并处理 ARP 请求.

解 :

第 1 步 : A 检查目的 IP 192.168.1.20 与自身在同一子网 ( 掩码 /24 ) ，决定直接通过链路层发送.

第 2 步 : A 查本地 ARP 缓存，若无 B 的 MAC 记录 → 发送 ARP 广播请求 :

  • 源 IP : 192.168.1.10，源 MAC : AA:AA
  • 目的 IP : 192.168.1.20，目的 MAC : FF:FF:FF:FF:FF:FF ( 广播 )
  • 内容 : “谁的 IP 是 192.168.1.20?”

第 3 步 : 所有同一子网的主机 ( 包括 B 和 C ) 都会收到 这个广播帧.

  • B 发现目的 IP = 自身 IP， 处理该请求 ，更新 ARP 缓存 ( 记录 A 的 IP↔︎MAC ) ，发送 ARP 单播应答给 A.
  • C 发现目的 IP ≠ 自身 IP， 丢弃该请求 ( 仅更新 ARP 缓存中 A 的记录 ).

第 4 步 : A 收到 B 的 ARP 单播应答 ( MAC = BB:BB ) ，更新 ARP 缓存，封装以太网帧 ( 目的 MAC = BB:BB ) 发送 IP 数据报.

真题 3 ( NAT 工作过程分析 )

内网有 3 台主机 ( 10.0.0.1，10.0.0.2，10.0.0.3 ) ，通过 NAT 路由器 ( 公网 IP = 202.1.1.1 ) 访问互联网. 分析下列数据报经 NAT 后的变化，并说明为什么需要 NAPT ( 端口转换 ).

解 :

NAT 表 ( NAPT 场景 ) :

内网地址:端口 公网地址:端口 状态
10.0.0.1:1025 202.1.1.1:60001 活跃
10.0.0.2:1025 202.1.1.1:60002 活跃
10.0.0.3:1080 202.1.1.1:60003 活跃

数据报转换 :

主机 10.0.0.1:1025 发往 8.8.8.8:53 :

  • 出境 ( 内→外 ) : 源地址 10.0.0.1:1025 → 202.1.1.1:60001
  • 入境 ( 外→内 ) : 目的地址 202.1.1.1:60001 → 10.0.0.1:1025

为什么需要 NAPT ( 而非纯 NAT ) :

若不使用端口转换，只替换 IP 地址，则 3 台主机同时向外发送时，都会映射到 202.1.1.1，收到回复时无法区分该转发给哪台内网主机. NAPT 通过 端口号 区分不同的内网会话，使一个公网 IP 可支持数万个并发连接.

真题 4 ( RIP 路由更新 + 无穷计数问题 )

三个路由器 A-B-C 线性连接，RIP 初始收敛后各路由器路由表如下 ( 跳数 ) :

  • A : 到 B=1，到 C=2
  • B : 到 A=1，到 C=1
  • C : 到 A=2，到 B=1

现在 B-C 之间的链路 断开 ，请描述 RIP 的更新过程 ( 未使用水平分割 ).

解 :

第 1 步 : B 检测到 B-C 链路断开，将路由表中到 C 的距离设为 \(\infty\) ( 16 ).

第 2 步 ( 30 s 后，RIP 正常更新 ) :

  • B 向 A 发路由表，含”到 C = 16 ( 不可达 ) “.
  • A 向 B 发路由表，含”到 C = 2” ( A 之前从 B 学到 C=2，现在 B 报 C=16，但 A 认为自己到 C=2，于是告诉 B “到 C = 2” ).

第 3 步 : B 收到 A 说”到 C = 2”，按 Bellman-Ford : 经 A 到 C = 2+1 = 3 ( 比 16 小 ) ，更新 B 到 C = 3. B 再告诉 A “到 C = 3”，A 更新到 C = 4…

如此循环，B 和 A 的到 C 距离从 3, 4, 5, 6… 不断递增，直到 16 ( 不可达 ) 才停止. 这就是 无穷计数 ( Count to Infinity ) 问题.

水平分割的解决效果 : 若 A 使用水平分割，由于 A 是从 B 学到的 C=2，A 不会将”到 C = 2”再发回给 B. 此时 B 更新到 C = 16，直接宣告 C 不可达， 一轮更新即可收敛 ( 实际上水平分割不能完全解决所有场景，三角形拓扑中仍可能发生 ).

真题 5 ( OSPF vs RIP 综合对比应用 )

某企业网络由 50 台路由器组成，其中有 3 条跨城市的广域网链路 ( 时延差异大 ) ，请分析应选用 RIP 还是 OSPF 作为 IGP，并给出理由.

解 :

应选用 OSPF ，理由如下 :

  1. 规模 : 50 台路由器超过 RIP 推荐的小型网络 ( RIP 最大 15 跳意味着网络直径不超过 15 跳，大型网络极易触发 16 跳不可达限制 ). OSPF 无跳数限制.

  2. 链路代价 : 广域网链路时延差异大，RIP 只以跳数计量，无法区分 1 Mbps 和 1 Gbps 链路; OSPF 的 Cost 与带宽成反比，能 选择带宽更高的路径 ，更符合实际需求.

  3. 收敛速度 : OSPF 使用触发更新 + 泛洪，拓扑变化后秒级收敛; RIP 需等 30 s 周期，收敛可能超过 180 s，故障恢复慢，影响业务.

  4. 可扩展性 : OSPF 支持区域划分，可将 50 台路由器按地区/功能划分为多个 Area，减少 LSA 泛洪范围，减轻路由器计算压力.

  5. 路由精确性 : OSPF 支持 VLSM、CIDR，支持等价多路径 ( ECMP ) 负载均衡; RIPv1 不支持子网掩码，灵活性差.

结论 : 对于有广域网链路、规模较大、对收敛要求高的企业网络， OSPF 是唯一合理选择. RIP 仅适合 10 台路由器以内的简单小型网络.
