一、数据链路层的功能

数据链路层的地位与功能

数据链路层位于 OSI 第二层，负责将网络层传来的数据在 相邻节点之间的链路 上可靠 ( 或尽力 ) 传输. 数据链路层的数据单元称为 帧 ( Frame ) .

五大核心功能 :

1. 封装成帧 ( Framing ) 将网络层的 IP 数据报封装成帧，在帧头/帧尾添加特定标志，使接收方能正确识别帧的起始与结束.

2. 透明传输 无论数据字段中含有何种比特组合 ( 包括与帧定界符相同的序列 ) ，都能正确传输. 解决方案 : 字符填充 ( 字节填充 ) 或位填充.

3. 差错检测与纠正 信道噪声导致传输错误，数据链路层通过 冗余编码 ( CRC、海明码等 ) 检测或纠正帧中的错误.

4. 流量控制 ( Flow Control ) 防止发送方速度过快导致接收方缓冲区溢出，通过滑动窗口等机制限制发送速率.

5. 可靠传输 ( Reliable Delivery ) 通过确认 ( ACK ) 和重传 ( ARQ ) 机制，确保帧的正确有序交付. 注意 : 有线以太网不提供可靠传输 ( 差错由上层处理 ) ，无线网络 ( 802.11 ) 才在链路层保证可靠传输.

二、组帧与差错控制

2.1 组帧方法

四种组帧方法

1. 字符计数法 帧首部用一个字段指明帧的字节总数. 接收方读取计数值，收满相应字节即为一帧.

  • 缺点 : 计数字段出错会导致后续帧全部错乱 ( 失去同步 ) ，不实用.

2. 字节填充的首尾定界符法 ( 字符填充法 ) 用特殊字节 ( 如 FLAG = 0x7E ) 标识帧的起止. 若数据中出现与 FLAG 相同的字节，则在其前插入 转义字符 ESC ( 0x7D ) ( 发送方填充，接收方去除 ).

  • 应用 : PPP 协议 ( Point-to-Point Protocol ) 采用此方法.

3. 比特填充法 ( 零比特填充 ) 用特殊比特串 01111110 ( 6个1 ) 作帧定界符. 发送方在数据中每遇到连续 5 个 1，自动在后面插入一个 0; 接收方每遇到连续 5 个 1 后的 0，自动删除.

  • 应用 : HDLC 协议采用此方法.

4. 物理层违规编码法 利用物理层编码中的”违规”信号 ( 正常数据不会出现的电平组合 ) 作帧定界符.

  • 应用 : 曼彻斯特编码中，高-高或低-低跳变在正常数据中不出现，可用于帧定界.

2.2 差错控制编码

奇偶校验码 ( Parity Check )

在数据位后附加 1 位校验位，使总 1 的个数为奇数 ( 奇校验 ) 或偶数 ( 偶校验 ).

  • 缺点 : 只能检测 奇数个比特 出错，不能检测偶数个比特同时出错，不能纠错，实际网络中很少单独使用.

循环冗余校验 ( CRC, Cyclic Redundancy Check )

CRC 是数据链路层 最常用的检错方法，检错能力强，实现简单 ( 硬件移位寄存器 ).

原理 :

  1. 发送方与接收方事先约定一个 生成多项式 \(G(x)\) ( 对应一个 \(r+1\) 位的二进制数，最高位和最低位均为 1 ).
  2. 设信息帧为 \(M(x)\) ( \(k\) 位 ) ，在末尾补 \(r\) 个 0，即 \(M(x) \cdot x^r\) ( 共 \(k+r\) 位 ).
  3. 用 \(M(x) \cdot x^r\) 除以 \(G(x)\)，得余数 \(R(x)\) ( \(r\) 位 ).
  4. 将余数附加在原始信息末尾，构成发送帧 \(T(x) = M(x) \cdot x^r + R(x)\) ( 共 \(k+r\) 位 ).
  5. 接收方用 \(T(x)\) 除以 \(G(x)\)，若余数为 0，则认为无误; 否则有错.

FCS ( 帧校验序列 ) : 即附加在帧尾的冗余校验位，就是 CRC 的余数 \(R(x)\).

CRC 的检错能力 :

  • 能检测所有 奇数个比特 错误.
  • 能检测所有 突发长度 \(\le r\) 的突发错误 ( \(r\) 为 FCS 位数 ).
  • 能检测所有 2位错.
  • 不能纠错 ( 只能检测是否有错，无法定位错误位置 ).

常用生成多项式 :

  • CRC-16 : \(x^{16}+x^{15}+x^2+1\) ( 用于 USB、磁盘 )
  • CRC-32 : \(x^{32}+x^{26}+\cdots+1\) ( 以太网 FCS 采用，即 \(G(x)\) 为 33 位 )

示例 1 ( CRC 计算过程 )

信息码 \(M = 10110\)，生成多项式 \(G(x) = x^3+x+1\) ( 对应 \(1011\)，\(r=3\) ).

发送方 :

补 3 个 0 后 : \(M \cdot x^3 = 10110\underline{000}\)

用 \(10110000\) 除以 \(1011\) ( 模 2 除法，即无进位的 XOR 除法 ) :

实际逐步计算 :

  • \(10110000\)，取前 4 位 \(1011\)，\(1011 \oplus 1011 = 0000\)，移下 \(0\)，得 \(00000\).
  • 前导 0 忽略，取 \(1\)，加入下一位 \(0\)，凑 4 位 : \(0000+\)下一位…

重新用竖式 ( 模2 ) :

正确计算 ( 逐步 XOR，仅当最高位为 1 时做 XOR ) :

发送帧 : \(T = 10110\underline{111}\).

接收方 :

用 \(10110111\) 除以 \(1011\)，若余数 \(= 000\)，无误. \(\checkmark\)

海明码 ( Hamming Code )

海明码是一种能检错且能纠错的线性分组码，适合纠正单比特错误.

基本原理 :

在数据位中插入若干 校验位，每个校验位 \(P_i\) 负责覆盖特定位置的比特. 接收方重新计算所有校验位，若出错，则错误位置 = 各校验位结果组合的二进制值.

校验位的数量要求 :

若数据有 \(m\) 位，需要 \(r\) 位校验位，满足 : \[2^r \ge m + r + 1\]

数据位 \(m\) 校验位 \(r\) 总编码长 \(m+r\)
1 2 3
2~4 3 5~7
5~11 4 9~15
12~26 5 17~31

校验位位置 : 放在编码的第 \(2^0=1, 2^1=2, 2^2=4, 2^3=8, \ldots\) 位.

最小码距 ( Hamming Distance ) 与检纠错能力 :

  • 检测 \(d\) 位错 : 需最小码距 \(d_{min} \ge d + 1\).
  • 纠正 \(d\) 位错 : 需最小码距 \(d_{min} \ge 2d + 1\).
  • 海明码最小码距为 3，可 纠 1 位错，检 2 位错.
  • 若加一位全局奇偶校验位 ( SECDED ) ，可 纠 1 位、检 2 位 ( SEC-DED ).

三、流量控制与可靠传输机制

3.1 停-等协议 ( Stop-and-Wait, SW )

基本思路

发送方每发送一帧后，必须 等待接收方的确认 ( ACK ) ，收到 ACK 后才能发送下一帧. 若超时未收到 ACK，则 重传 该帧.

帧序号 : 停-等协议只需 1 位 序号 ( 0 和 1 交替 ) ，用于区分新帧与重传帧 ( 防止重复帧被接收方当成新帧 ).

信道利用率 :

设 \(T_f\) 为帧的发送时延，\(T_p\) 为单向传播时延，\(T_a\) 为确认帧发送时延 ( 通常很小，近似为 0 ).

\[U = \frac{T_f}{T_f + 2T_p + T_a} \approx \frac{T_f}{T_f + 2T_p} = \frac{1}{1+2a}\]

其中 \(a = T_p / T_f\).

当 \(a \gg 1\) ( 长时延大带宽链路 ) ，\(U \approx 1/(2a) \to 0\)，信道利用率极低，这就是引入滑动窗口的动机.

3.2 滑动窗口机制

滑动窗口的基本思想

允许发送方在未收到确认的情况下，连续发送 多个帧，最多可发送 \(W\) ( 窗口大小 ) 个未被确认的帧. 这种方式将单帧的等待时间与多帧的发送时间 重叠 ( 流水线 ) ，大幅提升信道利用率.

信道利用率 ( 无差错情况 ) :

\[U = \begin{cases} \dfrac{W \cdot T_f}{T_f + 2T_p} = \dfrac{W}{1+2a} & W < 1+2a \\[10pt] 1 & W \ge 1+2a \end{cases}\]

使信道满负荷运行 ( \(U=1\) ) 的最小窗口 : \[W_{min} = 1 + 2a = 1 + \frac{2T_p}{T_f}\]

序号空间与窗口大小 :

若帧序号占 \(n\) 位，则序号空间大小为 \(2^n\)，但收发窗口大小有限制 ( 见下文 ).

3.3 回退 N 帧协议 ( Go-Back-N, GBN )

工作原理

发送窗口大小 \(W_s \le 2^n - 1\)，接收窗口大小 \(W_r = 1\).

接收方只接受 按序到达 的帧 :

  • 若收到期望帧 → 发送 ACK，接收窗口向前滑动.
  • 若收到非期望帧 ( 失序 ) → 丢弃，发送对最后一个正确帧的 累积确认 ( Cumulative ACK ) .

出错处理 :

  • 当发现错误 ( 超时或收到 NAK ) 时，发送方必须 重传出错帧及其后所有已发帧 ( “回退 N 个帧” ) ，即使后面的帧已正确到达接收方.

窗口大小限制 \(W_s \le 2^n - 1\) 的原因 :

若 \(W_s = 2^n\) ( 序号空间全用 ) ，会出现以下模糊性 : 发送方发出 \(2^n\) 个帧 ( 序号 \(0 \sim 2^n-1\) ) ，若所有 ACK 丢失，发送方重传序号 0; 而接收方已滑窗，期待新的序号 0，无法区分是重传帧还是新帧.

性能分析 ( 有差错时 ) :

设帧出错概率为 \(p\)，GBN 的有效吞吐率 : \[U_{GBN} = \frac{(1-p)}{(1-p)+W_s \cdot p} \cdot U_{no\_error}\]

近似 : 出一次错，需重传 \(W_s\) 个帧，代价较大.

3.4 选择重传协议 ( Selective Repeat, SR )

工作原理

发送窗口 \(W_s \le 2^{n-1}\)，接收窗口 \(W_r = W_s \le 2^{n-1}\).

接收方可以 接受失序帧 并缓存，等待缺失的帧到来后，再按序向上层交付.

出错处理 :

  • 出错时，只重传 出错的那一帧，不重传后续已正确接收的帧.
  • 代价小，效率高，但需要较大的接收缓冲区.

窗口大小限制 \(W_s \le 2^{n-1}\) 的原因 :

若 \(W_s = W_r = 2^n/2 + 1 > 2^{n-1}\)，会出现以下模糊性 : 假设 \(n=2\)，序号 \(\{0,1,2,3\}\)，\(W_s=W_r=3\). 发送方发 \(0,1,2\)，若所有 ACK 丢失，发送方重传 \(0,1,2\); 接收方已滑窗到 \(3,0,1\)，接收窗口包含序号 0，无法区分新帧 0 与重传帧 0.

示例 2 ( 滑动窗口信道利用率计算 )

某链路带宽 \(1\ \text{Mbps}\)，单向传播时延 \(270\ \text{ms}\) ( 卫星链路 ) ，帧长 \(1000\ \text{bit}\). (1) 求停-等协议的信道利用率. (2) 为使信道利用率达到 100%，滑动窗口至少需要多大? (3) 若序号字段 3 bit，GBN 和 SR 的最大窗口各为多少?

解 :

\(T_f = 1000 / 10^6 = 1\ \text{ms}\)，\(T_p = 270\ \text{ms}\)，\(a = T_p/T_f = 270\).

(1) 停-等信道利用率 : \[U_{SW} = \frac{1}{1+2 \times 270} = \frac{1}{541} \approx 0.185\%\]

极低，不适合卫星链路！

(2) 最小窗口 : \[W_{min} = 1+2a = 1 + 2 \times 270 = 541\]

(3) 序号字段 \(n=3\)，序号空间 \(= 2^3 = 8\).

  • GBN 最大窗口 : \(W_s \le 2^3 - 1 = 7\).
  • SR 最大窗口 : \(W_s \le 2^{3-1} = 2^2 = 4\).

因 GBN 窗口 7 < 541，信道利用率 \(U_{GBN} = 7/541 \approx 1.3\%\)，仍很低. 需要更大的序号字段 ( 如 \(n=10\)，\(W_{GBN} = 1023 > 541\) ) 才能充分利用信道.

四、介质访问控制 ( MAC )

介质访问控制的分类

当多个节点共享同一传输介质 ( 如无线信道、早期以太网总线 ) 时，需要协调各节点的发送权，防止冲突 ( 碰撞 ). 这就是 介质访问控制 ( MAC, Medium Access Control ) 问题.

分类 :

  1. 静态信道划分 : 预先将信道资源 ( 频率/时间/码型 ) 分配给各节点，无冲突，但资源利用率低 ( 如 FDM、TDM、CDMA ).
  2. 动态信道分配 :
    • 随机访问 : 节点按需竞争发送，可能碰撞，需碰撞检测/避免机制 ( 如 ALOHA、CSMA/CD、CSMA/CA ).
    • 受控访问 : 通过轮询或令牌机制有序分配信道 ( 如令牌环、令牌总线 ).

4.1 ALOHA 协议

纯 ALOHA ( Pure ALOHA )

最简单的随机访问协议 : 任何节点有数据就直接发送，不监听信道.

  • 碰撞 : 若两帧在时间上有任何重叠，则发生碰撞，两帧均损坏. 检测到碰撞后，各节点等待 随机时间 后重传.
  • 易受冲突的时间窗口 : \(2T_f\) ( 帧长的 2 倍时间内，若有其他帧开始发送则冲突 ).
  • 最大信道利用率 : \(U_{max} = \dfrac{1}{2e} \approx 18.4\%\) ( 当信道负载 \(G = 0.5\) 时 ).

时隙 ALOHA ( Slotted ALOHA )

将时间划分为等长时隙 ( = 帧传输时间 ) ，各节点只在 时隙开始处 发送.

  • 碰撞只发生在同一时隙，易受冲突窗口缩短为 \(T_f\) ( 减半 ).
  • 最大信道利用率 : \(U_{max} = \dfrac{1}{e} \approx 36.8\%\) ( 当 \(G = 1\) 时 ).

两者对比 :

纯 ALOHA 时隙 ALOHA
发送时机 随时 时隙开始处
最大利用率 \(1/(2e) \approx 18.4\%\) \(1/e \approx 36.8\%\)
实现复杂度 低 需要时钟同步

4.2 CSMA 协议

载波监听多路访问 ( CSMA, Carrier Sense Multiple Access )

在 ALOHA 基础上增加 先监听 ( 载波监听 ) 再发送的机制 : 发送前先检测信道是否空闲.

三种 CSMA 策略 :

1. 1-坚持 CSMA ( 1-persistent )

  • 信道空闲 → 立即发送 ( 概率 1 ).
  • 信道忙 → 持续监听，空闲后 立即 发送.
  • 缺点 : 多个节点同时等待时，信道一空闲就同时发送，冲突概率高.

2. 非坚持 CSMA ( non-persistent )

  • 信道空闲 → 立即发送.
  • 信道忙 → 等待随机时间 后再检测，不持续监听.
  • 优点 : 减少冲突 ( 多个节点等待时间不同 ).
  • 缺点 : 信道可能空闲一段时间无人发送 ( 浪费 ).

3. p-坚持 CSMA ( p-persistent )

  • 信道空闲 → 以概率 \(p\) 发送，以概率 \(1-p\) 等待一个时隙后再检测.
  • 信道忙 → 持续监听，空闲后以 \(p\) 概率发送.
  • 折衷方案，在冲突与利用率之间取得平衡.

CSMA 的局限 : 由于传播时延的存在，一个节点开始发送后，另一个节点可能尚未检测到信道忙，仍然发送，导致碰撞. CSMA 能 减少 但不能 消除 碰撞.

4.3 CSMA/CD ( 以太网核心 )

CSMA/CD ( Carrier Sense Multiple Access / Collision Detection )

在 CSMA 基础上增加 碰撞检测 : 发送过程中持续检测信道，一旦检测到碰撞立即停止发送，节省信道资源.

工作流程 ( “先听后说，边说边听” ) :

争用期 ( 冲突窗口 ) :

最坏情况下，碰撞检测需要等待 \(2T_p\) ( 端到端往返传播时延 ). 因此，将 \(2T_p\) 称为 争用期 或 碰撞窗口.

最小帧长推导 ( 重要 ) :

要保证发送方能在帧发送完毕前检测到碰撞 ( 否则碰撞发生后发送方已”收手”，不知道碰撞 ) ，必须满足 : \[T_f \ge 2T_p \implies \frac{L_{min}}{B} \ge 2T_p \implies \boxed{L_{min} = 2T_p \cdot B}\]

10BASE-T 以太网示例 :

  • 最大网段长度约 \(2500\ \text{m}\) ( 经过 4 个中继器 ) ，单向传播时延约 \(25.6\ \mu\text{s}\).
  • 争用期 \(= 2 \times 25.6 = 51.2\ \mu\text{s}\).
  • 带宽 \(B = 10\ \text{Mbps}\)，最小帧长 \(= 51.2 \times 10^{-6} \times 10 \times 10^6 = 512\ \text{bit} = 64\ \text{Byte}\).

这就是以太网最小帧长度 64 字节 的来历！若数据不足 46 字节，需 填充至 46 字节 ( 帧总长 = 6+6+2+46+4 = 64 B ).

二进制指数退避算法 ( Truncated Binary Exponential Backoff ) :

  1. 第 \(k\) 次碰撞后，从 \(\{0, 1, 2, \ldots, 2^{\min(k,10)}-1\}\) 中随机选一个整数 \(r\).
  2. 等待 \(r \times 2T_p\) ( \(r\) 个争用期 ) 后重新竞争.
  3. 超过 16 次仍碰撞，则报告发送失败.

关键特性 :

  • 退避时间 随冲突次数指数增大 ( 01→03→0~7→… ) ，减少网络负载高时的持续碰撞.
  • 碰撞 16 次后丢弃，不会无限重试.

考研计算示例 ( 深度理解 ) :

若发生第 3 次碰撞 (\(k=3\))，退避时间 \(r\) 从 \(\{0, 1, \dots, 2^{\min(3,10)}-1\} = \{0, 1, \dots, 7\}\) 中选取。

  • 选取 \(r=4\) 的概率是 \(1/8\).
  • 等待时间为 \(4 \times 2T_p\) ( 4 个争用期 ).
  • 若 \(k=11\)，则 \(r\) 从 \(0 \sim 2^{10}-1\) ( 即 \(0 \sim 1023\) ) 中选，而不是 \(2^{11}-1\) ( 上限截断为 10 ).

4.4 CSMA/CA ( 无线网络核心 )

CSMA/CA ( Carrier Sense Multiple Access / Collision Avoidance )

无线网络中 无法使用碰撞检测 ( CSMA/CD ) ，原因 :

  1. 发送时无线信号的自干扰使接收自身信号困难 ( 硬件难以实现全双工检测 ).
  2. 隐藏节点问题 : A 和 C 都在 B 的范围内，但 A 和 C 互不可见，无法检测彼此是否在发送.

因此 IEEE 802.11 采用 碰撞避免 ( 尽量不让碰撞发生 ) 而非碰撞检测.

工作机制 ( DCF 分布式协调功能 ) :

1. 帧间间隔 ( IFS, Inter-Frame Space )

  • SIFS ( Short IFS ) : 最短帧间隔，用于 ACK、CTS 等高优先级帧.
  • DIFS ( Distributed IFS ) : 数据帧发送前需等待的最小帧间隔，\(> SIFS\).

2. 退避机制 ( 随机退避，避免碰撞 ) :

发送数据前 :

  1. 监听信道是否空闲.
  2. 空闲时间达到 DIFS 后，启动随机退避计时器 ( 从 \([0, CW]\) 中随机选，\(CW\) 为竞争窗口大小 ).
  3. 退避计时器倒计时，仅在信道空闲时递减，信道忙时暂停.
  4. 计时器到 0，发送帧.
  5. 若碰撞，\(CW\) 加倍 ( 二进制指数退避 ) ，重新退避.

3. 确认机制 ( ACK ) :

接收方收到正确的帧后，等待 SIFS 后立即发 ACK，不进行退避 ( SIFS < DIFS，保证 ACK 优先发送 ). 发送方收到 ACK 才认为发送成功; 否则超时重传.

4. RTS/CTS 机制 ( 解决隐藏节点问题 ) :

可选机制，针对长数据帧 : 1. 发送方先发短的 RTS ( Request to Send ) 帧，包含将要传输的数据时长. 2. 接收方回复 CTS ( Clear to Send ) 帧 ( SIFS 后 ) ，所有听到 CTS 的节点设置NAV ( Network Allocation Vector，虚拟载波监听 ) ，在此时间内不发送. 3. 发送方收到 CTS 后 ( SIFS 后 ) 发数据，接收方收后 ( SIFS 后 ) 发 ACK.

五、局域网基本概念与体系结构

局域网的特点

  • 覆盖范围小 ( 同一建筑或园区 ) ，一般不超过 10 km.
  • 传输速率高 ( 10 Mbps ~ 100 Gbps ) ，误码率低 ( \(10^{-8} \sim 10^{-11}\) ).
  • 由单一组织拥有和管理.

IEEE 802 标准体系

IEEE 802 委员会将数据链路层分为两个子层 :

子层 全称 功能
LLC 逻辑链路控制 Logical Link Control 提供统一接口给网络层，负责差错控制、流量控制 ( 802.2 )
MAC 介质访问控制 Medium Access Control 负责成帧、编址 ( MAC 地址 ) 、介质访问 ( 802.3/802.11 等 )

实际应用中，LLC 层在 TCP/IP 网络中基本不用，以太网直接使用 MAC 层向上提供服务.

MAC 地址 ( 物理地址 ) :

  • 48 位 ( 6 字节 ) ，烧录在网卡 ROM 中 ( 理论上全球唯一 ).
  • 格式 : XX:XX:XX:XX:XX:XX ( 十六进制 ).
  • 前 24 位 ( OUI ) : 厂商标识，由 IEEE 分配.
  • 后 24 位 : 厂商自行分配的设备序号.
  • 广播地址 : FF:FF:FF:FF:FF:FF.

6.2 IEEE 802.11 ( 无线局域网，WiFi )

802.11 基本概念

基本服务集 ( BSS, Basic Service Set ) : 由一个 接入点 ( AP ) 和若干无线主机 ( 站点 ) 组成的基本网络单元.

服务集标识符 ( SSID ) : 无线网络的名称 ( 人们常说的”WiFi名字” ).

分布式系统 ( DS ) : 连接多个 BSS 的骨干网络 ( 有线以太网 ).

扩展服务集 ( ESS ) : 由多个 BSS 通过 DS 连接组成的大型无线网络.

Ad hoc 模式 ( IBSS ) : 无 AP，站点直接通信 ( 点对点临时网络 ).

802.11 帧结构 ( 简化 ) :

802.11 数据帧包含 4个地址字段 ( Address 1/2/3/4 ) ，用于支持 AP 转发 :

  • Address 1 : 接收方地址 ( RA ).
  • Address 2 : 发送方地址 ( TA ).
  • Address 3 : 目的/源地址 ( 用于 DS 中转 ).
  • Address 4 : 仅 WDS ( 无线分布式系统 ) 中使用.

6.3 令牌环 ( Token Ring, IEEE 802.5 )

令牌环工作原理

  • 所有站点连成 逻辑环形，一个特殊的控制帧 ( 令牌，Token ) 沿环依次传递.
  • 只有持有令牌的站点 才能发送数据帧.
  • 发送完毕后，将令牌传给下一个站点.
  • 无冲突，信道利用率高 ( 负载重时接近 100% ) ，时延确定.
  • 缺点 : 维护令牌 ( 令牌丢失或重复 ) 复杂; 实现和配置比以太网复杂; 速度较慢 ( 4 Mbps 或 16 Mbps ).
  • 现状 : 已基本被以太网取代，考研重点是概念与原理的对比.

令牌环 vs 以太网 :

比较 以太网 ( CSMA/CD ) 令牌环 ( Token Ring )
访问控制 竞争 ( 随机 ) 有序 ( 令牌轮询 )
轻载性能 好 ( 低延迟 ) 较差 ( 需等令牌 )
重载性能 差 ( 冲突增多 ) 好 ( 接近 100% 利用率 )
故障影响 节点故障不影响他人 节点故障可能中断全环

七、网桥与局域网交换机

7.1 网桥 ( Bridge )

网桥的功能与工作原理

网桥工作在 OSI 第二层 ( 数据链路层 ) ，根据帧的 MAC 地址 进行转发决策.

网桥的主要功能 :

  1. 过滤帧 : 读取帧的目的 MAC 地址，若目的地址在同一网段则 不转发 ( 过滤 ) ，否则转发到另一网段.
  2. 转发帧 : 将帧从一个端口转发到另一端口，隔离冲突域.
  3. 自学习 : 通过读取帧的源 MAC 地址，动态建立 MAC 地址表 ( 转发表 ).

自学习算法 :

  1. 收到一帧时，记录 源 MAC 地址 和 来源端口 ( 入端口 ) ，写入转发表 ( 超时老化，默认 300 s ).
  2. 查找 目的 MAC 地址 :
    • 若在转发表中，且目的端口与来源端口 不同 → 转发到对应端口.
    • 若在转发表中，且目的端口与来源端口 相同 → 过滤 ( 丢弃，目的在同侧 ).
    • 若 不在转发表 中 → 泛洪 ( Flood ) ，从除来源端口外的所有端口转发.

网桥 vs 中继器 vs 路由器 :

中继器 网桥 路由器
工作层次 物理层 数据链路层 网络层
地址识别 无 MAC 地址 IP 地址
冲突域 不隔离 隔离 隔离
广播域 不隔离 不隔离 隔离

7.2 以太网交换机 ( Ethernet Switch )

交换机的工作原理

交换机本质上是 多端口网桥，每个端口都有独立的冲突域，支持多对节点 同时并发通信.

三种帧转发方式 :

转发方式 开始转发时机 优点 缺点
存储转发 ( Store-and-Forward ) 收到完整帧后 可过滤错误帧 ( CRC 检验 ) 延迟最大
直通交换 ( Cut-Through ) 收到目的 MAC ( 14 字节 ) 后 延迟最小 不检测错误帧
无碎片转发 ( Fragment-Free ) 收到 64 字节后 过滤碰撞碎片 折衷方案

交换机的核心优势 :

  1. 每端口独享带宽 ( 100 Mbps 交换机每端口 100 Mbps，而集线器所有端口共享 100 Mbps ).
  2. 支持全双工 ( 无冲突，不需要 CSMA/CD ).
  3. 同时转发多个帧 ( 交换矩阵 ).
  4. 自学习，即插即用，无需人工配置.

冲突域与广播域 :

  • 交换机每个端口 = 独立冲突域 ( 彻底消除冲突 ).
  • 交换机所有端口 = 同一广播域 ( 广播帧仍会泛洪到所有端口 ).
  • 路由器每个接口 = 独立广播域 ( 路由器不转发广播帧 ).

VLAN ( 虚拟局域网 ) :

通过在交换机上配置 VLAN，可以将同一物理交换机的不同端口划分为不同的逻辑广播域，实现广播域的隔离 ( IEEE 802.1Q 标准 ).

802.1Q 帧格式 :

在源 MAC 地址和类型字段之间插入 4 字节 Tag.

  • TPID ( 2 字节 ) : 固定值 0x8100，标识这是一个 802.1Q 帧.
  • TCI ( 2 字节 ) : 包含优先级 ( PRI ) 、CFI 和 VID ( VLAN ID, 12 bit ) . VID 范围 0~4095 ( 0 和 4095 保留，可用 1~4094 ).

端口类型 ( 考研重点 ) :

  • Access 端口 : 连接主机，只处理 无标签帧. 发送时剥离 Tag，接收时打上默认 Tag ( PVID ).
  • Trunk 端口 : 连接交换机/路由器，允许多个 VLAN 通过. 发送时若 VID=PVID ( Native VLAN ) 则剥离 Tag，否则保留 Tag; 接收时若无 Tag 则打上 PVID，有 Tag 则透传.

生成树协议 ( STP, Spanning Tree Protocol, IEEE 802.1D )

为了提高网络可靠性，交换机之间通常部署 冗余链路. 但冗余链路会产生 环路，导致 :

  1. 广播风暴 : 广播帧在环路中无限循环，消耗全部带宽.
  2. MAC 地址表不稳定 : 同一 MAC 地址从不同端口出现，转发表振荡.
  3. 帧重复 : 目的主机收到同一帧的多份副本.

STP 解决方案 :

  1. 所有交换机交换 BPDU ( Bridge Protocol Data Unit ) 报文，选举出一个 根桥 ( Root Bridge ) .
  2. 每个非根交换机确定到根桥最短路径的 根端口 ( Root Port ) .
  3. 每个网段选出一个 指定端口 ( Designated Port ) .
  4. 其余端口设为 阻塞状态 ( Blocking ) ，不转发数据帧 ( 但仍接收 BPDU ).
  5. 网络拓扑形成 无环的逻辑树形结构 ( 生成树 ) .

RSTP ( 802.1w ) : 快速生成树协议，收敛时间从 STP 的 30~50 s 缩短到 1~2 s.

八、综合真题解析 ( Selected Problems )

真题 1 ( GBN vs SR 窗口大小判断 )

设滑动窗口协议的帧序号用 4 位二进制表示 ( 序号 \(0 \sim 15\) ). (1) GBN 协议的最大发送窗口是多少? (2) SR 协议的最大发送窗口和接收窗口各是多少? (3) 若某时刻 GBN 的发送方已发帧 3、4、5、6，接收方只正确收到帧 3，此时应如何处理?

解 :

\(n = 4\)，序号空间 \(= 2^4 = 16\).

  1. GBN 最大发送窗口 : \(W_s = 2^4 - 1 = \mathbf{15}\).

  2. SR 最大发送窗口 : \(W_s = 2^{4-1} = \mathbf{8}\); 最大接收窗口 : \(W_r = 2^{4-1} = \mathbf{8}\).

  3. GBN 接收窗口 \(= 1\)，只接受按序帧. 接收方正确收到帧 3，发 ACK3; 帧 4 到达时，若帧 4 损坏，接收方 丢弃帧 5、6，发 NAK4 ( 或等超时 ). 发送方收到 NAK4 或超时后，从帧 4 开始重传 4、5、6 ( 回退到 4，重发所有帧，包括已发过的 5 和 6 ).

真题 2 ( CSMA/CD 最小帧长计算 )

某以太网总线长 \(1\ \text{km}\)，传播速率 \(2 \times 10^8\ \text{m/s}\)，数据传输速率 \(10\ \text{Mbps}\). (1) 求争用期. (2) 求最小帧长. (3) 若某帧数据字段仅 20 字节，应如何处理?

解 :

  1. 单向传播时延 \(T_p = 1000 / (2 \times 10^8) = 5\ \mu\text{s}\). 争用期 \(= 2T_p = 10\ \mu\text{s}\).

  2. 最小帧长 : \[L_{min} = 2T_p \times B = 10 \times 10^{-6} \times 10 \times 10^6 = 100\ \text{bit} = 12.5\ \text{B}\]

取整为 13 字节 ( 实际标准取 64 字节是针对更大网络而言 ).

  1. 数据字段 20 字节，帧总长 \(= 6 + 6 + 2 + 20 + 4 = 38\ \text{字节} < 64\ \text{字节}\) ( 标准规定 ). 应在数据字段后 填充 26 字节 的无用字节，使数据字段达到 46 字节，总帧长 64 字节.

真题 3 ( 交换机自学习过程 )

如图所示，交换机有 3 个端口，初始转发表为空 :

事件序列 : ① A 向 C 发帧 ( 源 AA，目的 CC ) ② C 向 A 回帧 ( 源 CC，目的 AA ) ③ B 向 C 发帧 ( 源 BB，目的 CC )

请写出每次事件后转发表的变化及交换机的转发行为.

解 :

① A 向 C 发帧 ( AA → CC ) :

  • 收到帧，来自端口 1，源 MAC = AA.
  • 学习 : 记录 AA → 端口 1.
  • 查找目的 CC : 不在表中 → 泛洪 到端口 2 和端口 3.
  • 转发表 : {AA: 端口1}.

② C 向 A 回帧 ( CC → AA ) :

  • 收到帧，来自端口 3，源 MAC = CC.
  • 学习 : 记录 CC → 端口 3.
  • 查找目的 AA : 在表中，对应端口 1，与来源端口 ( 3 ) 不同 → 单播转发 到端口 1.
  • 转发表 : {AA: 端口1, CC: 端口3}.

③ B 向 C 发帧 ( BB → CC ) :

  • 收到帧，来自端口 2，源 MAC = BB.
  • 学习 : 记录 BB → 端口 2.
  • 查找目的 CC : 在表中，对应端口 3，与来源端口 ( 2 ) 不同 → 单播转发 到端口 3.
  • 转发表 : {AA: 端口1, CC: 端口3, BB: 端口2}.

结论 : 自学习完成后，后续通信全部单播，无泛洪，交换效率最高.

真题 4 ( 综合 : 信道利用率与窗口协议对比 )

卫星链路，带宽 \(1\ \text{Mbps}\)，单向传播时延 \(T_p = 250\ \text{ms}\)，帧长 \(L = 1000\ \text{bit}\)，ACK 帧可忽略.

  1. 求停-等协议、GBN ( 序号 3 bit ) 、SR ( 序号 3 bit ) 的信道利用率.
  2. 若要使信道利用率 \(\ge 80\%\)，SR 协议至少需要多少位序号?

解 :

\(T_f = L/B = 1000/10^6 = 1\ \text{ms}\)，\(a = T_p/T_f = 250\).

(1)

停-等 : \(U_{SW} = \dfrac{1}{1+2 \times 250} = \dfrac{1}{501} \approx 0.2\%\).

GBN ( \(n=3\)，\(W = 2^3-1 = 7\) ) : \[U_{GBN} = \frac{7}{1+2 \times 250} = \frac{7}{501} \approx 1.4\%\]

SR ( \(n=3\)，\(W = 2^{3-1} = 4\) ) : \[U_{SR} = \frac{4}{501} \approx 0.8\%\]

三种协议利用率均极低，说明卫星链路 必须用大窗口 才能充分利用信道.

(2) 使 \(U_{SR} \ge 80\%\) :

\[\frac{W}{1+2a} \ge 0.8 \implies W \ge 0.8 \times 501 = 400.8 \implies W \ge 401\]

SR 协议 \(W \le 2^{n-1}\)，需 : \[2^{n-1} \ge 401 \implies n-1 \ge \lceil \log_2 401 \rceil = 9 \implies n \ge 10\]

即至少需要 10 位序号，此时 SR 最大窗口 \(= 2^9 = 512 > 401\)，\(U = 512/501 > 1\)，取 \(U = 100\%\).

真题 5 ( CSMA/CD 与 CSMA/CA 工作差异分析 )

请分析以下场景，说明为什么有线以太网可以使用 CSMA/CD，而无线局域网必须使用 CSMA/CA.

解 :

有线以太网使用 CSMA/CD 的条件 :

  1. 可以检测碰撞 : 有线介质中，节点发送时可以同时监听信道，若发送信号与信道上的信号不同，说明发生了碰撞 ( 电信号叠加可检测 ).
  2. 信号能量可测 : 碰撞时信号幅度约为正常信号的 2 倍，硬件可可靠检测.
  3. 全双工困难但可实现 : 现代以太网交换机环境下已全双工，但早期总线以太网是半双工.

无线局域网不能使用 CSMA/CD 的原因 :

  1. 发送功率与接收功率差异极大 : 无线节点发送时，自身发出的信号功率远大于接收到的信号 ( 信号衰减 ) ，碰撞检测在硬件上极难实现 ( 相当于在打雷时听细雨声 ).
  2. 隐藏节点问题 : 节点 A 和节点 C 都在接入点 B 的覆盖范围内，但 A 和 C 互相不可见 ( 障碍物遮挡或距离过远 ). A 检测信道空闲 ( 没听到 C 的信号 ) 就发送，但 C 同时发送，在 B 处发生碰撞——A 和 C 都无法检测到这次碰撞.
  3. 暴露节点问题 : 节点 B 在发送给 A 时，C 检测到信道忙而不发，但 C 向 D 发送不会干扰 B→A，C 的等待是不必要的浪费 ( 碰撞避免过于保守 ).

因此，802.11 采用 CSMA/CA，通过 IFS 间隔、随机退避、ACK 确认、RTS/CTS 握手 来最大程度地 避免 碰撞的发生，而非碰撞后再处理.
