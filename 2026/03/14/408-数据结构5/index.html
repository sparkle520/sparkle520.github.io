第5章 图 ( Graph )

5.1 基本概念

5.1.1 图的定义

图 \(G = (V, E)\) 由顶点集 \(V\) ( Vertex ) 和边集 \(E\) ( Edge ) 组成. \(|V|\) 表示顶点数，\(|E|\) 表示边数，图不允许顶点集为空.

类型 边的表示 边数范围
无向图 边 \((u, v)\)，无方向 \(0 \leq \|E\| \leq \dfrac{\|V\|(\|V\|-1)}{2}\)
有向图 弧 \(\langle u, v \rangle\)，从 \(u\) 到 \(v\) \(0 \leq \|E\| \leq \|V\|(\|V\|-1)\)
完全图 任意两顶点间都有边 无向 \(\|E\| = \dfrac{n(n-1)}{2}\)，有向 \(\|E\| = n(n-1)\)

无向图 顶点 \(v\) 的度 \(\text{TD}(v)\) = 与 \(v\) 相连的边数.

有向图 * 入度 \(\text{ID}(v)\) = 以 \(v\) 为终点的弧数. * 出度 \(\text{OD}(v)\) = 以 \(v\) 为起点的弧数. * \(\text{TD}(v) = \text{ID}(v) + \text{OD}(v)\).

重要关系

\[\sum_{v \in V} \text{TD}(v) = 2|E| \quad \text{ ( 无向图 ) }\]

\[\sum_{v \in V} \text{ID}(v) = \sum_{v \in V} \text{OD}(v) = |E| \quad \text{ ( 有向图 ) }\]

  • 权 边或弧上的数值，表示代价、距离、时间等.
  • 网 ( 带权图 ) 边上带有权值的图.
  • 带权路径长度 路径上所有边权值之和.

5.1.2 路径与连通性

术语 定义
路径 从顶点 \(v\) 到顶点 \(w\) 经过的顶点序列
路径长度 路径上边的条数 ( 无权 ) 或边权之和 ( 有权 )
简单路径 路径中顶点 不重复 出现
回路 ( 环 ) 第一个和最后一个顶点相同的路径
简单回路 除首尾外顶点不重复的回路
概念 定义
连通 无向图中 \(u\) 到 \(v\) 有路径
连通图 无向图中任意两顶点连通
连通分量 无向图的极大连通子图
强连通 有向图中 \(u→v\) 且 \(v→u\) 均有路径
强连通图 有向图中任意两顶点均强连通
强连通分量 有向图的极大强连通子图

连通图 vs. 强连通图 连通图是无向图概念，强连通图是有向图概念，不可混用.

生成树 连通图 \(G\) 的一个 极小连通子图，包含 \(G\) 的所有 \(n\) 个顶点和 \(n-1\) 条边.

  • 生成树中去掉任一条边，图不再连通; 加入任一条边，必出现回路.
  • \(n\) 个顶点的连通图，生成树恰好有 \(n - 1\) 条边.

最小生成树 ( MST ) 带权连通图中，边权之和最小的生成树.

  • MST 不唯一 ( 边权相等时 )，但边权之和唯一.
  • MST 性质 若 \(e\) 是连接两个不同连通分量的最小权边，则 \(e\) 一定在某棵 MST 中 ( 切割性质 ).

5.2 图的存储

5.2.1 邻接矩阵

邻接矩阵 用 \(n \times n\) 矩阵 \(A\) 表示图，\(A[i][j]\) 表示顶点 \(i\) 到 \(j\) 是否有边 ( 无权图 ) 或边权 ( 有权图 ).

矩阵定义

\[A[i][j] = \begin{cases} 1 & \text{若 } (v_i, v_j) \in E \text{ ( 无权图 ) } \\ w_{ij} & \text{若 } (v_i, v_j) \in E \text{ ( 带权图 ) } \\ 0 / \infty & \text{不存在边} \end{cases}\]

  • 无向图邻接矩阵 对称，可只存上三角，节省空间.
  • 有向图邻接矩阵一般 不对称.
  • 顶点 \(i\) 的 度 ( 无向图 ) = 第 \(i\) 行 ( 或列 ) 中非零元素之和.
  • 顶点 \(i\) 的 出度 ( 有向图 ) = 第 \(i\) 行非零元素之和.
  • 顶点 \(i\) 的 入度 ( 有向图 ) = 第 \(i\) 列非零元素之和.
项目 分析
空间复杂度 \(O(\|V\|^2)\)，与边数无关
判断边是否存在 \(O(1)\)
遍历所有边 \(O(\|V\|^2)\)
适用场景 稠密图，\(\|E\|\) 接近 \(\|V\|^2\)

5.2.2 邻接表

邻接表 对每个顶点建立一个单链表，存储该顶点的所有邻接顶点，是图的链式存储.

  • 无向图中，每条边 \((u,v)\) 在两个顶点的链表中各出现一次.
  • 有向图中，每条弧 \(\langle u,v \rangle\) 只出现在 \(u\) 的链表中.
  • 顶点 \(i\) 的 出度 ( 有向图 ) = 顶点 \(i\) 链表的长度.
  • 顶点 \(i\) 的 入度 ( 有向图 ) 需遍历整个邻接表才能求得.
项目 分析
空间复杂度 \(O(\|V\| + \|E\|)\)
判断边是否存在 \(O(\|V\|)\) 最坏
遍历所有边 \(O(\|V\| + \|E\|)\)
适用场景 稀疏图，\(\|E\| \ll \|V\|^2\)

5.2.3 十字链表与邻接多重表 ( 了解 )

十字链表 专为有向图设计，每条弧有一个结点，同时链入起点的出弧链和终点的入弧链.

结点结构 | tailVex | headVex | weight | hlink | tlink |

  • 优点 同时方便求入度和出度，\(O(\text{deg}(v))\).
  • 适合有向图中需要频繁查询入度的场景.

邻接多重表 专为无向图设计，每条边只存储一个结点，避免邻接表中每条边重复存储的问题.

结点结构 | iVex | iLink | jVex | jLink | weight |

  • 优点 便于对边进行标记操作，删边只需修改指针.

5.2.4 四种存储方式综合对比

存储方式 空间 判边 求度 适用图
邻接矩阵 \(O(V^2)\) \(O(1)\) \(O(V)\) 稠密图、无向图
邻接表 \(O(V+E)\) \(O(V)\) 出度 \(O(1)\)，入度 \(O(V+E)\) 稀疏图
十字链表 \(O(V+E)\) \(O(V)\) 入出度均快 有向图
邻接多重表 \(O(V+E)\) \(O(V)\) 快 无向图

5.2.5 408 常考点

  • 邻接矩阵判边 \(O(1)\)，但遍历邻接点要扫描整行，整体遍历通常为 \(O(V^2)\)
  • 邻接表遍历邻接点按链表走，整体遍历通常为 \(O(V+E)\)
  • 稠密图优先矩阵，稀疏图优先邻接表
  • 无向图邻接矩阵第 \(i\) 行非零元素个数 = 顶点 \(i\) 的度
  • 有向图邻接矩阵第 \(i\) 行非零元素个数 = 出度，第 \(i\) 列非零元素个数 = 入度
  • 邻接表天然得到的是出度; 若题目频繁问入度，考虑建立逆邻接表或用十字链表
  • 无向图度之和为 \(2|E|\)，有向图入度和 = 出度和 = \(|E|\)
  • 完全图边数常与「允许自环」混淆，408 默认不允许自环

5.3 图的遍历 ( 必考 )

图的遍历

从图中某一顶点出发，访问图中所有顶点，且每个顶点仅被访问一次.

图的遍历需要 访问标记数组 visited[]，防止重复访问. 非连通图需要对每个未访问的顶点分别启动遍历.

5.3.1 深度优先搜索 ( DFS )

存储方式 时间复杂度 空间复杂度
邻接矩阵 \(O(\|V\|^2)\) \(O(\|V\|)\)
邻接表 \(O(\|V\| + \|E\|)\) \(O(\|V\|)\)

空间复杂度的 \(O(|V|)\) 来自递归调用栈或显式栈.

  • DFS 生成树 / 生成森林 遍历过程经过的边构成生成树.
  • 连通分量个数 调用 DFS 的次数即为连通分量数.
  • 拓扑排序 可基于 DFS 实现 ( 结点出栈顺序的逆序 ).
  • 关键路径 依赖拓扑序.
  • 欧拉路径 判断是否存在欧拉回路.

5.3.2 广度优先搜索 ( BFS )

存储方式 时间复杂度 空间复杂度
邻接矩阵 \(O(\|V\|^2)\) \(O(\|V\|)\)
邻接表 \(O(\|V\| + \|E\|)\) \(O(\|V\|)\)

空间复杂度的 \(O(|V|)\) 来自队列.

BFS 按层遍历，首次到达某顶点时经过的边数即为 最短路径长度 ( 无权图 ).

比较项 DFS BFS
数据结构 栈 ( 递归/显式 ) 队列
遍历顺序 纵向深入 横向逐层
空间复杂度 \(O(V)\) ( 最深递归深度 ) \(O(V)\) ( 最宽队列 )
找最短路 ( 无权 ) ✗ 不保证 ✓ 保证
连通分量 ✓ ✓
拓扑排序 ✓ ✓

5.3.3 408 常考点

  • 非连通图的遍历是「多次启动」DFS/BFS
  • 外层循环中调用 DFS/BFS 的次数 = 连通分量数 ( 无向图 )
  • DFS/BFS 过程中第一次访问某顶点所经过的边，构成对应的 DFS 生成树 / BFS 生成树
  • BFS 生成树中，从源点到任意顶点的树路径边数最少 ( 无权图最短路 )
  • 每次开始一次新的全图遍历前都要把 visited[] 清零
  • BFS 最短路若同时维护 dist[] / path[]，更新只发生在第一次到达该点时

5.4 图的应用

5.4.1 最小生成树 ( MST )

Prim 算法 ( 加点法 ) 从一个顶点出发，每次选取连接已选顶点集和未选顶点集的最小权边，将对应顶点加入已选集，直到所有顶点都被选.

项目 分析
时间复杂度 \(O(\|V\|^2)\) ( 邻接矩阵 ) ; 用优先队列优化后 \(O((\|V\|+\|E\|)\log\|V\|)\)
适用场景 稠密图 ( 边数多时更优 )

Kruskal 算法 ( 加边法 ) 将所有边按权值升序排序，依次尝试加入，若不形成回路则加入 MST，直到 MST 包含 \(n-1\) 条边.

项目 分析
时间复杂度 \(O(\|E\| \log \|E\|)\) ( 排序主导 )
适用场景 稀疏图 ( 边数少时更优 )

并查集是 Kruskal 判断是否成环的核心工具，路径压缩后近似 (O(1)) 判断连通性.

比较项 Prim Kruskal
思路 扩展顶点 扩展边
时间复杂度 \(O(V^2)\) \(O(E \log E)\)
优化后 \(O((V+E)\log V)\) \(O(E \log E)\)
适用图 稠密图 稀疏图
辅助结构 候选边数组 / 优先队列 并查集

5.4.2 最短路径

Dijkstra 算法 求单源最短路径 ( 源点 \(s\) 到所有顶点的最短距离 )，适用于非负权图.

项目 分析
时间复杂度 \(O(\|V\|^2)\) ( 邻接矩阵 ) ; 优先队列优化后 \(O((\|V\|+\|E\|)\log\|V\|)\)
能否处理负权边 ✗ 不能
能否处理负权环 ✗ 不能

Dijkstra 不适用于负权边. 存在负权边时应使用 Bellman-Ford 算法 ( 408 不要求，了解即可 ).

Floyd 算法 求所有顶点对之间的最短路径，允许 负权边 ( 不允许负权环 ).

项目 分析
时间复杂度 \(O(\|V\|^3)\)
空间复杂度 \(O(\|V\|^2)\)
能否处理负权边 ✓ 能 ( 不含负权环 )
适用图规模 顶点数较少时 ( \(\|V\| \leq 200\) )

Floyd 本质

Floyd 本质是 动态规划，([i][j]) 表示只经过顶点 ({0,1,,k}) 作为中间点时 (i) 到 (j) 的最短距离.

比较项 Dijkstra Floyd
问题类型 单源最短路 全源最短路
时间复杂度 \(O(V^2)\) / \(O((V+E)\log V)\) \(O(V^3)\)
负权边 ✗ ✓ ( 无负环 )
实现复杂度 中等 简单 ( 三重循环 )

5.4.3 拓扑排序

AOV 网 ( Activity On Vertex ) 用 顶点 表示活动，用 有向边 表示活动之间的前驱 / 后继关系的有向图.

拓扑排序 将 AOV 网中的顶点排列成线性序列，使得每条边 \(\langle u, v \rangle\) 对应的 \(u\) 都在 \(v\) 之前，该序列称为 拓扑序列.

  • 时间复杂度 \(O(|V| + |E|)\).
  • 拓扑序列不唯一 ( 可能存在多个入度为 0 的顶点 ).
  • 若拓扑排序结果不包含全部顶点，则图中有环.

三种判环方法

方法 原理 复杂度
拓扑排序 输出顶点数 < \(\|V\|\) 则有环 \(O(V+E)\)
DFS + 颜色标记 访问到 灰色 结点则有环 \(O(V+E)\)
并查集 Kruskal 中若两顶点同集合则加边成环 \(O(E\alpha(V))\)

5.4.4 关键路径

AOE 网 ( Activity On Edge ) 用 边 表示活动，用 顶点 表示事件 ( 活动的开始 / 结束 )，边上的权值为活动所需时间.

目标 求从源点到汇点的 最长路径 ( 关键路径 )，即完成整个工程的最短时间.

符号 名称 含义
\(ve(v_k)\) 最早发生时间 从源点到 \(v_k\) 的最长路径长度
\(vl(v_k)\) 最晚发生时间 不推迟工程完成前提下，\(v_k\) 最晚发生时间
\(e(a_i)\) 活动最早开始时间 \(e(a_i) = ve(v_j)\) ( \(v_j\) 为 \(a_i\) 的起点 )
\(l(a_i)\) 活动最晚开始时间 \(l(a_i) = vl(v_k) - w(a_i)\) ( \(v_k\) 为 \(a_i\) 的终点 )

时差 ( 松弛时间 ) \(d(a_i) = l(a_i) - e(a_i)\)，\(d(a_i) = 0\) 的活动称为 关键活动.

  1. 求 \(ve\) 按拓扑序从前往后计算，\(ve(\text{源点}) = 0\).

\[ve(v_k) = \max_{(v_j, v_k) \in E} \{ve(v_j) + w(v_j, v_k)\}\]

  1. 求 \(vl\) 按逆拓扑序从后往前计算，\(vl(\text{汇点}) = ve(\text{汇点})\).

\[vl(v_j) = \min_{(v_j, v_k) \in E} \{vl(v_k) - w(v_j, v_k)\}\]

  1. 求 \(e\)、\(l\)、\(d\) 对每条边 \(a_i = (v_j, v_k)\)
    • \(e(a_i) = ve(v_j)\)
    • \(l(a_i) = vl(v_k) - w(a_i)\)
    • \(d(a_i) = l(a_i) - e(a_i)\)
  2. 关键路径 所有 \(d(a_i) = 0\) 的活动构成关键路径，从源点到汇点.
  • 加速关键活动 才能缩短工程工期，加速非关键活动无效.
  • 关键路径 不唯一 ( 可能存在多条 )，需同时加速所有关键路径上的活动才能缩短工期.
  • 若某关键活动加速后时差仍为 0，则仍为关键活动; 若时差变为正数则成为非关键活动.

常见错误 认为减少某关键活动时间就一定能缩短工期 — 须注意该活动缩短后可能产生新的关键路径，实际缩短的时间取决于 所有关键路径的最小压缩量.

5.4.5 408 常考点

  • Dijkstra 只能处理非负权，题目出现负权边直接排除
  • Floyd 的三重循环最外层必须是中间点 \(k\); 题目常考循环顺序与初始化
  • MST 不一定唯一 ( 边权相等时 )，但总权值唯一
  • Prim 更像「扩点」，Kruskal 更像「选边 + 并查集判环」
  • 有拓扑序 ⇔ 是 DAG
  • Kahn 算法中若最终输出顶点数小于 \(|V|\)，则图中必有环
  • AOE 网的关键路径是「最长路径」，决定最短工期
  • 关键路径可能有多条，缩短工期要同时处理所有关键路径上的关键活动

5.5 例题精选

例题 1 ( 度的计算 ) 无向图有 8 条边，则所有顶点度之和为多少 ?

解

无向图度之和 = \(2|E| = 2 \times 8 = 16\).

例题 2 ( Prim 与 Kruskal 选择 )

图 \(G\) 有 1000 个顶点、5000 条边，选用 Prim 还是 Kruskal 更合适 ?

解

\(|E| = 5000 \ll |V|^2 = 10^6\)，属于稀疏图，优先使用 Kruskal 算法，时间复杂度 \(O(E \log E) = O(5000 \times 13) \approx 65000\)，优于 Prim 的 \(O(V^2) = 10^6\).

例题 3 ( Dijkstra 松弛过程 )

图中有顶点 \(A, B, C, D\)，边权 \(A→B=3, A→C=6, B→C=2, B→D=5, C→D=1\). 求 \(A\) 到各顶点最短距离.

解

步骤 加入 dist[A] dist[B] dist[C] dist[D]
初始 — 0 ∞ ∞ ∞
1 A 0 3 6 ∞
2 B — 3 5 8
3 C — — 5 6
4 D — — — 6

最短距离 \(A→B=3,\ A→C=5,\ A→D=6\).

例题 4 ( 拓扑排序判环 )

有向图顶点 \(1\sim5\)，边 \(1→2, 1→3, 2→4, 3→4, 4→5, 5→2\). 判断是否有环，若无则给出拓扑序列.

解

入度初始 \(\{1:0, 2:2, 3:1, 4:2, 5:1\}\).

入度为 0 的入栈 \([1]\). 弹出 1，输出 1，更新入度 \(\{2:1, 3:0\}\).

弹出 3，输出 3，更新入度 \(\{4:1\}\).

此时不存在入度为 0 的顶点 ( 2、4、5 的入度均为 1 )，算法无法继续.

输出顶点数 = 2 < 5，因此图中 有环 ( 环为 \(2 \to 4 \to 5 \to 2\) )，不存在拓扑序列.

例题 5 ( 关键路径 )

AOE 网如下，源点 \(v_1\)，汇点 \(v_6\). 活动 \(a_1(v_1→v_2,3)\)，\(a_2(v_1→v_3,2)\)，\(a_3(v_2→v_4,2)\)，\(a_4(v_3→v_4,4)\)，\(a_5(v_4→v_6,3)\).

解

\(ve\) 正向计算 \(ve(v_1)=0, ve(v_2)=3, ve(v_3)=2, ve(v_4)=\max(3+2,2+4)=6, ve(v_6)=6+3=9\).

\(vl\) 逆向计算 \(vl(v_6)=9, vl(v_4)=9-3=6, vl(v_2)=6-2=4, vl(v_3)=6-4=2, vl(v_1)=\min(4-3,2-2)=0\).

时差计算

活动 \(e\) \(l\) \(d\) 是否关键
\(a_1\) 0 1 1 否
\(a_2\) 0 0 0 是
\(a_3\) 3 4 1 否
\(a_4\) 2 2 0 是
\(a_5\) 6 6 0 是

关键路径 \(v_1 \xrightarrow{a_2} v_3 \xrightarrow{a_4} v_4 \xrightarrow{a_5} v_6\)，工期 = 9.

5.6 本章疑难点

1. 连通图 vs. 强连通图

  • 连通图 是无向图概念，任意两点可达; 强连通图 是有向图概念，任意两点互相可达.
  • 有向图中”有路径从 \(u\) 到 \(v\)“不等于”有路径从 \(v\) 到 \(u\)“.

2. 邻接表求入度

  • 邻接表天然存储出度，求入度需遍历所有邻接表 ( 逆邻接表可解决 ).
  • 考题若要频繁求有向图入度，优先考虑 十字链表.

3. Dijkstra 不能处理负权边

  • 负权边会导致已确定的最短路径被更新，贪心策略失效.
  • 处理负权边需用 Bellman-Ford ( \(O(VE)\) )，408 不作要求，了解即可.

4. 拓扑排序与有向无环图 ( DAG )

  • 有拓扑序 \(\Leftrightarrow\) 图是 DAG ( 有向无环图 ).
  • 拓扑序不唯一 ( 可能有多个源点 ).

5. 关键路径的多条问题

  • 工程中可能有 多条关键路径，只加速其中一条上的活动，若另一条不变则工期不变.
  • 必须同时加速 所有关键路径 上都包含的活动，才能保证工期缩短.

6. Floyd 初始化

  • \(\text{dist}[i][j]\) 初始化为邻接矩阵值，dist[i][i] = 0，不直接相连为 \(\infty\).
  • 三重循环的 最外层是中间顶点 \(k\)，不能颠倒循环顺序.

5.7 本章小结

  • 图是多对多关系的非线性结构，基本概念是后续所有算法的基础.
  • 邻接矩阵适合稠密图 ( 判边 \(O(1)\) )，邻接表适合稀疏图 ( 节省空间 ).
  • DFS 借助栈，BFS 借助队列，BFS 保证无权图最短路.
  • MST 的 Prim 适合稠密图，Kruskal 适合稀疏图，两者均基于贪心.
  • Dijkstra 解决非负权单源最短路，Floyd 解决全源最短路 ( 允许负权但无负环 ).
  • 拓扑排序判断 DAG，关键路径分析工程最短工期，两者是 AOV/AOE 网的核心应用.
