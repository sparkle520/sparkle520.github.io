sparkle5202026-03-142026-03-21

第7章 内部排序 ( Internal Sorting )

7.1 基本概念

7.1.1 核心术语

排序 将一组无序的记录按关键字的某种顺序 ( 递增或递减 ) 重新排列的过程. 内部排序 指排序期间所有数据均在内存中，外部排序 需借助外存.

术语 定义
稳定性 若排序前后，值相同的两个元素相对位置不变，则该算法 稳定
比较次数 算法执行过程中关键字比较的总次数
移动次数 算法执行过程中记录移动 ( 交换 ) 的总次数
原地排序 空间复杂度 \(O(1)\)，不使用额外数组
趟 对序列进行一次完整扫描或处理称为一趟

7.1.2 稳定性的重要性

  • 若记录有多个关键字，先按次关键字排序，再按主关键字 稳定 排序，可保留次关键字的顺序 ( 基数排序的基础 ).
  • 不稳定的排序算法 快速排序、堆排序、希尔排序、简单选择排序.
  • 稳定的排序算法 插入排序 ( 直接/折半 ) 、冒泡排序、归并排序、基数排序.

稳定性是算法本身的性质，不是输入数据的性质. 但不稳定排序在某些特定输入下可能表现为稳定，考试以最坏情况为准.

7.1.3 408 常考点

  • 稳定性的判定只看 关键字相等 的元素是否可能发生“跨越交换 / 跨组移动”
  • 多关键字排序常用技巧 先按次关键字排序，再对主关键字做 稳定排序，即可保留次关键字顺序
  • 记忆口诀 不稳定的只有 “快、堆、希、选”，其余 ( 插、冒、归、基 ) 稳定
  • “原地排序”指辅助空间 \(O(1)\) ( 递归栈一般不计入辅助数组，但 408 常单独讨论快排递归栈 )
  • 归并排序数组实现需要辅助数组 \(O(n)\)，但链表归并可以做到 \(O(1)\) 额外空间
  • 比较次数与移动次数要分开看 折半插入减少比较但不减少移动，因此总体仍是 \(O(n^2)\)
  • 重点会考 Partition 过程、快排最坏情况、建堆为何是 \(O(n)\)、归并趟数与稳定性
  • 选“最合适算法”题 按规模 / 有序性 / 稳定性 / 空间限制四个维度给理由

7.2 插入排序

7.2.1 直接插入排序

将第 \(i\) 个元素插入到前 \(i-1\) 个 已排好序 的子序列的合适位置.

情况 比较次数 移动次数 时间复杂度
最好 ( 正序 ) \(n-1\) \(0\) \(O(n)\)
最坏 ( 逆序 ) \(\dfrac{n(n-1)}{2}\) \(\dfrac{(n+1)(n-1)}{2}\) \(O(n^2)\)
平均 \(\dfrac{n^2}{4}\) \(\dfrac{n^2}{4}\) \(O(n^2)\)
  • 空间复杂度 \(O(1)\)，稳定，适合 基本有序 或 小规模 序列.

7.2.2 折半插入排序

折半插入排序

在直接插入的基础上，用 折半查找 确定插入位置，减少比较次数，但移动次数不变.

  • 比较次数 \(O(n \log n)\) ( 折半查找 ) ; 移动次数仍为 \(O(n^2)\).
  • 时间复杂度总体仍为 \(O(n^2)\) ( 移动主导 )，稳定.

7.2.3 希尔排序 ( Shell Sort )

希尔排序 按 步长 ( 增量序列 ) 分组，每组内做直接插入排序，逐步缩小步长直到 1.

  • 时间复杂度与步长序列有关，常见结果 \(O(n^{1.3}) \sim O(n^{1.5})\).
  • 空间复杂度 \(O(1)\)，不稳定 ( 相同元素可能因不同分组被打乱顺序 ).
  • 适合 中等规模 序列，是突破 \(O(n^2)\) 的第一个改进插入排序.

步长序列选择

步长序列 最坏复杂度
\(n/2, n/4, \ldots, 1\) ( Shell 原始 ) \(O(n^2)\)
Hibbard \(2^k - 1, \ldots, 3, 1\) \(O(n^{3/2})\)
Sedgewick 序列 \(O(n^{4/3})\)

7.3 交换排序

7.3.1 冒泡排序

每趟将最小元素”冒”到前端，flag 优化避免不必要的扫描.

情况 比较次数 交换次数 时间复杂度
最好 ( 正序 ) \(n-1\) \(0\) \(O(n)\)
最坏 ( 逆序 ) \(\dfrac{n(n-1)}{2}\) \(\dfrac{n(n-1)}{2}\) \(O(n^2)\)
平均 \(\dfrac{n^2}{4}\) \(\dfrac{n^2}{4}\) \(O(n^2)\)
  • 空间复杂度 \(O(1)\)，稳定.

7.3.2 快速排序

快速排序是 408 最高频 考点之一，需掌握 Partition 过程、递归实现、非递归实现、时间复杂度分析及最坏情况.

划分 ( Partition ) 选取枢轴 pivot，将序列划分为左 ≤ pivot、右 ≥ pivot 两部分.

调用 QuickSort(A, 1, n).

情况 递归深度 时间复杂度 说明
最好 \(O(\log n)\) \(O(n \log n)\) 每次划分均匀
最坏 \(O(n)\) \(O(n^2)\) 有序序列取首元素为枢轴
平均 \(O(\log n)\) \(O(n \log n)\) 期望 \(O(n \log n)\)
  • 空间复杂度 \(O(\log n)\) 平均 ( 递归栈 ) ; \(O(n)\) 最坏.
  • 不稳定 ( Partition 过程中远距离交换 ).
  1. 随机选取枢轴 随机选取 [low, high] 中的元素为枢轴，避免有序序列最坏情况.
  2. 三数取中 取 A[low]、A[mid]、A[high] 的中间值为枢轴，更稳定.
  3. 小规模切换 当子数组长度 \(\leq k\) ( 通常 \(k = 10 \sim 20\) ) 时切换为直接插入排序.
  4. 三向划分 将等于 pivot 的元素单独分区 ( 荷兰国旗问题 )，处理大量重复元素时效率更高.

7.3.3 408 常考点 ( 快速排序 )

  • Partition 结束后 pivot 一定就位，pivot 左侧元素均 ≤ pivot，右侧元素均 ≥ pivot
  • 题目手算时重点盯 low、high 的移动条件，以及“先从右找，再从左找”的赋值顺序
  • 取首元素作枢轴时，正序/逆序会退化到最坏 \(O(n^2)\)，递归深度也退化为 \(O(n)\)
  • 常见改进 随机枢轴、三数取中、小区间切插入、三向划分 ( 大量重复关键字 )
  • 快排 不稳定 的根因是跨区远距离交换
  • 空间复杂度平均为递归栈 \(O(\log n)\)，最坏为 \(O(n)\)，非递归实现用显式栈等价

7.4 选择排序

7.4.1 简单选择排序

每趟从未排序部分找最小元素，放到已排序末尾.

  • 比较次数 始终 \(\dfrac{n(n-1)}{2}\)，与输入无关.
  • 交换次数 最好 \(0\) ( 正序 )，最坏 \(n-1\).
  • 时间复杂度 始终 \(O(n^2)\); 空间复杂度 \(O(1)\).
  • 不稳定 ( 可能将当前最小值跳过相同元素交换到前面 ).

不稳定示例 \(\{3_a, 3_b, 1\}\) → 第 1 趟选 1 与 \(3_a\) 交换 → \(\{1, 3_b, 3_a\}\)，\(3_a\) 和 \(3_b\) 相对顺序改变.

7.4.2 堆排序

堆排序已在第 4 章 ( 堆 ) 中详细介绍，此处重点对比性能特点.

步骤 操作 时间复杂度
建堆 BuildHeap，从最后一个分支结点向上 siftDown \(O(n)\)
排序 循环 \(n-1\) 次，每次取堆顶并 siftDown \(O(n \log n)\)
总计 建堆 + 排序 \(O(n \log n)\)
  • 空间复杂度 \(O(1)\) ( 原地排序 ).
  • 不稳定 ( 堆顶与末尾互换后，相对顺序可能改变 ).
  • 最坏情况仍为 \(O(n \log n)\)，是唯一能保证最坏 \(O(n \log n)\) 且空间 \(O(1)\) 的排序.

7.5 归并排序

自底向上归并 从步长 1 开始，每轮步长翻倍，对相邻块两两合并.

  • 非递归版本不需要额外的递归栈，实际工程中更常用.
  • 时间复杂度 任何情况均为 \(O(n \log n)\)，共 \(\lceil \log_2 n \rceil\) 趟，每趟 \(O(n)\).
  • 空间复杂度 \(O(n)\) ( 辅助数组 ).
  • 稳定 ( Merge 中 B[i] <= B[j] 时优先取左边，保持相对顺序 ).

为什么归并是稳定的 Merge 时遇到相等元素，总是先取左子序列 ( 前者 )，保证原来靠前的元素仍在前面.

\(n\) 个元素的二路归并排序，共需 \(\lceil \log_2 n \rceil\) 趟归并.

趟数 有序子列长度 归并次数
1 2 \(\lfloor n/2 \rfloor\)
2 4 \(\lfloor n/4 \rfloor\)
\(k\) \(2^k\) \(\lfloor n/2^k \rfloor\)

示例 \(\{49, 38, 65, 97, 76, 13, 27\}\)

7.5.1 408 常考点 ( 归并排序 )

  • 二路归并排序需要 \(\lceil \log_2 n \rceil\) 趟，每趟都是“全数组级别”的合并，因此每趟 \(O(n)\)
  • 手算题常考 第 \(k\) 趟结束后有序子列最大长度为 \(\min(2^k, n)\)
  • 是否稳定只看 Merge 的相等处理 <= 取左边则稳定; 若写成 < 可能破坏稳定性
  • 数组归并通常需要辅助数组 \(O(n)\); 链表归并可做到 \(O(1)\) 额外空间

7.6 基数排序

基数排序 不比较关键字大小，而是按关键字 各位的值 分配到桶 ( 队列 ) 中，再收集，重复 \(d\) 次 ( \(d\) 为最大位数 )，完成排序.

  • LSD ( Least Significant Digit ) 从低位到高位排序 ( 最常用 ).
  • MSD ( Most Significant Digit ) 从高位到低位排序.

每趟过程

  1. 分配 按当前位值将元素分配到 \(r\) 个桶 ( \(r\) 为基数，十进制 \(r = 10\) ).
  2. 收集 按桶顺序将元素依次收回序列.

序列 \(\{329, 457, 657, 839, 436, 720, 355\}\)，三位十进制，LSD :

第 1 趟 ( 按个位 )

第 2 趟 ( 按十位 )

第 3 趟 ( 按百位 )

排序完成.

设 \(n\) 个元素，每个关键字有 \(d\) 位，每位基数为 \(r\) :

  • 每趟分配 \(O(n)\)，收集 \(O(r)\)，共 \(d\) 趟.
  • 时间复杂度 \(O(d(n + r))\)，通常视为 \(O(n)\) ( \(d, r\) 为常数时 ).
  • 空间复杂度 \(O(n + r)\) ( \(n\) 个指针节点 + \(r\) 个桶 ).
  • 稳定 ( 每趟分配和收集保持相对顺序，各趟均稳定，总体稳定 ).

基数排序是 非比较 排序，突破了基于比较的排序 (O(n n)) 下界. 但仅适用于关键字可分解为多个位的场景 ( 整数、字符串等 ).

桶排序 基数排序的简化版，将元素按值域分配到若干桶，每个桶内单独排序后合并.

  • 时间复杂度 \(O(n + k)\)，\(k\) 为桶的数量.
  • 适用于值域范围已知且分布均匀的场景.
  • 稳定.

7.6.1 408 常考点 ( 基数排序 )

  • \(d\) 是最大位数，\(r\) 是基数 ( 十进制 \(r=10\) )，时间复杂度通常写 \(O(d(n+r))\)
  • \(d,r\) 是否可视为常数取决于关键字范围 位数不受 \(n\) 增长影响时才可近似视为 \(O(n)\)
  • LSD 每一趟都要稳定分配与收集，才能保证最终整体稳定
  • 只能用于“可分位”的关键字 ( 整数、字符串 )，不适用于无法自然分位的对象
  • 基数排序不是“基于比较”的排序，因此不受 \(\Omega(n \log n)\) 比较下界约束

7.7 算法对比

7.7.1 综合对比表

这张表是 408 必考内容，需完整记忆. 重点关注不稳定算法 ( 快、堆、希、选 ) 和各算法最好情况.

算法 最好 平均 最坏 空间 稳定性
直接插入 \(O(n)\) \(O(n^2)\) \(O(n^2)\) \(O(1)\) 稳定
折半插入 \(O(n \log n)\)* \(O(n^2)\) \(O(n^2)\) \(O(1)\) 稳定
希尔排序 \(O(n)\) \(O(n^{1.3})\) \(O(n^2)\) \(O(1)\) 不稳定
冒泡排序 \(O(n)\) \(O(n^2)\) \(O(n^2)\) \(O(1)\) 稳定
快速排序 \(O(n \log n)\) \(O(n \log n)\) \(O(n^2)\) \(O(\log n)\) 不稳定
简单选择 \(O(n^2)\) \(O(n^2)\) \(O(n^2)\) \(O(1)\) 不稳定
堆排序 \(O(n \log n)\) \(O(n \log n)\) \(O(n \log n)\) \(O(1)\) 不稳定
归并排序 \(O(n \log n)\) \(O(n \log n)\) \(O(n \log n)\) \(O(n)\) 稳定
基数排序 \(O(d(n+r))\) \(O(d(n+r))\) \(O(d(n+r))\) \(O(n+r)\) 稳定

7.7.2 记忆口诀

不稳定的排序

“快 ( 快排 ) 堆 ( 堆排 ) 希 ( 希尔 ) 选 ( 简单选择 )” — 谐音”快堆希选”，记住这 4 个不稳定，其余均稳定.

空间 \(O(n)\) 的排序 只有 归并排序.

最坏仍 \(O(n \log n)\) 的排序 只有 堆排序 和 归并排序.

最好能到 \(O(n)\) 的排序 直接插入、冒泡 ( 含优化 ) 、希尔.

7.7.3 适用场景选择

规模 推荐算法 理由
\(n \leq 50\) ( 小规模 ) 直接插入、简单选择 常数小，实现简单
\(50 < n \leq 1000\) ( 中规模 ) 希尔排序 \(O(n^{1.3})\)，超越插入
\(n > 1000\) ( 大规模 ) 快速排序、堆排序、归并排序 \(O(n \log n)\)
要求 推荐算法
稳定性必须保证 归并排序 ( 大规模 ) 、插入 / 冒泡 ( 小规模 )
空间严格 \(O(1)\) 堆排序 ( 大规模 ) 、插入 / 冒泡 ( 小规模 )
链式存储 归并排序 ( 可原地归并 ) 、插入排序 ( 顺序插入 )
数据基本有序 直接插入 ( 接近 \(O(n)\) )
数据完全逆序 快速排序 ( 随机化 ) 、归并排序
关键字是整数且范围小 基数排序、桶排序
最坏情况有保证 堆排序 \(O(n \log n)\)
实践中最快 快速排序 ( 常数因子小，缓存友好 )

实际工程综合推荐

  • STL std::sort 内省排序 ( Introsort ) = 快速排序 + 堆排序 + 插入排序的三路混合，结合三者优势.
  • STL std::stable_sort 归并排序变体，保证稳定.
  • Python timsort 归并排序 + 插入排序，充分利用数据中的有序段.

408 考试中通常要求选择”最合适”的算法，答题时需说明理由 ( 规模、有序性、稳定性、空间限制 ).

7.7.4 408 常考点 ( 选算法题 )

  • 先说限制 是否要求稳定、是否空间受限、数据规模、是否基本有序、是否为链表
  • 再给结论 选哪种排序
  • 最后给理由 时间复杂度 ( 最好/平均/最坏 ) 、空间复杂度、稳定性
  • 基本有序 直接插入 ( 接近 \(O(n)\) )
  • 要最坏保证且空间 \(O(1)\) 堆排序
  • 要稳定且大规模 归并排序 ( 数组 ) 或链表归并 ( 链表 )

7.8 下界定理与基于比较的排序

排序下界定理

基于比较的排序算法，最坏情况下至少需要 \(\Omega(n \log n)\) 次比较.

证明思路 \(n\) 个元素的全排列有 \(n!\) 种，每次比较最多将可能性减半，需至少 \(\lceil \log_2(n!) \rceil \approx n \log_2 n - n \log_2 e = \Omega(n \log n)\) 次比较.

  • 堆排序和归并排序达到了这个下界 ( 最坏 \(O(n \log n)\) ).
  • 基数排序突破了这个下界，因为它 不是基于比较 的.

7.8.1 408 常考点

  • 下界结论仅针对“基于比较”的排序模型; 若题目给出“只能比较关键字大小”，则无法突破 \(\Omega(n \log n)\)
  • 非比较排序之所以可能更快，是因为利用了关键字结构 ( 位/字符/值域 )，相当于引入了额外信息与额外空间

7.9 例题精选

例题 1 ( 快速排序手动模拟 ) 对序列 \(\{49, 38, 65, 97, 76, 13, 27, 49'\}\) 进行快速排序 ( 取第一个元素为枢轴 )，写出第一次 Partition 后的结果.

解 pivot \(= 49\)，low \(= 1\)，high \(= 8\).

从右向左找 \(< 49\) 的 \(A[7] = 27 < 49\)，\(A[1] = 27\)，low 前进.

从左向右找 \(> 49\) 的 \(A[2] = 38 <49\) ( 跳过 ) \(\to\) \(A[3] = 65 > 49\)，\(A[7] = 65\)，high 后退.

继续 \(A[6] = 13 < 49\)，\(A[3] = 13\)，low 前进.

从左 \(A[4] = 97 > 49\)，\(A[6] = 97\)，high 后退.

继续 \(A[5] = 76 \ge 49\)，high 后退.

low \(= 4\)，high \(= 4\)，\(A[4] = 49\) ( pivot 就位 ).

结果 \(\{27, 38, 13, \mathbf{49}, 76, 97, 65, 49'\}\)，枢轴位于下标 4 ( 值 49 ).

例题 2 ( 归并排序趟数 ) 对 \(n = 10\) 个元素进行二路归并排序，共需几趟 ? 第 2 趟结束后有序子列最长长度是多少 ?

解 趟数 \(= \lceil \log_2 10 \rceil = 4\) 趟. 第 2 趟结束后，有序子列长度 \(= \min(2^2, n) = 4\)，每段最多 4 个元素.

例题 3 ( 稳定性判断 ) 判断以下场景哪些需要稳定排序

  1. 按姓名排序学生，可能同名.
  2. 电商平台按评分排序商品，评分相同时保持原始顺序 ( 上架时间序 ).
  3. 给整数数组从小到大排序，不关心相同值顺序.

解

  1. 姓名相同的学生排序后顺序无意义，不必稳定.
  2. 必须稳定 — 评分相同时需保留上架时间顺序.
  3. 不需要稳定.

例题 4 ( 场景选算法 ) 有 \(10^6\) 个记录，需要对其中最小的 1000 个记录进行排序输出，提出最高效方案.

解

先用 大顶堆 维护大小为 1000 的堆，扫描所有 \(10^6\) 个记录，时间 \(O(n \log k) = O(10^6 \times 10) = O(10^7)\).

再对堆中 1000 个元素进行 堆排序 输出，时间 \(O(k \log k)\). 总时间 \(O(n \log k)\)，远优于全部排序 \(O(n \log n)\).
