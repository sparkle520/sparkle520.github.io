sparkle5202026-03-142026-03-10

第6章 查找 ( Search )

6.1 基本概念

6.1.1 核心术语

查找 在数据集合中寻找满足某种条件的数据元素的过程. 查找的结果是找到或找不到.

术语 定义
查找表 由同一类型的数据元素 ( 或记录 ) 构成的集合
关键字 数据元素中唯一标识该元素的数据项，可以是主关键字或次关键字
静态查找 查找时不修改查找表，只做查找操作，如顺序查找、折半查找
动态查找 查找时可以插入或删除元素，如二叉排序树、散列表

6.1.2 平均查找长度 ( ASL )

平均查找长度 ASL ( Average Search Length ) 查找过程中 关键字比较次数 的期望值，是衡量查找算法效率的核心指标.

\[\text{ASL} = \sum_{i=1}^{n} P_i \times C_i\]

  • \(P_i\) 表示查找第 \(i\) 个元素的概率，通常假定 等概率 \(P_i = \dfrac{1}{n}\).
  • \(C_i\) 表示找到第 \(i\) 个元素所需的比较次数.
  • 需分别计算 查找成功 和 查找失败 的 ASL.

6.2 静态查找

6.2.1 顺序查找

哨兵的作用 将”循环是否越界”与”是否找到”合并为一次判断，减少比较次数.

设表长为 \(n\)，等概率查找

\[\text{ASL}_{\text{成功}} = \frac{1}{n}\sum_{i=1}^{n}(n-i+1) = \frac{n+1}{2}\]

\[\text{ASL}_{\text{失败}} = n + 1\]

  • 时间复杂度 \(O(n)\)，适用于 无序表 或 链式存储.

若表有序，查找失败时不必扫描到尾：

  • 扫描到第一个大于 key 的元素即可判定失败.
  • 失败时平均比较次数降低，但成功 ASL 不变.

\[\text{ASL}_{\text{失败 ( 有序 ) }} = \frac{1}{n+1}\sum_{i=1}^{n+1} i = \frac{n+2}{2}\]

6.2.2 折半查找 ( 二分查找 )

折半查找 仅适用于有序的顺序表，不适用于链表 ( 无法随机访问 ).

折半查找过程可用 判定树 ( 二叉查找树 ) 描述，树中第 \(k\) 层结点的比较次数为 \(k\).

设表长 \(n = 2^h - 1\) ( 满二叉树 )，高度 \(h = \log_2(n+1)\)

\[\text{ASL}_{\text{成功}} = \frac{1}{n}\sum_{j=1}^{h} j \times 2^{j-1} = \frac{n+1}{n}\log_2(n+1) - 1 \approx \log_2(n+1) - 1\]

\[\text{ASL}_{\text{失败}} \approx \log_2(n+1)\]

  • 时间复杂度 \(O(\log n)\).

对一般 \(n\)，折半查找的判定树构造规则

  • 根为 \(\lfloor (1 + n) / 2 \rfloor\)，左子树含 \(\lfloor (n-1)/2 \rfloor\) 个元素，右子树含 \(n - \lfloor (n-1)/2 \rfloor - 1\) 个元素.
  • 失败结点 ( 方形 ) 有 \(n+1\) 个，位于最后两层.
  • 查找失败比较次数 = 失败结点所在层数 ( 或层数 \(-1\)，取决于定义 ).

提示

408 真题常考”查找失败时的最多比较次数”= 判定树的 高度 (_2(n+1) ).

6.2.3 分块查找 ( 索引顺序查找 )

分块查找

将查找表分为若干块，块内元素 无序，块间元素 有序 ( 前一块的最大值 < 后一块的最小值 ). 另建索引表存储每块最大值及起始位置.

查找过程

  1. 在 索引表 中确定目标所在块 ( 顺序查找或折半查找 ).
  2. 在对应块内进行 顺序查找.

ASL 分析 ( 设 \(n\) 个元素分为 \(b\) 块，每块 \(s = n/b\) 个元素 )

  • 索引表用 顺序查找 \(\text{ASL} = \dfrac{b+1}{2} + \dfrac{s+1}{2}\)，令 \(b = \sqrt{n}\) 时最优，\(\text{ASL}_{\min} = \sqrt{n} + 1\).
  • 索引表用 折半查找 \(\text{ASL} = \lceil \log_2(b+1) \rceil + \dfrac{s+1}{2}\).
查找方式 时间复杂度 特点
顺序查找 \(O(n)\) 无序，无额外空间
折半查找 \(O(\log n)\) 需有序顺序存储
分块查找 \(O(\sqrt{n})\) 块间有序，块内无序，灵活

6.2.4 408 常考点

  • 哨兵让循环条件只需判断一次，尤其适合顺序表 ( 下标从 1 开始的写法 )
  • 有序顺序查找：一旦遇到第一个大于 key 的元素即可失败返回
  • 必须是有序的顺序表 ( 数组 )，链表不能折半
  • 判定树高度常用于求「查找失败的最多比较次数」：\(\lceil \log_2(n+1) \rceil\)
  • 拟合量级：成功与失败的 ASL 都是 \(O(\log n)\)
  • 块间有序 + 块内无序：先定位块，再块内顺序找
  • 索引顺序查找的经典最优：令块数 \(b \approx \sqrt{n}\) ( 索引用顺序查找时 )

6.3 动态查找 ( 树形结构 )

6.3.1 二叉排序树与 AVL 树

二叉排序树 ( BST ) 和平衡二叉树 ( AVL ) 已在第 4 章详细讨论，此处仅做查找性能总结.

结构 查找 ASL ( 成功 ) 查找时间 备注
BST ( 最优 ) \(O(\log n)\) \(O(\log n)\) 平衡时，形如完全二叉树
BST ( 最坏 ) \(O(n)\) \(O(n)\) 退化为单链 ( 有序插入 )
AVL 树 \(O(\log n)\) \(O(\log n)\) 任何情况，\(h = O(\log n)\)

6.3.2 B 树

B 树 ( 多路平衡查找树 ) 是一种 自平衡 的多路搜索树，所有叶结点在同一层，常用于外存 ( 磁盘 ) 索引.

\(m\) 阶 B 树定义

  • 每个结点最多有 \(m\) 棵子树，即最多 \(m-1\) 个关键字.
  • 根结点 ( 非叶 ) 至少有 2 棵子树，即至少 1 个关键字.
  • 非根非叶结点至少有 \(\lceil m/2 \rceil\) 棵子树，即至少 \(\lceil m/2 \rceil - 1\) 个关键字.
  • 所有 叶结点 在同一层 ( 失败结点，不含信息 ).
  • 结点关键字从左到右递增，子树指针与关键字交替排列.

一个含 \(k\) 个关键字的结点结构

  • \(P_0 < K_1 < P_1 < K_2 < \ldots < K_k < P_k\) ( 子树指针区间有序 ).
  • \(\lceil m/2 \rceil - 1 \leq k \leq m - 1\).
  • 每访问一个结点 = 一次磁盘 I/O.
  • 高度为 \(h\) 的 \(m\) 阶 B 树，查找最多需 \(h\) 次磁盘访问.
  • \(n\) 个关键字的 \(m\) 阶 B 树高度 \(h \leq \log_{\lceil m/2 \rceil}\left(\dfrac{n+1}{2}\right) + 1\).

插入规则

  1. 查找插入位置 ( 必为 最底层非叶结点 ).
  2. 插入关键字.
  3. 若该结点关键字数 \(> m-1\) ( 上溢 )，则 分裂 :
    • 取第 \(\lceil m/2 \rceil\) 个关键字上移到父结点.
    • 剩余分裂为两个结点.
  4. 若父结点也上溢，递归向上分裂，最坏情况分裂到根.

示例 ( 3 阶 B 树，结点最多 2 个关键字 )

删除分两种情况 :

  1. 删除非终端结点关键字 用其 中序前驱或后继 ( 必为终端结点的关键字 ) 替换，再删除终端结点中该关键字.

  2. 删除终端结点关键字

    • 若删后关键字数 \(\geq \lceil m/2 \rceil - 1\) ( 无下溢 ) 直接删除.
    • 若删后 下溢 ( 关键字数 \(< \lceil m/2 \rceil - 1\) )
      • 兄弟富余 向兄弟借一个关键字 ( 通过父结点中转 ).
      • 兄弟不富余 与兄弟及父结点中的一个关键字 合并，父结点关键字减少，可能引发递归合并.

高度与结点数关系 ( 考试常用 )

已知条件 结论
\(m\) 阶 B 树，\(n\) 个关键字 高度 \(h \leq \log_{\lceil m/2 \rceil}\dfrac{n+1}{2} + 1\)
高度 \(h\) 的 \(m\) 阶 B 树 最少关键字数 \(n \geq 2\lceil m/2 \rceil^{h-1} - 1\)
高度 \(h\) 的 \(m\) 阶 B 树 最多关键字数 \(n \leq m^h - 1\)

6.3.3 B+ 树

B+ 树

B 树的变体，是实际数据库 ( MySQL InnoDB 等 ) 索引的标准实现. 与 B 树的核心区别在于数据只存在 叶结点，内部结点仅存关键字 ( 索引 ).

B+ 树 vs B 树

比较项 B 树 B+ 树
数据存放位置 所有结点 仅叶结点
内部结点作用 索引 + 数据 纯索引
叶结点 失败结点，无数据 包含所有关键字和数据
叶结点连接 无 用链表 顺序相连 ( 支持范围查询 )
关键字重复 不重复 内部结点关键字在叶中 重复出现
顺序遍历 需中序遍历 遍历叶结点链表，\(O(n)\)
  • \(m\) 阶 B+ 树，内部结点最多 \(m\) 个子树，最多 \(m\) 个关键字 ( 与 B 树不同，B 树 \(m-1\) 个 ).
  • 非根内部结点至少 \(\lceil m/2 \rceil\) 个子树.
  • 所有叶结点包含全部关键字，按顺序排列，相邻叶结点用 双向链表 连接.
  • 查找路径：从根到叶结点，叶结点才是真正的数据.

B+ 树在数据库索引中的应用案例

MySQL InnoDB 存储引擎使用 B+ 树作为索引结构，原因如下

原因 说明
磁盘 I/O 最优 B+ 树高度低 ( 通常 3-4 层 )，每次查找最多 3-4 次磁盘访问
范围查询高效 叶结点链表支持 BETWEEN、ORDER BY、LIMIT 等范围操作
内部结点更紧凑 内部结点只存关键字不存数据，单页能存更多关键字，降低树高
顺序访问友好 遍历叶链表即可全表扫描，无需回溯

聚簇索引 vs 非聚簇索引

  • 聚簇索引 ( Primary Key ) 叶结点直接存储完整行数据，查找一次即可.
  • 非聚簇索引 ( Secondary Index ) 叶结点存储主键值，查找需回表 ( 两次 B+ 树查找 ).

实际高度估算 假设单页 16KB，关键字 + 指针约 14 字节，则单页约存 \(16384 / 14 \approx 1170\) 个条目. 高度为 3 的 B+ 树可存约 \(1170^2 \times n\) 条记录，支持数千万级别数据.

插入 与 B 树类似，叶结点上溢时从 中间关键字 分裂，但分裂时 中间关键字同时保留在叶结点 ( 上移的是副本 )，内部结点也获得该关键字.

删除 直接删叶结点中的关键字，下溢时从兄弟借或合并. 内部结点中的关键字不需删除 ( 仅起索引作用，保留副本即可 ).

B+ 树删除与 B 树的关键区别

删除叶结点关键字后，即使该关键字同时存在于内部结点 ( 作为分隔符 )，也 不必删除内部结点中的那份副本.

6.3.4 红黑树 ( 了解 )

红黑树 ( Red-Black Tree )

一种 自平衡 的二叉排序树，每个结点附加一个颜色属性 ( 红或黑 )，通过颜色约束保证树高 \(O(\log n)\).

五条性质

  1. 每个结点是 红色 或 黑色.
  2. 根结点是 黑色.
  3. 每个 叶结点 ( NIL 空结点 ) 是黑色.
  4. 若一个结点是 红色，则其两个子结点都是 黑色 ( 红结点的父/子不能是红色 ).
  5. 对任意结点，从该结点到其所有后代叶结点的路径上，黑色结点数量相同 ( 黑高相同 ).
  • 树高 含 \(n\) 个内部结点的红黑树高度 \(h \leq 2\log_2(n+1)\)，即 \(O(\log n)\).
  • 黑高 \(bh(T)\) 从任意结点到叶 ( 不含该结点 ) 的黑色结点数.
  • 最短路径 ( 全黑 ) 与最长路径 ( 红黑交替 ) 之比不超过 \(1 : 2\)，比 AVL 更宽松.
比较项 AVL 树 红黑树
平衡条件 \(\|\text{BF}\| \leq 1\)，严格 黑高相同，宽松
查找效率 略高 ( 树更矮 ) 略低
插入/删除旋转次数 较多 最多 \(O(1)\) 次旋转
适用场景 查找频繁 插删频繁 ( 如 Linux 进程调度、C++ map )

新结点默认 红色 插入 ( 不违反性质 5，尽量少破坏平衡 ).

插入后可能违反性质 4 ( 父红子红 )，通过 变色 + 旋转 修复

情况 叔父结点颜色 处理方法
Case 1 红色 父、叔变黑，祖父变红，祖父作为新结点递归处理
Case 2 黑色，新结点在内侧 对父做旋转，转化为 Case 3
Case 3 黑色，新结点在外侧 父变黑，祖父变红，对祖父做旋转

删除步骤

  1. 按 BST 方式删除结点.
  2. 若删除的是 红色结点，不影响黑高，直接完成.
  3. 若删除的是 黑色结点，黑高失衡，需通过 变色 + 旋转 修复 ( 共 4 种情况，较复杂，了解即可 ).

6.4 散列 ( Hash )

6.4.1 基本概念

散列表 ( 哈希表 )

根据关键字直接计算存储地址，理想情况下查找、插入、删除均为 \(O(1)\).

  • 散列函数 \(H(\text{key})\) 将关键字映射到存储地址的函数.
  • 冲突 不同关键字映射到同一地址，即 \(H(k_1) = H(k_2)\) 但 \(k_1 \neq k_2\).
  • 装填因子 \(\alpha = \dfrac{\text{表中记录数}}{\text{散列表长度}}\)，\(\alpha\) 越大冲突概率越高.

6.4.2 散列函数

\[H(\text{key}) = \text{key} \bmod p\]

  • \(p\) 通常选 不大于表长且最接近表长的素数 ( 质数 )，减少冲突.
  • 最常用，适用范围广.

\[H(\text{key}) = a \times \text{key} + b \quad (a, b \text{ 为常数})\]

  • 计算简单，无冲突 ( 线性映射 ).
  • 适合关键字 分布连续且范围已知 的场景.
  • 关键字范围稀疏时浪费空间.

分析关键字的数字分布，选取 分布较均匀的若干位 作为散列地址.

  • 例 手机号后 4 位 > 前 3 位 ( 运营商前缀相似 ).
  • 适合关键字 已知且分布有规律 的场景.

将关键字 平方，取中间若干位作为散列地址.

  • \(\text{key} = 1234 \Rightarrow 1234^2 = 1522756 \Rightarrow\) 取中间 3 位 \(= 227\).
  • 适合关键字位数较少、每位取值均匀时.

6.4.3 冲突处理方法

开放定址法 发生冲突时，按某种探测序列在散列表中 寻找下一个空槽 存入.

\[H_i = (H(\text{key}) + d_i) \bmod m\]

三种探测方式

方法 探测序列 \(d_i\) 特点
线性探测 \(1, 2, 3, \ldots, m-1\) 易产生 堆积，群集现象严重
二次探测 \(1^2, -1^2, 2^2, -2^2, \ldots\) 减少堆积，但不能探测所有槽
双重散列 \(d_i = i \times H_2(\text{key})\) 最优，减少聚集，探测序列随 key 变化

注意

线性探测产生的 一次聚集 是最严重的堆积问题 连续占用的槽越长，下一个元素落入该区域的概率越大，形成正反馈.

链地址法 ( 拉链法 ) 将散列到同一地址的所有关键字用 链表 链接，散列表中每个槽存储链表头指针.

  • 优点 无堆积问题，删除方便，装填因子可 \(> 1\).
  • 缺点 指针开销，链表在内存中不连续，缓存不友好.
比较项 开放定址 链地址
冲突处理 在表内探测 链表外延
装填因子 \(\alpha\) 必须 \(< 1\) 可 \(\geq 1\)
删除操作 复杂 ( 需标记墓碑 ) 简单
空间利用 无额外指针 有指针开销
适用场景 表大小已知、\(\alpha\) 较小 元素数量未知、频繁删除

6.4.4 散列表的操作与 ASL

  • 查找过程与插入路径完全相同，只是插入时找到空槽存入.

开放定址法删除的问题

  • 不能直接置空，否则会截断探测链，导致后续元素找不到.
  • 需使用“墓碑”标记 ( DELETED )，查找时跳过，插入时可复用.

链地址法删除 直接删除链表中对应结点，\(O(1)\).

设散列表长 \(m\)，装填因子 \(\alpha = n/m\)

线性探测开放定址法

\[\text{ASL}_{\text{成功}} \approx \frac{1}{2}\left(1 + \frac{1}{1-\alpha}\right)\]

\[\text{ASL}_{\text{失败}} \approx \frac{1}{2}\left(1 + \frac{1}{(1-\alpha)^2}\right)\]

链地址法

\[\text{ASL}_{\text{成功}} \approx 1 + \frac{\alpha}{2}\]

\[\text{ASL}_{\text{失败}} \approx \alpha + e^{-\alpha} \approx \alpha \quad (\alpha \text{ 较小时})\]

注意

408 考试中 ASL 通常直接根据散列过程 逐一计数，而非代公式. 建议画出散列表并统计每个关键字的比较次数.

选择散列表长度 \(m\) 的原则

  • \(m\) 选 素数 ( 尤其使用除留余数法时 )，减少系统性冲突.
  • 根据预期关键字数 \(n\) 和期望装填因子 \(\alpha\) 确定 \(m = \lceil n / \alpha \rceil\).
  • 一般 \(\alpha\) 控制在 \(0.5 \sim 0.75\) 之间，兼顾空间与效率.

6.4.5 408 常考点

  • 除留余数法最常考 \(H(k) = k \bmod p\)，\(p\) 往往取接近表长的素数
  • 题目同时给 \(m\) 与 \(p\) 时，以题目给定为准，不自行改表长
  • 查找与插入走同一条探测序列 ( 因此才有“查找失败时比较次数”一说 )
  • 删除不能直接置空 必须用墓碑 DELETED，否则会断开探测链
  • 线性探测最容易出现一次聚集，ASL 会随 \(\alpha\) 快速上升
  • 最稳的做法 按插入顺序画表，记录每个关键字的探测次数 ( 成功 ASL 按平均比较次数求 )
  • 失败 ASL 通常按“从 \(H(k)\) 出发直到遇到空槽”为一次失败查找的比较次数

6.5 各查找方法综合对比

方法 平均时间 最坏时间 有序要求 动态操作
顺序查找 \(O(n)\) \(O(n)\) 否 容易
折半查找 \(O(\log n)\) \(O(\log n)\) 必须有序 困难
分块查找 \(O(\sqrt{n})\) \(O(n)\) 块间有序 中等
BST \(O(\log n)\) \(O(n)\) 否 容易
AVL / 红黑树 \(O(\log n)\) \(O(\log n)\) 否 中等
B/B+ 树 \(O(\log n)\) \(O(\log n)\) 否 中等，适合外存
散列表 \(O(1)\) \(O(n)\) 否 容易

散列表查找为 \(O(1)\) 的前提 装填因子合理、散列函数均匀、冲突少. 最坏情况退化为 \(O(n)\) ( 全部冲突到同一链表 ).

6.6 例题精选

例题 1 ( 折半查找 ASL ) 有序表 \(\{1, 3, 5, 7, 9, 11, 13\}\)，共 7 个元素，求折半查找成功时的 ASL.

解

判定树 ( 二叉查找树 )

括号内为下标 ( 从 1 开始，根 = \(\lfloor(1+7)/2\rfloor = 4\)，值为 7 ).

比较次数 1→第3层，3→第2层，5→第3层，7→第1层，9→第3层，11→第2层，13→第3层.

修正计算 深度 1 有 1 个结点，深度 2 有 2 个，深度 3 有 4 个.

\[\text{ASL} = \frac{1}{7}(1 \times 1 + 2 \times 2 + 3 \times 4) = \frac{1+4+12}{7} = \frac{17}{7} \approx 2.43\]

例题 2 ( 散列表构造与 ASL )

关键字序列 \(\{19, 14, 23, 1, 68, 20, 84, 27, 55, 11, 10, 79\}\)，散列函数 \(H(k) = k \bmod 13\)，线性探测解决冲突，表长 \(m = 13\). 求查找成功的 ASL.

解

计算各关键字散列地址及探测次数

关键字 \(H(k)\) 实际存储位置 比较次数
19 6 6 1
14 1 1 1
23 10 10 1
1 1 2 ( 冲突 ) 2
68 3 3 1
20 7 7 1
84 7 8 ( 冲突 ) 2
27 1 4 ( 冲突 2 次 ) 3
55 3 5 ( 冲突 ) 3
11 11 11 1
10 10 12 ( 冲突 ) 3
79 1 9 ( 冲突多次 ) 9

\[\text{ASL}_{\text{成功}} = \frac{1}{12}(1+1+1+2+1+1+2+3+3+1+3+9) = \frac{28}{12} \approx 2.33\]

例题 3 ( B 树高度 )

一棵 5 阶 B 树，共有 300 个关键字，问该树的最大高度和最小高度分别是多少 ?

解

最大高度 ( 每个结点关键字数最少，树尽可能高 )

\(\lceil m/2 \rceil - 1 = \lceil 5/2 \rceil - 1 = 2\) ( 非根结点最少关键字数 )，根至少 1 个.

高度 \(h\) 时最少关键字数 \(n \geq 1 + 2 \times (2 + 1) \times \ldots = 2\lceil m/2\rceil^{h-1} - 1 = 2 \times 3^{h-1} - 1\).

令 \(2 \times 3^{h-1} - 1 \leq 300\)，解得 \(3^{h-1} \leq 150.5\)，\(h-1 \leq 4\)，\(h \leq 5\).

最小高度 ( 每个结点关键字数最多，树尽可能矮 )

每个结点最多 \(m-1 = 4\) 个关键字，高度 \(h\) 最多关键字数 \(5^h - 1 \geq 300\)，解得 \(h \geq \log_5 301 \approx 3.54\)，即 \(h_{\min} = 4\).

所以最大高度 = 5，最小高度 = 4.

例题 4 ( 分块查找 )

有 900 个元素分为 30 块，每块 30 个. 块间有序，块内无序. 索引表用顺序查找，求查找成功时的 ASL.

解

\[\text{ASL} = \frac{b+1}{2} + \frac{s+1}{2} = \frac{30+1}{2} + \frac{30+1}{2} = 15.5 + 15.5 = 31\]

6.7 本章疑难点

1. ASL 的计算基准

  • ASL 以 比较次数 为基准，不同算法的”一次比较”含义可能不同 ( 如折半查找每次与中间元素比较，可能等于、大于、小于三路 ).
  • 务必区分 查找成功 和 查找失败 的 ASL，两者计算方式不同.

2. 折半查找判定树的失败结点

  • 失败结点 ( 方形 ) 有 \(n+1\) 个，表示 \(n+1\) 个查找失败区间.
  • 失败结点的比较次数 = 其父结点所在层数 ( 即与父结点比较后判断失败 ).

3. B 树 vs B+ 树核心区别

  • B 树数据在 所有结点，B+ 树数据 仅在叶结点，内部结点是纯索引.
  • B+ 树叶结点有 顺序链表，支持高效范围查询，这是数据库选择 B+ 树的最重要原因.

4. 开放定址法删除必须用墓碑

  • 直接置空会断链，导致之后的查找失败 ( 本应存在的元素找不到 ).
  • 墓碑标记的槽在插入时可以复用，但查找时需跳过.

5. 装填因子 \(\alpha\) 的影响

  • \(\alpha\) 越大，冲突越多，ASL 越高. 通常 \(\alpha > 0.8\) 时性能急剧下降.
  • 链地址法中 \(\alpha\) 可以大于 1 ( 链表可以很长 )，但不推荐.

6. 红黑树的插入默认红色

  • 新结点默认红色，因为插入红结点只可能违反性质 4 ( 红红相邻 )，而插入黑结点必然违反性质 5 ( 黑高不均 )，修复代价更大.

6.8 本章小结

  • 静态查找的效率从低到高 顺序 \(O(n)\) → 分块 \(O(\sqrt{n})\) → 折半 \(O(\log n)\)，折半要求有序顺序表.
  • 动态查找树形结构 BST 平均 \(O(\log n)\) 但最坏退化，AVL 保证 \(O(\log n)\)，红黑树适合频繁增删.
  • B/B+ 树适用于外存，最小化磁盘 I/O. B+ 树因叶链表支持范围查询，成为数据库索引首选.
  • 散列表理想 \(O(1)\)，性能依赖散列函数和冲突处理. 除留余数法 + 链地址法是最常见组合.
  • ASL 是衡量所有查找算法的统一指标，考试必须熟练计算各方法的成功 / 失败 ASL.
