第2章 数据的表示和运算

2.1 数制与编码

2.1.1 进位计数制及其相互转换

进位计数制的核心概念

在进位计数制中，每个数位所代表的数值等于该位的数字乘以一个固定的常数，这个常数称为 位权.

设基数为 \(r\) ( \(r\) 进制 )，则数 \(N\) 可表示为 \[ N = \sum_{i=-m}^{n} k_i \times r^i \]

  • 基数 ( Radix ) : 每个数位上允许使用的数码个数 ( 如十进制为 0~9，基数为 10 ).
  • 位权 ( Weight ) : 数码在不同位置上的倍率值 ( 如十进制个位权 \(10^0\)，十位权 \(10^1\) ).

常见进制对照表

十进制 ( Decimal ) 二进制 ( Binary ) 八进制 ( Octal ) 十六进制 ( Hex )
0 0000 0 0

进制转换方法

( 1 ) 任意进制 \(\to\) 十进制

方法 : 按权展开相加法. 将每一位的数码乘以其对应的位权，然后相加.

例 : 将二进制数 \(1011.01_2\) 转换为十进制 \[ \begin{aligned} 1011.01_2 &= 1\times 2^3 + 0\times 2^2 + 1\times 2^1 + 1\times 2^0 + 0\times 2^{-1} + 1\times 2^{-2} \\ &= 8 + 0 + 2 + 1 + 0 + 0.25 \\ &= 11.25_{10} \end{aligned} \]

( 2 ) 十进制 \(\to\) 任意进制

方法 : 整数部分除基取余，小数部分乘基取整.

  • 整数部分 : 除以基数 \(r\)，取余数，直到商为 0，逆序 排列余数.
  • 小数部分 : 乘以基数 \(r\)，取整数，直到小数部分为 0 或达到精度要求，顺序 排列整数.

例 : 将 \(13.625_{10}\) 转换为二进制

  1. 整数部分 ( 13 ) :
    • \(13 \div 2 = 6 \dots\dots 1\) ( LSB )
    • \(6 \div 2 = 3 \dots\dots 0\)
    • \(3 \div 2 = 1 \dots\dots 1\)
    • \(1 \div 2 = 0 \dots\dots 1\) ( MSB )
    • 逆序结果 : \(1101_2\)
  2. 小数部分 ( 0.625 ) :
    • \(0.625 \times 2 = 1.25 \to\) 取整 1
    • \(0.25 \times 2 = 0.5 \to\) 取整 0
    • \(0.5 \times 2 = 1.0 \to\) 取整 1
    • 顺序结果 : \(0.101_2\)

最终结果 : \(13.625_{10} = 1101.101_2\)

( 3 ) 二进制 \(\leftrightarrow\) 八进制/十六进制

利用 \(2^3=8\) 和 \(2^4=16\) 的关系，可以进行快速分组转换.

  • 二转八 : 以小数点为界，向左/右每 3位 一组，不足补 0.
  • 二转十六 : 以小数点为界，向左/右每 4位 一组，不足补 0.

例 : 将二进制 \(11010110_2\) 转换为十六进制

  • 分组 : \(\underbrace{1101}_{D} \quad \underbrace{0110}_{6}\)
  • 结果 : \(D6_{16}\)

2.1.2 定点数的编码表示

定点数的概念

定点数是指 小数点位置固定不变 的数. 在计算机中，小数点通常固定在数值位的最高位之前 ( 定点小数 ) 或最低位之后 ( 定点整数 ).

  • 定点整数 : 小数点隐含在最低位之后，表示纯整数.
  • 定点小数 : 小数点隐含在符号位之后、数值位之前，表示纯小数.

例 : 定点整数 -5 ( 8 位 )，补码为 \(11111011\); 定点小数 -0.25 ( 8 位 )，补码为 \(1.1100000\).

真值与机器数

  • 真值 ( True Value ) : 带有 “正、负” 号的数，是机器数所代表的实际数值.
    • 例 : \(+15\), \(-8\), \(+1011\), \(-1100\).
  • 机器数 ( Machine Number ) : 把符号 “数字化” 的数，是数字在计算机中的二进制表示形式. 通常用 0 表示正，1 表示负.
    • 例 : \(01111\) ( 表示 \(+15\) ), \(11000\) ( 表示 \(-8\) ).

四种常见编码方式

( 1 ) 原码 ( True Form )
  • 定义 : 用最高位表示符号 ( 0 正 1 负 )，其余位表示数值的绝对值.
  • 特点 : 直观，易于人理解; 但 0 有两种表示 ( \(+0\) 和 \(-0\) )，加减运算复杂.
  • 范围 ( \(n+1\)位 ) :
    • 整数 : \(-(2^n-1) \sim 2^n-1\)
    • 小数 : \(-(1-2^{-n}) \sim 1-2^{-n}\)
( 2 ) 反码 ( Ones’ Complement )
  • 定义 : 正数的反码与原码相同; 负数 的反码是在原码基础上，符号位不变，数值位 按位取反.
  • 特点 : 0 仍有两种表示; 主要用于原码转补码的中间过渡.
  • 范围 : 同原码.
( 3 ) 补码 ( Two’s Complement )
  • 定义 : 正数的补码与原码相同; 负数 的补码是其反码末位加 1.
  • 特点 :
    • 0 有唯一表示 ( \([+0]_补 = [-0]_补 = 00\dots0\) ).
    • 能多表示一个最小负数 ( 如 8 位表示 \(-128\) ).
    • 符号位参与运算，减法可转换为加法.
  • 范围 ( \(n+1\)位 ) :
    • 整数 : \(-2^n \sim 2^n-1\)
    • 小数 : \(-1 \sim 1-2^{-n}\)
( 4 ) 移码 ( Excess Code )
  • 定义 : 在补码的基础上，将符号位 取反. 通常用于表示浮点数的 阶码.
  • 特点 : 保持了数据的大小顺序，便于比较大小; 0 有唯一表示.
  • 范围 : 同补码.

编码表示对比表 ( 以 8 位整数为例 )

编码类型 定义简述 正数 (+5) 负数 (-5) 零的表示 表示范围 ( 8位 )
原码 符号位+绝对值 \(00000101\) \(10000101\) \(\pm 0\) 不同 \(-127 \sim 127\)
反码 负数符号位不变，数值取反 \(00000101\) \(11111010\) \(\pm 0\) 不同 \(-127 \sim 127\)
补码 负数反码+1 \(00000101\) \(11111011\) 唯一 \(-128 \sim 127\)

注

  • 补码 是现代计算机存储整数的通用格式，其核心优势是消除了原码/反码中 +0 和 -0 的重复编码，因此能多表示一位负数 ( 如8位补码可表示-128，原码/反码仅能表示到-127 ).
  • 移码 主要用于浮点数的阶码部分，方便比较大小.
  • 特殊值 -128 : 8 位原码和反码无法表示 ( 原码/反码因存在重复的0编码，负数范围仅到-127 ); 8位补码中该值的编码为 \(10000000\)，是补码多出来的唯一负数.

2.1.3 整数的表示

无符号整数 ( Unsigned Integer )

  • 定义 : 整个寄存器的位数全部用于表示数值，没有符号位. 通常用于表示内存地址、计数等非负数.
  • 编码方式 : 直接使用二进制真值表示.
  • 表示范围 ( \(n\)位 ) : \[0 \sim 2^n - 1\]
    • 8位 : \(0 \sim 255\)
    • 16位 : \(0 \sim 65,535\)

有符号整数 ( Signed Integer )

  • 定义 : 最高位为 符号位 ( \(0\) 正，\(1\) 负 )，其余位为数值位.
  • 编码方式 : 现代计算机普遍采用 补码 表示.
  • 表示范围 ( \(n\) 位 ) :
    \[-2^{n-1} \sim 2^{n-1} - 1\]
    • 8位 : \(-128 \sim 127\)
    • 16位 : \(-32,768 \sim 32,767\)

为什么补码负数范围比正数多1? 因为补码中 0只有一种表示 ( \(00\dots0\) )，节省下来的 \(10\dots0\) ( 原码的 \(-0\) ) 被用来表示最小负数 \(-2^{n-1}\). 例如 8位补码

  • \(00000000 \to 0\)
  • \(01111111 \to +127\)
  • \(10000000 \to -128\) ( 人为定义，无对应的原码/反码 )

2.1.4 C语言中的整数类型及类型转换

C语言整数类型按”字节数”区分，类型转换分为 符号扩展 ( 有符号数 ) 和 零扩展 ( 无符号数 ).

类型转换示例

  1. 有符号数 \(\to\) 更长位数 ( 符号扩展 )
    • 规则 : 保持数值不变，用 符号位 填充新增的高位.
    • 例 : short x = -16 ( 16位补码 0xFFF0 ) \(\to\) int y = x ( 32位 ).
      • 结果 : y 的机器数为 0xFFFFFFF0，真值仍为 \(-16\).
  2. 无符号数 \(\to\) 更长位数 ( 零扩展 )
    • 规则 : 保持数值不变，用 0 填充新增的高位.
    • 例 : unsigned short x = 65520 ( 16位 0xFFF0 ) \(\to\) unsigned int y = x ( 32位 ).
      • 结果 : y 的机器数为 0x0000FFF0，真值仍为 \(65520\).
  3. 长位数 \(\to\) 短位数 ( 截断 )
    • 规则 : 直接 丢弃高位，保留低位. 可能导致数值改变.
    • 例 : int x = 0x000100F0 ( 十进制 \(65776\) ) \(\to\) short y = (short)x.
      • 结果 : y 保留低16位 0x00F0，真值为 \(240\).
  4. 有符号数 \(\leftrightarrow\) 无符号数 ( 同位数 )
    • 规则 : 位模式不变，仅改变解释方式.
    • 例 : short x = -1 ( 补码 0xFFFF ) \(\to\) unsigned short y = (unsigned short)x.
      • 结果 : y 的机器数仍为 0xFFFF，但被解释为无符号数，真值变为 \(65535\).

2.2 运算方法和运算电路

2.2.1 基本运算部件

计算机的运算核心是 ALU ( 算术逻辑单元 )，其基础是 加法器.

一位全加器 ( FA )

全加器 ( Full Adder ) 是实现两个1位二进制数及来自低位的进位输入相加的逻辑电路.

  • 输入 : 加数 \(A_i\)、加数 \(B_i\)、低位进位 \(C_{i-1}\).
  • 输出 : 本位和 \(S_i\)、向高位进位 \(C_i\).
  • 逻辑方程 : \[S_i = A_i \oplus B_i \oplus C_{i-1}\] \[C_i = A_i B_i + (A_i \oplus B_i)C_{i-1}\]

串行进位加法器 ( Ripple Carry Adder )

  • 原理 : 将 \(n\) 个全加器串联，进位信号从低位向高位逐级传递.
  • 特点 : 结构简单，但高位的计算必须等待低位进位产生，延迟大 ( 延迟与位数 \(n\) 成正比 ).

并行进位加法器 ( Look-ahead Carry Adder )

  • 原理 : 利用 超前进位 ( CLA ) 逻辑，同时生成各位的进位信号，不依赖低位运算结果.
  • 特点 : 运算速度快，延迟小 ( 延迟恒定 )，但电路结构复杂.

带标志加法器

  • 功能 : 在进行运算 ( 如加/减 ) 时，除了输出结果 \(F\)，还能生成 标志信息 ( PSW 状态字 ).
  • 四大标志位 :
    • OF ( Overflow ) : 有符号数 溢出判断. \(OF=1\) 表示溢出.
      • 逻辑 : \(OF = C_n \oplus C_{n-1}\) ( 最高位进位 \(\oplus\) 次高位进位 ).
    • SF ( Sign ) : 结果符号. 即结果的最高位， \(SF = F_{n-1}\).
    • ZF ( Zero ) : 零标志. 结果全为0时 \(ZF=1\).
    • CF ( Carry ) : 无符号数 进位/借位. \(CF = C_{out} \oplus C_{in}\) ( 进位输出 \(\oplus\) 加减控制信号 ).
      • 加法 (\(C_{in}=0\)): \(CF = C_{out}\).
      • 减法 (\(C_{in}=1\)): \(CF = C_{out} \oplus 1\) ( 即借位 ).

算术逻辑单元 ( ALU )

  • 功能 : 核心部件，能进行算术运算 ( 加减乘除 ) 和逻辑运算 ( 与或非异或 ).
  • 结构 : 由加法器、多路选择器 ( MUX )、累加器等组成，通过 操作控制信号 ( Opcode ) 选择具体功能.

并行加法器示例 ( 4 位 )

计算 \(3+5\) ( 二进制 \(0011+0101\) ) :

  • 串行进位 : 第1位算完进位传给第2位…直到第4位，耗时较长.
  • 并行进位 : 电路直接根据 \(A(0011)\) 和 \(B(0101)\) 的输入，瞬间 计算出所有位的进位，一次性输出结果 \(1000\) (8).

2.2.2 定点数的移位运算

移位运算通过改变数据位的位置来改变数值大小或调整数据结构，主要分为 算术移位、逻辑移位 和 循环移位.

算术移位 ( Arithmetic Shift )

  • 对象 : 有符号数. 移位过程中 符号位保持不变.
  • 规则 : 不同编码方式的填充规则不同 ( 重点 ).
编码方式 符号位 左移 ( \(\times 2\) ) 右移 ( \(\div 2\) ) 备注
原码 不变 补 \(0\) 补 \(0\) 无论正负，空位都补 \(0\)
反码 不变 补 \(\phi\) 补 \(\phi\) 正数补 \(0\)，负数补 \(1\) ( 与符号位一致 )
补码 不变 补 \(0\) 补 \(\phi\) 左移补 0，右移补符号位

注 \(\phi\) 表示符号位的值 ( 正数为 0，负数为 1 ).

  • 溢出风险 :
    • 左移 : 若最高有效位被移出 ( 如正数高位移出1，负数高位移出0 )，则发生 溢出，结果出错.
    • 右移 : 可能会丢失精度 ( 低位被移出 ).
  • 示例 ( 8位补码 ) : 以 \(-5\) ( \(11111011\) ) 为例.
    • 右移1位 ( \(\div 2\) ) : 符号位是1，高位补1 → \(11111101\) ( \(-3\), 精度丢失 ).
    • 左移1位 ( \(\times 2\) ) : 低位补0 → \(11110110\) ( \(-10\) ).

逻辑移位 ( Logical Shift )

  • 对象 : 无符号数.
  • 规则 : 整体移位，无论左移右移，空位一律补 \(0\).
  • 应用 : 比如 RGB 颜色值的提取、位掩码操作.
    • 例 : \(11111011\) 右移1位 → \(01111101\).

循环移位 ( Cyclic Shift )

  • 功能 : 移出的位不丢失，而是补到另一端. 适合 高低字节互换 ( 大小端转换 ) 或 加密算法.
  • 分类 :
    • 不带进位 ( CF ) : 移出的位直接填补空位.
    • 带进位 ( CF ) : 移出的位进入进位标志位 CF，原 CF 的值填补空位 ( 大循环 ).

2.2.3 定点数的加减运算 ( 补码 )

补码运算的核心优势在于 减法变加法，且符号位参与运算，硬件电路实现简单.

运算公式

  • 加法 : \([A+B]_{\text{补}} = [A]_{\text{补}} + [B]_{\text{补}} \pmod {2^{n+1}}\)
  • 减法 : \([A-B]_{\text{补}} = [A]_{\text{补}} + [-B]_{\text{补}} \pmod {2^{n+1}}\)
    • 求 \([-B]_{\text{补}}\) 的方法 : 将 \([B]_{\text{补}}\) 连同符号位 “按位取反，末位加1”.

加减运算电路 ( Hardware Implementation )

计算机内部使用 同一套电路 完成加法和减法，关键在于引入 控制信号 和 XOR 门.

  • 核心组件 : \(n\) 位加法器 ( Adder ) + \(n\) 个异或门 ( XOR ).
  • 控制信号 ( \(Sub\) ) :
    • \(Sub = 0\) : 执行 加法.
    • \(Sub = 1\) : 执行 减法.
  • 工作原理 :
    • 输入数据 \(Y\) 经过异或门 : \(Y_{out} = Y_{in} \oplus Sub\).
    • 加法 ( \(Sub=0\) ) :
      • \(Y \oplus 0 = Y\) ( 原始值 ).
      • 进位输入 \(C_{in} = Sub = 0\).
      • 结果 : \(X + Y + 0 = X + Y\).
    • 减法 ( \(Sub=1\) ) :
      • \(Y \oplus 1 = \overline{Y}\) ( 按位取反 ).
      • 进位输入 \(C_{in} = Sub = 1\) ( 末位加 1 ).
      • 结果 : \(X + \overline{Y} + 1 = X + [-Y]_{\text{补}} = X - Y\).

溢出判断 ( Overflow Check )

溢出 指运算结果超出了机器数所能表示的范围 ( 如8位补码范围 \(-128 \sim +127\) ).

方法 判据 说明
一位符号位法 \(V = A_S B_S \overline{S_S} + \overline{A_S} \overline{B_S} S_S\) 同号相加，结果异号 则溢出.
( 正+正=负 \(\rightarrow\) 上溢; 负+负=正 \(\rightarrow\) 下溢 )
进位判断法 \(V = C_{sign} \oplus C_{data}\) 符号位进位 \(\oplus\) 最高数值位进位 \(= 1\) 则溢出.
双符号位法 \(S_1 S_0\) ( 模 4 补码 ) \(00\) : 正无溢出; \(11\) : 负无溢出;
\(01\) : 正溢出 ( 上溢 ); \(10\) : 负溢出 ( 下溢 ).

注

  • \(A_S\) : 操作数 A 的符号位 ( Sign bit of A ). 在补码表示中，符号位为 0 表示正数，1 表示负数.
  • \(B_S\) : 操作数 B 的符号位 ( Sign bit of B ) 同上.
  • \(S_S\) : 加法结果 S 的符号位 ( Sign bit of the sum S ).
  • \(\overline{A_S}\) : \(A_S\) 的逻辑非 ( NOT )，即符号位的取反 ( 0 变 1，1 变 0 ).
  • \(\overline{B_S}\) : \(B_S\) 的逻辑非 ( NOT ).
  • \(\overline{S_S}\) : \(S_S\) 的逻辑非 ( NOT ).
  • \(V\) : 溢出标志 ( Overflow flag ). V = 1 表示发生溢出，V = 0 表示无溢出.
  • \(C_{sign}\) : 符号位的进位 ( Carry into the sign bit )，即从最高数值位 ( 最高数据位 ) 产生的进位，进入符号位.
  • \(C_{data}\) : 最高数值位的进位 ( Carry from the highest data bit )，有时也称为从次高位进入最高数值位的进位 ( 视具体实现而定 ). 在进位判断法中，通常指与符号位相关的两个相邻进位信号.
  • \(\oplus\) : 异或运算 ( XOR ). 如果两个操作数不同，结果为 1; 相同则为 0.
  • \(S_1 S_0\) : 双符号位 ( Double sign bits )，使用模 4 补码表示 ( Modified two’s complement ). 其中 \(S_1\) 是高符号位，\(S_0\) 是低符号位，用于更精确地检测溢出类型.
    • \(00\) : 正无溢出.
    • \(11\) : 负无溢出.
    • \(01\) : 正溢出 ( 上溢 ).
    • \(10\) : 负溢出 ( 下溢 ).

双符号位法存储时仅需 1 位，运算时才扩展为 2 位.

  1. 运算示例 例1 : 加法 \(3 + (-2)\) ( 8位 )
  • \([3]_{\text{补}} = 00000011\)，\([-2]_{\text{补}} = 11111110\).
  • 相加 : \[ \begin{array}{r} 00000011 \\ +\ 11111110 \\ \hline 100000001 \end{array} \]
  • 结果 : 舍弃最高进位，得 \(00000001\) ( \(+1\) ). 无溢出.

例2 : 减法与溢出 \(127 - (-1)\)

  • 相当于 \(127 + 1\). \([127]_{\text{补}} = 01111111\)，\([-(-1)]_{\text{补}} = [1]_{\text{补}} = 00000001\).
  • 相加 : \[ \begin{array}{r} 01111111 \\ +\ 00000001 \\ \hline 10000000 \end{array} \]
  • 结果 : \(10000000\).
  • 判断 :
    • 符号位法 : “正+正=负” (\(0+0=1\))，故发生 正溢出.
    • 双符号位法 : 扩展为 \(00\ 111... + 00\ 000... = 01\ 000...\) ( \(01\) 表示正溢出 ).

    • 进位位 C、寄存器 P、寄存器 Y 同步逻辑右移 :
      • 进位 C 移入 P 的最高位;
      • Y 的最低位移出，送控制逻辑判断;
    • 计数器 \(C_n\) 减 1，直到 \(C_n = 0\).

分”有符号数 ( int 型 )“和”无符号数 ( unsigned int 型 )“两种场景 :

  • 有符号数 ( int 型 ) 若乘积高 32 位的每一位都相同，且等于乘积低 32 位的符号位 → 不溢出; 否则溢出.
  • 无符号数 ( unsigned int 型 ) 若乘积的高 32 位全为 0 → 不溢出; 否则溢出.

除法运算是乘法的逆运算，用于求解 \(X \div Y = Q \cdots R\) ( 商 Q 和余数 R ). 定点除法主要分为 恢复余数法 和 加减交替法 ( 不恢复余数法 ).

  1. 初始化 :
    • 余数寄存器 R ← 被除数 \(|X|\)
    • 除数寄存器 Y ← \(|Y|\)
    • 商寄存器 Q ← 0
    • 计数器 \(C_n\) ← n ( 操作数位数 )
  2. 迭代过程 ( 重复 n 次 ) :
    • 试减 : \(R \leftarrow R - |Y|\)
    • 判断 :
      • 若 \(R \geq 0\) : 商上 1，\(R\) 左移 1 位
      • 若 \(R < 0\) : 商上 0，恢复余数 \(R \leftarrow R + |Y|\)，然后左移 1 位
    • 计数器减 1
  3. 确定符号 :
    • 商的符号 : \(Q_s = X_s \oplus Y_s\)
    • 余数的符号 : \(R_s = X_s\) ( 余数符号与被除数相同 )
  4. 输出结果 : 商 Q 和余数 R
加减交替法 ( 不恢复余数法 )

核心优化

  • 当余数为负时，不立即恢复 ( 加回 \(|Y|\) )，而是在下一次左移后加 \(|Y|\)
  • 数学等价性 : \(2(R + |Y|) - |Y| = 2R + |Y|\)
  • 优势 : 减少一次加法操作，提高效率

上商规则 :

  • 余数与除数 同号 → 商上 1，执行减法 ( \(R - |Y|\) )
  • 余数与除数 异号 → 商上 0，执行加法 ( \(R + |Y|\) )

2.3.1 浮点数的基本格式

浮点数的通用表示形式

\[N = (-1)^{\color{ #c92a2a}{S}} \times {\color{ #2f9e44}{M}} \times {\color{ #f08c00}{R}}^{\color{ #1971c2}{E}}\]

其中

  • S ( Sign ) : 符号位，0 表示正数，1 表示负数.
  • M ( Mantissa/Significand ) : 尾数 ( 有效数字部分 )，表示 精度.
  • R ( Radix ) : 基数 ( 隐含 )，二进制浮点数中为 2.
  • E ( Exponent ) : 阶码 ( 指数部分 )，表示 数值范围.

为什么叫”浮点”?

因为小数点的位置可以”浮动” :

  • \(1.234 \times 10^2 = 123.4\) ( 小数点右移 )
  • \(1.234 \times 10^{-2} = 0.01234\) ( 小数点左移 )

通过调整指数 \(E\)，小数点可以在数值中移动，从而表示不同量级的数.

规格化浮点数的特点

规格化 ( Normalized ) 表示

浮点数采用科学计数法，要求尾数 \(M\) 满足 \[1 \leq M < 2\]

即二进制形式为 \[M = 1.××××××_2\]

隐含位技术

  • 由于规格化后最高位必为 1，这个 1 不需要存储
  • 23 位尾数实际表示 24 位精度 ( 隐含的 1 + 23 位存储 )
  • 这样可以多获得 1 位精度

示例

  • 存储 : 10010000000000000000000 ( 23 位 )
  • 实际值 : 1.10010000000000000000000 ( 24 位 )

补充 : 原码与补码的规格化对比

补码规格化的核心原则

补码规格化的标志是：符号位与数值最高位不同.

  • 为什么?
    • 补码正数 \(0.0...\) 是非规格化，需左移直到 \(0.1...\) ( 同原码 )
    • 补码负数 \(1.1...\) ( 绝对值较小 ) 是非规格化，需左移直到 \(1.0...\) ( 绝对值变大 )
  • 特例 : \([-1]_{补} = 1.00...0\) 是规格化数 ( 满足 \(1.0\) 开头 )，且没有对应的原码规格化形式 ( 原码无法表示 -1 ).
  • 注意 : \([-1/2]_{补} = 1.10...0\) 不满足 符号位与数值位不同，因此在补码规则下通常视为 非规格化数，需左移一位变成 \(1.00...0\) ( 即 -1 ).

阶码的移码表示

阶码 \(E\) 采用 移码 ( Biased Representation ) 表示，而 非补码.

单精度 :

  • 偏移值 ( Bias ) = 127
  • 存储值 = 实际指数 + 127
  • 表示范围 : -126 ~ +127 ( 实际 ) → 1 ~ 254 ( 存储 )

为什么用移码?

  1. 便于比较大小 ( 移码天然有序 )
  2. 避免使用符号位
  3. 简化硬件比较电路

补充 : 整数移码 vs IEEE 754 移码

在定点整数中，移码通常用于表示有符号数，其偏置值为 \(2^{n-1}\); 而在 IEEE 754 浮点数标准中，为了更均衡地表示指数范围，偏置值取 \(2^{n-1} - 1\).

为什么 IEEE 754 偏置值是 127 而不是 128?

主要目的是为了 扩大正指数的范围，从而能表示更大的数值.

  • 若 Bias = 128 : 有效指数范围为 \(-127 \sim +126\).
  • 若 Bias = 127 : 有效指数范围为 \(-126 \sim +127\).

显然 \(+127\) 比 \(+126\) 大，能表示的绝对值更大的数 ( 虽然牺牲了一个负指数位，但通常大数的范围更重要 ).

非规格化数的计算与示例

当阶码 \(E\) 为全 0 ( 即 \(E=0\) ) 且尾数 \(M\) 不为 0 时，浮点数处于 非规格化 ( Denormalized ) 状态. 此时为了平滑过渡到 0，规定 :

  1. 阶码隐含值 : \(E_{real} = 1 - \text{Bias}\) ( 而不是 \(0 - \text{Bias}\) ).
    • 单精度 : \(1 - 127 = -126\).
    • 双精度 : \(1 - 1023 = -1022\).
  2. 尾数隐含位 : 隐含整数位为 0 ( 即 \(0.M\) ).

真值公式 : \[V = (-1)^S \times (0.M) \times 2^{1-\text{Bias}}\]

为什么要用 \(1-\text{Bias}\)?

为了实现 平滑下溢 ( Gradual Underflow ).

  • 最小的 规格化数 (\(E=1\)) : \(1.0 \times 2^{1-127} = 1.0 \times 2^{-126}\).
  • 最大的 非规格化数 (\(E=0, M=11...1\)) : \(0.11...1 \times 2^{-126} \approx 1.0 \times 2^{-126}\). 两者实现了无缝衔接.

计算示例 若单精度浮点数机器码为 0 00000000 10000000000000000000000 ( 十六进制 0x00400000 ).

  • S = 0 ( 正数 ).
  • E = 0 ( 全0，非规格化 ).
  • M = \(100...0\) ( 二进制 \(0.1_2 = 0.5\) ).
  • 计算 : \[V = +0.5 \times 2^{-126} = 2^{-1} \times 2^{-126} = 2^{-127}\]

溢出与下溢
  • 上溢 ( Overflow ) :
    • 正上溢 ( Positive Overflow ) : 数值大于最大正数 ( \(> MAX_{float}\) )，结果记为 \(+\infty\).
    • 负上溢 ( Negative Overflow ) : 数值小于最小负数 ( \(< MIN_{float}\) )，结果记为 \(-\infty\).
    • 后果 : 通常视为异常或错误.
  • 下溢 ( Underflow ) :
    • 数值绝对值太小，介于最大负数和最小正数之间 ( 即 \(( -MIN_{norm}, +MIN_{norm} )\) 之间，不含 0 ).
    • 处理 :
      • 结果记为 \(0\) ( 机器零 ).
      • 或者使用 非规格化数 ( Denormalized Number ) 提高精度.
    • 后果 : 精度丢失，但通常不报错.

注 计算机中的”下溢”通常指 绝对值过小 ( 趋近于 0 )，而非”数值过小” ( 趋近于负无穷 ). 趋近于负无穷属于”负上溢”.

定点数与浮点数的对比

定点数与浮点数是计算机中表示实数的两种基本方法，它们在范围、精度和实现复杂度上各有优劣.

比较维度 定点数 ( Fixed-Point ) 浮点数 ( Floating-Point )
小数点位置 固定不变 ( 隐含在特定位置 ) 浮动 ( 由阶码决定 )
数值范围 较小 ( 受位数限制 ) 很大 ( 受阶码限制 )
精度特性 绝对精度固定 ( 均匀分布 ) 相对精度固定 ( 稀疏分布，数值越大越稀疏 )
运算复杂度 简单 ( 类似整数运算 ) 复杂 ( 需对阶、规格化、舍入 )
溢出处理 容易溢出 ( 报错 ) 不易溢出 ( 上溢出为 \(\infty\)，下溢出为 \(0\) )
硬件成本 低 高 ( 需专门的 FPU )
典型应用 DSP、嵌入式控制、金融计算 ( 精度要求绝对固定 ) 科学计算、图形处理、通用计算机

核心区别总结

  • 定点数 就像一把 普通尺子，刻度间距是固定的，能测量的最大长度有限.
  • 浮点数 就像一把 皮尺，测量小物体时刻度很密 ( 精度高 )，测量大物体时刻度变稀 ( 精度低 )，但能测量的范围非常大.

浮点数加减运算比定点数复杂，硬件上需通过专门的浮点运算单元 ( FPU ) 完成. 核心流程可概括为 5 步法 : 对阶 \(\to\) 尾数加减 \(\to\) 规格化 \(\to\) 舍入 \(\to\) 判溢出.

目的 : 使两个操作数的阶码相同，以便尾数直接相加减.

  • 规则 : 小阶向大阶看齐.
    • 设 \(E_x < E_y\)，则将 \(x\) 的阶码增大至 \(E_y\).
    • 同时 \(x\) 的尾数 右移 ( 每右移一位，阶码加 1 ).
  • 为什么小向大看齐?
    • 小阶尾数右移，丢弃的是低位数据，仅影响精度.
    • 若大阶尾数左移，丢弃的是高位有效数据，会产生巨大误差.

操作 : 对阶后的尾数进行定点加减运算.

  • 注意 :
    • 浮点数尾数通常采用 补码 或 原码 表示.
    • 运算时需包含 隐含的 1 ( IEEE 754 规格化数 ).
    • 结果可能出现 临时溢出 ( 如 1.x + 1.x = 1x.x )，这不代表最终溢出，后续通过右规解决.

目的 : 保证结果符合 \(1.M\) 的格式 ( \(1 \leq |M| < 2\) ).

  • 右规 ( Right Shift ) :
    • 触发条件 : 尾数运算结果绝对值 \(\geq 2\) ( 双符号位不等 ).
    • 操作 : 尾数 右移 1 位，阶码 加 1.
    • 示例 : \(1.1 + 0.1 = 10.0 \xrightarrow{右规\hspace{2em}} 1.00 \times 2^1\).
  • 左规 ( Left Shift ) :
    • 触发条件 : 尾数运算结果绝对值 \(< 1\) ( 最高有效位不是 1 ).
    • 操作 : 尾数 逐位左移，阶码 逐位减 1，直到最高位为 1.
    • 示例 : \(1.001 - 1.000 = 0.001 \xrightarrow{左规\hspace{2em}} 1.0 \times 2^{-3}\).

目的 : 当尾数位数超过寄存器长度时，需舍弃多余位并保证精度.

  • 0 舍 1 入法 : 类似于四舍五入.
    • 舍弃位最高位为 0 \(\to\) 直接舍去.
    • 舍弃位最高位为 1 \(\to\) 尾数末位加 1.
  • 恒置 1 法 :
    • 无论舍弃位是什么，直接将尾数末位恒置为 1.
  • IEEE 754 默认 ( 就近舍入 ) :
    • 舍入到最近的偶数 ( Round to nearest, ties to even ).

判断 : 根据 最终阶码 \(E_{final}\) 判断结果是否合法.

  • 上溢 ( Overflow ) : \(E_{final} > E_{max}\) ( 如单精度 > 127 ).
    • 触发场景 :
      • 右规 时，阶码加 1 导致超过最大值.
      • 两个大数相乘 ( 指数相加 ) 导致阶码过大.
    • 处理 : 抛出异常 ( 硬件中断 ) 或置为 \(\infty\).
  • 下溢 ( Underflow ) : \(E_{final} < E_{min}\) ( 如单精度 < -126 ).
    • 触发场景 :
      • 左规 时，阶码减 1 导致小于最小值.
      • 两个小数相乘 ( 指数相加 ) 导致阶码过小.
    • 处理 : 通常置为 0 ( 机器零 ) 或非规格化数.

注意 只有 右规 可能导致上溢，只有 左规 可能导致下溢. 尾数溢出可以通过右规解决，不算真正的溢出;

阶码上溢才是真正的溢出.

详细运算示例 ( \(1.5 + 2.5\) )

Step 0 : 格式转换

  • \(1.5_{10} = 1.1_2 = 1.1 \times 2^0\)
  • \(2.5_{10} = 10.1_2 = 1.01 \times 2^1\)

Step 1 : 对阶

  • 小阶 ( \(2^0\) ) 向大阶 ( \(2^1\) ) 看齐.
  • \(1.5\) 变为 \(0.11 \times 2^1\) ( 尾数右移 1 位 ).

Step 2 : 尾数相加

  • 结果 : \(10.00 \times 2^1\).

Step 3 : 规格化 ( 右规 )

  • 尾数 \(10.00 \geq 2\)，需要 右规.
  • 尾数右移 1 位 (\(1.000\))，阶码 +1 (\(1+1=2\)).
  • 结果 : \(1.000 \times 2^2\).

Step 4 : 最终结果

  • \(1.000 \times 2^2 = 100_2 = 4.0_{10}\).
  • 检查阶码 \(2\) 在允许范围内，无溢出.

浮点数表示的局限性

精度陷阱与注意事项

  1. 大数吃小数 :
    • 当极大数与极小数相加时 ( 如 \(10^{20} + 1\) )，由于对阶时小数尾数右移过多被截断，导致 小数被忽略.
  2. 不满足结合律 :
    • \((a + b) + c \neq a + (b + c)\).
    • 编写代码时应尽量避免相差巨大的数直接运算.
  3. 十进制小数无法精确表示 :
    • 如 \(0.1\) 在二进制是无限循环小数，存储时必然存在误差.
    • 禁止 使用 if (f == 0.1) 进行等值比较，应使用 if (abs(f - 0.1) < epsilon).
