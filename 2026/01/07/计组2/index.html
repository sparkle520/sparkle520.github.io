2.1.2 定点数的编码表示

1. 定点数的概念

定点数是指 小数点位置固定不变 的数. 在计算机中，小数点通常固定在数值位的最高位之前 ( 定点小数 ) 或最低位之后 ( 定点整数 ).

  • 定点整数 : 小数点隐含在最低位之后，表示纯整数.
  • 定点小数 : 小数点隐含在符号位之后、数值位之前，表示纯小数.

2. 四种常见编码方式

(1) 原码 ( True Form )
  • 定义 : 用最高位表示符号 ( 0 正 1 负 )，其余位表示数值的绝对值.
  • 特点 : 直观，易于人理解; 但 0 有两种表示 ( 和 )，加减运算复杂.
  • 范围 ( 位 ) :
    • 整数 :
    • 小数 :
(2) 反码 ( Ones' Complement )
  • 定义 : 正数的反码与原码相同; 负数 的反码是在原码基础上，符号位不变，数值位 按位取反.
  • 特点 : 0 仍有两种表示; 主要用于原码转补码的中间过渡.
  • 范围 : 同原码.
(3) 补码 ( Two's Complement )
  • 定义 : 正数的补码与原码相同; 负数 的补码是其反码末位加 1.
  • 特点 :
    • 0 有唯一表示 ( ).
    • 能多表示一个最小负数 ( 如 8 位表示 ).
    • 符号位参与运算，减法可转换为加法.
  • 范围 ( 位 ) :
    • 整数 :
    • 小数 :
(4) 移码 ( Excess Code )
  • 定义 : 在补码的基础上，将符号位 取反. 通常用于表示浮点数的 阶码.
  • 特点 : 保持了数据的大小顺序，便于比较大小; 0 有唯一表示.
  • 范围 : 同补码.

编码表示对比表 ( 以8位整数为例 )

编码类型 定义简述 正数 (+5) 负数 (-5) 零的表示 表示范围 ( 8位 )
原码 符号位+绝对值 不同
反码 负数符号位不变，数值取反 不同
补码 负数反码+1 唯一
移码 补码符号位取反 唯一

注

  • 补码 是现代计算机存储整数的通用格式，其核心优势是消除了原码/反码中 +0 和 -0 的重复编码，因此能多表示一位负数 ( 如8位补码可表示-128，原码/反码仅能表示到-127 ).
  • 移码 主要用于浮点数的阶码部分，方便比较大小.
  • 特殊值 -128 : 8 位原码和反码无法表示 ( 原码/反码因存在重复的0编码，负数范围仅到-127 ); 8位补码中该值的编码为 ，是补码多出来的唯一负数.
