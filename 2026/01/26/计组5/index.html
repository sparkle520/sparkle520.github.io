第5章 中央处理器

5.1 CPU的功能和基本结构

5.1.1 CPU的功能

中央处理器 ( CPU ) 是计算机系统的核心部件，负责 获取指令、对指令进行译码 并 执行指令. 其核心功能可概括为以下五点 :

  1. 指令控制 ( Instruction Control )
    • 核心任务 : 程序的顺序控制.
    • 细节 : 严格按照程序规定的顺序，自动地从内存中取出指令并执行. ( 依赖 PC ).
  2. 操作控制 ( Operation Control )
    • 核心任务 : 产生控制信号.
    • 细节 : 将指令的操作码译码后，产生相应的控制信号，管理各部件按序工作.
  3. 时间控制 ( Timing Control )
    • 核心任务 : 时序同步.
    • 细节 : 对各种操作实施严格的时间定时，确保计算机有条不紊地自动工作.
  4. 数据加工 ( Data Processing )
    • 核心任务 : 算术/逻辑运算.
    • 细节 : 利用 ALU 对数据进行算术运算 ( 加减乘除 ) 和逻辑运算 ( 与或非 ).
  5. 中断处理 ( Interrupt Handling )
    • 核心任务 : 异常/中断响应.
    • 细节 : 处理运行过程中出现的异常情况 ( 如除0 ) 和外部设备的请求 ( 如 I/O 完成 ).

5.1.2 CPU的基本结构

CPU 主要由 运算器 和 控制器 两大部分组成，二者通过内部总线连接.

1. 运算器 ( Datapath )

  • 功能 : 数据的加工处理 ( “干活的” ).
  • 核心部件 :
    • ALU : 算术逻辑单元 ( 核心 ).
    • ACC : 累加寄存器.
    • PSW : 状态寄存器 ( 保存运算结果的标志，如 ZF, OF ).
    • GPRs : 通用寄存器组 ( R0~Rn ).
    • 暂存器 : 暂存中间数据.

2. 控制器 ( Control Unit )

  • 功能 : 协调和指挥整个计算机系统的操作 ( “指挥官” ).
  • 核心部件 :
    • PC : 程序计数器 ( 存放下一条指令地址 ).
    • IR : 指令寄存器 ( 存放当前指令 ).
    • ID : 指令译码器 ( 分析指令 ).
    • Timing : 时序发生器 ( 产生节拍信号 ).

5.1.3 CPU的寄存器

寄存器是 CPU 内部的高速存储单元. 根据 “程序员是否能通过指令访问”，通常将其分为两类 :

1. 用户可见寄存器 ( User Visible )

汇编程序员可以使用指令读取或修改这些寄存器的值.

  • 通用寄存器 ( GPRs, R0~Rn )
    • 功能 : 存放操作数、地址或中间结果.
    • 特点 : 数量较多 ( RISC 中可达 32 个 )，减少访存次数，提高效率.
  • 程序计数器 ( PC )
    • 功能 : 存放 下一条 指令的地址.
    • 更新 : 自动增量 ( PC + "1" ) 或由转移指令修改 ( JMP ).
  • 程序状态字寄存器 ( PSW / FLAGS )
    • 功能 : 存放算术运算的标志位和系统状态.
    • 核心标志位 :
      • ZF ( Zero ) : 结果为零标志 ( 1表示结果为0 ).
      • CF ( Carry ) : 进位/借位标志 ( 辅助判断 无符号数 溢出 ).
      • OF ( Overflow ) : 溢出标志 ( 辅助判断 有符号数 溢出 ).
      • SF ( Sign ) : 符号标志 ( 1表示结果为负 ).
  • 累加器 ( ACC )
    • 功能 : 运算器的核心寄存器，用于暂存操作数或运算结果.

2. 用户不可见寄存器 ( User Invisible )

对程序员 透明，由硬件或微指令自动管理，无法通过指令直接访问.

  • 指令寄存器 ( IR ) : 存放 当前正在执行 的指令.
    • 注 : 程序员只能将指令加载到内存，CPU 自动取指入 IR，程序员无法直接读写 IR.
  • 存储器地址寄存器 ( MAR ) : 存放欲访问的存储单元地址.
  • 存储器数据寄存器 ( MDR ) : 存放从内存读出或写入内存的数据.
  • 暂存寄存器 ( Temp ) : 暂存 ALU 的输入/输出数据.

考点速记 : 寄存器可见性辨析

  • 可见 : PC, PSW, ACC, 通用寄存器 ( R0… ), 基址寄存器, 变址寄存器.
  • 不可见 : IR, MAR, MDR, 暂存寄存器, Cache ( 对指令集透明 ). ( 技巧 : 凡是你在汇编语言里能操作的，就是可见的; 完全由硬件自动控制的，就是不可见的 )

字地址 vs 字节地址

在计算 PC 自增或寻址时，必须明确 “按什么编址” :

  • 按字节编址 ( Byte Addressing ) :
    • 地址的最小单位是 字节 ( 8bit ).
    • 若指令字长为 32位 ( 4字节 )，则下一条指令地址为 PC + 4.
    • ( 现代计算机通常采用此方式，如 x86, MIPS ).
  • 按字编址 ( Word Addressing ) :
    • 地址的最小单位是 机器字.
    • 若指令字长 = 机器字长，则下一条指令地址为 PC + 1.

转换关系 : 若机器字长为 \(W\) 字节，则 \[ \text{字地址} = \lfloor \frac{\text{字节地址}}{W} \rfloor \] ( 实际上就是去掉字节地址的低位，如 32位机器去掉低2位 ).

5.2 指令执行过程

5.2.1 指令周期

指令周期 : CPU 从主存取出并执行一条指令所需的 全部时间.

1. 完整的指令周期流程

一个完整的指令周期包含 4 个阶段 ( 机器周期 ) :

\[ \text{指令周期} = \underbrace{\text{取指周期 (FE)}}_{\text{必选}} + \underbrace{\text{间址周期 (IND)}}_{\text{可选}} + \underbrace{\text{执行周期 (EX)}}_{\text{必选}} + \underbrace{\text{中断周期 (INT)}}_{\text{可选}} \]

  • FE ( Fetch ) : 取指令，译码.
  • IND ( Indirect ) : 取有效地址 ( 仅用于间接寻址 ).
  • EX ( Execute ) : 取操作数，执行运算.
  • INT ( Interrupt ) : 响应中断，保存断点.

2. 各周期的标志触发器

为了区别 CPU 当前处于哪个周期，控制器内部通常设置 4 个标志触发器 :

周期名称 符号 触发器状态 任务简述
取指周期 FE FE=1 取指令，PC+1
间址周期 IND IND=1 取有效地址 ( 仅间接寻址时进入 )
执行周期 EX EX=1 取操作数，执行操作
中断周期 INT INT=1 保存断点，转中断服务程序

注意 、 1. FE 和 EX 是必选的 : 任何指令都需要取指和执行. 2. IND 和 INT 是可选的 : 只有间接寻址指令才有 IND，只有开中断且有请求时才有 INT. 3. 时钟周期关系 : 一个指令周期包含若干机器周期，一个机器周期包含若干时钟周期 ( 节拍 ).

5.2.2 指令周期的数据流

数据流是指令在各周期内，数据在寄存器、总线和存储器之间流动的路径. 数据流的流向取决于总线结构 ( 单总线/多总线 )，以下以 单总线结构 为例进行分析.

目标 : 从主存取出指令，并更新 PC 指向下一条指令.

  1. 流程描述 :
    • PC -> Bus -> MAR : 将 PC 中的指令地址送入 MAR.
    • 1 -> R : 向主存发送读命令.
    • M(MAR) -> DataBus -> MDR : 主存根据 MAR 的地址将指令读入 MDR.
    • MDR -> Bus -> IR : 将 MDR 中的指令送入 IR.
    • OP(IR) -> ID : 指令操作码送 ID 译码.
    • (PC) + 1 -> PC : 修改 PC 内容，指向下一条指令 ( 此时可能由专门的加法器或 ALU 完成 ).

常见指令举例 :

  • CLA ( Clear ACC ) :
    • 0 -> ACC ( 累加器清零，不访存 ).

访存目的辨析

虽然各周期都可能访问主存，但目的不同 :

  • 取指周期 : 取 指令 ( 读 ).
  • 间址周期 : 取 有效地址 ( 读 ).
  • 执行周期 : 取 操作数 ( 读 ) 或 存 结果 ( 写 ).
  • 中断周期 : 存 断点 ( 写 ).

5.2.3 指令执行方案

一个指令周期包含若干个机器周期 ( 或时钟周期 )，根据指令串行或并行执行的情况，可分为三种方案：

1. 单指令周期 ( Single-Cycle )

  • 定义 : 所有指令都在 固定的一个时钟周期 内完成.
  • 特点 :
    • 时钟周期 \(\tau\) 取决于 执行时间最长 的那条指令 ( 通常是 Load 指令 ).
    • CPI = 1.
  • 优缺点 : 控制简单，但时钟频率低，资源利用率低 ( 短指令要等长指令 ).

2. 多指令周期 ( Multi-Cycle )

  • 定义 : 将指令划分为若干个子过程，每个子过程占一个时钟周期.
  • 特点 :
    • 不同指令所需的时钟周期数不同 ( 如 ADD 需 3个，Load 需 5个 ).
    • CPI > 1.
  • 优缺点 : 允许各指令按需消耗周期，提高了效率; 但 控制电路复杂 ( 需要保存状态 ).

3. 流水线方案 ( Pipelining )

  • 定义 : 多条指令在不同执行阶段 并行工作.
  • 特点 :
    • 在理想情况下，每个时钟周期完成一条指令.
    • 理想 CPI \(\approx\) 1.
  • 优缺点 : 吞吐率极高 ( 现代 CPU 主流方案 ); 但硬件开销大，需解决冒险问题.

三种方案对比总结

方案 周期长度 CPI 硬件利用率 控制复杂度
单周期 最长指令 1 低 简单
多周期 可变 >1 中 复杂
流水线 最短阶段 \(\approx\) 1 高 最复杂

5.3 数据通路的功能和基本结构

5.3.1 数据通路的功能

数据通路 ( Datapath ) : 实现 CPU 内部各部件间数据传输的路径. ( ALU, 寄存器, 总线 ).

5.3.2 数据通路的组成

  1. 专用数据通路 : 点对点连接 ( 硬件量大，速度快 ).
  2. 总线数据通路 :
    • 单总线 : 所有部件挂在一组总线上 ( 冲突多，需暂存器 ).
    • 多总线 : 双总线、三总线 ( 效率高 ).

5.3.3 数据通路的操作举例 ( 单总线 )

MOV (R0), R1 ( 将 R1 的值写入 R0 指向的内存单元 )

  1. 取指 : PC -> Bus -> MAR, 1->R, Mem -> Bus -> MDR -> IR.
  2. 执行 :
    • R0 -> Bus -> MAR ( 送地址 )
    • R1 -> Bus -> MDR ( 送数据 )
    • 1 -> W ( 写控制 )

5.4 控制器的功能和工作原理

5.4.1 控制器的结构和功能

控制器负责 “解释指令” 和 “产生微操作信号”.

  • 输入 : 指令 OP、标志位 PSW、时钟 CLK.
  • 输出 : 控制信号 ( 如 PC_Out, ALU_Add, Mem_Read ).

5.4.2 硬布线控制器 ( Hardwired )

  • 原理 : 使用 组合逻辑电路 ( 门电路 ) 直接产生控制信号.
  • 逻辑 : \(C_i = f(OP, \text{State}, \text{Time})\).
  • 特点 :
    • 速度快 ( 纯硬件 ).
    • 设计复杂，难以修改 ( 扩充指令需重做电路 ).
    • 应用 : RISC 计算机.

5.4.3 微程序控制器 ( Microprogrammed )

  • 原理 : 仿照程序设计思想，将微操作信号编写成 微指令，存放在 控制存储器 ( CM ) 中.
  • 核心部件 :
    • 控制存储器 ( CM/CS ) : 存放微程序 ( ROM ).
    • 微指令寄存器 ( CMDR/uIR ) : 存放当前微指令.
    • 微地址寄存器 ( CMAR/uAR ) : 存放下一条微指令地址.
  • 特点 :
    • 设计规整，易于修改 ( 改 ROM 即可 ).
    • 速度较慢 ( 需访存 CM ).
    • 应用 : CISC 计算机.

微指令编码方式

  1. 直接编码 : 每一位代表一个控制信号 ( 字长最长，速度最快 ).
  2. 字段直接编码 : 将互斥信号分组编码 ( 缩短字长，需译码 ).
  3. 混合编码.

5.5 异常和中断机制

5.5.1 异常和中断的基本概念

  • 异常 ( Exception ) : CPU 内部 产生的意外 ( 如除0、缺页、非法指令 ). 也称 内中断.
  • 中断 ( Interrupt ) : CPU 外部 产生的请求 ( 如 I/O 完成、时钟中断 ). 也称 外中断.

5.5.2 异常和中断的分类

类别 来源 示例 处理时机
故障 ( Fault ) 指令执行中 缺页、TLB Miss 可恢复，重执当前指令
自陷 ( Trap ) 指令预设 System Call ( 软中断 ) 这里的”异常”是预期的，执行下一条
终止 ( Abort ) 硬件故障 校验错、总线错 不可恢复，终止程序
外部中断 I/O 设备 键盘、打印机 指令周期结束时 响应

5.5.3 异常和中断响应过程

  1. 关中断 : 保存现场期间不允许被干扰.
  2. 保存断点 : 将 PC 和 PSW 压栈.
  3. 引出程序 : 获取中断服务程序入口地址 ( 硬件向量法 / 软件查询法 ).
  4. 执行服务程序.
  5. 恢复现场.
  6. 开中断 & 返回.

5.6 指令流水线

5.6.1 指令流水线的基本概念

将指令执行过程分解为若干个子过程 ( 段 )，每个子过程由专门的硬件并行执行.

  • 五级流水线 ( MIPS ) :
    1. IF ( 取指 )
    2. ID ( 译码/取数 )
    3. EX ( 执行 )
    4. MEM ( 访存 )
    5. WB ( 写回 )

5.6.2 流水线的基本实现

需要 流水线寄存器 ( Pipeline Register ) 在各段之间保存中间结果 ( 如 IF/ID, ID/EX ).

5.6.3 流水线的冒险 ( Hazard )与处理

流水线因相关性而阻塞，导致性能下降.

  1. 结构冒险 ( 资源冲突 )
    • 原因 : 多条指令争用同一硬件 ( 如同时访问内存 ).
    • 对策 : 哈佛结构 ( 指令 Cache 和数据 Cache 分离 ).
  2. 数据冒险 ( Data Hazard )
    • 原因 : 后一条指令依赖前一条的结果 ( RAW ).
    • 对策 :
      • 暂停 ( Stall ) : 插入气泡.
      • 数据旁路 ( Forwarding ) : 将 ALU 结果直接送入下一级 ALU，不经过寄存器堆.
      • 编译优化 : 调整指令顺序.
  3. 控制冒险 ( Control Hazard )
    • 原因 : 转移指令导致 PC 改变，流水线中的预取指令失效.
    • 对策 :
      • 分支预测 ( 静态/动态 ).
      • 延迟槽 ( 总是执行跳转后的那条指令 ).

5.6.4 流水线的性能指标

设指令条数为 \(n\)，级数为 \(k\)，时钟周期为 \(\Delta t\).

  1. 吞吐率 ( TP ) : \(TP = n / T_{total}\).
  2. 加速比 ( S ) : \(S = T_{serial} / T_{pipeline} = \frac{n \cdot k}{k + n - 1} \approx k\) ( 当 \(n \gg k\) ).

5.6.5 高级流水线技术

  1. 超标量 ( Superscalar ) : 每个周期发射多条指令 ( 多个执行部件 ).
  2. 超流水线 ( Superpipeline ) : 细分流水级，提升主频.
  3. 超长指令字 ( VLIW ) : 编译器将多条指令打包成一条长指令.

5.7 多处理器的基本概念

5.7.1 SISD、SIMD、MIMD

基于 Flynn 分类法 ( 指令流 + 数据流 ).

  1. SISD : 单指令流单数据流 ( 传统 PC ).
  2. SIMD : 单指令流多数据流 ( 向量处理器，GPU ). 一个控制器控制多个 ALU 对不同数据做相同操作.
  3. MISD : 多指令流单数据流 ( 极少见 ).
  4. MIMD : 多指令流多数据流 ( 多核处理器, 集群 ).

5.7.2 硬件多线程的基本概念

在单核 CPU 中维护多个线程上下文，快速切换以隐藏延迟.

  1. 细粒度 : 每个时钟周期切换线程.
  2. 粗粒度 : 遇长延迟 ( 如 Cache Miss ) 才切换.
  3. 同时多线程 ( SMT ) : 超标量 + 多线程 ( 如 Intel Hyper-Threading ). 在同一周期内发射不同线程的指令.

5.7.3 多核处理器的基本概念

将多个完整的 CPU 核心集成在同一芯片上 ( CMP ). 共享 Cache ( 通常 L3 共享 ).

5.7.4 共享内存多处理器 ( SMP )

所有处理器共享同一物理内存，访问延迟相同 ( UMA ). 需解决 Cache 一致性 问题 ( MESI 协议 ).

5.8 本章小结

本章是计组的核心，重点掌握 CPU 结构、指令执行数据流、微程序设计、流水线冒险 及 性能计算.

5.9 常见问题和易混淆知识点

1. 指令周期、机器周期、时钟周期

  • 指令周期 : 执行一条指令的时间. ( 包含多个机器周期 ).
  • 机器周期 ( CPU 周期 ) : 完成一个基本操作 ( 如取指、访存 ) 的时间. ( 包含多个时钟周期 ).
  • 时钟周期 ( 节拍 ) : CPU 主频的倒数，最基本的时间单位. \[ \text{指令周期} > \text{机器周期} > \text{时钟周期} \]

2. 异常 vs 中断

  • 异常 : 同步的，源自 CPU 内部 ( 比如除 0 )，与当前指令有关.
  • 中断 : 异步的，源自 CPU 外部 ( 比如鼠标点击 )，与当前指令无关.
