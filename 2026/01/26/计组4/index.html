第4章 指令系统

4.1 指令系统

4.1.1 指令集体系结构 ( ISA )

指令集体系结构 ( ISA, Instruction Set Architecture ) 是软件和硬件之间的界面 ( Interface ).

  • 软件视角 : 程序员 ( 或编译器 ) 能看到的计算机属性 ( 指令格式、寻址方式、寄存器、中断机制等 ).
  • 硬件视角 : 处理器必须实现的逻辑规范.

4.1.2 指令的基本格式

一条指令通常由 操作码 ( Opcode ) 和 地址码 ( Operand / Address ) 两部分组成.

\[ \text{指令} = \text{操作码 (OP)} + \text{地址码 (A)} \]

  • 操作码 ( OP ) : 指出指令中该进行什么操作 ( 如 ADD, MOV, JMP ).
  • 地址码 ( A ) : 指出被操作信息的地址 ( 寄存器编号、内存地址、立即数 ).

按地址码数目分类

类型 格式 含义 示例
零地址 OP 不需要操作数 ( 如停机 ) 或 隐含操作数 ( 如堆栈顶 ). RET, NOP
一地址 OP A1 (A1) OP -> A1 ( 单目运算 ) 或 (ACC) OP (A1) -> ACC. INC A, LOAD A
二地址 OP A1 A2 (A1) OP (A2) -> A1 ( 通常 A1 为目的操作数 ). ADD AX, BX
三地址 OP A1 A2 A3 (A1) OP (A2) -> A3. ( RISC 常见 )
四地址 OP A1 A2 A3 A4 (A1) OP (A2) -> A3, A4 为下一条指令地址. ( 早期计算机 )

4.1.3 定长操作码指令格式

  • 定义 : 指令字长固定，且操作码长度固定.
  • 优点 : 译码电路简单，执行速度快.
  • 缺点 : 浪费信息位，指令灵活性差.
  • 应用 : RISC ( 精简指令集计算机 ) 多采用定长操作码.

4.1.4 扩展操作码指令格式

  • 定义 : 指令字长固定，但操作码长度可变. ( 利用地址码的位来扩展操作码 ).
  • 原则 :
    1. 短码不能是长码的前缀 ( 类似哈夫曼编码，保证唯一译码 ).
    2. 使用频率高的指令分配短操作码，频率低的分配长操作码.

4.1.5 指令的操作类型

  1. 数据传送 : MOV, LOAD, STORE, PUSH, POP.
  2. 算术逻辑运算 : ADD, SUB, MUL, AND, OR, NOT, SHL ( 移位 ).
  3. 移位操作 : 算术移位、逻辑移位、循环移位.
  4. 程序控制 :
    • 无条件转移 : JMP.
    • 条件转移 : JZ ( 结果为0转 ), JO ( 溢出转 ).
    • 调用与返回 : CALL, RET.
    • 陷阱 ( Trap ) : 意外中断.
  5. 输入输出 : IN, OUT ( 独立编址时使用 ).

4.2 指令的寻址方式

寻址方式 : 确定 本条指令 的数据地址 以及 下一条指令 的地址的方法.

4.2.1 指令寻址和数据寻址

  • 指令寻址 : 寻找下一条将要执行的指令地址.
    • 顺序寻址 : PC = PC + 1 ( 1 代表指令字长 ).
    • 跳跃寻址 : PC = 目标地址 ( 由 JMP, CALL 等指令给出 ).
  • 数据寻址 : 寻找操作数的地址.
    • 格式 : OP + 寻址特征 + 形式地址 (A)
    • 有效地址 ( EA, Effective Address ) : 操作数的真实物理地址.

4.2.2 常见的数据寻址方式

Immediate Addressing

  • 格式 : OP #A
  • 原理 : 形式地址 A 就是操作数本身 ( 补码表示 ).
  • EA : 无需访存.
  • 特点 :
    • 优点 : 速度最快 ( 取指即取数 )，不访存.
    • 缺点 : 位数受限，只能给常数.

Direct Addressing

  • 格式 : OP A
  • 原理 : A 即为操作数的有效地址.
  • EA : \(EA = A\)
  • 特点 :
    • 优点 : 简单，无需计算 EA.
    • 缺点 : A 的位数限制了寻址范围.

Indirect Addressing

  • 格式 : OP @A ( 或 (A) )
  • 原理 : A 指向内存单元，该单元的内容才是 EA.
  • EA : \(EA = (A)\) ( 一次间址 ) 或 \(EA = ((A))\) ( 多次间址 ).
  • 特点 :
    • 优点 : 扩大寻址范围，便于编制程序 ( 如指针 ).
    • 缺点 : 多次访存，速度慢.

Register Addressing

  • 格式 : OP Ri
  • 原理 : 操作数在寄存器 \(R_i\) 中.
  • EA : \(EA = R_i\) ( 这里的 EA 指寄存器编号 ).
  • 特点 :
    • 优点 : 速度快 ( 不访存 )，指令字短.
    • 缺点 : 寄存器数量有限.

Register Indirect Addressing

  • 格式 : OP (Ri)
  • 原理 : 寄存器 \(R_i\) 的内容是操作数的内存地址.
  • EA : \(EA = (R_i)\)
  • 特点 :
    • 优点 : 比间接寻址快 ( 少访存一次 )，便于循环 ( 修改 \(R_i\) ).

Displacement Addressing

  • 原理 : \(EA = (R) + A\). ( 寄存器内容 + 形式地址 ).
  1. 相对寻址 ( Relative ) :
    • 基准 : 程序计数器 PC.
    • EA : \(EA = (PC) + A\). ( A 为偏移量，可正可负 ).
    • 用途 : 浮动程序 ( 代码在内存中移动不影响执行 )，条件转移.
    • ( 注 : 取指后 PC 已自增，计算时用的是更新后的 PC ).
  2. 基址寻址 ( Base ) :
    • 基准 : 基址寄存器 BR ( OS 管理 ).
    • EA : \(EA = (BR) + A\).
    • 用途 : 多道程序设计 ( 逻辑地址 -> 物理地址 )，用户不可修改 BR.
  3. 变址寻址 ( Indexed ) :
    • 基准 : 变址寄存器 IX ( 用户管理 ).
    • EA : \(EA = (IX) + A\).
    • 用途 : 数组处理、循环 ( A 为数组首地址，IX 为下标 ).

Stack Addressing

  • 原理 : 操作数在堆栈顶 ( 硬堆栈-寄存器 / 软堆栈-内存 ).
  • EA : 隐含在堆栈指针 SP 中.
  • 特点 : 0 地址指令.

4.3 程序的机器级代码表示

4.3.1 常用汇编指令介绍 ( x86 )

  • 数据传送 :
    • mov eax, ebx : 将 ebx 的值送入 eax.
    • push eax : 压栈 ( SP-4, 存入 eax ).
    • pop eax : 出栈 ( 弹出至 eax, SP+4 ).
  • 算术运算 :
    • add eax, 10 : eax = eax + 10.
    • sub eax, ebx : eax = eax - ebx.
    • inc eax : eax++.
  • 逻辑运算 : and, or, xor, not, shl, shr.

4.3.2 选择语句的机器级表示

选择结构 ( if-else ) 通常通过 条件码 ( Flags ) 和 条件转移指令 实现.

  1. CMP 指令 : cmp a, b ( 计算 \(a-b\)，不保存结果，只更新标志位 ).
    • ZF ( Zero Flag ) : 结果为 0 ( 即 a=b ) 时置 1.
    • SF ( Sign Flag ) : 结果为负 ( 即 a<b ) 时置 1.
    • CF ( Carry Flag ) : 无符号借位.
    • OF ( Overflow Flag ) : 有符号溢出.
  2. 跳转指令 :
    • je ( Jump if Equal ) : ZF=1 时跳.
    • jg ( Jump if Greater ) : \(a>b\) 时跳.
    • jl ( Jump if Less ) : \(a<b\) 时跳.
    • jmp : 无条件跳.

4.3.3 循环语句的机器级表示

循环 ( for, while ) 本质是 “条件判断 + 向后跳转”.

  • loop 指令 : 专用于循环，默认用 ecx 计数. ( ecx--, 若 ecx != 0 则跳 ).

4.3.4 过程调用的机器级表示

函数调用 ( Function Call ) 依赖 栈 ( Stack ) 来保存上下文.

  • 调用者 ( Caller ) :
    1. 保存 调用者保存寄存器.
    2. 参数压栈 ( 从右向左 ).
    3. call 指令 : 将 返回地址 (下一条指令) 压栈，跳转到目标函数.
  • 被调用者 ( Callee ) :
    1. push ebp ; mov ebp, esp ( 建立新的栈帧 ).
    2. 保存 被调用者保存寄存器.
    3. 执行函数体.
    4. leave ( 恢复栈指针 ) ; ret ( 弹出返回地址并跳转 ).

4.4 CISC 和 RISC 的基本概念

指令集架构的设计主要有两种流派：CISC ( Complex Instruction Set Computer, 复杂指令系统计算机 ) 和 RISC ( Reduced Instruction Set Computer, 精简指令系统计算机 ).

4.4.1 设计理念对比

核心理念 : “以硬代软”.

通过增加硬件的复杂性 ( 复杂的指令系统 ) 来简化编译器的设计工作.

  • 初衷 : 早期的存储资源昂贵，为了缩短目标程序长度，设计了功能强大的指令 ( 一条指令能完成高级语言的一句复杂语句 ).
  • 代表架构 : Intel x86 架构.

核心理念 : “二八定律”.

基于统计发现，程序中 80% 的时间在执行 20% 的简单指令.

  • 初衷 : 简化指令系统，使其适合 流水线 工作，从而提高主频和执行效率. 复杂的任务通过组合多条简单指令完成.
  • 代表架构 : ARM, RISC-V, MIPS.

对比维度 CISC ( 如 x86 ) RISC ( 如 ARM ) 优势方
1. 指令系统 复杂庞大，各类指令都有 精简，只包含最常用的简单指令 -
2. 指令长度 不固定 ( 变长，1-15字节 ) 固定 ( 定长，通常 4字节 ) RISC ( 译码快 )
3. 寻址方式 多且复杂 ( 可直接处理内存数据 ) 少且简单 RISC
4. 访存指令 不限 ( 任何指令都可访存 ) 只有 Load/Store 指令 RISC ( 规整 )
5. 指令周期 差异很大 大多数指令在 一个时钟周期 内完成 ( CPI \(\approx\) 1 ) RISC
6. 控制器实现 微程序控制 为主 组合逻辑控制 ( 硬布线 ) 为主 RISC ( 速度快 )
7. 寄存器数量 较少 很多 ( 利用寄存器窗口技术 ) RISC
8. 流水线支持 难以实现高效流水线 易于实现高效流水线 RISC
9. 编译器设计 难 ( 但生成的目标代码短 ) 易 ( 需优化流水线，生成代码长 ) CISC ( 代码密度高 )

4.4.3 现代融合趋势

知识拓展 : 界限模糊

现代高性能处理器往往融合了二者的优点

  • x86 ( CISC ) : 内部将复杂指令翻译成微操作 ( Micro-ops )，内核实际上是 RISC 架构.
  • ARM ( RISC ) : 为了提高性能，也引入了复杂的专用指令 ( 如 NEON 向量指令 ).

4.5 本章小结

本章重点在于理解 指令格式 ( 操作码+地址码 )、寻址方式 ( 如何找到操作数 ) 以及 CISC/RISC 的设计哲学.

  • 计算题高频点 : 扩展操作码的计算、相对寻址的 EA 计算 ( 注意 PC 自增 )、CISC/RISC 对比.
  • 概念题高频点 : 各种寻址方式的特点 ( 谁最快，谁范围最大 ).

4.6 常见问题和易混淆知识点

1. 基址寻址 vs 变址寻址

  • 基址寻址 ( Base ) : 面向 系统 ( OS ).用于程序重定位. 基址寄存器内容不变 ( 用户不可动 )，偏移量可变.
  • 变址寻址 ( Index ) : 面向 用户.用于数组遍历. 变址寄存器内容可变 ( 用户自增 )，偏移量不变 ( 数组首地址 ). ( 口诀 : 基址不动偏移变，变址在变偏移定 )

2. 相对寻址的 PC 值

在计算 \(EA = (PC) + A\) 时，PC 必须是更新后的值. 若指令字长为 2 字节，取出指令后 \(PC = PC_{old} + 2\). 计算 EA 时要用 \(PC_{old} + 2\).
